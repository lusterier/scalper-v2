## NEW H-038 hazard text (BRIEF §20, after the H-037 block; L-026 non-partial invariant + L-036 func-name Test citation)

```
### H-038 — SL/TP/BE/trail prices MUST be tick-quantized side-aware before set_trading_stop, and the quantized value persisted

**Context.** `services/execution/app/placement.py` (and the lifecycle BE/trail monitor) computed SL/TP prices as raw `compute_sl_price`/`compute_tp_price` (and `_compute_be_sl_price`/`_compute_trail_sl_price`) Decimals and sent them to `set_trading_stop` UNQUANTIZED. Bybit silently snaps a sub-tick price to the instrument `priceFilter.tickSize` (T-556 live-diagnostic: SL 76017.546→76017.50); a stricter instrument/endpoint may instead *reject* it → SL unset (capital risk). The DB persisted the raw value while the exchange held the snapped one → DB↔exchange drift (§N1 / reconciliation). Triage finding #2 (operator-prioritized ahead of triage C as a direct capital risk).

**Policy.** Every SL/TP/BE/trail price MUST be passed through `quantize_price(price, side, instrument_info.tick_size)` (T-558a; `buy`→ROUND_FLOOR / `sell`→ROUND_CEILING — SL never tighter, TP never further) BEFORE `set_trading_stop`, and the SAME quantized Decimal MUST be the value persisted (`persist_placement_tx` / `position_state` / BE-trail SL updates) so DB == exchange. In `placement.py` the quantize is applied once at the SL/TP compute site, BEFORE the paper/live fork, so paper, live, and persist receive the identical quantized value (parity preserved). (placement SL/TP: T-558b1; BE/trail enforcement + tests: T-558b2.)

**Test.** `test_placement_sl_tp_tick_quantized_at_set_trading_stop_and_persist` + `test_placement_quantize_price_uses_request_side_and_instrument_tick`.

H-038 numbering note: F6 triage-finding-#2 (surfaced by the T-556 live-diagnostic 2026-05-18; operator-prioritized ahead of triage C as direct capital risk). Sibling-class of H-037 (same pre-flight/instrument-filter capital-safety discipline as the qty side) but a distinct failure mode (price tick vs partial-TP min-lot), distinct origin, distinct policy (quantize-and-persist vs reject). Placement SL/TP enforced + tested in T-558b1; BE/trail enforcement + tests appended by T-558b2 to this same block. Not in the H-030..H-036 closed audit cluster.
```
