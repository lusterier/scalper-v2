### `services/execution/app/main.py` — create_app factory + lifespan

Skeleton lifespan: open pool + bus, attach to `app.state`, yield, reverse-shutdown.

```python
"""FastAPI app factory + lifespan for execution-service (§9.5, §N6)."""
from __future__ import annotations
from contextlib import asynccontextmanager
from typing import TYPE_CHECKING

from fastapi import FastAPI

from packages.bus import NatsClient
from packages.db import create_pool
from packages.observability import (
    configure,
    get_logger,
    make_metrics_asgi_app,
    make_registry,
)

from .config import Settings
from .health import router as health_router

if TYPE_CHECKING:
    from collections.abc import AsyncIterator

__all__ = ["create_app"]


def create_app(settings: Settings | None = None) -> FastAPI:
    if settings is None:
        settings = Settings()  # type: ignore[call-arg]

    configure(level=settings.log_level)
    logger = get_logger(settings.service_name, "system")
    registry_metrics = make_registry()

    @asynccontextmanager
    async def lifespan(app: FastAPI) -> AsyncIterator[None]:
        pool = await create_pool(
            settings.database_url,
            application_name=settings.service_name,
        )
        bus = NatsClient(
            servers=[settings.nats_url],
            name=settings.service_name,
            logger=logger,
        )
        await bus.connect()

        app.state.pool = pool
        app.state.bus = bus

        logger.info("service_started", http_port=settings.http_port)
        try:
            yield
        finally:
            await bus.close()
            await pool.close()
            logger.info("service_stopped")

    app = FastAPI(lifespan=lifespan)
    app.state.settings = settings
    app.state.logger = logger
    app.include_router(health_router)
    app.mount("/metrics", make_metrics_asgi_app(registry_metrics))
    return app
```

~100 LOC including module docstring. Smaller than T-109 final state (139 LOC) because no JSONB codec, no pipeline composition — those land in their owner tasks.

## Lifecycle

Lifespan order (T-214 minimum):
1. `pool = await create_pool(database_url, application_name="execution-service")`.
2. `bus = NatsClient(servers=[nats_url], name="execution-service", logger=logger); await bus.connect()`.
3. `app.state.pool = pool; app.state.bus = bus`.
4. `yield`.
5. `await bus.close()`.
6. `await pool.close()`.

T-215+ extends this: between steps 2 and 3, build `adapters: dict[BotId, ExchangeClient]` from `bots` table + per-bot env keys. Other tasks add APScheduler (T-220), reconciliation (T-221), pipeline classes (T-216, T-218).

## Edge cases

- **`database_url` is required**: pydantic-settings raises `ValidationError` at `Settings()` construction if env var unset. Mirror T-109 / T-100 behavior — service fails to start, healthcheck never green.
- **NATS connect failure**: `bus.connect()` raises in lifespan startup → uvicorn fails to come up. Test path `test_lifespan_propagates_bus_connect_failure` mirror T-109.
- **Reverse shutdown ordering**: `bus.close` before `pool.close` per T-109 precedent. Future T-216 publish flow may have an in-flight publish that depends on the pool — closing pool first would crash the publish. T-214 docstring locks the order so T-218 onwards inherits cleanly.
- **`/ready` reason precedence**: bus → db. Cheapest check first. Mirror T-109.
- **No `ws` field in `/ready` reason**: execution-service has no public-WS server. T-209 (private Bybit WS) is a long-lived OUTBOUND connection; readiness for it is wired at T-209 plan-doc time (likely a third reason key like `"exchange_ws"`); T-214 ships only `bus` + `db` reasons.
