# T-516b — Shadow variants per-trade drill-down section

**Phase:** F5
**Spec:** BRIEF §13.6 (Dashboard integration: per-trade drill-down shows all 5 variants alongside live outcome); ADR-0010 (parent_kind discriminator)
**Blocked-by:** T-516a1 (DONE — paper-trades backend), T-516a2 (DONE — paper-trades UI routes)
**Blocks:** T-517 (per-symbol best-variant aggregate + rejected explorer routes)
**Branch:** `feat/T-516b-shadow-variants-section`

## Purpose

Replaces placeholder #4 (Shadow variants) in BOTH `trades.$tradeId.tsx` (live) + `paper-trades.$paperTradeId.tsx` (paper) drill-down routes with a real `<ShadowVariantsView />` component that fetches all variants for the parent trade and renders them alongside the live outcome.

Operator OQs resolved at plan-stage (2026-05-09):

- **OQ-1 = Split per parent_kind**: TWO endpoints — `GET /api/trades/{trade_id}/shadow-variants` (parent_kind="live") + `GET /api/paper-trades/{paper_trade_id}/shadow-variants` (parent_kind="paper"). Mirror existing `/api/trades/{id}/*` + `/api/paper-trades/{id}/*` convention. parent_kind hardcoded per route handler.
- **OQ-2 = 7-col compact table**: variant_name + side + entry_price + qty + terminal_outcome (status pill) + realized_pnl (PriceDelta) + MFE/MAE % delta. Active variants (terminated_at NULL) render `—` for terminal_outcome / realized_pnl + "active" pill.
- **OQ-3 = Separate parent row in same table**: BRIEF §13.6 verbatim "all 5 variants alongside live outcome". `<ShadowVariantsView />` renders 1 special "Live" row at top (from `parent` prop) + N variant rows. Distinct visual pill `Live` / `Variant`. Parent passed as prop from parent route (Trade | PaperTrade union).
- **OQ-4 = Full mirror tests**: comprehensive coverage with parent_kind=live vs paper distinct cases for router + UI + DB query. ~350 LOC tests total.

## Public interface

### NEW DB helper `packages/db/queries/shadow.py`

```python
async def select_shadow_variants_by_parent(
    conn: asyncpg.Connection,
    *,
    parent_trade_id: int,
    parent_kind: Literal["live", "paper"],
) -> list[ShadowVariantRow]:
    """Return all variants (terminated + active) for parent_trade_id + parent_kind.

    ORDER BY variant_name ASC for stable render order. Used by T-516b
    /api/trades/{id}/shadow-variants + /api/paper-trades/{id}/shadow-variants.
    """
```

SQL: `SELECT {_SHADOW_VARIANT_BASE_COLUMNS} FROM shadow_variants WHERE parent_trade_id = $1 AND parent_kind = $2 ORDER BY variant_name ASC`. Returns `list[ShadowVariantRow]` (empty list if none).

### Endpoint additions

**ADD to `services/analytics_api/app/routers/trades.py`**:

```python
@router.get("/{trade_id}/shadow-variants", response_model=ShadowVariantListResponse)
async def list_shadow_variants_for_trade(
    trade_id: int,
    pool: Annotated[asyncpg.Pool, Depends(get_pool)],
) -> ShadowVariantListResponse:
    """All shadow variants for a live trade (parent_kind='live')."""
    async with pool.acquire() as conn:
        rows = await select_shadow_variants_by_parent(
            conn, parent_trade_id=trade_id, parent_kind="live"
        )
    return ShadowVariantListResponse(
        variants=[ShadowVariantResponse(**asdict(r)) for r in rows]
    )
```

**ADD to `services/analytics_api/app/routers/paper_trades.py`** — identical handler with `parent_kind="paper"` + URL `{paper_trade_id}`. No 404 required when zero variants — empty list is valid (parent trade may not have any shadow variants).

### NEW UI component `ui/src/components/ShadowVariantsView.tsx`

```tsx
export function ShadowVariantsView({
  parentTradeId,
  parentKind,
  parent,
}: {
  parentTradeId: string;
  parentKind: "live" | "paper";
  parent: Trade | PaperTrade | undefined;
}): React.JSX.Element {
  const url = parentKind === "live"
    ? `/api/trades/${parentTradeId}/shadow-variants`
    : `/api/paper-trades/${parentTradeId}/shadow-variants`;

  const variantsQuery = useQuery({
    queryKey: ["shadow-variants", parentKind, parentTradeId],
    queryFn: () => apiFetch<ShadowVariantListResponse>(url),
    staleTime: 30_000,
    retry: false,
  });

  // Root container uses data-testid="shadow-variants-view".
  // Loading skeletons use data-testid="shadow-variants-loading".
  // Component MUST NOT render any data-testid="timeline-placeholder"
  // (placeholder count discipline per AC#15a / WG#10).
  //
  // Render 7-col table:
  //   1. type (pill: "Live" | "Variant")
  //   2. variant_name (or "live" for parent row)
  //   3. side
  //   4. entry_price (mono)
  //   5. qty (mono)
  //   6. terminal_outcome / status (pill: terminated outcome OR "active" OR "open")
  //   7. realized_pnl (PriceDelta or — for active/null)
  //   + MFE/MAE pair (cell with two values)
}
```

## Lifecycle

UI: `<ShadowVariantsView />` mounts when parent route resolves; useQuery fetches variants; unmount on navigation away. 30s staleTime matches drill-down convention.

Backend: stateless GET handler; pool-backed; no transaction needed (single SELECT).

DB: read-only single SELECT `WHERE parent_trade_id = $1 AND parent_kind = $2 ORDER BY variant_name ASC`.

## Edge cases

1. **Zero variants for parent** — backend returns `{ variants: [] }`. UI shows "Live" parent row only + empty-state message "No shadow variants for this trade".
2. **Parent prop undefined (parent route still loading)** — UI renders skeleton placeholder for Live row + "Loading variants…" for variants table. Avoids double-loading-state (parent + variants both fetched in parallel by parent route).
3. **All variants active (terminated_at NULL)** — render `—` for terminal_outcome + realized_pnl, "active" pill in status column. MFE/MAE may be null (if FSM hasn't recorded yet) → render `—`.
4. **parent_kind mismatch (paper trade ID lookup against live route)** — backend returns empty list (no rows match `parent_trade_id=X AND parent_kind='live'` if paper trade); UI renders empty state. NOT a 404.
5. **Trade exists but no shadow runtime ran** — same as zero variants case.
6. **Decimal precision** — backend Pydantic serializes Decimal as string per §5.3; UI renders via mono span (no arithmetic).
7. **Active variant → realized_pnl null** — guard render via `?? "—"`.
