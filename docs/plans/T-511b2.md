```python
async def emit_post_commit_close_event(
    *,
    bus: BusProtocol,
    bot_id: str,
    correlation_id: CorrelationId,
    order_closed_payload: OrderClosed,
    trade_id: int,
    closed_at: datetime,
    bound_logger: BoundLogger,
) -> None:
    """Post-commit publisher for ``OrderClosed`` + ``TradeClosedPayload``.

    Two publishes — OrderClosed to ``orders.events.<bot_id>`` (production wire)
    + TradeClosedPayload to ``trade.closed.<bot_id>`` (internal H-016 shadow
    cancellation hook per T-511b2). Both best-effort; per-publish try/except
    so first publish failure does NOT short-circuit second publish (mirror
    :func:`emit_post_commit_events` WG#6 pattern at placement_persist.py:478).
    """
    from packages.bus import MessageEnvelope as _Env
    from packages.bus.payloads import TradeClosedPayload, subject_for_trade_closed

    # Publish 1: OrderClosed (production wire — analytics-api consumes).
    try:
        envelope = _Env(
            correlation_id=correlation_id,
            publisher="execution-service",
            payload=order_closed_payload.model_dump(mode="json"),
        )
        await bus.publish(subject_for_orders_event(bot_id), envelope)
    except Exception as exc:
        bound_logger.error(
            "execution.event_publish_failed",
            bot_id=bot_id,
            event_type="order_closed",
            error=str(exc),
        )

    # Publish 2: TradeClosedPayload (internal H-016 hook — T-511b2).
    try:
        trade_closed_payload = TradeClosedPayload(
            parent_trade_id=trade_id,
            bot_id=str(bot_id),
            closed_at=closed_at,

            correlation_id=correlation_id,
            publisher="execution-service",
            payload=trade_closed_payload.model_dump(mode="json"),
        )
        await bus.publish(subject_for_trade_closed(str(bot_id)), trade_closed_envelope)
    except Exception as exc:
        bound_logger.error(
            "execution.trade_closed_publish_failed",
            bot_id=bot_id,
```

```python
async def _on_parent_close_envelope(self, envelope: MessageEnvelope) -> None:
    """NATS handler: parse envelope → dispatch to typed handler."""
    payload = TradeClosedPayload.model_validate(envelope.payload)
    await self._on_parent_close(payload)

async def _on_parent_close(self, payload: TradeClosedPayload) -> None:
    """Cancel all in-flight variant tasks for the closing parent trade.

    Per BRIEF §20 H-016 policy: when parent trade closes, cancel all child
    variant tasks immediately. Each cancelled task's H-016 try/finally
    finalizer (T-511b1 shipped) handles its own bus_unsubscribe + registry
    cleanup. Idempotent on duplicate close events (registry empty after
    first cancellation).
    """
    tasks = self._active_tasks.get(payload.parent_trade_id)
    if not tasks:
        return
    for task in tasks:
        if not task.done():
            task.cancel()
    # Do NOT await tasks here — finalizer cleanup happens lazily as each
    # CancelledError propagates through the task body. Awaiting would
    # block the close-event handler indefinitely if any task's finally
    # block is slow (NATS unsubscribe under network failure).
```

## Write-time guidance (consume verbatim during implementation)

1. **Reuse VariantSpec from packages.bus.payloads at strategy-engine import**: do NOT redefine in strategy-engine; import from `packages.bus.payloads` to keep wire-shape consistent. Verify mapping `ShadowVariant.overrides: dict[str, Decimal]` (scoring) → `VariantSpec.overrides: dict[str, Decimal | int]` (bus) is structurally compatible (Decimal is subset of `Decimal | int`; existing scoring dict keys all map correctly).

2. **OrderRequest schema test backfill**: existing `test_orders_schema.py` constructs OrderRequest with all kwargs. After adding 2 fields, existing tests still work because new fields have defaults (`Field(default_factory=list)` + `None`). Brief-reviewer must verify no existing test broke.

3. **Emit-from-placement.py conditional guard**: `if request.shadow_variants:` (truthy check on list) catches both `[]` (default) AND non-list nulls; explicit empty-list short-circuit avoids needless emit. Test #12 verifies.

4. **`_on_parent_close` non-await semantic**: cancel-only, do NOT await tasks. Awaiting in close-handler context blocks the bus consumer indefinitely if a task's finally is slow (NATS network down). Each cancelled task's own try/finally (T-511b1 shipped) handles its bus_unsubscribe lazily. Idempotent: duplicate close events fire `task.cancel()` on already-cancelled tasks (no-op).

5. **`_active_tasks[parent_trade_id]` reads in `_on_parent_close`**: registry mutation happens in `_run_shadow_variant`'s finally block (each task removes itself). `_on_parent_close` reads + cancels concurrently. asyncio is single-threaded; the read+cancel sequence is atomic w.r.t. the registry mutation (no event-loop interleave).
