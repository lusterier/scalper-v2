## Goal

Ship the shadow-worker producer half — paper-aware per ADR-0010:

1. **LIVE open-side emit**: `emit_post_commit_shadow_start_event` in `placement_persist.py` publishing `ShadowStartPayload` (parent_kind='live') to `shadow.start.<bot_id>` AFTER trade commit; called from `placement.py:328` site only when the parent OrderRequest carries `shadow_variants`.
2. **PAPER open-side emit**: same helper called from `placement.py:240-252` paper-fork BEFORE early-return; `parent_trade_id = OrderPlaceResult.paper_trade_id` (NEW field; PaperExchange populates from `_persist_open` insert_paper_trade return), `parent_kind='paper'`.
3. **LIVE close-side emit**: extend `emit_post_commit_close_event` in `reconcile.py` to ALSO publish `TradeClosedPayload(parent_kind='live')` to `trade.closed.<bot_id>` alongside `OrderClosed`.
4. **PAPER close-side emit**: PaperExchange `_persist_close` directly publishes `TradeClosedPayload(parent_kind='paper')` post-commit, gated by NEW ctor flag `emit_parent_lifecycle: bool = False` (default False for variant PE; primary bot PE in adapter pool wiring sets True).
5. **Parent-close H-016 hook**: ShadowWorker subscribes to `trade.closed.>` (in addition to existing `shadow.start.>`); `_on_parent_close` cancels all `_active_tasks[parent_trade_id]` tasks per BRIEF §20 H-016 policy. Topic mode-agnostic — both live and paper events flow through same subscription.
6. **Per-bot YAML config plumbing**: extend `OrderRequest` wire schema with `shadow_variants` + `shadow_max_duration_hours` (additive non-breaking; schema_version stays "1.0" per CONCERN 5 fix); strategy-engine `consumer.py:_publish_order_request` populates from `bot_config.shadow` when enabled + maps `BotConfig.exchange.mode` → ShadowStartPayload.parent_kind ("paper" → "paper"; "live"/"testnet" → "live").
7. **main.py wire**: construct `ShadowWorker` in execution-service lifespan; `start()` after pool/bus ready; `stop()` AFTER `bus.close()` (per main.py:300-330 existing convention; main.py shutdown BLOCKER 2 fix).
8. **BRIEF §13.7 parity test**: `test_variant_step_transitions_match_live_lifecycle_fsm` (CONCERN 4 verbatim name fix) cross-checks the 3 BE/trail helpers in `shadow_worker.py` against verbatim source `lifecycle.py:233-268` — anti-DRY guard for T-511b1 verbatim-copy decision.

```python
async def emit_post_commit_shadow_start_event(
    *,
    bus: BusProtocol,
    bot_id: BotId,
    correlation_id: CorrelationId,
    parent_trade_id: int,
    parent_kind: Literal["live", "paper"],
    symbol: str,
    side: Literal["buy", "sell"],
    entry_price: Decimal,
    qty: Decimal,
    shadow_variants: list[VariantSpec],
    bound_logger: BoundLogger,

    """Post-commit publisher for ``ShadowStartPayload`` (mirror :func:`emit_post_commit_events`).

    Best-effort: publish failure logged but does NOT raise — DB tx already
    committed. parent_kind discriminator routes parent_trade_id to either
    trades.id (live) or paper_trades.id (paper) per ADR-0010 + T-511b2a foundation.

    from packages.bus.payloads import ShadowStartPayload, subject_for_shadow_start

    payload = ShadowStartPayload(
        parent_trade_id=parent_trade_id,
        parent_kind=parent_kind,
        bot_id=str(bot_id),

        side=side,
        entry_price=entry_price,
        qty=qty,

            correlation_id=correlation_id,

            payload=payload.model_dump(mode="json"),

            "execution.shadow_start_publish_failed",

            parent_trade_id=parent_trade_id,
            parent_kind=parent_kind,
            error=str(exc),
```

```python
# New handlers:
async def _on_parent_close_envelope(self, envelope: MessageEnvelope) -> None:
    """NATS handler: parse envelope → dispatch to typed handler."""
    payload = TradeClosedPayload.model_validate(envelope.payload)
    await self._on_parent_close(payload)

async def _on_parent_close(self, payload: TradeClosedPayload) -> None:
    """Cancel all in-flight variant tasks for the closing parent trade.

    Per BRIEF §20 H-016 policy: when parent trade closes, cancel all child
    variant tasks. Each cancelled task's H-016 try/finally finalizer
    (T-511b1 shipped) handles its own bus_unsubscribe + registry cleanup.
    Cancel-only (no await) — duplicate close events fire .cancel() on
    already-cancelled tasks (no-op).

    parent_kind from payload is informational only (logging/metrics);
    cancellation logic depends only on parent_trade_id (registry key).
    """
    tasks = self._active_tasks.get(payload.parent_trade_id)
    if not tasks:
        return
    for task in tasks:
        if not task.done():
            task.cancel()
```
