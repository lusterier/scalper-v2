## Edge cases

- **`--plugin-registry-path` unset + bot YAML uses plugin condition**: `load_bot_config_from_string` (`yaml_loader.py:223 _parse_plugin`) raises `ValueError("plugin condition requires plugin_registry")`. CLI re-raises as SystemExit(2) with operator-actionable message *"--plugin-registry-path is required when bot config uses plugin conditions"*. CONCERN #4 addressed.
- **`--config-path` file missing**: argparse `type=Path` + manual `path.is_file()` check → SystemExit(2).
- **`--config-path` file invalid YAML**: `yaml.safe_load` raises `YAMLError` → SystemExit(2) with file-name + line-number context.
- **`--override` path doesn't exist on raw YAML dict**: ValueError with explicit *"--override path 'X.Y.Z' does not exist on YAML"* → SystemExit(2).
- **`--override` value type mismatch on BotConfig validation**: pydantic ValidationError surfaces field-specific message → SystemExit(2).
- **`bots` row not found for --bot**: insert_backtest_run FK constraint fails → SystemExit(1) with "register bot first via UI or seed bots table" hint.

## --override syntax precision (CONCERN #4 from prior REVISE)

Format: `'<dotted-path>=<value>'`, repeatable.

### Path syntax

Per OQ-A=A (raw bytes config), path resolves against the **raw YAML dict** (NOT `BotConfig.model_dump()` shape). Examples:
- `'trading.universe=["BTCUSDT","ETHUSDT"]'` (top-level YAML key per BRIEF §B.1)
- `'execution.fee_rate=0.0005'`
- `'execution.slippage_params.fixed_slippage_pct=0.001'`
- `'scoring.rules.0.threshold=0.7'` (integer segment for list index)
- `'signals.ttl_seconds=180'`

### Value typing

- Bool: `'true'` / `'false'` / `'True'` / `'False'`
- Int: `'42'`
- Float: `'0.5'`
- Str: fallback (any non-numeric, non-bool)
- List: YAML literal `'[...]'` (parsed via `yaml.safe_load` of value)

### Apply algorithm

```python
def _apply_overrides(yaml_dict: dict, overrides: list[str]) -> dict:
    """Walk + mutate raw YAML dict per --override spec."""
    for kv in overrides:
        path, _, value_raw = kv.partition("=")
        value = yaml.safe_load(value_raw)  # auto-typing
        segments = path.split(".")
        cursor = yaml_dict
        for seg in segments[:-1]:
            if seg.isdigit():
                cursor = cursor[int(seg)]
            elif isinstance(cursor, dict) and seg in cursor:
                cursor = cursor[seg]
            else:
                raise ValueError(
                    f"--override path {path!r} does not exist on YAML"
                )
        last = segments[-1]
        if last.isdigit():
            cursor[int(last)] = value
        else:
            cursor[last] = value
    return yaml_dict
```

After mutation: `config_yaml = yaml.safe_dump(yaml_dict, sort_keys=False)`; `config_hash = sha256(config_yaml.encode("utf-8")).hexdigest()`. Then validate via `load_bot_config_from_string(config_yaml, plugin_registry=...)` — pydantic catches type mismatches with field-specific errors.

For no-override case: identity preserved (raw bytes from file = `config_yaml` stored = hash input).

**Hash non-identity caveat (CONCERN #4 from second REVISE)**: `yaml.safe_dump` is non-identity round-trip even for value-preserving overrides — comments stripped, numeric/string formatting normalised. Consequence: an override invocation that doesn't change any value (e.g., `--override 'symbols=["BTCUSDT"]'` where symbols is already `["BTCUSDT"]`) produces a `config_yaml` whose hash differs from the no-override hash. **Operator-acknowledged**: hash semantically pins the canonicalised YAML, NOT the user input. Cross-source audit lookups (analytics-api UI hash vs CLI hash) reliably match for no-override invocations only; with overrides the hash diverges from any user-pasted YAML even if functionally equivalent. Document this in operator-runbook for T-509 follow-up.

### `_publish_signals` helper (BLOCKER #3 fix + OQ-D=C clock advancement)

```python
async def _publish_signals(
    source: HistoricalSignalSource,
    bus: BusProtocol,
    bot_config: BotConfig,
    replay_clock: ReplayClock,
) -> None:
    """Yield SignalRow from T-504; advance replay_clock; reconstruct SignalValidated; publish.

    SignalRow is @dataclass (NOT Pydantic). CLI explicitly constructs
    SignalValidated from the 8 SignalRow fields + computes expires_at from
    received_at + bot's signal TTL (BotConfig.signals.ttl_seconds; H-008).
    """
    async for signal_row in source:
        # OQ-D=C: advance replay_clock to signal timestamp BEFORE publishing
        # so subsequent handler invocations see consistent now() context.
        replay_clock.set(signal_row.received_at)
        signal_validated = SignalValidated(
            schema_version="1.0",
            source=signal_row.source,
            idempotency_key=signal_row.idempotency_key,
            received_at=signal_row.received_at,
            symbol=signal_row.symbol,
            original_symbol=signal_row.original_symbol or signal_row.symbol,
            action=signal_row.action,
            expires_at=signal_row.received_at
            + timedelta(seconds=bot_config.signals.ttl_seconds),
            payload=signal_row.payload,
        )
        envelope = MessageEnvelope(
            correlation_id=CorrelationId(f"backtest-{uuid4()}"),
            published_at=signal_row.received_at,  # heap-sort key per T-502
            publisher="backtest-cli",
            payload=signal_validated.model_dump(),
        )
        await bus.publish("signals.validated", envelope)
```

### `ReplayClock` helper (OQ-D=C, NEW `scripts/replay_clock.py`)

```python
"""Replay-virtual clock for backtest CLI (T-507b OQ-D=C Belt-and-suspenders).

Stateful object advanced by paper.run_replay (per OHLC bucket_start) and
signal publisher (per signal.received_at). All components receive
`now_fn=replay_clock.now` so timestamps stay consistent across replay
(paper trades + scoring evaluations + dispatched orders + reconcile flow).

Without ReplayClock the wall-clock now_fn produces today's timestamps for
backtest of historical data — paper_trades.closed_at = today, while
signal.received_at = weeks ago. Inconsistent semantic + breaks
max_signal_age_seconds DB lookup window (consumer.py:143).
"""
from __future__ import annotations

from dataclasses import dataclass, field
from datetime import datetime

__all__ = ["ReplayClock"]


@dataclass
class ReplayClock:
    """Monotonic replay-virtual clock; latest set wins (must be >= current)."""

    initial: datetime
    _t: datetime = field(init=False)

    def __post_init__(self) -> None:
        self._t = self.initial

    def set(self, t: datetime) -> None:
        """Advance clock to t (no-op if t < current; non-monotonic protection)."""
        if t > self._t:
            self._t = t

    def now(self) -> datetime:
        """Return current replay-virtual time."""
        return self._t
```

### `_load_bot_config_with_overrides` helper (BLOCKER #6 fix — raw bytes per OQ-A=A)

```python
def _load_bot_config_with_overrides(
    *, config_path: Path, overrides: list[str],
    plugin_registry: PluginRegistry | None,
) -> tuple[str, str, BotConfig]:
    """Load --config-path bytes, apply --override patches via raw YAML dict mutation,
    re-serialise, hash, validate. Returns (config_yaml, config_hash, BotConfig).

    Raw-bytes-first per OQ-A=A: config_hash matches analytics-api convention
    (`backtests.py:206`) so cross-source audit lookups work. No-override case:
    identity preserved (raw bytes = stored = hashed).
    """
    raw_bytes = config_path.read_bytes()
    if overrides:
        yaml_dict = yaml.safe_load(raw_bytes)
        yaml_dict = _apply_overrides(yaml_dict, overrides)
        config_yaml = yaml.safe_dump(yaml_dict, sort_keys=False, allow_unicode=True)
    else:
        config_yaml = raw_bytes.decode("utf-8")
    config_hash = hashlib.sha256(config_yaml.encode("utf-8")).hexdigest()
    bot_config = load_bot_config_from_string(
        config_yaml, plugin_registry=plugin_registry,
    )
    return config_yaml, config_hash, bot_config
```

### `_compute_summary` helper (CONCERN #2 from prior — explicit Decimal/float boundary; CONCERN #6 ADR-0008 PF=None)

```python
async def _compute_summary(conn: _DbExecutor, run_id: UUID) -> dict[str, Any]:
    """5 aggregates over backtest_trades. Decimal/float boundary explicit:
    pnl + mdd are Decimal (NUMERIC sums per §5.3); wr + pf are float
    (proportions/dimensionless ratio). PF=None per ADR-0008 when undefined.
    """
    rows = await conn.fetch(
        "SELECT realized_pnl FROM backtest_trades "
        "WHERE run_id = $1 AND status = 'closed' ORDER BY closed_at",
        run_id,
    )
    if not rows:
        return {"total_trades": 0, "wr": None, "pnl": "0", "pf": None, "mdd": "0"}

    pnls: list[Decimal] = [Decimal(r["realized_pnl"]) for r in rows]
    total_trades = len(pnls)
    wins = sum(1 for p in pnls if p > 0)
    wr = round(wins / total_trades, 4)

    pnl: Decimal = sum(pnls, start=Decimal("0"))
    gross_wins: Decimal = sum((p for p in pnls if p > 0), start=Decimal("0"))
    gross_losses: Decimal = sum((-p for p in pnls if p < 0), start=Decimal("0"))

    # ADR-0008: PF=None for both edge cases (no losses OR no wins).
    if gross_wins == 0 or gross_losses == 0:
        pf: float | None = None
    else:
        # Explicit Decimal→float cast at dimensionless-ratio boundary.
        pf = round(float(gross_wins) / float(gross_losses), 4)

    cum, peak, mdd = Decimal("0"), Decimal("0"), Decimal("0")
    for p in pnls:
        cum += p
        if cum > peak:
            peak = cum
        drawdown = peak - cum
        if drawdown > mdd:
            mdd = drawdown

    return {
        "total_trades": total_trades,
        "wr": wr,
        "pnl": str(pnl),
        "pf": pf,
        "mdd": str(mdd),
    }
```

## Drain coordination semantics

The replay completes when **all 4 conditions** hold:
1. `signal_source` async iterator exhausted
2. `paper.run_replay()` returned (HistoricalOHLCSource exhausted)
3. `bus.run_until_empty()` drained (no envelopes left)
4. `dispatcher_task` cancelled (consume loop blocks on `paper.stream_executions()` infinite async-for; cancel + suppress is the only termination)

If any task raises during gather → cleanup cancels dispatcher + propagates → outer except marks run as FAILED.

## ADR-0008 — backtest summary PF semantics

(Separate file `docs/adr/0008-backtest-summary-pf-semantic.md` shipped with T-507b commit.)

**Decision**: PF in `backtest_runs.summary` is `null` when `gross_wins == 0` OR `gross_losses == 0`. Otherwise `pf = float(gross_wins / gross_losses)` rounded to 4 decimals.

**Rationale**: operator-friendly JSON serialisation; avoids JSON-incompatible `Inf` and ambiguous `0.0` for no-wins.

**Trade-off**: deviation from finance convention (PF=∞ for no losses; PF=0 for no wins). Operator may need explicit conversion when comparing T-507b output to external tools (TradingView, Backtrader). Documented.
