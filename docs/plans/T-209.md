## OQ resolutions (operator-approved 2026-04-30 — "use defaults")

1. **OQ-1 — Layout & WS class shape**: **A** — new file `packages/exchange/bybit_v5/ws.py` containing class `BybitV5PrivateWs` (own lifecycle, own state). Adapter receives the constructed ws via ctor kwarg `ws: BybitV5PrivateWs` (DI mirror of `client: BybitV5Client`). Mirror `BinanceWsClient` layering.

2. **OQ-2 — Queue overflow strategy**: **A** — internal `asyncio.Queue()` (unbounded) for executions + positions. Writer = receive loop, never blocks. Bounded + drop violates H-009 (consumer dedup by `execId`, drop = silent loss). PaperExchange's bounded queue is acceptable in-process; live WS is not. Add depth-observability log (WARN at 1000 depth) deferred to T-F2+ if observability shows pressure.

3. **OQ-3 — Backoff helper**: **A** — re-derive `_full_jitter_backoff_delays(rng)` privately in `ws.py` (~10 LOC). Cross-package coupling `exchange → market` for one helper is worse than ~10 LOC duplication. Promote to `packages.core` if and only if a third site emerges (rule-of-three).

4. **OQ-4 — Auth-failure handling on reconnect**: **A** — on `auth_data.get("success") is False`: log ERROR `bybit_private_ws_auth_failed`, increment counter `bybit_private_ws_auth_failures_total`, continue the reconnect loop (does NOT raise into the task). H-007 mandates unconditional reconnect; permanent HMAC mis-sign is surfaced via metric → F5 alerting (out of T-209 scope).

5. **OQ-5 — Heartbeat**: **A** — app-level `{"op": "ping"}` JSON frame every 18s (port v1; Bybit V5 expects JSON ping, native control PING may not satisfy server idle timeout). Native `websockets.connect(..., ping_interval=20, ping_timeout=10)` as transport-level backstop.

6. **OQ-6 — Category filter**: **A** — drop frames where `data[i].get("category") != "linear"` per v1 port. F2 single-bot scope = linear futures only. Filter applied per data item, not per topic frame (Bybit may batch heterogeneous categories; we ignore non-linear items, keep linear).

7. **OQ-7 — WS lifecycle wiring**: **A** — `BybitV5PrivateWs.run()` is the blocking driver: open → auth → subscribe → receive → reconnect-loop, until `close()`. T-215 composition root will spawn `asyncio.create_task(ws.run())` after constructing both `client` and `ws` and the adapter. `BybitV5Adapter.close()` calls `await self._ws.close(); await self._client.close()` (in that order — close ws BEFORE client to drain WS state cleanly).

8. **OQ-8 — WS URL ctor default**: **A** — `ws_url: str = "wss://stream.bybit.com/v5/private"` default; testnet/demo URLs are ctor-kwarg overrides supplied by composition root (mirror T-207 `base_url`).

9. **OQ-9 — PositionEvent mapper**: **A** — duplicate the field-mapping logic locally in `ws.py` as `_map_position_event(item: dict) -> PositionEvent`. 6 fields + `occurred_at = datetime.fromtimestamp(int(updatedTime)/1000, tz=UTC)`. Document alignment with T-208b `_map_position_row` in module docstring. Surface-expansion guard: NO shared `PositionMapper` class extracted across REST and WS.

10. **OQ-10 — Test strategy**: **A** — unit-only with mocked websockets: a `FakeWebSocket` async-iterable yields canned frames; `websockets.asyncio.client.connect` patched to return it. Backoff determinism via injected `rng=random.Random(0)`. ~15 tests covering auth-signing hand-compute (HMAC equality), reconnect-with-injected-backoff, frame demux execution vs position, category filter (drop non-linear), queue draining, close idempotency + cancels receive task, ExecutionEvent/PositionEvent mapping H-015 round-trip, auth-failure path (counter + continued reconnect), pong handling. No testnet integration tests in T-209 (deferred to T-222 E2E smoke per §11.6).

11. **OQ-11 — Pre-emptive split**: **A** — keep T-209 unified. Estimated ~250 LOC src (220 ws.py + 30 adapter delta) + ~280 LOC tests. Under §0.3 cap (400) and at L-006 watch threshold (280). Combined review is more valuable (auth + reconnect + delegation in one diff). Mid-write contingency: if `ws.py` exceeds 280 LOC mid-write, split per L-007 — extract `_FrameDemux` or factor auth signing into helper module.

   ```python
   def _full_jitter_backoff_delays(rng: random.Random | None) -> Iterator[float]:
       rand = rng or random
       attempt = 0
       while True:
           ceiling = min(1.0 * (2**attempt), 60.0)
           yield rand.uniform(0.0, ceiling)
           attempt += 1
   ```

## Public interface (T-209 scope)

### `packages/exchange/bybit_v5/ws.py` (new file)

```python
"""§11.2 BybitV5PrivateWs — long-lived authenticated WS for Bybit V5.

T-209 ships:

* :class:`BybitV5PrivateWs` — owns the receive loop + auth handshake +
  reconnect with full-jitter backoff (H-007). Demuxes inbound frames by
  ``topic`` to per-stream :class:`asyncio.Queue` instances.
* ``executions()`` / ``positions()`` — :class:`AsyncIterator` factories
  draining the per-stream queues. Mirrors :class:`PaperExchange` Protocol
  contract per T-201 OQ-1 (``def`` not ``async def``).
* ``close()`` — idempotent lifecycle drain.

Not on the ExchangeClient Protocol — adapter delegates to this class.
"""

# Module constants per Decision #10 (protocol-binding, not operationally tunable):
_BACKOFF_BASE_S = 1.0
_BACKOFF_CAP_S = 60.0
_AUTH_PAYLOAD_PREFIX = "GET/realtime"
_AUTH_EXPIRES_OFFSET_S = 5

# Default ctor kwargs (operator-tunable per §N9 / Decision #11):
_DEFAULT_WS_URL = "wss://stream.bybit.com/v5/private"
_DEFAULT_PING_INTERVAL_S = 18.0
_DEFAULT_LONG_DISCONNECT_THRESHOLD_S = 60.0


def _full_jitter_backoff_delays(
    rng: random.Random | None = None,
) -> Iterator[float]:
    rand = rng if rng is not None else random
    attempt = 0
    while True:
        ceiling = min(_BACKOFF_BASE_S * (2**attempt), _BACKOFF_CAP_S)
        yield rand.uniform(0.0, ceiling)
        attempt += 1


def _map_execution_event(item: dict[str, Any]) -> ExecutionEvent:
    side_raw = item["side"]
    side: Literal["buy", "sell"] = "buy" if side_raw == "Buy" else "sell"
    return ExecutionEvent(
        exchange_exec_id=item["execId"],
        exchange_order_id=item["orderId"],
        symbol=item["symbol"],
        side=side,
        price=Decimal(str(item["execPrice"])),
        qty=Decimal(str(item["execQty"])),
        fee=Decimal(str(item["execFee"])),
        executed_at=datetime.fromtimestamp(int(item["execTime"]) / 1000, tz=UTC),
    )


def _map_position_event(item: dict[str, Any]) -> PositionEvent:
    side_raw = item["side"]
    if side_raw == "Buy":
        side: Literal["buy", "sell"] | None = "buy"
    elif side_raw == "Sell":
        side = "sell"
    else:
        side = None
    avg_price = item.get("avgPrice")
    entry_price = Decimal(str(avg_price)) if avg_price not in ("", None) else None
    leverage_raw = item.get("leverage")
    leverage = int(str(leverage_raw)) if leverage_raw not in ("", None) else None
    unrealized_raw = item.get("unrealisedPnl")
    unrealized_pnl = (
        Decimal(str(unrealized_raw)) if unrealized_raw not in ("", None) else None
    )
    return PositionEvent(
        symbol=item["symbol"],
        side=side,
        size=Decimal(str(item["size"])),
        entry_price=entry_price,
        leverage=leverage,
        unrealized_pnl=unrealized_pnl,
        occurred_at=datetime.fromtimestamp(int(item["updatedTime"]) / 1000, tz=UTC),
    )


class ConnectionState(Enum):
    DISCONNECTED = "disconnected"
    CONNECTING   = "connecting"
    CONNECTED    = "connected"
    RECONNECTING = "reconnecting"
    CLOSED       = "closed"


class BybitV5PrivateWs:
    def __init__(
        self,
        *,
        api_key: str,
        api_secret: str,
        logger: BoundLogger,
        ws_url: str = _DEFAULT_WS_URL,
        ping_interval_s: float = _DEFAULT_PING_INTERVAL_S,
        long_disconnect_threshold_s: float = _DEFAULT_LONG_DISCONNECT_THRESHOLD_S,
        rng: random.Random | None = None,
        now_fn: Callable[[], float] = time.time,
    ) -> None:
        self._api_key = api_key
        self._api_secret = api_secret
        self._logger = logger
        self._ws_url = ws_url
        self._ping_interval_s = ping_interval_s
        self._long_disconnect_threshold_s = long_disconnect_threshold_s
        self._rng = rng
        self._now_fn = now_fn
        self._state = ConnectionState.DISCONNECTED
        self._ws: ClientConnection | None = None
        self._execution_queue: asyncio.Queue[ExecutionEvent] = asyncio.Queue()
        self._position_queue: asyncio.Queue[PositionEvent] = asyncio.Queue()
        self.long_disconnect_count: int = 0
        self.auth_failure_count: int = 0

    @property
    def state(self) -> ConnectionState:
        return self._state

    async def run(self) -> None:
        if self._state is not ConnectionState.DISCONNECTED:
            raise NotConnectedError(
                f"run() called in state {self._state.value!r}; expected 'disconnected'"
            )
        await self._reconnect_loop()

    async def close(self) -> None:
        if self._state is ConnectionState.CLOSED:
            return
        self._state = ConnectionState.CLOSED
        if self._ws is not None:
            with contextlib.suppress(WebSocketException):
                await self._ws.close()
        self._ws = None

    def executions(self) -> AsyncIterator[ExecutionEvent]:
        return self._iter_executions()

    async def _iter_executions(self) -> AsyncIterator[ExecutionEvent]:
        while True:
            yield await self._execution_queue.get()

    def positions(self) -> AsyncIterator[PositionEvent]:
        return self._iter_positions()

    async def _iter_positions(self) -> AsyncIterator[PositionEvent]:
        while True:
            yield await self._position_queue.get()

    # _reconnect_loop, _connect_once, _authenticate, _subscribe,
    # _ping_loop, _receive_until_closed, _on_frame, _gen_auth_frame
    # — internal methods.
```

## Hand verification

T-209 is largely I/O orchestration (WS lifecycle, frame demux). Math content limited to: HMAC-SHA256 signature (cryptographic, not financial), one millisecond-to-datetime division (`int(execTime)/1000`), and Decimal round-trip on price/qty/fee.

Math-validator (Gate 4) ACTIVE per CLAUDE.md gate-4 conditional rule (diff touches `packages/exchange/`). Expected verdict: **VERIFIED — H-015 string round-trip preservation; no Decimal→float silent casts; no new arithmetic outside Decimal sums or sub-aggregates**.

### §F.1 — Auth HMAC hand verification

Fixed inputs:
- `api_key = "test_key"` (string)
- `api_secret = "test_secret"` (string)
- `now_fn() = 1700000000.0` (float seconds since epoch)
- `_AUTH_EXPIRES_OFFSET_S = 5`

Computed:
- `expires = int((1700000000.0 + 5) * 1000) = 1700000005000`
- `payload = "GET/realtime1700000005000"`
- `signature = hmac.new(b"test_secret", b"GET/realtime1700000005000", hashlib.sha256).hexdigest()`

Hand computed (Python repl reference, NOT pinned in plan to avoid drift; test recomputes via `hmac.new` and asserts equality with frame.args[2]).

Frame: `{"op": "auth", "args": ["test_key", 1700000005000, signature]}`.

### §F.2 — Reconnect backoff hand verification

`rng = random.Random(0)`, sequence `_full_jitter_backoff_delays(rng)`:

| Attempt | Ceiling      | `rng.uniform(0, ceiling)` (seed 0) |
|--------:|-------------:|------------------------------------:|
|       0 |  1.0         | 0.844422 (≈0.84)                    |
|       1 |  2.0         | 1.518009 (≈1.52)                    |
|       2 |  4.0         | 1.665398 (≈1.67)                    |
|       3 |  8.0         | 3.404557 (≈3.40)                    |
|       4 | 16.0         | 6.785317 (≈6.79)                    |
|       5 | 32.0         | 13.04...                            |
|       6 | 60.0         | (clamped from here on)              |

Test asserts the first 5 delays of an injected `random.Random(0)` instance match `pytest.approx` to 0.001 tolerance against re-computed values from the same seeded sequence. This pins the formula `min(base*2**n, cap)` and the full-jitter `uniform(0, ceiling)` shape; exact decimals are verified via Python's deterministic Mersenne Twister (NOT against hand-computed values, which would be a floating-point-precision-quagmire).

## Lifecycle

`BybitV5PrivateWs` lifecycle:

1. **Construction** (sync): state = `DISCONNECTED`; queues empty; ws=None.
2. **`run()` first call**: state-guard (must be `DISCONNECTED`) → enter `_reconnect_loop`.
3. **`_reconnect_loop` iteration**:
   a. State = `CONNECTING` (or `RECONNECTING` if not first iteration).
   b. `async with ws_connect(ws_url, ping_interval=20, ping_timeout=10) as ws`.
   c. State = `CONNECTED`. Reset attempt counter, disconnect timer.
   d. Send auth frame. Recv response. If `success=false` → log + counter + `await ws.close()` → fall through to except branch (will reconnect).
   e. Send subscribe frame. (No ack-tracking; subsequent data frames confirm liveness.)
   f. Spawn `_ping_loop` task.
   g. Drive `_receive_until_closed(ws)`: async-iter over WS, JSON-decode, demux each frame.
   h. On WS exception or normal close: cancel ping task, set ws=None, fall through.
   i. On exception: compute disconnect elapsed; emit long-disconnect log if threshold crossed; sleep `next(delays)`; loop.
4. **`close()` (any time)**: state = `CLOSED`; suppress WS close exceptions; ws=None. `_reconnect_loop` exits via state check on next iteration. In-flight `_receive_until_closed` exits when ws closes.
5. **State after `close()`**: `executions()` / `positions()` async iterators continue to drain queues until empty (consumers must check shutdown signal externally OR the queues block forever — T-218 dispatcher will own its own shutdown logic).

## Edge cases

1. **Reconnect mid-auth**: WS exception during auth handshake (timeout, server disconnect) → catch in reconnect loop, treat as disconnected, backoff + retry. Auth re-handshake on next iteration per §11.2.

2. **Subscribe ack lost**: no ack tracking; subsequent data frames imply success. If subscribe silently fails, no events arrive and the WS appears idle; long-disconnect log fires after 60s of no frames OR timeout. Acceptable per §0.8 (anti-hypothetical).

3. **Pong reply absent**: native `ping_interval=20, ping_timeout=10` triggers transport-level disconnect after 30s of no pong. App-level pings (`{op:"ping"}`) don't have explicit pong-tracking; if Bybit fails to respond, transport-level pong-tracking catches it.

4. **Frame missing `topic`**: `msg.get("topic", "")` returns `""`; routes to neither queue; logged DEBUG. Non-data frames (auth ack, subscribe ack, pong) typically lack `topic` → harmlessly skipped.

5. **Frame with unknown topic**: same as #4 — skipped silently. Future Bybit topic additions don't crash the receive loop.

6. **`category`-less data item**: defensive — `item.get("category") != "linear"` is False if missing; skipped.

7. **Queue depth growth**: unbounded queue; T-218 dispatcher must drain. If dispatcher dies, queue grows until OOM. Per §0.8 anti-hypothetical, no explicit guard in T-209; observability via metric `bybit_private_ws_queue_depth` deferred to T-F2+.

8. **`run()` re-entry**: state guard raises `BybitWsStateError` (ws-local error class). Mirror BinanceWsClient.

9. **`close()` after `close()`**: idempotent no-op (state check).

10. **`run()` after `close()`**: state guard raises (state == `CLOSED`, not `DISCONNECTED`). Caller must construct a new `BybitV5PrivateWs` instance.

## Risks (post-resolution)

- **Risk: `websockets` library version drift** (e.g., `ws_connect` API changes between versions). Mitigation: pin same version as `packages.market` (already in workspace); pyproject explicit pin.
- **Risk: Bybit V5 auth payload format drifts** (e.g., they change `"GET/realtime"` prefix or expires units). Mitigation: T-222 testnet smoke (deferred) catches; production smoke at deploy. T-209 tests pin our format against Bybit V5 docs as of 2026-04.
- **Risk: queue depth grows unbounded under T-218 dispatcher slowness or crash** → OOM. Mitigation: T-F2+ task to add depth-observability + alerting hook; not in T-209 scope (per §0.8).
- **Risk: `time.fromtimestamp(int(execTime)/1000, tz=UTC)` precision loss for sub-millisecond `execTime`**: Bybit ms-precision is sufficient for trade-event ordering at scalper-v2 scope. F5+ may need ns-precision via `executed_at_ns` int field; tracked deferred.
- **Risk: app-level ping interval drift relative to Bybit's actual idle window** (Bybit could change from 20s to 30s without notice). Mitigation: ctor kwarg `ping_interval_s` operator-tunable per §N9.
- **Risk: `random.Random(0)` deterministic sequence varies across Python minor versions** (Mersenne Twister output is documented stable). Mitigation: pinned Python `>=3.12` per workspace; tests use `pytest.approx(0.001)` tolerance against re-computed values from same seeded `random.Random(0)`, NOT hard-coded floats.
