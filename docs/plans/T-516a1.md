## Architecture

**Mirror live `/api/trades/*` stack 1:1**. paper_trades schema (migration 0008) is structurally identical to trades schema (migration 0005) — same column names + types + nullability + semantic. The only differences:

| Aspect | trades | paper_trades |
|---|---|---|
| Source FK | `signal_id → signals.id` direct | `signal_id` nullable BIGINT (no FK; mirror live convention) |
| Open FK | `open_order_id → orders.id` | `open_order_id → paper_orders.id` |
| close_order_id | plain BIGINT (no FK; T-202 W#4) | plain BIGINT (no FK; T-202 W#4 mirror) |
| Volume | live trading flow | paper simulation flow |

T-516a1 backend is purely a read-side analytics view; FK targets are irrelevant to query helpers (we don't traverse FKs). Mirror live's signature byte-for-byte.

**Endpoint shape (mirror `/api/trades/`)**:
```
GET /api/paper-trades/?bot_id=X&symbol=Y&status=Z&from=ISO&to=ISO&limit=50&offset=0
GET /api/paper-trades/{paper_trade_id}
```

Response envelope mirrors live (`paper_trades: list[...]` instead of `trades: list[...]` for the list endpoint).
