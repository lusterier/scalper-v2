## Architecture

**Replay flow (per-row)**:

```
1. Decode meta from row.meta JSONB (T-513a shipped:
   {"virtual_entry_price": str, "sl_pct": str, "tp_pct": str, "be_trigger": str, "be_sl_level": str}).
2. Compute remaining timer: created_at + observation_minutes (Decimal-clean wall-clock carry-over).
3. WG-equiv #5: replay window cap pre-check (settings.shadow_rejected_replay_query_window_max_hours).
   If window exceeded → finalize SHUTDOWN_MID_REPLAY + warn-log + return.
4. WG-equiv #3: timer carry-over pre-check (now >= expires_at).
   If elapsed → finalize NO_TRIGGER (or WOULD_BE if be_triggered sticky from prior session,
   but be_triggered is observation-state-only — not persisted to DB; default NO_TRIGGER) + return.
   NOTE: T-513b1 cannot recover prior be_triggered sticky state across restart since
   T-513a does not persist it. Plan §"BE-trigger restart deficiency" addresses this.
5. virtual_entry == 0 defensive early-return (T-513a precedent): finalize NO_TRIGGER + return
   without subscribing.
6. Compute thresholds via T-513a `_compute_thresholds(side, entry, sl_pct, tp_pct, be_trigger)`.
7. Construct obs_state dict (best=entry, worst=entry, be_triggered=False, outcome=None).
8. Construct candle handler via T-513a `_make_observation_candle_handler` factory.
9. Iterate ohlc_1m cursor [created_at, now()] per candle invoke handler.
   After each candle: check terminal_future.done().
10. Detect terminal via terminal_future.done() post-each-candle (SAME pattern as T-512a;
    handler sets terminal_future on threshold cross).
11. Per-task compute timeout wraps replay loop (settings.shadow_rejected_replay_per_observation_timeout_seconds).
12. Terminal during replay → finalize with classified outcome + computed MFE/MAE pcts via
    T-513a `_compute_mfe_mae_pcts(side, entry, best, worst)`.
13. Replay window cap exceeded OR per-task timeout fired → finalize SHUTDOWN_MID_REPLAY.
14. Else (window exhausted no terminal) → spawn live continuation:
    - subscribe `market.ohlc.1m.<symbol>` with same handler (continues threshold check)
    - asyncio.wait_for(terminal_future, remaining_timeout)
    - on terminal: finalize with classified outcome + MFE/MAE
    - on TimeoutError: finalize WOULD_BE (if be_triggered) or NO_TRIGGER
    - try/finally: bus_unsubscribe
    Register the live continuation task via shadow_rejected_worker.register_resume_task(rejected_id, task).
```

**BE-trigger restart deficiency (acknowledged limitation)**:
T-513a `obs_state["be_triggered"]` is in-memory only (not persisted to `shadow_rejected.meta`). On restart, replay starts with `be_triggered=False` regardless of whether the prior session crossed BE threshold. The restart replay re-iterates all OHLC candles from `created_at`, so if a prior candle crossed BE and a later candle did not retract, replay will re-detect BE during iteration (sticky-flag re-asserted). Edge case: if BE was crossed but then SL also fired in same prior session pre-restart, only the first terminal (SL via SL-first conservative bias) is reported — same outcome regardless of restart timing. Edge case: if BE was crossed and signal-window expired without SL/TP, replay correctly classifies WOULD_BE. **No data loss across restart for this stickiness invariant** because OHLC replay is deterministic per BRIEF §13.7 invariant.

## Hand verification

Per L-014 active control + math-validator gate. T-513b1 reuses T-513a shipped math (`_compute_thresholds`, `_compute_mfe_mae_pcts`); replay infrastructure does not introduce new financial math. Hand verification covers:

**Wall-clock carry-over math (mirror T-512a §Hand verification)**:

```
created_at = now - 130 minutes (synthetic restart-after-130-min scenario)
observation_minutes = 60 (Settings default)
expires_at = created_at + timedelta(minutes=60)
remaining_seconds = (expires_at - now).total_seconds()
                  = (created_at + 60min - now).total_seconds()
                  = (-70min).total_seconds()
                  = -4200 seconds

Since remaining_seconds < 0, replay returns NO_TRIGGER terminal immediately
(timer carry-over pre-check at step 4).
```

**Threshold computation (T-513a verbatim, no new math)**:

```
side = "buy" (LONG signal rejected)
entry = 65000 (Decimal)
sl_pct = 0.005, tp_pct = 0.01, be_trigger = 0.005

tp_threshold = 65000 * (1 + 0.01) = 65650.000 (Decimal-exact)
sl_threshold = 65000 * (1 - 0.005) = 64675.000 (Decimal-exact)
be_threshold = 65000 * (1 + 0.005) = 65325.000 (Decimal-exact)
```

These match T-513a unit test fixtures verbatim. T-513b1 replay invokes these via the
imported `_compute_thresholds` function — no math drift possible.

**MFE/MAE computation (T-513a verbatim)**:

```
side = "buy"
entry = 65000
best = 65500 (set by replay during candle iteration)
worst = 64900

mfe_pct = (best - entry) / entry = 500 / 65000 ≈ 0.00769 (float per DOUBLE PRECISION DB column)
mae_pct = (entry - worst) / entry = 100 / 65000 ≈ 0.00154
```

T-513b1 invokes `_compute_mfe_mae_pcts` from T-513a; no math drift possible.
