## Public interface

### Migration 0011 (`audit_events` hypertable)

```sql
CREATE TABLE audit_events (
    id              BIGSERIAL,
    occurred_at     TIMESTAMPTZ NOT NULL,
    actor           TEXT NOT NULL,
    action          TEXT NOT NULL,
    entity_type     TEXT NOT NULL,
    entity_id       TEXT NOT NULL,
    before_state    JSONB,
    after_state     JSONB,
    correlation_id  TEXT,
    meta            JSONB NOT NULL DEFAULT '{}'::jsonb,
    PRIMARY KEY (occurred_at, id)
);
SELECT create_hypertable('audit_events', 'occurred_at', chunk_time_interval => interval '30 days');
CREATE INDEX ae_entity ON audit_events (entity_type, entity_id, occurred_at DESC);
```

Verbatim per BRIEF §7.2:1108-1126. Mirror migration 0010 (scoring_evaluations) shape:
- 10 columns; `before_state` + `after_state` + `correlation_id` nullable; `meta` server default `'{}'::jsonb`.
- Composite PK `(occurred_at, id)` per TimescaleDB hypertable convention.
- Hypertable on `occurred_at` with **30-day chunks** per §7.2:1124 (note: not 7-day like signals/features; audit events are sparser).
- Index `ae_entity (entity_type, entity_id, occurred_at DESC)` for T-405 reader (typical query: "show all events for entity_type='symbol_map' entity_id='BTCUSDT.P' last N days").
- No FK on any column (audit-stream stands alone — mirror `trade_pnl_deltas` + `scoring_evaluations`).
- No UNIQUE constraint at DB layer (idempotency is application concern; @non_idempotent helper).

### `packages/db/queries/audit.py` (NEW module)

```python
@dataclass(frozen=True, slots=True)
class AuditEventRow:
    """Read projection of audit_events row (consumed by T-405 viewer)."""
    id: int
    occurred_at: datetime
    actor: str
    action: str
    entity_type: str
    entity_id: str
    before_state: dict[str, Any] | None
    after_state: dict[str, Any] | None
    correlation_id: str | None
    meta: dict[str, Any]


@non_idempotent
async def insert_audit_event(
    conn: _DbExecutor,
    *,
    occurred_at: datetime,           # caller passes now_fn() result
    actor: str,                       # 'lan:<source_ip>' per §16.8 dashboard write origin
    action: str,                      # 'symbol_map.create' / etc — caller-defined namespace
    entity_type: str,                 # 'symbol_map' / 'bot' / 'config' — caller-defined
    entity_id: str,                   # PK string of the mutated entity
    before_state: dict[str, Any] | None,    # JSONB; None for create
    after_state: dict[str, Any] | None,     # JSONB; None for delete
    correlation_id: str | None,             # request-scoped trace id
) -> int:
    """INSERT one row into audit_events; return generated id.

    @non_idempotent per §N3 — caller MUST not retry on failure. Caller
    invokes this INSIDE the same conn.transaction() as the business
    mutation; tx rollback on audit failure rolls back the business
    write too (atomic). T-401b is the first consumer (symbol_map CRUD).
    """
```

`json.dumps` + `::jsonb` cast for before_state/after_state (default asyncpg codec doesn't auto-convert dicts — same pattern as signal_gateway `insert_signal`).

### `services/analytics_api/app/routers/bots.py` (NEW)

```python
router = APIRouter(prefix="/api/bots", tags=["bots"])

@router.get("/", response_model=BotListResponse)
async def list_bots(pool: Annotated[asyncpg.Pool, Depends(get_pool)]) -> BotListResponse:
    async with pool.acquire() as conn:
        rows = await select_all_bots(conn)
    return BotListResponse(
        bots=[BotResponse(**asdict(r)) for r in rows]
    )

@router.get("/{bot_id}", response_model=BotResponse)
async def get_bot(bot_id: str, pool: Annotated[asyncpg.Pool, Depends(get_pool)]) -> BotResponse:
    async with pool.acquire() as conn:
        row = await select_bot_by_id(conn, bot_id)
    if row is None:
        raise HTTPException(status_code=404, detail=f"bot {bot_id!r} not found")
    return BotResponse(**asdict(row))
```
