## Migration shape (committed)

```python
"""features hypertable.

Revision ID: 0004
Revises: 0003
Create Date: 2026-04-26

Fourth migration for scalper-v2 (brief §7.2 ``features`` DDL,
§18.3 retention/compression, §19 F1 bullet 4, §9.3 line 1494
ON CONFLICT contract).

[full module docstring per the test_0003 precedent — table shape +
lifecycle policies + nullability + GIN deferral + compression orderby
justification + downgrade contract]
"""

from __future__ import annotations

import sqlalchemy as sa
from alembic import op

revision: str = "0004"
down_revision: str | None = "0003"
branch_labels: tuple[str, ...] | None = None
depends_on: tuple[str, ...] | None = None


def upgrade() -> None:
    op.create_table(
        "features",
        sa.Column("feature_name", sa.Text(), nullable=False),
        sa.Column("symbol", sa.Text(), nullable=False),
        sa.Column("computed_at", sa.TIMESTAMP(timezone=True), nullable=False),
        # sa.Double() compiles to PG "DOUBLE PRECISION" (8 bytes, 53-bit
        # mantissa), matching §7.2 line 907 verbatim and the wire-side
        # `float` of §8.4 FeatureUpdate. sa.Float() would compile to
        # "FLOAT" / "real" (4 bytes, 24-bit mantissa) — silent data-loss.
        sa.Column("value_num", sa.Double(), nullable=True),
        sa.Column("value_bool", sa.Boolean(), nullable=True),
        sa.Column("value_json", sa.dialects.postgresql.JSONB(), nullable=True),
        sa.Column("source_version", sa.Text(), nullable=False),
        sa.PrimaryKeyConstraint(
            "feature_name", "symbol", "computed_at", "source_version",
            name="features_pkey",
        ),
    )

    op.execute(
        "SELECT create_hypertable("
        "'features', 'computed_at', "
        "chunk_time_interval => interval '7 days'"
        ")"
    )

    op.execute(
        "CREATE INDEX features_latest ON features "
        "(feature_name, symbol, computed_at DESC)"
    )

    op.execute(
        "ALTER TABLE features SET ("
        "timescaledb.compress, "
        "timescaledb.compress_segmentby = 'feature_name, symbol', "
        "timescaledb.compress_orderby = 'computed_at DESC'"
        ")"
    )
    op.execute("SELECT add_compression_policy('features', INTERVAL '30 days')")
    op.execute("SELECT add_retention_policy('features', INTERVAL '180 days')")


def downgrade() -> None:
    # Symmetric-stub downgrade per §N8 forward-only policy. Drop policies
    # before the table so bgw_job entries do not leak when the
    # underlying hypertable disappears.
    op.execute("SELECT remove_retention_policy('features', if_exists => true)")
    op.execute("SELECT remove_compression_policy('features', if_exists => true)")
    op.drop_table("features")
```

## Decisions committed (operator: "use defaults")

1. **`value_num` storage type** = DOUBLE PRECISION (verbatim §7.2 line 907) implemented via `sa.Double()` (compiles to PG `DOUBLE PRECISION`, 8 bytes / 53-bit mantissa). `sa.Float()` would silently downgrade to PG `real` (4 bytes / 24-bit mantissa) and trip the test_0004 column-shape assertion.
2. **No DB-side CHECK constraint** on exactly-one-of value_num/value_bool/value_json — `FeatureValue.__post_init__` (T-106) enforces app-layer invariant; T-110 is the only writer and is pre-validated.
3. **No GIN index on `value_json`** — §7.2 line 914 specifies only `features_latest` btree; F1+ entry queued for F4+ analytics-driven JSONB queries.
4. **`compress_orderby = 'computed_at DESC'`** — mirrors T-103 precedent for `ohlc_1m`, matches natural "latest first" query pattern. Brief §18.3 line 1182 specifies only `compress_segmentby`; orderby is an opt-in optimisation.
5. **F1+ entry: GIN deferral** — added to `### F1+ opportunistic` in TASKS.md during close-out.
6. **Migration test value_* fixture** — INSERT 3 rows (one per variant) with asyncpg type round-trip assertion; covers storage shape without depending on app-layer FeatureValue.
7. **asyncpg JSONB codec** registered explicitly in the end-to-end smoke test — `set_type_codec("jsonb", encoder=json.dumps, decoder=json.loads, schema="pg_catalog")` so round-trip yields Python dict not JSON string. No global codec exists in conftests; per-connection registration is the right scope.
