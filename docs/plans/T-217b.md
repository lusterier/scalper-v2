## Scope

T-217b extends T-217a's `run_position_monitor_for_trade` body with BE trigger + trail SL adjustment logic per §9.5:1589-1590:

1. **BE trigger** — read `select_trade_fsm_params(trade_id)` per tick; if `position_state.sl_type == 'protective'` AND favorable move ≥ `be_trigger` → `adapter.set_trading_stop(tpsl_mode='Full', sl_price=entry ± be_sl_level)` per H-013 → UPDATE `position_state.sl_price` + `sl_type='be'`. Fires exactly once (next tick reads sl_type='be' → does not re-trigger).
2. **Trail SL update** — if `position_state.sl_type == 'trail'` (T-218b promotes on partial_tp per ADR-0005) AND `best_price` moved favorably since last tick → new SL = `best_price ∓ trail_pct` → `adapter.set_trading_stop(tpsl_mode='Full', sl_price=new_trail)` → UPDATE `position_state.sl_price`. No-op if best_price unchanged.
3. **Failure handling** — `set_trading_stop` exception → log ERROR `execution.lifecycle_be_set_failed` or `_trail_set_failed` + continue loop. Adapter's 3× retry handles transient failures; monitor never crashes on adapter exception (defensive vs. T-216b2 emergency_close which is initial-setup only).

**Out of scope (T-217a-shipped)**: per-tick MFE/MAE/best_price/running_pnl computation; spawn/cancel infrastructure; lifespan integration; Settings.

**Out of scope (T-219)**: close flow; cumulative-delta P&L.

## Decisions (operator-approved 2026-05-02 "use defaults")

- **OQ-A** ✅ — BE-trigger percentage-of-entry: long `(current - entry) / entry >= be_trigger`; new SL = `entry * (1 + be_sl_level)`. Short symmetric. Fires once via `sl_type == 'protective'` guard.
- **OQ-B** ✅ — Trail SL percentage-of-best: long `best_price * (1 - trail_pct)`; short `best_price * (1 + trail_pct)`. Update only on best_price movement (anti-spam).
- **OQ-C** ✅ — NEW query helper `update_position_state_sl(bot_id, symbol, sl_price, sl_type, updated_at)` composite-PK UPDATE; column-disjoint from fill-flow + monitor-tick UPDATEs.
- **OQ-D** ✅ — Failure handling: log ERROR + continue loop. Adapter 3× retry handles transient; monitor task never crashes on `set_trading_stop` exception (no emergency_close from runtime path; that's T-216b2 initial-setup only).

## NEW `packages/db/queries/execution.py` `update_position_state_sl` (+~30 LOC)

```python
async def update_position_state_sl(  # noqa: PLR0913
    conn: _DbExecutor,
    *,
    bot_id: str,
    symbol: str,
    sl_price: Decimal,
    sl_type: Literal["protective", "be", "trail"],
    updated_at: datetime,
) -> None:
    """Composite-PK UPDATE for SL-adjustment paths (T-217b BE / trail).

    H-018-symmetric: composite PK ``(bot_id, symbol)`` only. Column-disjoint
    from :func:`update_position_state_after_fill` (writes ``remaining_qty``)
    and :func:`update_position_state_monitor_tick` (writes monitor-only fields)
    — distinct writer for SL-adjustment flow.
    """
    await conn.execute(
        """
        UPDATE position_state
        SET sl_price = $1, sl_type = $2, updated_at = $3
        WHERE bot_id = $4 AND symbol = $5
        """,
        sl_price,
        sl_type,
        updated_at,
        bot_id,
        symbol,
    )
```

## MODIFIED `services/execution/app/lifecycle.py` (+~110 LOC)

Three new module-private helpers + caller block in `run_position_monitor_for_trade`:

```python
def _check_be_trigger(
    side: str,
    current_price: Decimal,
    entry_price: Decimal,
    be_trigger: Decimal,
) -> bool:
    """Return True iff favorable move >= be_trigger.

    Long: (current - entry) / entry >= be_trigger.
    Short: (entry - current) / entry >= be_trigger.
    """
    if side == "buy":
        return (current_price - entry_price) / entry_price >= be_trigger
    return (entry_price - current_price) / entry_price >= be_trigger


def _compute_be_sl_price(
    side: str,
    entry_price: Decimal,
    be_sl_level: Decimal,
) -> Decimal:
    """Long: entry * (1 + be_sl_level); Short: entry * (1 - be_sl_level)."""
    if side == "buy":
        return entry_price * (Decimal("1") + be_sl_level)
    return entry_price * (Decimal("1") - be_sl_level)


def _compute_trail_sl_price(
    side: str,
    best_price: Decimal,
    trail_pct: Decimal,
) -> Decimal:
    """Long: best * (1 - trail_pct); Short: best * (1 + trail_pct)."""
    if side == "buy":
        return best_price * (Decimal("1") - trail_pct)
    return best_price * (Decimal("1") + trail_pct)
```

## Hand verification (Gate 4 math-validator critical scope)

T-217b introduces Decimal arithmetic in 3 helpers + branch comparisons. Hand-computable fixtures:

**Fixture A: BE trigger long, on the threshold**
- entry=`Decimal("100")`, current=`Decimal("100.5")`, be_trigger=`Decimal("0.005")`, be_sl_level=`Decimal("0.003")`, side='buy'
- `(100.5 - 100) / 100 = 0.500 / 100 = 0.005`. `0.005 >= 0.005` → True (boundary).
- new SL = `100 * (1 + 0.003) = 100 * 1.003 = 100.300` (NUMERIC(30,12) preserves precision).

**Fixture B: BE trigger long, below threshold**
- entry=`Decimal("100")`, current=`Decimal("100.4")`, be_trigger=`Decimal("0.005")`, side='buy'
- `(100.4 - 100) / 100 = 0.4 / 100 = 0.004`. `0.004 >= 0.005` → False (no trigger).

**Fixture C: BE trigger short, on the threshold**
- entry=`Decimal("100")`, current=`Decimal("99.5")`, be_trigger=`Decimal("0.005")`, be_sl_level=`Decimal("0.003")`, side='sell'
- `(100 - 99.5) / 100 = 0.5 / 100 = 0.005`. True → trigger.
- new SL = `100 * (1 - 0.003) = 100 * 0.997 = 99.700`.

**Fixture D: Trail SL long, best moved up**
- best_price=`Decimal("110")`, trail_pct=`Decimal("0.005")`, side='buy'
- new trail SL = `110 * (1 - 0.005) = 110 * 0.995 = 109.450`.

**Fixture E: Trail SL short, best moved down**
- best_price=`Decimal("90")`, trail_pct=`Decimal("0.005")`, side='sell'
- new trail SL = `90 * (1 + 0.005) = 90 * 1.005 = 90.450`.

**Fixture F: Trail SL no-op (best unchanged)**
- ps.best_price=`Decimal("110")`, new_best=`Decimal("110")` (same), sl_type='trail' → branch is `new_best != ps.best_price` is False → no `set_trading_stop` call → no UPDATE.

**Decimal precision invariants**:
- All Decimal × Decimal → Decimal (exact); no float coercion; no `getcontext().prec` mutations.
- `Decimal("1") + be_sl_level` and `Decimal("1") - trail_pct` use string-constructed Decimal literals.
- Division `(current - entry) / entry` is Decimal/Decimal → Decimal (default precision 28 digits; sufficient for percentage comparison).
- `position_state.sl_price` is NUMERIC(30,12) per migration 0006 — store SL prices to 12-dp precision; new BE SL `100 * 1.003 = 100.300000000000` fits column scale exactly.

## Lifecycle

**Pre-T-217b master**: T-217a monitor reads price + computes monitor metrics + UPDATEs monitor fields per tick. SL-adjustment branches absent — sl_type stays `'protective'` until T-218b promotes to `'trail'` on partial_tp; SL never moves to BE; trail SL never updates from best_price movement.

**Post-T-217b master**: monitor body extended with BE branch (fires once on favorable move ≥ be_trigger when sl_type='protective') + trail branch (updates SL from best_price ± trail_pct on movement when sl_type='trail'). `set_trading_stop` calls always use H-013 explicit `tpsl_mode='Full'`.

**Unblocks T-219** (close flow can now trust that sl_type transitions are deterministic).

## Edge cases

- **`fsm_params` returns None mid-loop** (trade row deleted by future T-219 atomic close): defensive `continue` — next tick re-reads ps which would also be None and triggers monitor self-exit via T-217a-shipped path.
- **`set_trading_stop` raises**: log ERROR with named key + continue loop. Monitor task does NOT crash. Adapter has 3× retry per H-013; if exhausted, next tick retries from monitor.
- **best_price unchanged** (no movement since last tick): trail branch is a no-op; no `set_trading_stop` call; no UPDATE.
- **BE trigger on exact boundary** (`favorable == be_trigger`): triggers (>= operator). Decimal exact comparison.
- **sl_type='be' on tick after BE fire**: BE branch is `sl_type == 'protective'` only → does NOT re-fire. UPDATE persists `sl_type='be'` so subsequent reads see new state.
- **sl_type='trail' transitions back to 'be'**: not possible per current state machine (protective → be OR protective → trail; trail does not regress). T-217b body assumes monotonic state.
- **Race with T-218b dispatcher writes** (sl_type promotion to 'trail' on partial_tp): monitor reads ps with sl_type='trail' → trail branch fires; column-disjoint UPDATEs are MVCC-safe.

## Write-time guidance (for plan-reviewer carry-forward)

1. **`Decimal("1")` literal in formulas**: never bare `1` (Python int × Decimal works but explicit Decimal literal is consistent with §5.3 precision invariant + L-008 active control).
2. **Adapter ctor kwarg**: `run_position_monitor_for_trade` signature extended with `adapter: ExchangeClient`; T-217a's `placement.py` spawn site threads `adapter` from closure (already in scope per `make_per_bot_handler` ctor).
3. **`set_trading_stop` H-013 explicit kwarg**: every call site uses `tpsl_mode="Full"` literal (Literal["Full","Partial"] narrowing). Brief-reviewer at SHIP gate verifies grep for `set_trading_stop` calls in lifecycle.py emits only `tpsl_mode="Full"`.
4. **try/except `Exception` around `set_trading_stop`**: catches all adapter errors (AuthError, OrderRejected, NetworkTimeout, RateLimitError, UnknownState — same family as T-216b2). Rationale: monitor must NEVER crash from runtime adapter failure; emergency_close is initial-setup-only path.
5. **`else` clause on try/except**: UPDATE only runs on `set_trading_stop` SUCCESS. If adapter raised, no UPDATE → next tick will retry the BE/trail flow (idempotent at adapter level).
6. **Trail branch best_price movement guard**: `ps.best_price is not None AND new_best != ps.best_price`. The None check is defensive; first tick post-T-218b promotion to 'trail' will have `ps.best_price == None` initially → no trail update yet (waiting for T-217a monitor's next tick to seed best_price). Avoids spurious set_trading_stop on the very first 'trail' tick.
7. **BE/trail are mutually exclusive `if/elif`** because sl_type is a single value at any moment. BE fires when sl_type='protective'; trail fires when sl_type='trail'; sl_type='be' has neither branch fire.
8. **`select_trade_fsm_params` per-tick query cost**: 1 extra SELECT per tick on `trades.meta` JSONB. At 1Hz × N trades, this is bounded; sub-ms PG read on PK lookup. No caching needed at F2 scope.
9. **Decimal precision for division** in `_check_be_trigger`: `(current - entry) / entry` produces a Decimal at default precision (28 digits). For `be_trigger` percentage comparison this is sufficient — the bot config typical `be_trigger=Decimal("0.005")` is 3 dp; division result e.g. `Decimal("0.500") / Decimal("100") = Decimal("0.005")` exact at 3 dp. Hand verification fixtures use exact-comparable values.
10. **Test fixture `set_trading_stop` mock**: AsyncMock on adapter; assert called with explicit `tpsl_mode='Full'` + computed sl_price; verify `update_position_state_sl` called with matching sl_price + sl_type per branch.
11. **No new query helper for FSM params**: `select_trade_fsm_params` was T-217a-shipped; T-217b is its first runtime consumer. Verify codec round-trip works in actual runtime path (testcontainer integration optional but L-008 says non-trivial SQL with JSONB read should have integration test — `select_trade_fsm_params` reads simple JSONB column without compute, mock-only test sufficient per existing pattern).
12. **No-op trail when first 'trail' tick has best_price=None**: documented in WG#6; T-217a monitor seeds best_price from first successful tick post-T-218b promotion. Trail branch waits for second tick onward.
