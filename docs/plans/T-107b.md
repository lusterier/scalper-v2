### VwapFeature
```python
class VwapFeature:
    """Cumulative session VWAP — daily session (UTC midnight reset).

    typical_price_i = (high_i + low_i + close_i) / 3
    VWAP = sum(typical_i * volume_i) / sum(volume_i)
        over candles within the latest session.

    The engine (T-110) supplies an arbitrarily-wide window; this
    indicator filters to candles whose ``bucket_start.date()`` matches
    the latest candle's UTC date (single session). Older candles are
    silently dropped.

    Zero-volume edge: if ``sum(volume_i) == 0`` over the filtered
    session, raise :class:`FeatureUnderflowError` — VWAP is undefined
    on zero-volume input. The engine sees this as "no VWAP for this
    bar" rather than a silent fallback that would drift the published
    value into nonsense.
    """

    def __init__(self, session: str = "daily", interval: str = "1m") -> None:
        if session != "daily":
            raise ValueError(
                f"session must be 'daily', got {session!r} "
                "(weekly/hourly are F1+)"
            )
        self.session = session
        self.interval = interval
        # Static "_session" suffix per §B.2 line 3079 verbatim spec
        # ("ind.{symbol}.1m.vwap_session"). Session enum stays as a
        # constructor parameter for forward-compat (hourly/weekly when
        # F1+ unlocks them) but is NOT part of the name identifier —
        # the registry layer (T-111) carries `params: {session: ...}`
        # alongside the name in the YAML config.
        self.name_template = "ind.{symbol}.{interval}.vwap_session"
        self.source_version = "builtin.vwap.v1"
        self.warmup_candles = 1

    def compute(self, candles: Sequence[OhlcCandle]) -> FeatureValue:
        if len(candles) < self.warmup_candles:
            raise FeatureUnderflowError(
                f"VWAP requires >= {self.warmup_candles} candles, got {len(candles)}"
            )
        latest_date = candles[-1].bucket_start.date()
        session_candles = [c for c in candles if c.bucket_start.date() == latest_date]
        weighted = Decimal(0)
        total_volume = Decimal(0)
        for c in session_candles:
            typical = (c.high + c.low + c.close) / Decimal(3)
            weighted += typical * c.volume
            total_volume += c.volume
        if total_volume == 0:
            raise FeatureUnderflowError(
                f"VWAP undefined on zero-volume session "
                f"({len(session_candles)} candles, sum(volume)=0)"
            )
        return FeatureValue(value_num=weighted / total_volume)
```

### BollingerFeature
```python
class BollingerFeature:
    """Bollinger Bands — SMA middle ± k·σ_pop bands.

    middle = SMA(close, period)
    σ_pop  = sqrt(sum((close_i - middle)^2 for i) / period)   # population, /n
    upper  = middle + std_dev_multiplier * σ_pop
    lower  = middle - std_dev_multiplier * σ_pop

    Returns ``FeatureValue(value_json={"upper", "middle", "lower"})``
    with Decimal sub-keys; T-110 converts to float at the wire seam.

    **Hashability:** the returned :class:`FeatureValue` is the
    ``value_json`` variant, which per T-106 contract is **not**
    hashable (Mapping/dict instances are not). Callers requiring
    set/dict membership must normalise first.
    """

    def __init__(
        self,
        period: int = 20,
        std_dev_multiplier: Decimal = Decimal("2"),
        interval: str = "15m",
    ) -> None:
        if period < 2:
            raise ValueError(f"period must be >= 2, got {period}")
        if std_dev_multiplier <= 0:
            raise ValueError(
                f"std_dev_multiplier must be > 0, got {std_dev_multiplier}"
            )
        self.period = period
        self.std_dev_multiplier = std_dev_multiplier
        self.interval = interval
        # Stringify multiplier for name_template (Decimal("2") → "2", not "2E+0")
        mult_str = format(std_dev_multiplier.normalize(), "f")
        self.name_template = (
            f"ind.{{symbol}}.{{interval}}.bollinger_{period}_{mult_str}"
        )
        self.source_version = "builtin.bollinger.v1"
        self.warmup_candles = period

    def compute(self, candles: Sequence[OhlcCandle]) -> FeatureValue:
        if len(candles) < self.warmup_candles:
            raise FeatureUnderflowError(
                f"Bollinger(period={self.period}) requires >= {self.warmup_candles} "
                f"candles, got {len(candles)}"
            )
        closes = [c.close for c in candles[-self.period :]]
        period_dec = Decimal(self.period)
        middle = sum(closes, Decimal(0)) / period_dec
        variance = sum((c - middle) ** 2 for c in closes) / period_dec
        sigma = variance.sqrt()
        offset = self.std_dev_multiplier * sigma
        return FeatureValue(value_json={
            "upper": middle + offset,
            "middle": middle,
            "lower": middle - offset,
        })
```

### MacdFeature + helper
```python
def _sma_seeded_ema_series(values: Sequence[Decimal], period: int) -> list[Decimal]:
    """SMA-seeded EMA series matching :class:`EmaFeature` semantics.

    Returns the EMA value at each bar from index ``period - 1`` onwards
    (inclusive). Output length is ``len(values) - period + 1``.

    **Index alignment** (load-bearing for :class:`MacdFeature`): output
    index ``i`` corresponds to input bar index ``i + period - 1``. So
    ``_sma_seeded_ema_series(closes, 26)`` produces values for bars
    25, 26, 27, …; ``_sma_seeded_ema_series(closes, 12)`` produces
    values for bars 11, 12, 13, …. Two series with different periods
    can be aligned to the larger period's first-bar index by dropping
    the leading ``slow - fast`` values from the shorter-period series.

    Used by :class:`MacdFeature` (which needs the SERIES of fast/slow
    EMAs to derive the MACD-line series). A single-Decimal convenience
    equivalent to :class:`EmaFeature`'s output is just
    ``_sma_seeded_ema_series(values, period)[-1]`` — pinned by the
    test_builtins_macd golden cross-check.
    """
    if len(values) < period:
        raise ValueError(
            f"_sma_seeded_ema_series requires >= {period} values, got {len(values)}"
        )
    period_dec = Decimal(period)
    sma_seed = sum(values[:period], Decimal(0)) / period_dec
    alpha = Decimal(2) / (period_dec + 1)
    one_minus_alpha = Decimal(1) - alpha
    series = [sma_seed]
    for v in values[period:]:
        series.append(v * alpha + series[-1] * one_minus_alpha)
    return series


class MacdFeature:
    """MACD — fast/slow EMA crossover indicator with signal line.

    macd_line[i]   = fast_ema[i] - slow_ema[i]   (aligned at slow's first-bar index)
    signal_line[j] = SMA-seeded EMA of macd_line series, period = signal
    histogram[j]   = macd_line[j] - signal_line[j]

    Both fast_ema and slow_ema use :func:`_sma_seeded_ema_series`
    (same algorithm as :class:`EmaFeature`), aligned at bar index
    ``slow - 1`` so they share the same time axis.

    Warmup math (fast=12, slow=26, signal=9 default):
      bars 0..25  (26 closes) → first slow_ema at bar 25 (SMA seed)
      bars 0..25  → fast_ema series of length 15 (bars 11..25)
      bar 25       → first MACD-line value
      bars 26..33 → 8 more MACD-line values (total 9)
      bar 33       → first signal_line (SMA seed of 9 MACD-line values)
                    → first complete (macd, signal, histogram) triple
      Required: 34 closes = slow + signal - 1 = 26 + 9 - 1.

    Returns ``FeatureValue(value_json={"macd", "signal", "histogram"})``
    with Decimal sub-keys; T-110 converts to float at the wire seam.

    **Hashability:** as with :class:`BollingerFeature`, the returned
    :class:`FeatureValue` is the ``value_json`` variant — not hashable
    per T-106 contract.
    """

    def __init__(
        self,
        fast: int = 12,
        slow: int = 26,
        signal: int = 9,
        interval: str = "15m",
    ) -> None:
        if fast < 1 or slow < 1 or signal < 1:
            raise ValueError(
                f"fast/slow/signal must be >= 1, got fast={fast} slow={slow} signal={signal}"
            )
        if fast >= slow:
            raise ValueError(
                f"fast must be < slow, got fast={fast} slow={slow}"
            )
        self.fast = fast
        self.slow = slow
        self.signal = signal
        self.interval = interval
        self.name_template = (
            f"ind.{{symbol}}.{{interval}}.macd_{fast}_{slow}_{signal}"
        )
        self.source_version = "builtin.macd.v1"
        self.warmup_candles = slow + signal - 1

    def compute(self, candles: Sequence[OhlcCandle]) -> FeatureValue:
        if len(candles) < self.warmup_candles:
            raise FeatureUnderflowError(
                f"MACD(fast={self.fast},slow={self.slow},signal={self.signal}) "
                f"requires >= {self.warmup_candles} candles, got {len(candles)}"
            )
        closes = [c.close for c in candles]
        slow_ema = _sma_seeded_ema_series(closes, self.slow)
        fast_ema_full = _sma_seeded_ema_series(closes, self.fast)
        # _sma_seeded_ema_series(closes, slow) starts at bar (slow - 1);
        # _sma_seeded_ema_series(closes, fast) starts at bar (fast - 1).
        # Drop the first (slow - fast) fast_ema values to share slow_ema's
        # first-bar index. After this slice, fast_ema[i] and slow_ema[i]
        # are at the same bar.
        fast_ema = fast_ema_full[self.slow - self.fast :]
        macd_line = [f - s for f, s in zip(fast_ema, slow_ema, strict=True)]
        signal_line_series = _sma_seeded_ema_series(macd_line, self.signal)
        return FeatureValue(value_json={
            "macd": macd_line[-1],
            "signal": signal_line_series[-1],
            "histogram": macd_line[-1] - signal_line_series[-1],
        })
```

## Decisions committed (operator-approved)

Operator decisions 1-10 from the T-107a open-questions list apply unchanged. T-107b-specific decisions:

1. **VWAP session boundary** = UTC date equality on `bucket_start.date()` for the latest candle; older candles silently filtered out.
2. **VWAP zero-volume** = raise `FeatureUnderflowError` (degenerate input class, fail-loud per §0.4).
3. **VWAP warmup_candles** = 1 (minimum: VWAP is technically defined from the first session candle).
4. **VWAP across session boundaries** = use only the latest (current) UTC-date session.
5. **Bollinger std_dev_multiplier type** = `Decimal` (consistency with Decimal-throughout).
6. **Bollinger output sub-key precision** = Decimal in `value_json` mapping (T-110 converts to float at wire seam).
7. **MACD seed-EMA paradigm** = same SMA seed + alpha = 2/(p+1) as `EmaFeature`/T-107a; both fast and slow EMAs use this; signal-line is EMA on the macd-line series with the same algorithm.
8. **MACD's EMA helper** = module-level function `_sma_seeded_ema_series(values, period) -> list[Decimal]` in `macd.py`. Returns a list (not just the latest Decimal) because MACD needs the SERIES of fast/slow EMAs to derive the MACD-line series — single-Decimal helper would force O(n²) iteration. Golden test pins helper-tail-equals-`EmaFeature` (mathematical consistency cross-check).
9. **MACD warmup_candles = slow + signal − 1** for default fast=12 slow=26 signal=9 → 34 closes. Hand-verified by operator.
10. **Bollinger period < 2 reject** in constructor — period=1 → σ_pop=0 → upper=middle=lower is degenerate.
11. **VWAP zero-volume test** explicit in test fixture set per operator refinement.
12. **Test fixtures** = TDD-first per §N4, hand-computable expected values documented in test docstring, ≥3 cases per indicator.
13. **VWAP name_template** = static `"ind.{symbol}.{interval}.vwap_session"` (per §B.2 verbatim) — session parameter affects compute logic (cutoff strategy), not the identifier. F1+ sessions (hourly/weekly) would need a follow-up decision on whether to keep static `_session` or evolve to `_{session}`.
14. **MacdFeature/BollingerFeature output hashability** = `value_json` variant, non-hashable per T-106; documented explicitly in both class docstrings.
15. **`_sma_seeded_ema_series` index alignment** = output index `i` ↔ input bar index `i + period - 1`; documented explicitly in helper docstring + `MacdFeature.compute` alignment comment.
