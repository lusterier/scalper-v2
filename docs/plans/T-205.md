## ADR-0003 baked-in decisions

Per ADR-0003 (commit `dcf98bd`):

1. **3 bucket families**: `bybit:<sub_account>:orders` (rate=10/s, capacity=20), `bybit:<sub_account>:positions` (rate=10/s, capacity=20), `bybit:ip:global` (rate=120/s, capacity=240).
2. **Pause flag**: KV key `bybit:ip:pause`, value = ISO-8601 expiry timestamp; default duration 500ms; env `RATE_LIMIT_PAUSE_MS`.
3. **Optimistic CAS**: `update(key, value, last=revision)`; retry-once-on-conflict; fail-open after 3 conflicts.
4. **All params as Settings env vars**: `RATE_LIMIT_ORDERS_RATE/_CAPACITY`, `RATE_LIMIT_POSITIONS_RATE/_CAPACITY`, `RATE_LIMIT_IP_GLOBAL_RATE/_CAPACITY`, `RATE_LIMIT_PAUSE_MS`.
5. **DI via constructor**: T-215 instantiates one shared `SharedRateLimiter`; passed to every `BybitV5Adapter` ctor; PaperExchange does NOT consume.
6. **`sub_account` keying**: from `bots` table column (T-103a F1 ship); caller-provided per `acquire()` call.

## OQ resolutions (operator-approved 2026-04-29 — "use defaults")

1. **OQ-1 — `acquire()` API surface**: **default A** — `async def acquire(sub_account: str, endpoint_group: Literal["orders", "positions"]) -> None`. Blocks-and-retries when local bucket empty (waits for refill); raises `RateLimitError` only when upstream pause-flag active (caller propagates to retry layer per §11.3). Cleaner async semantics than tuple-return; T-208 just `await limiter.acquire(...)`.

2. **OQ-2 — IP-global bucket coupling**: **default A** — every `acquire(sub_account, group)` debits BOTH `bybit:<sub_account>:<group>` AND `bybit:ip:global` — sequenced, but treated as one logical operation (if IP-global fails after sub-account succeeded, the sub-account debit is NOT rolled back; instead the next acquire will see less-than-expected capacity which is acceptable safety-side bias). Mirror Bybit's actual enforcement.

3. **OQ-3 — Refill mechanics**: **default A** — lazy refill on each `acquire`: `tokens_now = min(capacity, last_tokens + (now - last_refill_at) * rate)`. Persist updated `(tokens, last_refill_at)` to KV via CAS. No background timer task; lifecycle simpler.

4. **OQ-4 — `RateLimitError` sourcing on Bybit response (T-208 binding)**: **default A** — when T-208 receives upstream Bybit `RateLimitError` (HTTP 429), it calls `await limiter.signal_upstream_rate_limit()` which writes the `bybit:ip:pause` flag to KV with `expires_at = now + RATE_LIMIT_PAUSE_MS`. Limiter does NOT call this internally — caller-driven from actual exchange response. Decouples T-205 from HTTP layer.

5. **OQ-5 — KV bucket bootstrap**: **default A** — assumes `rate_limits` KV bucket exists (provisioned by T-019 nats-bootstrap or F2+ pre-flight check at execution-service startup). Limiter does NOT create the bucket; `kv_get` on missing bucket raises `PublishError` per existing `kv_put` precedent.

6. **OQ-6 — Test strategy**: **default A** — env-gated integration tests against real NATS (mirror T-104a/b pattern; ci-full has NATS testcontainer); unit tests with KV mock for refill math + CAS retry logic + pause-flag interaction. Hand-verifiable token-debit fixtures at exact Decimal values.

7. **OQ-7 — LOC scope estimate**: **default A** — ~140-160 LOC src for `rate_limiter.py` + ~50-60 LOC for `packages/bus/client.py` extensions (`kv_get` + `kv_update`) = ~190-220 src total. Tests ~250-300 LOC. Surface-expansion guard 200 LOC src mid-write per L-006 watch — STOP if approaching.

## Decisions committed

Per the T-204 → T-213c calibration trajectory.

1. **OQ-defaults applied throughout**; ADR-0003 baked-in; no unresolved decisions remain.

2. **Two-package scope**:
   - `packages/bus/client.py` — extended with `kv_get(bucket, key) -> tuple[bytes, int] | None` and `kv_update(bucket, key, value, last_revision) -> int` per Decision #3 / OQ-7.
   - `packages/exchange/rate_limiter.py` — NEW; `SharedRateLimiter` class with `acquire()`, `signal_upstream_rate_limit()`, `_refill_and_debit()` private helper.

3. **Time source DI'd via `now_fn: Callable[[], datetime] = now_utc`** per T-213b precedent — test-deterministic refill math; mirrors PaperExchange clock-DI pattern.

4. **`@idempotent` marker on `acquire()`**: technically debiting a token is non-idempotent at first glance, BUT semantically `acquire()` represents a "wait until I can proceed" gate — replaying it does NOT double-debit beyond the natural rate-limiting effect. Per §N3 + brief §11.4 implication ("each call debits one token"), we mark `acquire()` as `@non_idempotent` because retry-on-failure WOULD double-debit; T-208 retry matrix (§11.2) treats acquire-then-call as one logical operation with no internal retry.

5. **`@idempotent` marker on `signal_upstream_rate_limit()`**: writing the same pause-flag value twice yields the same KV state (last-write-wins). Per `kv_put` precedent (`@idempotent` per current `client.py:276`). Rebroadcast on multiple adapters seeing the same `RateLimitError` is the intended fail-safe.

6. **`@idempotent` markers on new bus methods**: `kv_get` is read-only (idempotent by definition); `kv_update` is CAS-guarded — replaying with the same `last_revision` will fail (revision now stale), so semantically once-only — but the SAME caller calling it twice with the same intent will retry through `_refill_and_debit` with fresh revision. Mark `kv_get` as `@idempotent`; mark `kv_update` as `@non_idempotent` (matches CAS semantics — replay yields different outcome based on intervening state).

7. **`SharedRateLimiter` is per-process singleton-instance, DI'd, NOT module-level** per §N6. T-215 composition root instantiates one instance; passes to N adapters.

8. **Settings ingestion via constructor kwargs**, NOT module-level Settings.parse_env() reads. Mirrors T-213b ctor-DI pattern. Composition root (T-215) reads env, builds limiter:
   ```python
   limiter = SharedRateLimiter(
       bus=bus,
       orders_rate=settings.rate_limit_orders_rate,
       orders_capacity=settings.rate_limit_orders_capacity,
       ...
       pause_ms=settings.rate_limit_pause_ms,
   )
   ```

9. **`acquire()` blocking semantics**: when local bucket empty, `acquire()` does `await asyncio.sleep(wait_seconds)` where `wait_seconds = (1 - tokens) / rate`. Then re-tries refill+debit. After 3 conflicts on CAS, fails open + WARN log.

10. **Pause-flag check is the FIRST step of `acquire()`**: if pause flag active (`now < expires_at`), `await asyncio.sleep((expires_at - now).total_seconds())`, then proceed. No bucket-debit during pause; tokens accumulate naturally during pause via lazy-refill on next call.

11. **Pause flag's `expires_at` is ISO-8601 UTC** per §N1; serialized as `bytes` for KV compatibility (`expires_at.isoformat().encode("utf-8")`).

12. **Bucket value serialization**: `(tokens: float, last_refill_at: datetime)` JSON-serialized as `{"tokens": 19.5, "last_refill_at": "2026-04-29T12:00:00.123456+00:00"}` then `.encode("utf-8")` for KV bytes value. **`tokens` as float, NOT Decimal** because rate-limit budget is approximate-by-design — sub-token precision is irrelevant; lazy-refill arithmetic over fractional seconds favors float speed over Decimal exactness. NOT a §N1 violation: §N1 / §5.13 invariant is for financial math (P&L, prices, qty), NOT for operational counters where rounding to nearest integer is the natural unit.

13. **Conventional housekeeping**:
    - Plan-doc `## Hand verification`: §F section for lazy-refill math (closed-form) + CAS retry decision tree.
    - `docs/modules/exchange.md`: skip (defer to F2 close-out per T-213a/T-213b/T-213c precedent).
    - Branch name: `feat/T-205-rate-limiter`.

14. **No per-call jitter on lazy-refill wait or CAS retry** (Decision binding H-025): brief §20 H-025 line 2802 says "Exponential backoff with jitter per endpoint group". This requirement is implemented at the IP-coordination layer via the **500ms shared `bybit:ip:pause` flag** (ADR-0003 §2; OQ-2 default A) — when one bot hits Bybit's `RateLimitError`, ALL adapters on the IP observe the shared pause and sleep until `expires_at`. The shared-pause-flag IS the H-025 mechanism; per-call jitter on the lazy-refill wait formula `(1 - tokens) / rate` would be redundant (refill wait is deterministic by design — token bucket is approximate, not contention-resolving) and per-call jitter on CAS retry would mask the 3-conflict fail-open invariant from ADR-0003 §3 (CAS conflicts are diagnostic — repeated jitter-spread retries would hide the burst-load signal). Brief §11.2 retry matrix `[0.5, 1.0, 2.0]s + jitter` is a SEPARATE concern bound at T-208 BybitV5Adapter (HTTP-level retry on transient network/auth failures); rate-limiter-side handles the rate-limit-induced backoff via the pause-flag mechanism only.

### `packages/bus/client.py` (extended from T-008)

Two new methods on `NatsClient` mirroring `kv_put` shape.

```python
@idempotent
async def kv_get(self, bucket: str, key: str) -> tuple[bytes, int] | None:
    """Read ``value`` + revision from JetStream KV ``bucket`` under ``key``.

    Returns ``(value, revision)`` if key exists, ``None`` if missing
    (NATS distinguishes "key never set" from "key deleted with tombstone";
    we collapse both to None for caller simplicity — rate limiter treats
    missing as "fresh bucket starting at full capacity").

    Bucket must be pre-provisioned per kv_put precedent.
    """
    if self._state is not ConnectionState.CONNECTED or self._js is None:
        raise NotConnectedError(f"kv_get called in state {self._state.value!r}")
    try:
        kv = await self._js.key_value(bucket)
        entry = await kv.get(key)
    except nats.errors.KeyNotFoundError:
        return None
    except nats.errors.Error as exc:
        ...  # log + raise PublishError
    return (entry.value, entry.revision)


@non_idempotent
async def kv_update(
    self, bucket: str, key: str, value: bytes, last_revision: int,
) -> int:
    """CAS update: write only if current revision == last_revision.

    Raises :class:`PublishError` (with ``__cause__`` set to
    ``nats.errors.KeyWrongLastSequenceError``) if revision mismatched —
    caller must re-read + retry. Returns new revision on success.
    """
    if self._state is not ConnectionState.CONNECTED or self._js is None:
        raise NotConnectedError(f"kv_update called in state {self._state.value!r}")
    try:
        kv = await self._js.key_value(bucket)
        revision = await kv.update(key, value, last=last_revision)
    except nats.errors.Error as exc:
        ...  # log + raise PublishError
    return revision
```

### `packages/exchange/rate_limiter.py` (NEW)

Single module with `SharedRateLimiter` class + small helper. Lazy-refill token bucket with NATS KV CAS persistence.

```python
"""§11.4 + ADR-0003 NATS KV token-bucket shared rate limiter.

T-205 ships SharedRateLimiter; T-208 (BybitV5Adapter REST methods) consumes
via DI'd ctor kwarg; T-215 (adapter pool composition root) instantiates
one shared instance for the live-adapter pool.

PaperExchange does NOT consume the limiter (no upstream rate limit).
"""
from __future__ import annotations

import asyncio
import json
import logging
from datetime import datetime, timedelta
from typing import TYPE_CHECKING, Literal

from packages.bus.errors import PublishError
from packages.core import idempotent, non_idempotent, now_utc
from packages.exchange.errors import RateLimitError

if TYPE_CHECKING:
    from collections.abc import Callable
    from packages.bus import NatsClient


__all__ = ["SharedRateLimiter"]

EndpointGroup = Literal["orders", "positions"]

_BUCKET = "rate_limits"
_PAUSE_KEY = "bybit:ip:pause"
_IP_GLOBAL_KEY = "bybit:ip:global"
_MAX_CAS_RETRIES = 3

logger = logging.getLogger(__name__)


class SharedRateLimiter:
    """Per-process token-bucket coordinator (per ADR-0003)."""

    def __init__(
        self,
        *,
        bus: NatsClient,
        orders_rate: float,
        orders_capacity: float,
        positions_rate: float,
        positions_capacity: float,
        ip_global_rate: float,
        ip_global_capacity: float,
        pause_ms: int,
        now_fn: Callable[[], datetime] = now_utc,
    ) -> None:
        self._bus = bus
        self._params: dict[str, tuple[float, float]] = {
            "orders": (orders_rate, orders_capacity),
            "positions": (positions_rate, positions_capacity),
            "_ip_global": (ip_global_rate, ip_global_capacity),
        }
        self._pause_ms = pause_ms
        self._now_fn = now_fn

    @non_idempotent
    async def acquire(
        self, sub_account: str, endpoint_group: EndpointGroup,
    ) -> None:
        """Block until a token can be debited from sub-bucket + IP-global.

        Decision #9 — when local bucket empty, asyncio.sleep((1-tokens)/rate)
        then retry refill+debit. Decision #10 — pause-flag check FIRST.
        Decision #3 (ADR-0003) — CAS retry-once + fail-open after 3 conflicts.
        """
        await self._wait_if_paused()
        sub_key = f"bybit:{sub_account}:{endpoint_group}"
        await self._refill_and_debit(sub_key, endpoint_group)
        await self._refill_and_debit(_IP_GLOBAL_KEY, "_ip_global")

    @idempotent
    async def signal_upstream_rate_limit(self) -> None:
        """Write the bybit:ip:pause flag (called by T-208 on Bybit 429).

        OQ-4 default A: caller-driven; limiter does NOT detect 429
        internally. expires_at = now + RATE_LIMIT_PAUSE_MS.
        """
        expires_at = self._now_fn() + timedelta(milliseconds=self._pause_ms)
        await self._bus.kv_put(
            _BUCKET, _PAUSE_KEY, expires_at.isoformat().encode("utf-8"),
        )

    async def _wait_if_paused(self) -> None:
        """Decision #10 — pause-flag check first."""
        result = await self._bus.kv_get(_BUCKET, _PAUSE_KEY)
        if result is None:
            return
        value, _revision = result
        try:
            expires_at = datetime.fromisoformat(value.decode("utf-8"))
        except ValueError:
            logger.warning("rate_limiter.malformed_pause_flag", extra={"value": repr(value)})
            return
        now = self._now_fn()
        if now < expires_at:
            await asyncio.sleep((expires_at - now).total_seconds())

    async def _refill_and_debit(self, key: str, group: str) -> None:
        """Lazy-refill + CAS-debit one token (per Decision #9 / ADR-0003 §3)."""
        rate, capacity = self._params[group]
        for attempt in range(_MAX_CAS_RETRIES):
            result = await self._bus.kv_get(_BUCKET, key)
            now = self._now_fn()
            if result is None:
                tokens, last_refill_at, revision = capacity, now, None
            else:
                # CONCERN 2 fix — corrupted KV value (mid-write crash, non-JSON
                # bytes) falls back to fresh-bucket; mirror pause-flag malformed
                # handler shape per Edge case #2.
                try:
                    state = json.loads(result[0].decode("utf-8"))
                    tokens = state["tokens"]
                    last_refill_at = datetime.fromisoformat(state["last_refill_at"])
                except (json.JSONDecodeError, KeyError, ValueError):
                    logger.warning(
                        "rate_limiter.malformed_bucket_state",
                        extra={"key": key, "value": repr(result[0])},
                    )
                    tokens, last_refill_at = capacity, now
                    revision = None
                else:
                    revision = result[1]
            elapsed = (now - last_refill_at).total_seconds()
            tokens = min(capacity, tokens + elapsed * rate)
            if tokens < 1.0:
                wait_seconds = (1.0 - tokens) / rate
                await asyncio.sleep(wait_seconds)
                continue
            tokens -= 1.0
            new_value = json.dumps({
                "tokens": tokens,
                "last_refill_at": now.isoformat(),
            }).encode("utf-8")
            try:
                if revision is None:
                    await self._bus.kv_put(_BUCKET, key, new_value)
                else:
                    await self._bus.kv_update(_BUCKET, key, new_value, revision)
            except PublishError:
                continue  # CAS conflict — retry
            return
        # Fail-open after 3 CAS conflicts (ADR-0003 §3 / Decision #6).
        logger.warning("rate_limiter.cas_failover_open", extra={"key": key, "group": group})
```

## Hand verification

### §F.1 — Lazy-refill formula (closed-form)

Given:
- `rate = 10 tokens/s` (orders bucket)
- `capacity = 20 tokens`
- `last_refill_at = 2026-04-29T12:00:00.000000+00:00`
- `last_tokens = 5.0`
- `now = 2026-04-29T12:00:00.500000+00:00` (500ms later)

Computation:
- `elapsed = (now - last_refill_at).total_seconds() = 0.5`
- `tokens_now = min(capacity, last_tokens + elapsed * rate) = min(20, 5.0 + 0.5 * 10) = min(20, 10.0) = 10.0`
- After debit: `tokens = 10.0 - 1.0 = 9.0`

Test fixture `test_acquire_lazy_refill_formula_pinned`:
- `tokens_before = 5.0`, `elapsed = 0.5`, `rate = 10` → `tokens_after_refill = 10.0` → `tokens_after_debit = 9.0` ✓

### §F.2 — CAS retry decision tree

Given local bucket empty, no pause flag, NATS revision = 7:

1. `kv_get` → `(value={"tokens": 0.5, "last_refill_at": "..."}, revision=7)`.
2. Compute refill: `tokens_now = min(capacity, 0.5 + elapsed * rate)`.
3. If `tokens_now < 1.0` → `asyncio.sleep((1-tokens_now)/rate)` → loop.
4. If `tokens_now >= 1.0` → debit (`tokens = tokens_now - 1`); `kv_update(value, last_revision=7)`.
5. `kv_update` succeeds → return.
6. `kv_update` raises `PublishError` (cause = `KeyWrongLastSequenceError`) → loop attempt 2.
7. After 3 failures → WARN log + return (fail-open per ADR-0003 §3).

Test fixtures:
- `test_acquire_retries_on_cas_conflict_up_to_3_times_then_fails_open` — `bus.kv_update.side_effect = [PublishError, PublishError, PublishError]` → 3 retries + WARN log + no exception raised.
- `test_acquire_creates_fresh_bucket_when_kv_get_returns_none` — `bus.kv_get.return_value = None` → starts at full capacity; uses `kv_put` not `kv_update`.

### §F.3 — Pause-flag wait formula

Given:
- `now = 2026-04-29T12:00:00.000000+00:00`
- `expires_at = 2026-04-29T12:00:00.300000+00:00` (300ms in future)

Computation:
- `wait_seconds = (expires_at - now).total_seconds() = 0.3`
- `await asyncio.sleep(0.3)` → blocks 300ms.

Test fixture `test_acquire_blocks_when_pause_flag_active_until_expires_at`:
- `kv_get(_PAUSE_KEY)` → `(b"2026-04-29T12:00:00.300000+00:00", revision=42)`.
- Patch `asyncio.sleep` → assert called with `0.3`.

Math-validator (Gate 4) ACTIVE per CLAUDE.md gate-4 conditional rule (diff touches `packages/exchange/`). Expected verdict: **VERIFIED** — closed-form arithmetic + hand-computable fixtures + no Decimal/precision boundary issues.

## Lifecycle

`SharedRateLimiter` instance lifecycle (T-205 scope):

1. Constructor (T-205): store bus + 3 bucket params + pause_ms + now_fn. NO I/O.
2. `acquire(sub_account, endpoint_group)` (T-205 body):
   - (a) Pause-flag check via `_wait_if_paused()`; if active, sleep until expires.
   - (b) `_refill_and_debit(f"bybit:{sub}:<group>", group)` for sub-account bucket.
   - (c) `_refill_and_debit(_IP_GLOBAL_KEY, "_ip_global")` for IP-global bucket.
   - (d) Return None on success; WARN log + return on 3-conflict fail-open.
3. `signal_upstream_rate_limit()` (T-205 body): writes `bybit:ip:pause = (now + pause_ms).isoformat()`.

No `start_consuming()` / `close()` lifecycle methods — the limiter is stateless (state lives in NATS KV).

## Edge cases

1. **Missing `rate_limits` bucket**: `kv_get` raises `PublishError` (per existing `kv_put` precedent); propagates to caller; T-208 receives error → composition-root crash at first acquire. Acceptable per OQ-5 — bucket is infra responsibility.

2. **Malformed pause-flag value** (corrupt `expires_at` string): caught at `datetime.fromisoformat` → WARN log + skip pause (treated as not-paused). Defends against bad operator KV writes.

3. **Negative `tokens` after refill** (clock skew, NTP jump): theoretically `last_refill_at > now` → negative `elapsed` → tokens decrease. `min(capacity, tokens + neg)` still yields a valid value (might be < tokens). Test pin: `test_acquire_handles_clock_skew_negative_elapsed`.

4. **Concurrent `signal_upstream_rate_limit` from N adapters**: idempotent per Decision #5 — last-write-wins on `bybit:ip:pause`; multiple writes within 500ms yield same effective behavior (pause persists until last-written `expires_at`).

5. **CAS conflict storm under burst load**: 3 retries + fail-open is the documented behavior. WARN log gives operator visibility. ADR-0003 §3 acceptance.

6. **`kv_get` returns None for fresh sub_account**: limiter treats as "fresh bucket starting at full capacity" (initial state `(capacity, now, None)`); first debit via `kv_put` (no revision yet). After `kv_put`, subsequent calls use `kv_update` with revision.

7. **Two acquires for same group within `1/rate` second**: first debits; second sees `tokens < 1.0` → sleeps `(1-tokens)/rate`; refills + debits.

8. **Pause flag activates mid-acquire after `_wait_if_paused()` returned**: race window between pause-flag check and sub-account debit can let a debit proceed even though the IP just got rate-limited. Acceptable per ADR-0003 fail-open posture — sub-account debit goes through; the next `acquire()` call (any group, any bot) will see the new pause flag and sleep. Worst case: 1 extra request lands at Bybit and gets the next `RateLimitError`, which itself triggers `signal_upstream_rate_limit()` rebroadcast. The pause-flag mechanism is self-recovering. Documented per ADR-0003 §"Consequences" — fail-open is bounded by upstream Bybit per-IP `RateLimitError` + 500ms pause-flag rebroadcast as the recovery surface.

## Risks (post-resolution)

- **Risk: CAS retry loop infinite under sustained burst**: 3-retry hard limit + WARN log + fail-open is the documented bound per ADR-0003. Mitigation: WARN log gives operator visibility; metrics counter could be added in T-F2+ if observability shows fail-opens are persistent.
- **Risk: Float arithmetic in `tokens` accumulation under millions of calls**: f64 has ~15 decimal digits of precision; rate-limit budget never exceeds ~240 tokens; lazy-refill arithmetic over fractional seconds yields enough headroom for years of operation. Documented in Decision #12.
- **Risk: NATS KV revision-number wraparound**: NATS revision is 64-bit; not load-bearing.
- **Risk: `bus.kv_update` raises `PublishError` for non-CAS errors (network, etc.)**: caught + retried as if CAS conflict. After 3 retries, fail-open; same surface as legitimate CAS conflict. Acceptable; the retry-once budget is symmetric across error types.
- **Risk: `signal_upstream_rate_limit` is called concurrently by N adapters**: idempotent per Decision #5; last-write-wins; pause persists until last `expires_at`. Acceptable per ADR-0003.
