## Numeric precision & Decimal→float boundary (Gate-4)

Two numeric boundaries, both intentional + documented:

**(a) Persistence scale.** T-530 `AccountBalance` carries unbounded `Decimal` (Bybit `Decimal(str(totals))` 1:1; paper seed+Σrealized). Persisted into `Numeric(20,4)` → PostgreSQL round-half-even to scale 4 at INSERT, mirroring the `0009_trade_pnl_deltas` / `0006_position_state.py:116 running_pnl` USD-money convention. Accepted, NOT a silent degradation: equity snapshot is a monitoring time-series, NOT P&L-truth — financial truth is the T-220 cumulative-delta audit (ADR-0006, §Hazards). $0.0001 USD granularity is ample for equity-trend monitoring; the choice is repo-convention-aligned and L-005-pinned in test_0019 so a future regression to bare `Numeric()` is caught.

**(b) Gauge.** `prometheus_client.Gauge.set()` requires float → `metrics.virtual_balance.labels(bot_id=bot_id).set(float(bal.total_equity))`. Sole Decimal→float; monitoring boundary; lossy-OK; gauge not read by sizing/P&L (those use `AccountBalance` Decimal directly). The gauge is set from the **pre-persist in-memory `bal.total_equity`** (full T-530 Decimal precision, then float-cast), while the persisted row is scale-4 NUMERIC — they may diverge at the 5th+ decimal. Intentional + documented: both are monitoring views, neither is P&L-truth; no consumer compares gauge vs row. T-531 performs zero arithmetic.

## Hand verification

- **Paper:** T-530 → `AccountBalance(wallet=available=total_equity=margin=Decimal("10250.5000"), unrealized=Decimal("0"))` [seed 10000 + Σrealized 250.50, T-530-shipped math-validated]. Row: id auto (1), bot_id="alpha", snapshot_at=2026-05-15T12:00:00+00:00, 5×Numeric(20,4) = 10250.5000 / 10250.5000 / 10250.5000 / 10250.5000 / 0.0000 — all ≤4dp, exact at scale 4, NO truncation. gauge `virtual_balance{bot_id="alpha"}` = float(Decimal("10250.5000")) = 10250.5. Zero arithmetic (records 1:1).
- **Same-instant fan-out (OQ-4=A, shared live sub-account "subA", bots alpha+beta):** `get_account_balance("subA")` ONCE → `AccountBalance(total_equity=Decimal("125000.1234"))` [≤4dp USD money — exact at scale 4]. 2 rows: (id=1,bot_id="alpha",snapshot_at=T,total_equity=125000.1234), (id=2,bot_id="beta",snapshot_at=T,total_equity=125000.1234) — identical except (id,bot_id); composite PK (snapshot_at,id) unique. Misfire re-fire → id=3,id=4 (distinct, no PK collision). gauges `virtual_balance{alpha}` = `virtual_balance{beta}` = 125000.1234.
- **Truncation worked example (documents boundary (a)):** Bybit returns `"totalEquity":"125000.12345678"` → T-530 `AccountBalance.total_equity = Decimal("125000.12345678")` → INSERT into `Numeric(20,4)` → PG round-half-even: 5th-decimal digit is 5 with non-zero tail (…45678) → stored row = `125000.1235`. gauge = float(Decimal("125000.12345678")) ≈ 125000.12345678 (set from pre-persist `bal`, full precision). Row 125000.1235 vs gauge 125000.12345678 divergence at decimal 5+ is the documented (b) intentional behavior — both monitoring, neither P&L-truth.
- **Negative:** `AccountBalance(unrealized_pnl=Decimal("-125.2500"))` → row Numeric(20,4) = -125.2500 signed exact (≤4dp; mirror T-530 negative `totalPerpUPL` golden).
- No EMA/Wilder/seed/division — not an indicator/P&L computation; only numeric concerns = the two documented boundaries (a)+(b).
