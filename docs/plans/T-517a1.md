## Purpose

Backend half of per-symbol best-variant aggregate surface. Closes BRIEF §13.6 second bullet "which variant would have been best over last N trades?" Operator drills into a specific symbol → backend aggregates terminated `shadow_variants` rows by `variant_name` → returns multi-metric table (n_trades + win_rate + total_pnl + avg_pnl + best_pnl + worst_pnl + avg_mfe_pct + avg_mae_pct).

Pre-emptive sub-split T-517a → T-517a1 (this; backend) + T-517a2 (UI) per L-007 + operator OQ-3=A 2026-05-12. Combined T-517a est ~470 LOC src (backend SQL JOIN + Python aggregation + router ~230 + UI ~240) would trip §0.3 cap; sub-split keeps each well under cap. Mirror T-517b sub-split precedent + T-516a1+T-516a2 split precedent.

Operator OQs resolved at T-517a plan-stage 2026-05-12:

- **OQ-1 = A**: Multi-metric table (variant_name + n_trades + win_rate + total_pnl + avg_pnl + best_pnl + worst_pnl + avg_mfe_pct + avg_mae_pct). Default sort by total_pnl DESC.
- **OQ-2 = A**: Time-based window (TimeRangePicker default 30d via `?from=` / `?to=` ISO-8601 created_at filter). Mirror trades.index/paper-trades.index/shadow.rejected UX.
- **OQ-3 = A**: Sub-split T-517a → T-517a1 (backend) + T-517a2 (UI).
- **OQ-4 = A**: Optional `bot_id` filter; default all-bots aggregated for symbol.

## Architecture decision: SELECT-then-Python aggregation (NOT SQL GROUP BY) + dynamic predicate builder mirror sibling T-517b1

Mirror existing analytics router pattern (compute_expectancy, compute_hourly_heatmap, compute_pnl_series at `services/analytics_api/app/analytics_compute.py:98+158+185`):

1. DB helper `select_shadow_variants_for_aggregate(symbol, bot_id, from_at, to_at)` fetches all matching terminated variants as `ShadowVariantAggregateRow` list.
2. Python aggregator `compute_variant_aggregate(rows) -> list[VariantAggregateMetrics]` groups by `variant_name` + computes 8 metrics per variant.
3. Router orchestrates: fetch → compute → Pydantic-narrow → return envelope.

**Rationale (fetch-then-compute):** SQL GROUP BY would couple aggregation logic to PG dialect + diverge from established analytics_compute pattern. In-memory aggregation is testable without DB (unit-test the math), reuses dataclass narrowing convention, mirrors compute_expectancy 1:1.

**Volume bound:** shadow_variants per-symbol per-month is bounded (≤5 variants × parent trades; in F5 paper-mode the operator runs ≤100 trades/day; 30d window × 5 variants × ~3000 trades = ~15000 rows worst case). In-memory aggregation acceptable; matches `select_trades_for_analytics` charter at `analytics.py:1726` (limit-less; routers enforce reasonable filter windows).

**Rationale (dynamic predicate builder):** Per plan-reviewer Gate 1 CONCERN-2 — sibling `_build_shadow_rejected_where_clause` (T-517b1, `shadow.py:407-450`) shipped 2 days ago uses dynamic predicate-list builder. T-517a1 mirrors that idiom for intra-module consistency. Charter predicates (`COALESCE(t.symbol, pt.symbol) = $1::text`, `v.terminated_at IS NOT NULL`, `v.realized_pnl IS NOT NULL`) become "always-included" predicates in the builder; optional filters (bot_id, from_at, to_at) become conditionally-appended predicates with direct column comparisons (`v.bot_id = $N`, `v.created_at >= $N`, `v.created_at < $N`). **L-021 trigger removed:** dynamic builder produces direct column comparisons (no `$N::type IS NULL OR ...` pattern) — PG type-inference resolves correctly without explicit `::timestamptz` casts on optional filters; only the always-included `$1::text` cast on symbol remains (defensive, COALESCE result is TEXT). Testcontainer-gated integration test still required per L-021 active control for the JOIN behavior verification.

async def select_shadow_variants_for_aggregate(
    conn: _DbExecutor,
    *,
    symbol: str,
    bot_id: str | None,
    from_at: datetime | None,
    to_at: datetime | None,

    JOINs ``shadow_variants`` with parent ``trades`` (parent_kind='live') OR
    ``paper_trades`` (parent_kind='paper') by ``parent_trade_id`` to pull
    parent symbol. Filter ``COALESCE(t.symbol, pt.symbol) = $1::text``.

    Variants in ``shutdown_mid_replay`` terminal state where realized_pnl is
    NULL are excluded.

    ORDER BY ``created_at DESC`` for deterministic iteration; in-memory
    aggregator is order-invariant for total/avg/min/max metrics.
    """

SQL shape — module-level constants for SELECT prefix + ORDER BY suffix; WHERE clause assembled per-call by `_build_shadow_variant_aggregate_where_clause` (mirror sibling T-517b1 `_build_shadow_rejected_where_clause`):

```python
_SELECT_AGGREGATE_SQL_PREFIX = (
    "SELECT COALESCE(t.symbol, pt.symbol) AS parent_symbol, v.bot_id, "
    "v.variant_name, v.realized_pnl, v.mfe_pct, v.mae_pct, v.parent_kind, "
    "v.created_at "
    "FROM shadow_variants v "
    "LEFT JOIN trades t ON v.parent_kind = 'live' AND v.parent_trade_id = t.id "
    "LEFT JOIN paper_trades pt ON v.parent_kind = 'paper' AND v.parent_trade_id = pt.id "
)
_SELECT_AGGREGATE_SQL_SUFFIX = " ORDER BY v.created_at DESC"


def _build_shadow_variant_aggregate_where_clause(
    *,
    symbol: str,
    bot_id: str | None,
    from_at: datetime | None,
    to_at: datetime | None,
) -> tuple[str, list[Any]]:
    """Mirror :func:`_build_shadow_rejected_where_clause` (T-517b1).

    Charter predicates are ALWAYS included (per analytics_compute charter
    'only finalized P&L counts for stats' at analytics.py:1735):
      - COALESCE(t.symbol, pt.symbol) = $1::text  (defensive cast; COALESCE
        result is TEXT)
      - v.terminated_at IS NOT NULL
      - v.realized_pnl IS NOT NULL

    Optional filters appended as direct column comparisons (no L-021 cast
    needed — PG infers from column type):
      - bot_id → v.bot_id = $N
      - from_at → v.created_at >= $N
      - to_at → v.created_at < $N

    Returns ``("WHERE <predicates>", [bind args in $N order])``. symbol is
    always $1; optional filters take $2..$4 if present.
    """
    bind_args: list[Any] = [symbol]
    predicates: list[str] = [
        "COALESCE(t.symbol, pt.symbol) = $1::text",
        "v.terminated_at IS NOT NULL",
        "v.realized_pnl IS NOT NULL",
    ]
    if bot_id is not None:
        bind_args.append(bot_id)
        predicates.append(f"v.bot_id = ${len(bind_args)}")
    if from_at is not None:
        bind_args.append(from_at)
        predicates.append(f"v.created_at >= ${len(bind_args)}")
    if to_at is not None:
        bind_args.append(to_at)
        predicates.append(f"v.created_at < ${len(bind_args)}")
    return ("WHERE " + " AND ".join(predicates), bind_args)
```

Per-call assembly:
```python
where_clause, where_args = _build_shadow_variant_aggregate_where_clause(...)
sql = f"{_SELECT_AGGREGATE_SQL_PREFIX}{where_clause}{_SELECT_AGGREGATE_SQL_SUFFIX}"
rows = await conn.fetch(sql, *where_args)
```

#### NEW aggregator function `compute_variant_aggregate`

```python
def compute_variant_aggregate(
    rows: list[ShadowVariantAggregateRow],
) -> list[VariantAggregateMetrics]:
    """Group rows by variant_name + compute 8 metrics per variant.

    Returns variants sorted by ``(-total_pnl, variant_name)`` for
    deterministic order: primary descending by total_pnl (operator-visible
    "best variant" highlight); secondary tie-break ascending by variant_name
    (alphabetical) — guarantees stable output independent of SQL row
    iteration order or dict-insertion order. Empty input → empty list.

    Metrics:
        n_trades = count of rows for variant
        win_count = count where realized_pnl > 0
        win_rate = win_count / n_trades (float in [0, 1])
        total_pnl = sum(realized_pnl) (Decimal, precision-preserved)
        avg_pnl = total_pnl / n_trades (Decimal)
        best_pnl = max(realized_pnl) (Decimal)
        worst_pnl = min(realized_pnl) (Decimal)
        avg_mfe_pct = mean of non-None mfe_pct (float; None if all None)
        avg_mae_pct = mean of non-None mae_pct (float; None if all None)

    Empty-bucket guard: division-by-zero precluded by n_trades > 0
    (variant in dict only if at least one row).
    """
    # ... compute per-variant metrics into list[VariantAggregateMetrics]
    return sorted(metrics, key=lambda m: (-m.total_pnl, m.variant_name))
```

### NEW router `services/analytics_api/app/routers/shadow_aggregate.py`

```python
router = APIRouter(prefix="/api/shadow/aggregate", tags=["shadow-aggregate"])


@router.get("/{symbol}", response_model=VariantAggregateListResponse)
async def get_variant_aggregate(
    symbol: str,
    pool: Annotated[asyncpg.Pool, Depends(get_pool)],
    bot_id: Annotated[str | None, Query()] = None,
    from_at: Annotated[
        datetime | None,
        Query(alias="from", description="created_at >= from (ISO-8601)."),
    ] = None,
    to_at: Annotated[
        datetime | None,
        Query(alias="to", description="created_at < to (ISO-8601)."),
    ] = None,
) -> VariantAggregateListResponse:
    """Aggregate terminated shadow variants for ``symbol`` by variant_name."""
    async with pool.acquire() as conn:
        rows = await select_shadow_variants_for_aggregate(
            conn,
            symbol=symbol,
            bot_id=bot_id,
            from_at=from_at,
            to_at=to_at,
        )
    metrics = compute_variant_aggregate(rows)
    return VariantAggregateListResponse(
        symbol=symbol,
        variants=[VariantAggregateResponse(**asdict(m)) for m in metrics],
        bot_id=bot_id,
        from_at=from_at,
        to_at=to_at,
    )
```

## Edge cases

1. **Symbol not in any variants** — `SELECT` returns `[]`; `compute_variant_aggregate` returns `[]`; envelope: `{symbol, variants: [], ...}`. NOT a 404 (consistent with paper-trades / shadow.rejected empty-result convention; mirror analytics router non-404 pattern).
2. **Window with no rows** — same as #1.
3. **All variants have `realized_pnl=NULL`** (e.g., all `shutdown_mid_replay`) — filtered out by `realized_pnl IS NOT NULL` predicate; aggregate empty.
4. **Single variant only** — aggregate returns 1 row; default sort by total_pnl DESC trivial.
5. **Variant with 0 wins** — `win_rate = 0.0`; `best_pnl` is the least-loss row (max of negatives); `worst_pnl` is the worst loss.
6. **Variant with n=1** — `total_pnl == avg_pnl == best_pnl == worst_pnl`; `win_rate ∈ {0.0, 1.0}`.
7. **All `mfe_pct` / `mae_pct` are None** — `avg_mfe_pct = None` / `avg_mae_pct = None`. (Edge case for `shutdown_mid_replay` rows where MFE/MAE were never computed before observation cancelled. But MFE/MAE-NULL with realized_pnl-NOT-NULL is unusual; defensive guard.)
8. **bot_id filter narrows aggregate** — fewer rows; same per-variant computation.
9. **from/to filter** — backend filter on `created_at`; Python aggregator unaware of window.
10. **L-021 cast sites** — Per Architecture decision dynamic builder pattern: only `$1::text` defensive cast on always-included symbol predicate (`COALESCE(t.symbol, pt.symbol) = $1::text`). Optional filters (bot_id, from_at, to_at) when present become direct column comparisons (`v.bot_id = $N`, `v.created_at >= $N`, `v.created_at < $N`) — PG type-inference resolves correctly without explicit casts. **No `$N::type IS NULL OR ...` pattern in generated SQL** (this pattern was the original static-WHERE approach; eliminated by dynamic builder).
