## Goal

Ship the schema foundation that unblocks T-511b2 paper-mode-aware producer:

1. **Migration 0015**: drop `shadow_variants.parent_trade_id → trades(id) ON DELETE CASCADE` FK (introduced 0014); add `parent_kind: TEXT NOT NULL` discriminator column. parent_trade_id stays BIGINT but no FK (mirror shadow_rejected's no-FK pattern per T-510a OQ-6=A; integrity at app layer via `parent_kind` discriminator). Allows shadow_variants to point at EITHER `trades.id` (live) OR `paper_trades.id` (paper).
2. **`packages/db/queries/shadow.py` helper updates**: `insert_shadow_variant` adds `parent_kind: Literal["live", "paper"]` kwarg (keyword-only, no default — caller must specify); `ShadowVariantRow` dataclass adds `parent_kind` field; SQL INSERT + SELECT projections updated.
3. **`packages/bus/payloads.py` delta**: `ShadowStartPayload` adds `parent_kind: Literal["live", "paper"]` field (keyword-only, no default).
4. **T-511b1 shipped consumer-side update**: `services/execution/app/shadow_worker.py:_run_shadow_variant` passes `parent_kind=payload.parent_kind` to `insert_shadow_variant`. Mandatory: payload schema change forces consumer update — not a silent refactor but a required propagation.

## Scope (in)

### NEW: `migrations/versions/0015_shadow_variants_relax_parent_fk.py` (~80 LOC migration §0.3-excluded)

```python
"""shadow_variants: drop parent_trade_id FK + add parent_kind discriminator.

T-511b2a: relax migration 0014 FK constraint so shadow_variants can
reference EITHER `trades.id` (live execution) OR `paper_trades.id`
(paper execution). parent_kind discriminator carries the routing
context; integrity enforced at application layer (BotConfig.exchange.mode
flows via OrderRequest → ShadowStartPayload → insert).

Mirrors T-510a OQ-6=A no-FK convention for shadow_rejected.signal_id
(signals composite-PK rejects FK on id alone). Same pattern here:
parent_trade_id BIGINT, no FK, discriminator-routed.

Backfill: parent_kind defaults to 'live' for any existing rows. In
practice the table is empty at 0015 time (T-510a shipped 0014, T-510b
shipped helpers; no producer until T-511b2). Default-then-NOT-NULL
two-step is defensive for future migrations.

Downgrade: re-add FK to trades(id) ON DELETE CASCADE + drop
parent_kind. Note: downgrade WILL FAIL if any rows have
parent_kind='paper' AND parent_trade_id ∉ trades.id (FK violation).
Operator-acknowledged risk; downgrade only viable on empty / live-only
state.
"""

from __future__ import annotations

import sqlalchemy as sa
from alembic import op

revision = "0015"
down_revision = "0014"
branch_labels = None
depends_on = None


def upgrade() -> None:
    # Drop FK constraint introduced in migration 0014.
    op.drop_constraint(
        "shadow_variants_parent_trade_id_fkey",
        "shadow_variants",
        type_="foreignkey",
    )
    # Add parent_kind discriminator. Default 'live' for backfill (empty in
    # practice); two-step ALTER drops the default after NOT NULL applies.
    op.add_column(
        "shadow_variants",
        sa.Column(
            "parent_kind",
            sa.Text(),
            server_default=sa.text("'live'"),
            nullable=False,
        ),
    )
    op.alter_column("shadow_variants", "parent_kind", server_default=None)


def downgrade() -> None:
    op.drop_column("shadow_variants", "parent_kind")
    op.create_foreign_key(
        "shadow_variants_parent_trade_id_fkey",
        "shadow_variants",
        "trades",
        ["parent_trade_id"],
        ["id"],
        ondelete="CASCADE",
    )
```
