## Purpose

3rd execution-service APScheduler scheduled job (sibling to T-220b P&L audit + T-531 equity tick; shares the lifespan-owned `AsyncIOScheduler`). Periodically verifies that every **tracked** open position still has an exchange-side stop-loss. A position that exists on Bybit (`size>0`) AND in `position_state` whose exchange SL has vanished (`Position.sl_price is None or <= 0`, per T-534a decode) for **N consecutive ticks** is emergency-flattened + DB-closed via the shipped `emergency_close_tracked_position` helper (T-534b1). Mid-session naked-position protection. Live/testnet only — paper bots skipped (synthetic-internal paper SL has no exchange-drop failure mode; OQ-4=A / H-031 precedent). Ships the **formal H-028 §20 catalog allocation** (mirror T-525a1 H-027 mechanism; §0.8 — slot reserved by ADR-0011, allocated at the implementing task).

## Operator OQ answers (resolved, baked in)

- **OQ-1=A**: per-bot loop. `bot_ids = sorted(str(b) for b in adapter_pool.adapters if b not in adapter_pool.paper_bot_ids)`; one `select_position_states_for_bots(conn, bot_ids)`; per-bot `adapter.get_positions()`; match by symbol (verbatim `restart.py:86-93` shape). Wiring mirrors T-531.
- **OQ-2=A**: the counter advances ONLY on a consecutive run of *observed* matched+SL-missing ticks. Any **observed** ineligibility — SL restored, position flat/absent (size==0 / no exchange position / no DB ps_row), or just emergency-closed — zeroes the `(bot,symbol)` entry AND prunes it from the dict (no unbounded growth; clean re-open semantics).
- **OQ-3=A**: `get_positions()` failure is **NOT an observation** → neither increment nor reset nor prune that bot's keys (the streak survives a transient blip). Explicit `except (NetworkTimeout, RateLimitError, AuthError)` → `bound_logger.warning` + `continue`; broad `except Exception` → `bound_logger.error` + `continue`. Never raise into the scheduled job (would kill the safety job / drop the other bots that tick); never increment toward emergency-close on uncertainty (H-028 false-positive guard — the entire point of the hazard).
- **Derived composition of OQ-2=A ∧ OQ-3=A (baked, not a new OQ)**: prune drops a counter key ONLY if its bot was *observed this tick* (`get_positions` succeeded) AND the key was not re-confirmed eligible. Keys belonging to bots whose `get_positions` errored are *preserved untouched* (no observation = no state change). This is the exact reconciliation of "ineligibility resets" (observed) with "error skip-no-increment" (no-observation = no-reset).
- **OQ-4=A / OQ-B=A / OQ-C=A**: paper-skip (`paper_bot_ids`); act only on matched (exchange size>0 + DB ps_row) + SL-missing; orphan-exchange (no DB ps_row) and orphan-db (no exchange position) → defer-`info`-log only, NO reconcile (T-221 owns it); close semantics (`close_reason='emergency'` / `event_type='sl_watchdog_emergency_close'`) all delivered inside the shipped T-534b1 helper — T-534b2 only calls it.

   > ### H-028 — Periodic SL watchdog must not false-positive on transient errors
   >
   > **Context.** A mid-session naked position (open on the exchange with no stop-loss — operator removed it manually, a `set_trading_stop` silently failed, or Bybit dropped it) is unbounded-loss exposure. A periodic watchdog polls `ExchangeClient.get_positions()` and emergency-closes any tracked position whose `Position.sl_price` (T-534a) is absent. The hazard (ADR-0011 anticipated): a transient `get_positions` failure (`NetworkTimeout` / `RateLimitError` / `AuthError`) or an empty/garbled response must NOT be read as "SL dropped" — a false-positive would emergency-close a real, correctly-protected position (capital event from a network blip). Formal §20 allocation lands with T-534b2 (the watchdog tick) per §0.8; the close primitive is T-534b1.
   >
   > **Policy.** The watchdog acts only on a **confirmed, consecutive** observation: a position present in BOTH the exchange snapshot (`size>0`) AND `position_state`, whose `sl_price is None or <= 0`, on **N consecutive successful ticks** (`execution_sl_watchdog_missing_threshold_ticks`, default 3, §N9). The in-memory `(bot_id,symbol)` counter advances ONLY on a successful `get_positions` returning that matched-and-SL-missing state; a transient/any-exception `get_positions` is treated as *no observation* — the tick is skipped for that bot, the counter is neither incremented nor reset (the streak survives the blip), the job never raises (other bots still verified, next tick still fires). Any observed ineligibility (SL restored, position flat/absent, post-close) resets+prunes the counter. Reaching the threshold calls `emergency_close_tracked_position` (T-534b1: reduce-only flatten + atomic DB close `close_reason='emergency'` + `sl_watchdog_emergency_close` audit + `OrderClosed` emit). Paper bots are skipped (`AdapterPoolResult.paper_bot_ids` — synthetic-internal paper SL has no exchange-drop mode; H-031 precedent). Untracked exchange positions and orphan DB rows are defer-logged only — reconciliation is T-221's domain, never the watchdog's. The counter is process-memory; a restart resets it (a fresh process re-derives within N ticks; durable risk-latch persistence is H-027's domain, not this — a missing SL is re-observable, not a latched decision).
   >
   > **Test.** `test_sl_watchdog_emergency_closes_after_n_consecutive_missing` (N-1 ticks no close → Nth tick fires `emergency_close_tracked_position` once + counter cleared) + `test_sl_watchdog_transient_error_does_not_increment_counter` (matched+missing tick → count 1; next tick `get_positions` raises `NetworkTimeout` → count UNCHANGED, no emergency close, streak preserved) + `test_sl_watchdog_sl_restored_resets_counter` + `test_sl_watchdog_skips_paper_bots`.

|---|---|---|---|---|---|---|
| 1 | ok | yes | `None` | missing | (absent)→1 | no |
| 2 | ok | yes | `None` | missing | 1→2 | no |
| 3 | raises `NetworkTimeout` | — | — | no-observation (OQ-3=A) | 2→2 (untouched; bot∉observed → not pruned) | no |
