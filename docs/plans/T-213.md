4. **Slippage model semantics** per §12.1 line 1937 verbatim:
   - `fixed_pct`: `slippage_amount = price * fixed_slippage_pct` (constant pct of price).
   - `proportional_to_qty`: `slippage_amount = price * qty_slippage_coeff * qty` (scales with order size).
   - `half_spread`: `slippage_amount = (high - low) / 2 * half_spread_factor` (uses candle's high-low range as spread proxy in F2; in F5 backtest uses real bid/ask spread).

   Each model has a per-bot **coefficient parameter**: `fixed_slippage_pct`, `qty_slippage_coeff`, `half_spread_factor`. **OQ-3 below** asks how these per-bot parameters are passed; default A: extend constructor with `slippage_params: dict[str, Decimal]` (single dict scoped to active model).

5. **Fee computation** per §12.1 line 1940 verbatim: `fee = abs(qty) * fill_price * fee_rate`. Always positive. Deducted from realized P&L at fill time. Stored in `paper_executions.fee NUMERIC(20, 8)` (mirror live precision).

6. **Decimal arithmetic precision**: Python `decimal` default context (28 significant digits) for all intermediate computation. Final values coerced to schema NUMERIC scales **at the persistence boundary** (T-213b scope, not T-213a). T-213a's slippage / fee functions return raw Decimal at full precision.

7. **Sign convention** for buy/sell fill prices:
   - Buy: fill_price = mid_price + slippage_amount (slippage moves AGAINST you).
   - Sell: fill_price = mid_price - slippage_amount.

   Default `mid_price = OHLC-1m close` per Q4-A (F2 simplification; F5 backtest uses true mid-price from intra-candle replay).

8. **SL/TP cross detection on OHLC-1m candle**: a candle closes at time T with `(open, high, low, close)`. For an open BUY position with SL at `sl_price`:
   - If `low ≤ sl_price ≤ high`: SL crosses during this 1m bar. Fill at `sl_price` (no slippage on SL fill; brief is silent — see OQ-3 sub-question).
   - Else: no SL trigger from this candle.

   For TP: same logic with `tp_price`. SELL positions: SL above entry, TP below entry, cross logic flipped.

   **F2 simplification per Q4-A**: F2 does NOT model intra-candle path; if both SL AND TP cross in the same candle (rare; only possible for tight SL/TP), pessimistic = SL fills first (worst-case for paper P&L; conservative). F5 backtest harness §12.2 generates intra-candle deterministic path (O→toward-extreme-first→other-extreme→C) and resolves correctly.

### `packages/exchange/paper/slippage.py` (NEW — extracted from adapter.py)

Pure-functional slippage model implementations. Each function is module-level, takes Decimal inputs, returns Decimal slippage_amount. No state, no I/O.

```python
"""§12.1 line 1937 slippage models (pure-functional).

Each function computes ``slippage_amount`` for a fill given the
current candle context + per-bot config. Sign convention applied at
caller site (buy: add to mid_price; sell: subtract).

Hand-verifiable per docs/plans/T-213.md §Hand verification.
"""
from __future__ import annotations
from decimal import Decimal


def fixed_pct(*, price: Decimal, fixed_slippage_pct: Decimal) -> Decimal:
    """``slippage = price * fixed_slippage_pct``. Constant pct of price.

    Example: price=65000, pct=0.0005 → slippage=32.5.
    """
    return price * fixed_slippage_pct


def proportional_to_qty(
    *, price: Decimal, qty: Decimal, qty_slippage_coeff: Decimal
) -> Decimal:
    """``slippage = price * qty_slippage_coeff * abs(qty)``.

    Models market impact: larger orders pay more slippage.
    Example: price=65000, qty=0.5, coeff=0.0001 →
    slippage = 65000 * 0.0001 * 0.5 = 3.25.
    """
    return price * qty_slippage_coeff * abs(qty)


def half_spread(
    *, high: Decimal, low: Decimal, half_spread_factor: Decimal
) -> Decimal:
    """``slippage = (high - low) / 2 * half_spread_factor``.

    Uses candle's high-low range as spread proxy in F2 (F5 backtest
    uses real bid/ask spread). ``half_spread_factor`` is per-bot
    multiplier (typically 1.0 for full half-spread; 0.5 for inside
    half-spread; etc.).
    Example: high=65100, low=64900, factor=1.0 →
    slippage = (65100 - 64900) / 2 * 1.0 = 100.
    """
    return (high - low) / Decimal("2") * half_spread_factor
```

"""§12.1 line 1940 fee computation."""
from __future__ import annotations
from decimal import Decimal


def compute_fee(
    *, qty: Decimal, fill_price: Decimal, fee_rate: Decimal
) -> Decimal:
    """``fee = abs(qty) * fill_price * fee_rate``. Always positive.

    Example: qty=0.5, price=65032.5, rate=0.0006 →
    fee = 0.5 * 65032.5 * 0.0006 = 19.50975.
    Stored as NUMERIC(20, 8) per paper_executions schema (T-212).
    """
    return abs(qty) * fill_price * fee_rate

### §A — Slippage models

#### §A.1 — `fixed_pct`

Inputs: `price = Decimal("65000")`, `fixed_slippage_pct = Decimal("0.0005")` (5 bps).

Hand compute:
```
slippage = 65000 * 0.0005 = 32.5
```

Test fixture: `assert slippage.fixed_pct(price=Decimal("65000"), fixed_slippage_pct=Decimal("0.0005")) == Decimal("32.5000")`.

Decimal exactness: `65000 * 0.0005` in Python decimal default context = `32.5000` (4 trailing zeros from operand scale union). Test assertion uses `==` which is exact for Decimal.

#### §A.2 — `proportional_to_qty`

Inputs: `price = Decimal("65000")`, `qty = Decimal("0.5")`, `qty_slippage_coeff = Decimal("0.0001")`.

Hand compute:
```
slippage = 65000 * 0.0001 * abs(0.5)
         = 6.5 * 0.5
         = 3.25
```

Test fixture: `assert slippage.proportional_to_qty(price=Decimal("65000"), qty=Decimal("0.5"), qty_slippage_coeff=Decimal("0.0001")) == Decimal("3.25000")`.

#### §A.3 — `half_spread`

Inputs: `high = Decimal("65100")`, `low = Decimal("64900")`, `half_spread_factor = Decimal("1.0")`.

Hand compute:
```
range = 65100 - 64900 = 200
half_range = 200 / 2 = 100
slippage = 100 * 1.0 = 100.0
```

Test fixture: `assert slippage.half_spread(high=Decimal("65100"), low=Decimal("64900"), half_spread_factor=Decimal("1.0")) == Decimal("100.0")`.

### §B — Fee computation

Inputs: `qty = Decimal("0.5")`, `fill_price = Decimal("65032.5")`, `fee_rate = Decimal("0.0006")` (6 bps).

Hand compute:
```
fee = abs(0.5) * 65032.5 * 0.0006
    = 0.5 * 65032.5 * 0.0006
    = 32516.25 * 0.0006
    = 19.509750
```

Test fixture: `assert compute_fee(qty=Decimal("0.5"), fill_price=Decimal("65032.5"), fee_rate=Decimal("0.0006")) == Decimal("19.50975000")`.

### §C — Market order fill price

#### §C.1 — Buy market order

Setup:
- Last observed `OhlcCandlePayload` for `BTCUSDT`: `close = Decimal("65000")`, `high = 65100`, `low = 64900`.
- Bot config: `slippage_model = "fixed_pct"`, `fixed_slippage_pct = Decimal("0.0005")`, `fee_rate = Decimal("0.0006")`.
- Order: `side = "buy"`, `qty = Decimal("0.5")`.

Hand compute:
```
mid_price = last_close = 65000
slippage_amount = fixed_pct(price=65000, fixed_slippage_pct=0.0005) = 32.5
fill_price = mid_price + slippage_amount = 65000 + 32.5 = 65032.5  (buy: ADD)
fee = compute_fee(qty=0.5, fill_price=65032.5, fee_rate=0.0006) = 19.50975
```

Test fixture asserts `fill_price == Decimal("65032.5000")` and `fee == Decimal("19.50975000")`.

#### §C.2 — Sell market order

Same setup, but `side = "sell"`:

```
fill_price = mid_price - slippage_amount = 65000 - 32.5 = 64967.5  (sell: SUBTRACT)
fee = compute_fee(qty=0.5, fill_price=64967.5, fee_rate=0.0006) = 19.49025
```

Test fixture asserts `fill_price == Decimal("64967.5000")` and `fee == Decimal("19.49025000")`.

### §D — SL/TP cross detection

**SL/TP fill price + fee semantics** (BLOCKER 5 resolution): per OQ-3 sub-question default, fills at exact trigger price (no slippage on SL/TP fills). Per §12.1 line 1940 ("fees deducted at fill time, same as live"), fees apply on SL/TP closes too. Each §D case includes hand-computed fee value alongside fill_price.

#### §D.1 — SL cross on buy position

Setup:
- Open BUY position: `entry_price = Decimal("65000")`, `qty = Decimal("0.5")`, `sl_price = Decimal("64500")`, `tp_price = Decimal("65500")`, `tpsl_mode = "Full"`, `fee_rate = Decimal("0.0006")`.
- Closed candle: `open = 65000`, `high = 65050`, `low = 64400`, `close = 64600`.

Hand compute:
```
SL cross check: sl_price = 64500. Is 64500 in [low=64400, high=65050]? YES.
SL fill_price = sl_price = 64500.

TP cross check: tp_price = 65500. Is 65500 in [low=64400, high=65050]? NO (65500 > 65050).
TP does not fill.

Fee on SL fill = compute_fee(qty=0.5, fill_price=64500, fee_rate=0.0006)
              = abs(0.5) * 64500 * 0.0006
              = 0.5 * 64500 * 0.0006
              = 32250 * 0.0006
              = 19.35

Outcome: PendingSLTPFill(symbol="BTCUSDT", side="buy", qty=0.5, trigger_price=64500,
         triggered_at=now_fn(), kind="sl", tpsl_mode="Full"); fee=Decimal("19.350000").
```

#### §D.2 — TP cross on buy position

Same setup, different candle: `open = 65000, high = 65600, low = 64900, close = 65500`.

```
SL cross: sl_price = 64500 in [64900, 65600]? NO (64500 < 64900).
TP cross: tp_price = 65500 in [64900, 65600]? YES.
TP fill_price = tp_price = 65500.

Fee on TP fill = compute_fee(qty=0.5, fill_price=65500, fee_rate=0.0006)
              = 0.5 * 65500 * 0.0006
              = 32750 * 0.0006
              = 19.65

Outcome: PendingSLTPFill(..., kind="tp", trigger_price=65500, tpsl_mode="Full");
         fee=Decimal("19.650000").
```

#### §D.3 — Both SL and TP cross (pessimistic SL-first per Q4-A)

Setup:
- Open BUY position: `entry = 65000, sl = 64500, tp = 65500, tpsl_mode = "Full", fee_rate = 0.0006`.
- Volatile candle: `open = 65000, high = 65700, low = 64300, close = 64600`.

Hand compute:
```
SL cross: 64500 in [64300, 65700]? YES.
TP cross: 65500 in [64300, 65700]? YES.

Both cross. F2 simplification: pessimistic SL-first (worst-case for paper P&L).
SL fill_price = 64500 (TP discarded for this candle).

Fee on SL fill = compute_fee(qty=0.5, fill_price=64500, fee_rate=0.0006)
              = 19.35

Outcome: single PendingSLTPFill(kind="sl", trigger_price=64500, tpsl_mode="Full");
         fee=Decimal("19.350000"); TP NOT enqueued.
```

(F5 backtest harness §12.2: replay intra-candle path O→toward-extreme-first→other-extreme→C; deterministically resolves which fills first based on close-vs-open direction. F2 paper simplification chosen for Q4-A scope discipline.)

#### §D.4 — Sell position SL cross (inverted)

Sell positions: SL is ABOVE entry, TP is BELOW entry.

Setup:
- Open SELL position: `entry = 65000, qty = 0.3, sl = 65500, tp = 64500, tpsl_mode = "Partial", tp_size = 0.1, fee_rate = 0.0006`.
- Closed candle: `high = 65600, low = 64900, close = 65500`.

Hand compute:
```
SL cross: 65500 in [64900, 65600]? YES.
TP cross: 64500 in [64900, 65600]? NO (64500 < 64900).
SL fill_price = 65500.

Fee on SL fill = compute_fee(qty=0.3, fill_price=65500, fee_rate=0.0006)
              = abs(0.3) * 65500 * 0.0006
              = 0.3 * 65500 * 0.0006
              = 19650 * 0.0006
              = 11.79

Outcome: PendingSLTPFill(side="sell", qty=0.3, trigger_price=65500,
         kind="sl", tpsl_mode="Partial"); fee=Decimal("11.790000").
```

**H-013 cross-reference for §D.4**: tpsl_mode propagated through cross detection without `'Full'` default. PendingSLTPFill carries the explicit `Partial` value from `set_trading_stop` registration → `_check_sl_tp_crosses` read → enqueue. Future T-213b consumer (persistence + emission) reads `PendingSLTPFill.tpsl_mode` to decide partial-vs-full close semantics.

## Lifecycle

PaperExchange instance lifecycle (T-213a scope):
1. Constructor: store all DI'd values + initialise empty `_last_price` / `_last_candle` caches.
2. `start_consuming()`: subscribe to `market.ohlc.1m.>` via NATS bus.
3. Each closed candle → `_on_candle` updates caches + invokes `_check_sl_tp_crosses` (T-213a logic) → enqueues fill events (T-213b drains).
4. `place_market_order` etc.: T-213a computes fill_price + slippage + fee; raises `NotImplementedError("T-213b owns persistence")` after computation.
5. `close()`: no-op stays from T-211.

T-213b extends: pool injection, `_check_sl_tp_crosses` writes to paper_*, async-iterator drain of fill queue.

## Edge cases

- **No observed price for symbol**: `place_market_order` raises `PaperExchangeError("No last-observed price for {symbol}; subscribe to market.ohlc.1m.{symbol} first")`. Caller (T-215 adapter pool init) is responsible for `start_consuming()` before first order.
- **In-progress candle**: `_on_candle` ignores `is_closed=False` per Q4-A F2 simplification (only closed candles update last_price + trigger SL/TP checks).
- **SL+TP both cross in same candle**: pessimistic SL-first per Q4-A.
- **Negative qty (sell)**: `abs(qty)` in slippage `proportional_to_qty` + `compute_fee` handles. Sign carries through fill_price direction.
- **Zero qty**: `compute_fee` returns 0; `place_market_order` should reject upstream (T-216 placement pipeline validates non-zero qty before adapter call).
- **Decimal precision overflow**: with default 28-digit context, all real-world scenarios fit. NUMERIC(30,12) coerce happens at T-213b persistence boundary.
