## Public API (committed shape, T-100 mirror verbatim)

### `app/main.py`
```python
def create_app(settings: Settings | None = None) -> FastAPI:
    """Build a configured FastAPI for feature-engine.

    Mirror of T-100 services/market_data/app/main.py minus the
    BinanceWsClient / SubscriptionManager / OhlcPipeline / OhlcBackfill
    composition. T-109 ships pool + bus only; T-110 will extend the
    lifespan with the dispatcher + registry + per-feature consumer.

    Composition split (mirrors T-100 / T-015a/T-015b2a):
    * Synchronous primitives (Settings, structlog logger, Prometheus
      registry) instantiated in create_app body and attached to app.state
      immediately so dependency providers see them before lifespan runs.
    * Asynchronous resources (asyncpg Pool, NatsClient) live in lifespan
      and attach inside the async with block. Teardown in reverse order.
    """
    if settings is None:
        settings = Settings()  # type: ignore[call-arg]

    configure(level=settings.log_level)
    logger = get_logger(settings.service_name, "system")
    registry = make_registry()

    @asynccontextmanager
    async def lifespan(app: FastAPI) -> AsyncIterator[None]:
        # 1. asyncpg pool — packages.db.create_pool wrapper does DSN
        #    scheme validation + application_name injection (per-service
        #    log attribution in PG; T-110 will be the first writer).
        pool = await create_pool(
            settings.database_url,
            application_name=settings.service_name,
        )
        # 2. NATS bus — JetStream context acquired; subscriptions/publishes
        #    ready for T-110's market.ohlc.> consumer + features.updated.*
        #    publisher.
        bus = NatsClient(
            servers=[settings.nats_url],
            name=settings.service_name,
            logger=logger,
        )
        await bus.connect()

        app.state.pool = pool
        app.state.bus = bus

        logger.info("service_started", http_port=settings.http_port)
        try:
            yield
        finally:
            # Reverse order: bus first (drains in-flight publishes),
            # then pool. T-110 will extend with pipeline.stop before
            # bus.close so per-feature task teardown lands while bus
            # is still open.
            await bus.close()
            await pool.close()
            logger.info("service_stopped")

    app = FastAPI(lifespan=lifespan)

    # Sync state attach happens here (not in lifespan) so deps.py providers
    # see typed primitives immediately after create_app() returns. Defensive
    # against tests that hit endpoints outside a TestClient context and
    # never enter the lifespan.
    app.state.settings = settings
    app.state.logger = logger

    app.include_router(health_router)
    app.mount("/metrics", make_metrics_asgi_app(registry))
    return app
```

### `app/health.py`
```python
"""Liveness and readiness routes for feature-engine (§9.3, §18.4).

* GET /health — liveness. Returns 200 {} (T-100 verbatim shape).
* GET /ready — readiness. Returns 200 {"ready": true} when both
  (a) NatsClient is CONNECTED and (b) asyncpg pool acquires inside 1 s.
  Otherwise 503 {"ready": false, "reason": <bus|db>}. Reason precedence
  is bus → db (cheapest check first; first failing reason wins).

No ws-equivalent — feature-engine has no analogous long-lived external
connection. T-110's dispatcher reuses the bus. The reason="db" key
matches T-100's market-data-svc verbatim so monitoring alert rules
work uniformly across services.
"""

_POOL_ACQUIRE_TIMEOUT_SECONDS = 1.0
router = APIRouter(tags=["health"])

@router.get("/health", status_code=status.HTTP_200_OK)
async def health() -> dict[str, str]:
    """Liveness probe — always 200 while the process is alive."""
    return {}

@router.get("/ready")
async def ready(
    bus: Annotated[NatsClient, Depends(get_bus)],
    pool: Annotated[asyncpg.Pool, Depends(get_pool)],
) -> JSONResponse:
    """Readiness probe — 200 when bus + db both healthy."""
    if bus.state is not BusConnectionState.CONNECTED:
        return JSONResponse(
            status_code=status.HTTP_503_SERVICE_UNAVAILABLE,
            content={"ready": False, "reason": "bus"},
        )
    try:
        async with pool.acquire(timeout=_POOL_ACQUIRE_TIMEOUT_SECONDS):
            pass
    except (TimeoutError, asyncpg.InterfaceError, asyncpg.PostgresError):
        return JSONResponse(
            status_code=status.HTTP_503_SERVICE_UNAVAILABLE,
            content={"ready": False, "reason": "db"},
        )
    return JSONResponse(
        status_code=status.HTTP_200_OK,
        content={"ready": True},
    )
```

4. No `FEATURE_ENGINE_FEATURES` env-stopgap — §0.8 anti-hypothetical, T-110 adds when it has a real consumer.
5. Minimal lifespan — pool + bus + state attach + reverse-order shutdown. No `asyncio.create_task` on long-running consumer.
6. HTTP port 8000 (compose service-name DNS isolates).
7. UID/GID 10003 (signal=10001, market=10002).
8. `compose.yaml` mirrors market-data block. No env-stopgap; no nginx/cloudflared route (internal-only).
9. Prometheus scrape entry mirrors market-data job def.
10. ~15 tests total mirroring T-100 minus WS cases.
11. Cap-relevant LOC ~390 source — at the §0.3 cap edge; pre-emptive split deferred (mirror lacks WS/holder/race complexity).
12. No `services/__init__.py` chain extension needed.
13. **`/health` body** = `{}` verbatim from T-100. NOT `{"status": "ok"}`.
14. **`/ready` body** = `{"ready": True}` (success) / `{"ready": False, "reason": <bus|db>}` (fail) verbatim from T-100. Monitoring alert rules pivot on the `ready` boolean and `reason` enum string.
15. **`/ready` reason key for pool failure** = `"db"` (T-100 verbatim, monitoring-alert-stable).
16. **`/metrics`** = `app.mount("/metrics", make_metrics_asgi_app(registry))` from `packages.observability`.
17. **Lifespan instantiates pool + bus only.** Settings, logger, registry instantiated synchronously in `create_app` body before `FastAPI(lifespan=...)` so deps.py providers see them outside lifespan startup.
18. **`pool = await create_pool(...)` from `packages.db`** (T-009 wrapper) — DSN scheme validation + `application_name=settings.service_name` injection for per-service log attribution in PG.
19. **`NatsClient(servers=[settings.nats_url], name=settings.service_name, logger=logger)`** — kwargs match T-100 verbatim.
20. **Pool acquire exception set** = `(TimeoutError, asyncpg.InterfaceError, asyncpg.PostgresError)` verbatim from T-100. Builtin `TimeoutError` (Python 3.12: `asyncio.TimeoutError` is alias).
21. **`LogLevel = Literal["DEBUG", "INFO", "WARNING", "ERROR", "CRITICAL"]`** alias exported.
22. **`get_logger_dep`** name (with `_dep` suffix) avoids collision with `packages.observability.get_logger`.

## Test strategy

15 tests total mirroring T-100 minus WS cases.

### `test_config.py` (~3-4 cases + 4-param env parametrize)
- Defaults: `Settings(database_url="...")` sets service_name/log_level/http_port/nats_url to documented defaults.
- `database_url` required (raises `ValidationError` when unset).
- `extra="ignore"` accepts unknown env vars.
- Env override: `service_name` / `log_level` / `nats_url` overridable (4-param parametrize).

### `test_health.py` (1 case)
- `GET /health` → 200, body `{}`.

### `test_ready.py` (5 cases)
- Happy: bus CONNECTED + pool acquire OK → 200, `{"ready": True}`.
- Bus fail: bus DISCONNECTED → 503, `{"ready": False, "reason": "bus"}`.
- Pool fail (PostgresError): bus CONNECTED + pool.acquire raises → 503, `{"ready": False, "reason": "db"}`.
- Pool fail (TimeoutError): pool.acquire timeout → 503, `{"ready": False, "reason": "db"}`.
- Precedence: bus DISCONNECTED + pool would also fail → 503 `reason="bus"`.

### `test_app_factory.py` (5 cases)
- Factory returns `FastAPI` with expected route set: `/health`, `/ready`, plus `/metrics` ASGI mount.
- `/metrics` ASGI mount serves Prometheus default-collector text.
- Sync state attach pre-lifespan: `app.state.settings` and `app.state.logger` present immediately after `create_app()`.
- Async state attach post-lifespan: `app.state.pool` + `app.state.bus` present after lifespan startup.
- Lifespan reverse-order shutdown: `bus.close` called before `pool.close` (assertion via call-order list on the fakes).

Coverage target: ≥95% on `config.py`/`health.py`; ~80% on `main.py`/`deps.py` is acceptable (T-100 ratio precedent).
