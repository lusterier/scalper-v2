## Public interface

```python
@dataclass(frozen=True, slots=True)
class RuleOutcome:
    """§10.6 plugin Rule output. Adapter target for Condition Protocol tuple."""
    result: bool
    metadata: dict[str, Any] | None = None


class Rule(ABC):
    """§10.6 plugin Rule base class. Plugins inherit and implement evaluate."""
    name: ClassVar[str]
    version: ClassVar[str]

    @abstractmethod
    def __init__(self, params: dict[str, Any]) -> None: ...

    @abstractmethod
    def evaluate(self, ctx: RuleContext) -> RuleOutcome: ...


# Re-export RuleContext from conditions.base for §10.6 verbatim import path.
from packages.scoring.conditions.base import RuleContext

__all__ = ["Rule", "RuleContext", "RuleOutcome"]
```

NEW `packages/scoring/registry.py` (~85 LOC):

```python
def load_plugin_registry(path: Path) -> dict[tuple[str, str], type[Rule]]:
    """Parse plugin_registry.yaml + resolve entry_points to Rule subclasses.

    YAML schema:
        rules:
          - name: oi_squeeze
            version: "2"
            entry_point: plugins.rules.oi_squeeze:OISqueezeRule

    Returns dict keyed on (name, version) tuple → Rule subclass.

    Errors:
    * FileNotFoundError if path missing.
    * ValueError if YAML structure invalid.
    * ImportError if entry_point module not importable.
    * AttributeError if class not in module.
    * TypeError if class doesn't inherit Rule.
    * Pydantic ValidationError if name/version/entry_point malformed.
    """
```

NEW `packages/scoring/conditions/plugin.py` (~75 LOC):

```python
class PluginCondition(BaseModel):
    """§10.2:1707 + §10.6 plugin variant — wraps an instantiated Rule.

    Constructed by T-308 YAML loader: looks up rule_class in registry by
    (name, version), instantiates with params, wraps in PluginCondition.
    PluginCondition.evaluate(ctx) calls Rule.evaluate(ctx) → RuleOutcome
    and adapts to (result, metadata) tuple per Condition Protocol.
    """
    model_config = ConfigDict(frozen=True, arbitrary_types_allowed=True)
    type: Literal["plugin"] = "plugin"
    name: str
    version: str
    rule: Rule  # arbitrary_types_allowed=True — Rule is non-Pydantic ABC

    def evaluate(self, ctx: RuleContext) -> tuple[bool, dict[str, Any] | None]:
        outcome = self.rule.evaluate(ctx)
        return outcome.result, outcome.metadata
```

`packages/scoring/conditions/__init__.py` re-exports `PluginCondition`. `packages/scoring/__init__.py` re-exports `Rule`, `RuleOutcome` from `protocol.py`.

## Validation rules

1. **Plugin Rule class MUST inherit `Rule` ABC** — `load_plugin_registry` raises `TypeError("entry_point '<...>' is not a Rule subclass")` if not.
2. **`name` / `version` ClassVar match registry keys** — `registry[("oi_squeeze", "2")]` resolves the class with `name="oi_squeeze"` and `version="2"` ClassVar attrs. Loader checks `rule_class.name == yaml_name and rule_class.version == yaml_version`; raises `ValueError` on mismatch.
3. **Duplicate `(name, version)` in YAML** — loader raises `ValueError("duplicate plugin entry: name=<X> version=<Y>")`.
4. **YAML structure**: top-level `rules` list; each item has `name: str`, `version: str`, `entry_point: str` (`module:Class` format). Pydantic schema validates.
5. **`entry_point` format**: must contain exactly one `:` separating module path from class name. Loader raises `ValueError("invalid entry_point format: '<X>'; expected 'module.path:ClassName'")`.
6. **`Rule.__init__` signature**: `__init__(self, params: dict[str, Any]) -> None`. Plugins must accept a single `params` dict (operator YAML supplies). Loader does NOT instantiate at registry-load time; that is T-308 YAML loader's job per-rule.
7. **PluginCondition is `frozen=True, arbitrary_types_allowed=True`** — `strict=True` is omitted because Pydantic strict-mode rejects arbitrary-type fields. Defensive: `rule` field is enforced as non-None at construction (Pydantic required field).

## Edge cases

- **Empty `rules` list in YAML**: returns empty dict. Valid (operator may have no plugins).
- **YAML file missing**: `FileNotFoundError`. Caller (T-309 lifespan) decides whether to fail or default to empty.
- **Plugin module raises ImportError on import**: surfaces at registry-load time. Caller fail-fast.
- **Plugin class exists but has no `name` or `version` ClassVar**: `AttributeError` at registry-load.
- **Two plugins with same `(name, version)`**: `ValueError` at registry-load.
- **Plugin `__init__` raises with operator-supplied params**: NOT T-305's concern — that surfaces at T-308 YAML load when PluginCondition is constructed.
- **Plugin `evaluate` raises**: NOT T-305's concern — surfaces at T-307 evaluator call. T-307's `on_error` policy applies.
- **`PluginCondition.rule` is `None`**: Pydantic required field — construction without `rule=` raises ValidationError.

- **Registry load name/version ClassVar mismatch with YAML**: YAML name=foo, class.name="bar" → `ValueError`.
- **PluginCondition construction with valid Rule instance**: succeeds.
- **PluginCondition construction without `rule` field**: ValidationError.
- **PluginCondition.evaluate**: passes ctx to `rule.evaluate`, gets `RuleOutcome(result=True, metadata={"k": "v"})`, returns `(True, {"k": "v"})`. Adapter pattern.
- **PluginCondition.evaluate with metadata=None**: returns `(True, None)` — None passes through.
