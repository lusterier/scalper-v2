## Algorithm specification

Given OHLC values `(o, h, l, c)` where `l ≤ o, c ≤ h` and `l ≤ h`:

| Case | Direction | first_extreme | second_extreme |
|------|-----------|---------------|----------------|
| `c > o` (bullish) | toward high first | `h` | `l` |
| `c < o` (bearish) | toward low first | `l` | `h` |
| `c == o` (doji) | toward high first (per OQ-2=A) | `h` | `l` |

Output: `(o, first_extreme, second_extreme, c)` — a 4-tuple.

The 3 segments `(o → first_extreme), (first_extreme → second_extreme), (second_extreme → c)` define a piecewise-linear price path that visits BOTH extremes within a single 1-minute candle (matching TradingView "Replay" semantics).

### Validation rules (raise ValueError per OQ-3=A)

1. `low > high` → `ValueError("intra_candle: low > high (...)") `.
2. `open < low` or `open > high` → `ValueError("intra_candle: open outside [low, high] (...)")`.
3. `close < low` or `close > high` → `ValueError("intra_candle: close outside [low, high] (...)")`.

(No validation on Decimal type — type-checked statically; runtime trust caller per existing paper-engine convention.)

## Public interface

```python
"""Intra-candle deterministic path generator (T-505 / brief §12.2:1961-1963).

Pure function. No I/O, no globals, no side effects. Output drives F5
PaperExchange replay-mode SL/TP cross detection (T-506) at intra-candle
granularity, replacing T-213a's Q4-A pessimistic SL-first simplification.
"""

from __future__ import annotations

from decimal import Decimal

__all__ = ["generate_intra_candle_path"]


def generate_intra_candle_path(
    *,
    open: Decimal,
    high: Decimal,
    low: Decimal,
    close: Decimal,
) -> tuple[Decimal, Decimal, Decimal, Decimal]:
    """Return deterministic 4-point intra-candle price path per §12.2:1961-1963.

    Path: ``(open, first_extreme, second_extreme, close)`` where
    ``first_extreme = high if close >= open else low`` (doji per OQ-2=A
    behaves as bullish — toward high first).

    Raises ``ValueError`` on malformed OHLC (low > high; open/close
    outside [low, high]).
    """
    if low > high:
        msg = f"intra_candle: low > high (low={low!r}, high={high!r})"
        raise ValueError(msg)
    if open < low or open > high:
        msg = f"intra_candle: open outside [low, high] (open={open!r}, low={low!r}, high={high!r})"
        raise ValueError(msg)
    if close < low or close > high:
        msg = (
            f"intra_candle: close outside [low, high] "
            f"(close={close!r}, low={low!r}, high={high!r})"
        )
        raise ValueError(msg)

    if close >= open:  # bullish or doji (OQ-2=A)
        return (open, high, low, close)
    return (open, low, high, close)
```

## Hand verification

### §A — Bullish candle (c > o)

Input: `o=Decimal("100"), h=Decimal("110"), l=Decimal("95"), c=Decimal("105")`.

Path: `(100, 110, 95, 105)` — visits `H=110` first (toward-high direction since c=105 > o=100), then `L=95`, then closes at `C=105`.

Segments:
- `100 → 110`: rises 10 (touches H).
- `110 → 95`: falls 15 (touches L).
- `95 → 105`: rises 10 (closes at C).

### §B — Bearish candle (c < o)

Input: `o=Decimal("100"), h=Decimal("110"), l=Decimal("95"), c=Decimal("97")`.

Path: `(100, 95, 110, 97)` — visits `L=95` first (toward-low direction since c=97 < o=100), then `H=110`, then closes at `C=97`.

Segments:
- `100 → 95`: falls 5 (touches L).
- `95 → 110`: rises 15 (touches H).
- `110 → 97`: falls 13 (closes at C).

### §C — Doji (c == o)

Input: `o=Decimal("100"), h=Decimal("110"), l=Decimal("95"), c=Decimal("100")`.

Per OQ-2=A: doji behaves as bullish (toward high first because `close >= open` covers `==`). Path: `(100, 110, 95, 100)`.

### §D — Constant candle (o == h == l == c)

Input: `o=h=l=c=Decimal("100")`.

Path: `(100, 100, 100, 100)`. All 4 prices equal. Validation passes (`low ≤ open ≤ high` with all equal).

### §E — Open at extreme (o == l, bullish)

Input: `o=Decimal("95"), h=Decimal("110"), l=Decimal("95"), c=Decimal("105")`.

Path: `(95, 110, 95, 105)`. First-extreme step `95 → 110` rises full range; second `110 → 95` returns to low (== open); close `95 → 105` rises 10. Note: `first_extreme=h=110, second_extreme=l=95=o` — degenerate but algorithmically correct per spec.

### §F — Decimal precision preserved

Input: `o=Decimal("65000.123456789012"), h=Decimal("65010.987654321098"), l=Decimal("64990.111111111111"), c=Decimal("65005.555555555555")`.

Path: `(Decimal("65000.123456789012"), Decimal("65010.987654321098"), Decimal("64990.111111111111"), Decimal("65005.555555555555"))`. All 12 fractional digits preserved; no float coercion.

### §G — Validation: low > high

Input: `o=100, h=95, l=110, c=100` (high < low). Raises `ValueError` mentioning `low > high`.

### §H — Validation: open outside [low, high]

Input: `o=120, h=110, l=95, c=105`. Raises `ValueError` mentioning `open outside [low, high]`.
