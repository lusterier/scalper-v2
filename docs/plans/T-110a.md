## Public API (committed shape)

```python
# packages/features/buffers/registry.py

from __future__ import annotations

from collections import deque
from contextlib import AbstractContextManager
from typing import TYPE_CHECKING, Self

if TYPE_CHECKING:
    from collections.abc import Mapping
    from types import TracebackType

    from packages.features.types import OhlcCandle


class BufferRegistry:
    """Reference-counted rolling-candle buffer registry (H-014).

    One in-memory deque per (symbol, interval) key, sized to the
    pre-declared capacity. Multiple features that share the same
    (symbol, interval) acquire individual handles on the same buffer;
    the buffer is allocated on the 0->1 acquire transition and
    deallocated on the 1->0 release transition. push() routes a candle
    to the buffer if any handle is held; otherwise drops silently
    (defensive against NATS delivering candles for not-yet-acquired
    keys during T-110b startup ordering).

    Pure domain - no asyncio, no NATS, no DB. Single-threaded asyncio
    callers (T-110b feature-engine event loop) serialize access via
    the event loop; sync methods never await, so no Lock is needed.
    """

    def __init__(self, capacity_map: Mapping[tuple[str, str], int]) -> None: ...

    def acquire(self, symbol: str, interval: str) -> BufferHandle:
        """Bump refcount on (symbol, interval); return a fresh handle.

        Raises KeyError if (symbol, interval) is not in capacity_map.
        """

    def push(self, symbol: str, interval: str, candle: OhlcCandle) -> None:
        """Append candle to the buffer for (symbol, interval).

        No-op if the key is not in capacity_map OR has zero acquired
        handles. Buffer enforces fixed maxlen - oldest candle is
        dropped on overflow (deque.append semantics).
        """


class BufferHandle(AbstractContextManager["BufferHandle"]):
    """One reference into a (symbol, interval) buffer.

    Context-manager-style: enter on acquire, exit decrements refcount.
    On the 1->0 transition the underlying buffer is deallocated. After
    __exit__, tail() raises RuntimeError. Double-exit is idempotent.
    """

    @property
    def symbol(self) -> str: ...
    @property
    def interval(self) -> str: ...

    def tail(self, n: int) -> tuple[OhlcCandle, ...]:
        """Return the last <=n candles as an immutable snapshot.

        If buffer holds fewer than n candles, returns what's there
        (shorter tuple). Caller (a Feature) is responsible for
        FeatureUnderflowError on insufficient warmup.

        Raises RuntimeError if the handle has been released.
        """

    def __enter__(self) -> Self: ...
    def __exit__(
        self,
        exc_type: type[BaseException] | None,
        exc: BaseException | None,
        tb: TracebackType | None,
    ) -> None: ...
```

## Hazards (§20)

**H-014 — rolling buffer refcount (generalised from WS).**

Multiple features that consume `(symbol, interval) = (BTCUSDT, 15m)` (e.g., EMA-20, EMA-50, RSI-14) hold individual `BufferHandle` instances on the same shared buffer. When EMA-20 unsubscribes (operator unloads it via T-111 plugin reload, or shutdown order), the buffer must remain alive for EMA-50 and RSI-14. Failure mode = silent dealloc → next push() drops → FeatureUnderflowError on next compute() for the still-active features.

Test: `test_refcount_buffer_survives_one_holder_releasing` (mirrors `test_refcount_sub_survives_one_caller_releasing` from T-101c). Asserts buffer still readable after one of three holders releases.

## Hand verification — refcount lifecycle trace

Goal: walk the H-014 contract step by step so the reviewer / future-me can audit the implementation against an explicit invariant table without re-deriving the semantics.

Setup:
```
capacity_map = {("BTCUSDT", "15m"): 50}
registry = BufferRegistry(capacity_map)
# internal state pre-step-1:
#   _buffers = {}        (lazy-create on first acquire)
#   _counts  = {}
```

| step | action                                                | _counts             | _buffers (key → len/maxlen) | invariant check                          |
|------|-------------------------------------------------------|---------------------|-----------------------------|------------------------------------------|
| 1    | `hA = registry.acquire("BTCUSDT","15m")`              | `{(BTC,15m): 1}`    | `{(BTC,15m): 0/50}`         | 0→1: buffer created                      |
| 2    | `hB = registry.acquire("BTCUSDT","15m")`              | `{(BTC,15m): 2}`    | `{(BTC,15m): 0/50}`         | refcount++; same buffer                  |
| 3    | `hC = registry.acquire("BTCUSDT","15m")`              | `{(BTC,15m): 3}`    | `{(BTC,15m): 0/50}`         | refcount++; same buffer                  |
| 4    | `registry.push("BTCUSDT","15m", candle1)`             | `{(BTC,15m): 3}`    | `{(BTC,15m): 1/50}`         | candle visible to all three holders      |
| 5    | `assert hB.tail(1) == (candle1,)`                     | `{(BTC,15m): 3}`    | `{(BTC,15m): 1/50}`         | snapshot read                            |
| 6    | `hA.__exit__(None,None,None)`                         | `{(BTC,15m): 2}`    | `{(BTC,15m): 1/50}`         | **H-014: buffer survives one releaser**  |
| 7    | `assert hB.tail(1) == (candle1,)`                     | `{(BTC,15m): 2}`    | `{(BTC,15m): 1/50}`         | hB still reads same data                 |
| 8    | `registry.push("BTCUSDT","15m", candle2)`             | `{(BTC,15m): 2}`    | `{(BTC,15m): 2/50}`         | further pushes work                      |
| 9    | `hB.__exit__(None,None,None)`                         | `{(BTC,15m): 1}`    | `{(BTC,15m): 2/50}`         | refcount--; buffer alive                 |
| 10   | `hC.__exit__(None,None,None)`                         | `{}`                | `{}`                        | 1→0: buffer deallocated; key removed     |
| 11   | `registry.push("BTCUSDT","15m", candle3)`             | `{}`                | `{}`                        | no handle held → silent drop (defensive) |
| 12   | `hA.tail(1)` (already released at step 6)             | n/a                 | n/a                         | raises `RuntimeError("handle released")` |
| 13   | `hA.__exit__(None,None,None)` (already released)      | `{}`                | `{}`                        | idempotent no-op                         |
| 14   | `registry.acquire("ETHUSDT","15m")`                   | n/a                 | n/a                         | raises `KeyError` — ETH not in capacity_map |
| 15   | `registry.acquire("BTCUSDT","15m")` (post-dealloc)    | `{(BTC,15m): 1}`    | `{(BTC,15m): 0/50}`         | fresh buffer, **NOT** the old one        |

Critical invariants (each row in the table above must hold):
- I1. `(_buffers.keys() == _counts.keys())` after every step.
- I2. After every step, `_counts[k] >= 1` ⇔ `k in _buffers`.
- I3. `acquire` followed by `release` of the same handle is reference-balanced.
- I4. Buffer at step 15 is a **fresh** deque, not the one from step 1 (no resurrection).
- I5. Each `BufferHandle` instance holds **exactly one** ref; no handle-internal counter.

## Decisions committed

1. Sharing granularity = per-(symbol, interval) shared buffer with refcount. Mirrors T-101c SubscriptionManager + v1 PriceManager.
2. Buffer creation timing = lazy on first acquire. Refcount governs lifetime. Push to no-holders drops silently.
3. Capacity policy = static at registry construction (max warmup of features known at startup, computed by T-110b composition root from `indicators.yaml`). T-111 dynamic plugin reload defers capacity-resize to a future ADR.
4. Tail under-fill = return shorter tuple. Features themselves raise FeatureUnderflowError; registry is dumb storage.
5. tail() return type = `tuple[OhlcCandle, ...]` (immutable snapshot). Avoids accidental mutation; copy cost negligible at warmup ≤ 200 candles.
6. Push to unknown key (not in capacity_map) = silent drop. Defensive against T-110b dispatcher routing a candle whose (symbol, interval) key was filtered out at startup.
7. Push to known key with refcount=0 = silent drop. Defensive against NATS delivering candles for symbols whose features haven't yet acquired (e.g., during warmup ordering).
8. Sync vs async = sync. T-110b is single-threaded asyncio; sync methods that don't await are atomic across the event loop. Pure domain → no asyncio dependency.
9. Handle re-entry = one-shot. After `__exit__`, handle is invalid. Matches SubscriptionManager pattern.
10. Double-release = idempotent no-op.
11. Module location = `packages/features/buffers/` directory, not flat `buffers.py`. Consistent with `builtins/` directory pattern.
12. `packages/features/__init__.py` NOT re-exporting `BufferRegistry`. Buffers are an internal T-110b dep; explicit `from packages.features.buffers import …` keeps the public feature API surface narrow.
13. Generic-over-T = no, specific to `OhlcCandle`. §0.8 anti-hypothetical.

Trace already says "via push-after-acquire showing prior data is gone" — preserve that wording in the test docstring so the intent doesn't get refactored away.
4. `BufferHandle` carries no internal refcount per decision #5 (I5). When implementing, ensure `__exit__` flips a single `_released: bool` flag (not a counter) and `_released=True` short-circuits both `tail()` and double-`__exit__`. Idempotent double-release (test 13) tests this directly.
5. `tail()` returns `tuple[OhlcCandle, ...]` — when constructing the snapshot from the underlying `deque`, slice via `tuple(itertools.islice(deque, max(0, len(deque)-n), len(deque)))` or equivalent, NOT `tuple(list(deque)[-n:])` which materialises the full deque. At capacity 50 this is academic, but the idiom matters for §0.8 anti-future-resize-pain.
