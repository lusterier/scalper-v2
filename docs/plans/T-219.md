## Scope

T-219 replaces `services/execution/app/reconcile.py:reconcile_close`'s `NotImplementedError("T-219: ...")` stub with the full body per §9.5:1594-1599 + ADR-0006:

1. **Acquire per-sub-account `asyncio.Lock`** (D4) — protects against shared-sub-account snapshot interleaving across bots.
2. **Snapshot BEFORE** — `before_total = await adapter.get_closed_pnl_cumulative(sub_account)`.
3. **Sleep** `Settings.execution_closed_pnl_post_close_sleep_s` (default 2.0; D2; H-011).
4. **Snapshot AFTER** — `after_total = await adapter.get_closed_pnl_cumulative(sub_account)`.
5. **Compute delta** — `delta = after_total - before_total`.
6. **`update_trade_close`** (D5) — single PK-keyed UPDATE per H-018: `realized_pnl=delta`, `fees_paid=ps.fees_paid_at_close`, `close_reason=<derived from exec_type>`, `exit_price=final_fill_price`, `closed_at`, `close_order_id` (synthetic-fill resolution via `select_open_order_id_by_trade_id` per ADR-0006 D5 amendment).
7. **`delete_position_state`** (D5) — composite-PK DELETE; T-217a/T-217b lifecycle monitor self-exits next tick (T-217a-shipped graceful-exit path).
8. **Emit `OrderClosed`** to `orders.events.<bot_id>` post-commit (Q2 publish-after-persist; OUTSIDE caller's tx).

**Lifespan integration**: `closed_pnl_locks: dict[str, asyncio.Lock]` keyed on `sub_account` string (D4); built from `adapter_pool.adapters` at lifespan setup; threaded into `ExecutionDispatcher` ctor; dispatcher propagates lock + sub_account + adapter + sleep_s to `reconcile_close`.

**Out of scope (T-220 owns)**: periodic 5-min P&L audit loop + Migration 0009 `trade_pnl_deltas` correction writes.

**Out of scope (T-221 owns)**: post-restart reconciliation of orphan DB ↔ exchange position pairs.

## NEW `services/execution/app/reconcile.py` body (~120 LOC; replaces 9-line stub)

**Split per Gate-1 REVISE BLOCKER #1**: `reconcile_close` is the in-tx persist body (snapshot pair + DB writes); caller-side `emit_post_commit_close_event` publishes `OrderClosed` post-tx-commit (mirror T-216b2 `persist_placement_tx` + `emit_post_commit_events` pattern at `services/execution/app/placement_persist.py:466-500`). Dispatcher's `_process` orchestrates: invokes `reconcile_close` inside its `conn.transaction()`, captures returned `(OrderClosed payload, correlation_id)`, then exits tx and invokes `emit_post_commit_close_event`.

```python
"""§9.5:1592 + §9.5:1594-1599 close-flow reconciliation (T-219 owner per ADR-0006).

Cumulative-delta strategy per ADR-0006 D1-D5 + D6 hazard-test mapping.
Acquires per-sub-account asyncio.Lock (D4), snapshots ``closed_pnl_total``
BEFORE close, sleeps ``Settings.execution_closed_pnl_post_close_sleep_s``
(D2; H-011), snapshots AFTER, delta = realized P&L (full delta to single trade
per D3); writes via ``update_trade_close`` PK-only (D5; H-018) + drops
``position_state`` row inside caller's tx; returns ``OrderClosed`` payload +
``correlation_id`` for caller-side post-commit emit (Q2 publish-after-persist
mirror T-216b2 pattern).
"""

from __future__ import annotations

from decimal import Decimal
from typing import TYPE_CHECKING

from packages.bus.schemas.orders import OrderClosed, subject_for_orders_event
from packages.db.queries.execution import (
    delete_position_state,
    select_open_order_id_by_trade_id,
    update_trade_close,
)

if TYPE_CHECKING:
    import asyncio
    from datetime import datetime

    import asyncpg
    from asyncpg.pool import PoolConnectionProxy
    from structlog.stdlib import BoundLogger

    from packages.bus import NatsClient
    from packages.exchange.protocols import ExchangeClient

    type _DbExecutor = (
        asyncpg.Connection[asyncpg.Record] | PoolConnectionProxy[asyncpg.Record]
    )


__all__ = ["reconcile_close"]


_EXEC_TYPE_TO_CLOSE_REASON = {
    "close": "manual",
    "sl": "sl",
    "trail": "trail",
    "unknown": "unknown",
}


async def reconcile_close(  # noqa: PLR0913 — ctor-style call site
    *,
    conn: _DbExecutor,
    adapter: ExchangeClient,
    bound_logger: BoundLogger,
    bot_id: str,
    symbol: str,
    sub_account: str,
    closed_pnl_lock: asyncio.Lock,
    closed_pnl_post_close_sleep_s: float,
    trade_id: int,
    close_order_id: int | None,
    exec_type: str,
    fees_paid_at_close: Decimal | None,
    final_fill_price: Decimal,
    closed_at: datetime,
) -> tuple[OrderClosed, str, str]:
    """T-219 cumulative-delta close-flow PERSIST body per ADR-0006 D1-D5.

    Returns ``(OrderClosed_payload, correlation_id, exchange_order_id)`` for
    caller-side post-commit emit per Q2 publish-after-persist contract (mirror
    :func:`persist_placement_tx` + :func:`emit_post_commit_events` split from
    T-216b2). Caller (T-218b dispatcher ``_process``) captures the return tuple
    INSIDE the tx, exits the tx (commit), then invokes
    :func:`emit_post_commit_close_event` post-commit.
    """
    import asyncio  # noqa: PLC0415 — runtime sleep, not type-only

    # D5 — synthetic-fill close_order_id resolution.
    if close_order_id is None:
        resolved = await select_open_order_id_by_trade_id(conn, trade_id)
        if resolved is None:
            bound_logger.error(
                "execution.reconcile_close_orphan_trade",
                bot_id=bot_id, trade_id=trade_id, symbol=symbol,
            )
            msg = f"orphan trade: trade_id={trade_id} not found in trades table"
            raise RuntimeError(msg)
        close_order_id = resolved

    # Read correlation_id + exchange_order_id from the close_order_id row
    # (per Gate-1 REVISE BLOCKER #2 + CONCERN #4 — both needed for OrderClosed envelope).
    order_meta = await select_order_meta_by_id(conn, close_order_id)
    if order_meta is None:
        bound_logger.error(
            "execution.reconcile_close_order_meta_missing",
            bot_id=bot_id, trade_id=trade_id, close_order_id=close_order_id,
        )
        msg = f"orders row id={close_order_id} not found"
        raise RuntimeError(msg)
    correlation_id, close_exchange_order_id = order_meta

    # D4 — per-sub-account Lock; narrow scope (BEFORE→sleep→AFTER triplet only)
    # per Gate-1 CONCERN #1. Cross-bot Bybit-ledger correctness explained in
    # plan §"Edge cases" (Bybit is single source of truth for delta).
    async with closed_pnl_lock:
        before_total = await adapter.get_closed_pnl_cumulative(sub_account)
        # D2 — single sleep before AFTER snapshot only.
        await asyncio.sleep(closed_pnl_post_close_sleep_s)
        after_total = await adapter.get_closed_pnl_cumulative(sub_account)

    # D3 — full delta to single trade.
    delta = after_total - before_total

    # D5 — close_reason derivation per ADR-0006.
    close_reason = _EXEC_TYPE_TO_CLOSE_REASON.get(exec_type, "unknown")

    # D5 — atomic close persistence (caller's tx).
    await update_trade_close(
        conn,
        trade_id=trade_id,
        exit_price=final_fill_price,
        realized_pnl=delta,
        fees_paid=fees_paid_at_close,
        closed_at=closed_at,
        close_reason=close_reason,
        close_order_id=close_order_id,
    )
    await delete_position_state(conn, bot_id=bot_id, symbol=symbol)

    bound_logger.info(
        "execution.reconcile_close_persisted",
        bot_id=bot_id, trade_id=trade_id, symbol=symbol,
        realized_pnl=str(delta), close_reason=close_reason,
    )

    order_closed_payload = OrderClosed(
        bot_id=bot_id,
        order_id=close_order_id,
        exchange_order_id=close_exchange_order_id,
        symbol=symbol,
        timestamp=closed_at,
        realized_pnl=delta,
        close_reason=close_reason,
    )
    return order_closed_payload, correlation_id, close_exchange_order_id


async def emit_post_commit_close_event(
    *,
    bus: NatsClient,
    bot_id: str,
    correlation_id: str,
    order_closed_payload: OrderClosed,
    bound_logger: BoundLogger,
) -> None:
    """Post-commit publisher for ``OrderClosed`` (mirror :func:`emit_post_commit_events`).

    Wraps payload in :class:`MessageEnvelope` with correlation_id + publisher
    string per audit-grade event contract. KV-fail-still-X semantic: publish
    failure logged but does NOT raise (DB tx already committed; T-220 audit
    catches missing event via DB-vs-NATS divergence).
    """
    from packages.bus import MessageEnvelope as _Env  # noqa: PLC0415 — avoid cycle

    try:
        envelope = _Env(
            correlation_id=correlation_id,
            publisher="execution-service",
            payload=order_closed_payload.model_dump(mode="json"),
        )
        await bus.publish(subject_for_orders_event(bot_id), envelope)
    except Exception as exc:
        bound_logger.error(
            "execution.event_publish_failed",
            bot_id=bot_id,
            event_type="order_closed",
            error=str(exc),
        )
```

**Tx boundary refactor**: T-218b's `_process` body has the close-trigger nested deep inside `async with self._pool.acquire() as conn, conn.transaction():`. T-219 must restructure so the post-commit emit happens AFTER `conn.transaction()` exits. Two options:

- **(a)** Capture `(order_closed_payload, correlation_id)` as locals inside the close-trigger branch; after the `async with` blocks exit (= commit), emit. Requires moving emit call outside the transaction context manager.
- **(b)** Split `_process` into `_process_in_tx` (returns optional close-event payload tuple) + `_process_post_tx` (emits if payload present). Caller orchestrates.

**Default (a)** — minimal restructure; emit call is at the END of `_process` after the `async with` blocks; if close-trigger didn't fire, the captured payload is None and emit is skipped. ~5 LOC added to T-218b's body.

**Edge case for `fees_paid_at_close`**: T-218b's `_process` body invokes `update_trade_fees_incremental(trade_id, fee_delta=message.fee)` BEFORE the close-trigger branch. So at the point reconcile_close fires, `trades.fees_paid` is already updated for THIS final fill. We need to read it back to thread into `update_trade_close`. Two options:
- **(a)** Add a SELECT before `reconcile_close`: `current_fees = await select_trade_fees(conn, trade_id)`.
- **(b)** Pass `fees_paid_at_close = ps.fees_paid_running` if T-217a's monitor tracks running fees (it doesn't — fees are trade-level, not position_state-level).
- **(c)** Have `update_trade_close` skip writing `fees_paid` (preserve existing value via partial UPDATE).

**Default (c)** — make `fees_paid` a NULL-passthrough kwarg in `update_trade_close`; if None, omit from SET clause. Cleanest — avoids extra SELECT, keeps fees_paid as the running incremental sum from T-218b. Requires `update_trade_close` signature extension.

**`correlation_id` + `exchange_order_id` threading via `select_order_meta_by_id` helper** (per Gate-1 BLOCKER #2 + CONCERN #4 fix): both fields needed for `OrderClosed` envelope. Sourced from orders table by `close_order_id`:

- `correlation_id`: T-216b1 `insert_order` writes it per `services/execution/app/placement_persist.py:382`.
- `exchange_order_id`: T-216b1 `insert_order` writes it; for synthetic close (no orders.id match), `close_order_id` resolves to OPEN order's id per ADR-0006 D5 amendment, and OPEN order's `exchange_order_id` is the entry order's exchange-side id.

**New query helper `select_order_meta_by_id(conn, order_id) -> tuple[str, str] | None`** returning `(correlation_id, exchange_order_id)` — single SELECT WHERE id=$1 PK; trivial; mock-only test per L-008 (no CAST/COALESCE/CASE).

The empty-string TODO from prior draft is GONE — synthetic-close emit now carries the OPEN order's `exchange_order_id` (defensible audit-grade trail; T-220 audit can correlate via shared exchange_order_id of the entry vs close).

## Hand verification (Gate 4 math-validator critical scope)

**Decimal arithmetic**: single subtraction `delta = after_total - before_total`. Both Decimal. Exact.

**Fixture A: H-001 orderId-permutation invariance**
- before_total = `Decimal("100")`, after_total = `Decimal("125.50")`, delta = `Decimal("25.50")`.
- Bybit `closed-pnl` rows produced between snapshots: `[{closedPnl: "12.25", orderId: "ID_X"}, {closedPnl: "13.25", orderId: "ID_Y"}]`. The reconciler reads ONLY the cumulative `total - total`, never per-row `orderId`. Permuting orderIds → same delta = `25.50`.

**Fixture B: H-002 identical-prior-trade isolation**
- Pre-existing closed trade with same `(symbol, qty, entry_price)` already contributed to `before_total`. New close adds `Decimal("8.00")` to `after_total`. delta = 8.00 (only the new contribution; not corrupted by entry+qty match).

**Fixture C: H-011 sleep timing**
- `closed_pnl_post_close_sleep_s = 2.0`. Test `monkeypatch.setattr(reconcile_mod.asyncio, "sleep", ...)` to capture sleep call. Assert sleep called once with arg=2.0 between BEFORE and AFTER snapshots.

**Fixture D: H-012 closed-pnl over WS-accumulation**
- Pre-tick `position_state.running_pnl = Decimal("19.50")` (WS-accumulated, may be stale). closed_pnl_delta = `Decimal("23.75")`. `update_trade_close` is called with `realized_pnl=Decimal("23.75")` (NOT 19.50). Assertion: kwarg matches Decimal-exact.

**Fixture E: D3 full-delta single-trade attribution**
- One trade closes with delta=`Decimal("50.00")`. `update_trade_close` writes 50.00 to ONE trade row. `position_state` deleted via composite PK. Idempotent within tx.

**Fixture F: D4 cross-bot lock serialization**
- Two reconcile_close calls running concurrently with shared lock — second blocks until first releases. Test uses `asyncio.gather` + asserts call ordering on `adapter.get_closed_pnl_cumulative`. Pseudocode:
```python
lock = asyncio.Lock()
adapter.get_closed_pnl_cumulative.side_effect = [Decimal("100"), Decimal("125"), Decimal("125"), Decimal("130")]
# 4 sequential calls; 1st bot: BEFORE=100, AFTER=125, delta=25; 2nd bot: BEFORE=125 (post-1st), AFTER=130, delta=5.
```

**Decimal precision**: all `Decimal × Decimal → Decimal` (subtraction only); no float; no `getcontext().prec` mutations. Bybit `closed-pnl` strings round-trip via `Decimal(item["closedPnl"])` per T-208a-shipped pattern (preserves precision).

## Edge cases

- **`get_closed_pnl_cumulative` raises mid-snapshot**: Lock holds; exception propagates; caller's tx rolls back. `RateLimitError` triggers SharedRateLimiter pause per T-205. Re-tick from dispatcher (next ExecutionEvent for same exec_id is dropped per H-009 dedup ring; recovery via T-221 post-restart reconciliation OR operator-initiated close).
- **Cross-bot Lock contention** (per Gate-1 CONCERN #2 — explicit rationale): bot alpha holds Lock 2s+; bot beta queues; beta's reconcile_close starts after alpha releases. Beta's `before_total` includes alpha's just-recorded close → beta's delta = beta-only. **Why this is correct even though alpha's DB tx may not have committed yet when beta acquires Lock**: Bybit `closed-pnl` REST API reflects exchange ledger truth, NOT execution-service DB state. After Lock release, alpha's close is already recorded in Bybit's ledger (the H-011 sleep ensures Bybit materialized the close-pnl row). Beta's snapshot pair sees alpha's contribution regardless of alpha's DB tx commit timing. Bybit is single source of truth for delta computation per H-012; DB-side persistence is a record-keeping consequence.
- **Lock scope choice — narrow** (per Gate-1 CONCERN #1 — explicit interpretation): ADR-0006 D4 line 89 has both narrow and wide formulations. Plan picks narrow (Lock around BEFORE→sleep→AFTER triplet ONLY; releases before `update_trade_close` + `delete_position_state` + emit). Rationale: minimizes cross-bot blocking; per-trade UPDATE/DELETE/emit are PK-independent and don't need cross-bot serialization. ADR amendment retroactive note may follow post-T-219 ship.
- **`asyncio.Lock` deadlock risk**: single-process scope; Lock is acquired only by reconcile_close; no nested acquisition; no callers hold any other lock first. Deadlock-free by topology.
- **`update_trade_close` fees_paid=None partial update**: SQL becomes `UPDATE trades SET exit_price=$1, realized_pnl=$2, closed_at=$3, close_reason=$4, close_order_id=$5 WHERE id=$6` (5 SET items vs 6 when fees_paid included). Backward-compat for T-216b1's emergency_close path which does pass `fees_paid=Decimal("0")` (placeholder).
- **`OrderClosed` emit failure**: log ERROR + return (no raise). Persistence stands. T-220 audit catches missing event via DB-vs-NATS divergence (but T-220 is audit-only, not replay).
- **`close_order_id=None` via synthetic SL/trail with missing trade row**: `select_open_order_id_by_trade_id` returns None → ERROR + raise → tx rollback. Defensive (matches T-218b's orphan_synthetic_fill pattern).
- **PaperExchange `sub_account` validation**: paper requires `sub_account == str(bot_id)` per Decision #8; lifespan threads `str(bot_id)` for paper bots, real sub_account string for Bybit bots.

## Write-time guidance (for plan-reviewer carry-forward)

1. **`asyncio.sleep` import inside function**: mirror T-217a pattern — `import asyncio  # noqa: PLC0415` inside function body so test fixtures can `monkeypatch.setattr(reconcile_mod.asyncio, "sleep", ...)`.

2. **Lock acquisition spans BEFORE→sleep→AFTER triplet only**: not extended over `update_trade_close` + `delete_position_state` + emit. Rationale: post-snapshot DB writes don't need cross-bot serialization (per-trade UPDATE is independent); minimizing Lock hold time reduces cross-bot blocking.

3. **`fees_paid=None` partial update in `update_trade_close`**: SQL builder pattern — if kwarg is None, omit `fees_paid = $X` from SET clause. Test pin `test_update_trade_close_omits_fees_paid_from_set_when_kwarg_none` verifies. T-216b1's `emergency_close` callers pass `Decimal("0")` not None, so backward-compat preserved.

5. **`OrderClosed` envelope `realized_pnl` field**: schema must include this field. Verify `packages/bus/schemas/orders.py:OrderClosed` already has it (T-216a-shipped per §8.4:1356-1363).

6. **`close_reason` mapping defensiveness**: `_EXEC_TYPE_TO_CLOSE_REASON.get(exec_type, "unknown")` — unknown exec_type falls back to "unknown" close_reason. T-220 audit catches.

7. **`exec_type` threading from T-218b dispatcher**: `_process` already has `exec_type` in scope post-`_derive_exec_type`. T-218b's reconcile_close call adds `exec_type=exec_type` kwarg.
