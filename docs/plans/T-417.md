## Purpose

Ship Section 6 Feature inspector per BRIEF §14.3:2065 — feature browser (filter by name prefix) + select feature+symbol → chart historical values + current stale/fresh status indicator. Single TanStack Router file `routes/features.tsx` per OQ-1=A (selection state internal — feature_name has dots like `ind.btcusdt.15m.ema_20` which complicate URL encoding; same-page state pattern simpler than `/features/$featureKey`).

**Polymorphic values per OQ-2=B**: features have `value_num | value_bool | value_json` (only ONE populated per feature definition; backend returns the other 2 as null). Numeric features → Recharts `<LineChart>` (mirror T-413 PnlChart). Non-numeric features → "Not chartable — JSON/bool view" placeholder + simple table of last 100 historical values.

**Staleness threshold per OQ-3=A**: client-side computation (backend stays policy-free per T-404 OQ-6=C). Single `STALENESS_MS = 5 * 60_000` named constant (5 minutes). Status: `now - latest.computed_at > STALENESS_MS` → stale (red dot); else fresh (green dot).

**Time-range picker per OQ-4=A**: TimeRangePicker (1h / 24h / 7d / 30d / custom). Default 24h.

**History page size per OQ-5=A**: backend default 1000 (chart-resolution scale; max 5000). T-417 ships single fetch (no pagination) — chart consumes the 1000-point batch.

T-417 reuses T-413 PnlChart pattern + T-411 DataTable + format-time helpers. F4-specific shape: polymorphic value rendering (numeric chart vs non-numeric table-fallback).

**Explicitly out of scope for T-417**: per-feature threshold metadata (F5+ — backend would need to expose feature config); JSON-tree drill-down for `value_json` features (F5+ — T-417 ships JSON.stringify pretty-print only); manual refresh button on top of TanStack Query staleTime (F5+); pin/star favorite features (F5+); side-by-side compare 2 features (F5+); CSV export of history (F5+); per-feature alert config (F5+); error boundary (T-418+); CEST/UTC toggle (F5+); mobile responsive (per §14.2:2056); Storybook for chart variants; per-symbol filter dropdown above prefix (operator types prefix as feature_name fragment which already disambiguates per-symbol via row).

### Behavioral contract — query layer

**Latest features** (`/features`):
- `useQuery(["features-latest", prefix], () => apiFetch<FeatureLatestListResponse>("/api/features/latest?prefix=" + encoded + "&limit=100"), { staleTime: STALE_MS })` — 30s staleTime; manual refetch via prefix change.
- Empty `prefix` → omit `?prefix=` URL param entirely (backend default = no filter; same-as-trades-status="all" pattern).

**Feature history** (chart):
- `useQuery(["feature-history", selected, fromIso, toIso], queryFn, { enabled: selected !== null })` — only fires when feature selected.
- URL: `/api/features/history?feature_name=<encoded>&symbol=<encoded>&from=<ISO>&to=<ISO>&limit=1000`.

### Behavioral contract — staleness

```tsx
const STALENESS_MS = 5 * 60_000; // 5 minutes — UI cache TTL aligned w/ feature-engine emit cadence; per L-001 named.

function isStale(computedAtIso: string, thresholdMs: number = STALENESS_MS): boolean {
  return Date.now() - new Date(computedAtIso).getTime() > thresholdMs;
}
```

`<StalenessDot computedAt={row.computed_at} />` — green (fresh) if `!isStale(...)`; red (stale) if `isStale(...)`. Tooltip: `"computed " + minutes + " min ago"` (computed via `Math.floor((Date.now() - new Date(computedAtIso).getTime()) / 60_000)`).

### Behavioral contract — polymorphic value display

`FeatureRow` may have `value_num: number | null` OR `value_bool: boolean | null` OR `value_json: object | array | null`. T-417 display logic:

```tsx
function renderValue(row: FeatureRow): React.ReactNode {
  if (row.value_num !== null) return <span className="font-mono">{row.value_num.toFixed(4)}</span>;
  if (row.value_bool !== null) return <span>{String(row.value_bool)}</span>;
  if (row.value_json !== null) return <span className="text-xs text-muted-foreground">JSON</span>;
  return <span className="text-muted-foreground">—</span>;
}
```

Chart panel logic per OQ-2=B:
- if `selected?.value_num !== null` (latest row had numeric value) → render `<FeatureChart>` from history.
- else → render placeholder + simple `<table>` of last 100 history rows with computed_at + value_bool/value_json display.

**Per WG#X — `value_num` truthiness check**: use `!== null` not `!== undefined` (Pydantic serializes null fields as `null` per T-404 OQ-5 + WG echo). `0` is a valid numeric value (RSI=0 e.g.); avoid `if (value_num)` falsy bug.

### Behavioral contract — feature browser

`<DataTable>` columns: feature_name + symbol + value (rendered via `renderValue`) + computed_at (formatUtcDateTime) + StalenessDot.

`onRowClick` callback: set `selected = { feature_name, symbol, value_num, value_bool, value_json }`. Triggers history fetch + chart render.

### Behavioral contract — time-range picker

Top-bar TimeRangePicker (default 24h). Range applies to chart window only — browser table always fetches latest snapshot (no time filter on `/api/features/latest`).

## Write-time guidance

(plan-reviewer Gate 1 APPROVE 2026-05-05; 10 WG items — must be addressed during implementation; brief-reviewer Gate 3 verifies each verbatim.)

1. **STALENESS_MS rationale comment fix.** Plán uvádza "5min covers feature-engine emit cadence per BRIEF §9.3" — toto je nepresné, §9.3 nešpecifikuje 5min cadence (cadence = candle-close per-feature interval). BRIEF §10.3:1727 má backend staleness ako `2 × interval_seconds` per-feature. STALENESS_MS = 5min je **pragmatic UX threshold** (NIE mathematically derived). Inline komentár musí znieť doslovne v duchu: `// STALENESS_MS = 5 * 60_000 — pragmatic UX threshold for fresh/stale dot; NOT business-logic gating (BRIEF §10.3:1727 backend uses per-feature 2 × interval_sec). Per-feature threshold deferred to F5+ when feature config metadata is exposed via API; per L-001 named constant; per §N9 candidate for VITE_ env exposure if operator wants it tweakable.` Rationale dôvod: drift-checker / brief-reviewer overí citáciu §9.3 — chybná citácia by spadla pri Gate 3.

2. **Polymorphic value `!== null` invariant in tests.** Test `renderValue` (alebo ekvivalent) musí explicitne pokrývať `value_num=0` (RSI=0 alebo MACD=0 valid-numeric edge case). Plán spomína `!== null not falsy` — v teste musí byť assertion `renderValue({ value_num: 0, ... })` renderne `"0.0000"` cez Recharts pipeline (nie `—` / placeholder). Rovnako `value_bool=false` ako valid-bool edge case (nie placeholder).

3. **`prefix` URL param empty-omit symmetry.** Plán deklaruje `?prefix=` URL omitted ak empty. Test #2 (`prefix filter appends ?prefix= URL param (empty omits)`) musí asserovať OBOJE: prefix="ind.btc" → URL obsahuje `?prefix=ind.btc`; prefix="" → URL nemá `prefix=` substring vôbec (nie `?prefix=`). Zachytáva ekvivalent T-414 status="all" precedens behaviour.

4. **`useQuery` enabled gate na history.** History fetch sa nesmie odpáliť ak `selected === null`. Test musí asserovať že na initial mount (selected=null) NIE JE žiadny `/api/features/history` request. Až po row-click sa volá. Použiť MSW spy alebo TanStack Query state introspection.

5. **`computed_at` Z-suffix per §N1.** Filter `from`/`to` na chart history fetch musí skončiť cez `.toISOString()` (Z-suffix UTC). Test #6 explicitne asserujte URL contains `from=...Z` + `to=...Z`. Mirror T-413/T-414 precedens.

6. **Recharts `data=[]` bezpečnostný check pre FeatureChart.** Plán predpisuje "No numeric data" placeholder. Test #9 musí asserovať placeholder render BEZ Recharts ResponsiveContainer mounted (žiadny ResizeObserver chyba pri 0-data). Existing `pnl-chart-empty` testid pattern.

7. **Filter null `value_num` rows pre chart input.** `valueToChartNumber` helper vracia null pre null `value_num`. Test #10 explicitne asserujte: `data = [{value_num: 1.5}, {value_num: null}, {value_num: 2.5}]` → chart renderuje 2 body (nie 3, nie crash). Filter musí byť `data.filter(r => r.value_num !== null)` PRED Recharts mappingom.

8. **`source_version` field passive mirror.** API-types musí obsahovať `source_version: string` (per FeatureResponse mirror), ale browser DataTable ho NEMUSÍ render — plán to deklaruje ako F5+ drill-down detail. WG fix: typecheck-only mirror; nepoužitý field cez `// eslint-disable-next-line @typescript-eslint/no-unused-vars` ak ESLint hlási. Alebo (preferované): pri `FeatureRow` destructure do row-render skip prejde čisto.

9. **`value_json` discriminated union TS type.** Plán deklaruje `Record<string, unknown> | unknown[] | null`. Backend Pydantic union: `dict[str, Any] | list[Any] | None`. Verify TS types renderujú v `renderValue` cez branched check (`Array.isArray(row.value_json) ? "JSON array" : "JSON object"` ak chcete granularitu, alebo unifikovaný "JSON" placeholder per plánovaná simple form). Plán má unifikovaný "JSON" placeholder — drift-checker overí že implementácia nezavádza per-shape branching ktorý plán nedeklaruje.

10. **Left-nav 6th link delta scope.** `routes/__root.tsx` update musí byť čisto +1 NavLink — žiadny iný refactor (plán: ~5 LOC delta). Drift-checker pri >10 LOC delta v __root.tsx vráti DRIFT.
