## Goal

Introduce `packages/bus/protocol.py` with `BusProtocol` (Python `typing.Protocol`) declaring the bus surface that all consumers depend on:
- `async def publish(subject, envelope)`
- `async def subscribe(subject, handler) -> object`
- `async def close()`
- `async def kv_get(bucket, key) -> tuple[bytes, int] | None`
- `async def kv_put(bucket, key, value) -> int`
- `async def kv_update(bucket, key, value, last_revision) -> int`

Retype 4 consumer signatures from `bus: NatsClient` → `bus: BusProtocol` so they accept either `NatsClient` (production / live) or `ReplayBus` (T-507b backtest CLI). T-507b downstream is then mypy-strict-clean when injecting `ReplayBus()` into `make_signal_handler` / `ExecutionDispatcher` / `PaperExchange` / `FeatureResolver`.

Patch `ReplayBus` to satisfy the Protocol:
- Convert `subscribe(...)` from `def` to `async def` (matches NatsClient signature; existing call-sites all `await bus.subscribe(...)` so this is the de-facto contract — confirmed via grep on 3 call-sites in strategy-engine main + execution main + paper adapter)
- Add `async def kv_get` returning `None` always (replay has no KV state; FeatureResolver `_try_kv` already short-circuits on `kv_result is None` per `packages/scoring/resolver.py:181-182`, so this is sufficient for DB-fallback feature path per OQ-5=A)
- Add `async def kv_put` raising `NotImplementedError` (write op is meaningless in replay — fail loud per §0.4)
- Add `async def kv_update` raising `NotImplementedError` (same rationale)

Per OQ-5=A *DB-only fallback in replay*: no FeatureResolver changes. Existing `_try_kv` returns `(None, None)` on `kv_result is None` → resolver falls through to `_try_db` (DB-only feature lookup). Operator backfills features via existing `scripts/backfill_features.py` before running T-507b CLI.

## Public interface

### `BusProtocol` (Protocol)

```python
"""Bus interface contract — Python typing.Protocol declared once;
implementations: NatsClient (live) + ReplayBus (in-process replay).

T-507a: introduced so strategy-engine + execution-service + paper-engine
+ scoring-resolver consumer signatures can accept either implementation
without per-call-site cast / type-ignore. Mypy strict-clean.
"""
from __future__ import annotations

from typing import TYPE_CHECKING, Protocol

if TYPE_CHECKING:
    from packages.bus.client import Handler
    from packages.bus.envelope import MessageEnvelope


class BusProtocol(Protocol):
    """Structural type for bus implementations.

    Implementations:
    - :class:`packages.bus.NatsClient` — production / live NATS JetStream
    - :class:`packages.bus.ReplayBus` — in-process timestamp-ordered replay (T-502)

    `subscribe` returns `object` because NatsClient returns
    `nats.aio.subscription.Subscription` and ReplayBus returns
    `ReplaySubscription`; no current consumer reads the returned value
    (3 call-sites grep-verified at T-507a plan-time:
    strategy_engine/app/main.py:139, execution/app/main.py:176,
    exchange/paper/adapter.py:249 — all `await bus.subscribe(...)`
    discarding the return).

    KV methods: NatsClient implements via JetStream KV; ReplayBus stubs
    return None for kv_get and raise NotImplementedError for
    kv_put/kv_update (replay has no KV write semantic). FeatureResolver
    `_try_kv` handles None gracefully via existing fallback to `_try_db`.
    """

    async def publish(self, subject: str, envelope: MessageEnvelope) -> None: ...
    async def subscribe(self, subject: str, handler: Handler) -> object: ...
    async def close(self) -> None: ...
    async def kv_get(self, bucket: str, key: str) -> tuple[bytes, int] | None: ...
    async def kv_put(self, bucket: str, key: str, value: bytes) -> int: ...
    async def kv_update(
        self, bucket: str, key: str, value: bytes, last_revision: int,
    ) -> int: ...
```

### `ReplayBus` patches

```python
# Convert subscribe to async (no semantic change; matches NatsClient + 3 await call-sites).
async def subscribe(
    self, subject_pattern: str, handler: Handler,
) -> ReplaySubscription:
    ...  # body unchanged from T-502

# NEW: KV stubs.
async def kv_get(self, bucket: str, key: str) -> tuple[bytes, int] | None:
    """Replay has no KV state — always return None.

    FeatureResolver._try_kv handles None via DB-fallback per
    packages/scoring/resolver.py:181-182. Args ignored; signature
    matches NatsClient.kv_get for BusProtocol satisfaction.
    """
    return None

async def kv_put(self, bucket: str, key: str, value: bytes) -> int:
    """Replay does not support KV writes — fail loud per §0.4."""
    msg = "ReplayBus.kv_put: KV writes are not supported in replay mode"
    raise NotImplementedError(msg)

async def kv_update(
    self, bucket: str, key: str, value: bytes, last_revision: int,
) -> int:
    """Replay does not support KV CAS — fail loud per §0.4."""
    msg = "ReplayBus.kv_update: KV CAS writes are not supported in replay mode"
    raise NotImplementedError(msg)
```

### `packages/bus/tests/test_bus_protocol.py` (NEW, ~35 LOC)

Per plan-reviewer concern #5 — use `inspect.getmembers` enumeration pattern (precedent `packages/exchange/tests/test_protocol_conformance.py:50-68`); catches scenario "developer adds method to BusProtocol but forgets to add to ReplayBus stub":

```python
import inspect
from packages.bus import BusProtocol, NatsClient, ReplayBus

def _protocol_method_names() -> set[str]:
    """Enumerate Protocol method names (excluding dunder + typing internals)."""
    return {
        name for name, _ in inspect.getmembers(BusProtocol)
        if not name.startswith("_") and inspect.isfunction(getattr(BusProtocol, name))
    }

def test_natsclient_satisfies_bus_protocol() -> None:
    methods = _protocol_method_names()
    for name in methods:
        assert hasattr(NatsClient, name), f"NatsClient missing BusProtocol.{name}"

def test_replaybus_satisfies_bus_protocol() -> None:
    methods = _protocol_method_names()
    for name in methods:
        assert hasattr(ReplayBus, name), f"ReplayBus missing BusProtocol.{name}"
```

Mypy is the primary structural-typing check at compile time; these runtime tests catch additions where ReplayBus stub is forgotten.
