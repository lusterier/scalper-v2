## L-032 (append to docs/review-lessons.md post-resolution)

**Pattern:** an ops-framed task that `git rm`/renames/edits git-tracked files OR removes compose service-blocks which tests depend on silently regresses the suite — ops-framing is NOT a gate waiver for git-committed deletions (CLAUDE.md "anything into git = development task"). Born from `1750021` (deleted `configs/bots/{alpha,beta}.yaml` + `strategy-engine-beta` block under ops-framing; 8 tests red on master, caught only by CI).
**Active control:** before committing ANY ops-framed task: (1) `git diff --staged --name-status` → for every Deleted/Renamed path, `grep -rlF <basename> tests/ packages/*/tests/ services/*/tests/`; (2) for Modified `compose*.yaml`, `git diff --staged -- compose*.yaml | grep -E '^-[[:space:]]+[a-z0-9-]+:'` to list removed service-block keys, then `grep -rlF <removed-service-name> tests/`; any hit in (1) or (2) ⇒ run those tests (or the fast suite) before commit. The ops/dev label does NOT waive this. Sibling of L-030 (real-config↔test coupling) + L-028 (gates-green-while-reality-differs).
