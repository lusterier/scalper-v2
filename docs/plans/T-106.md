## Public API (committed shape)

```python
# packages/features/types.py
@dataclass(frozen=True, slots=True)
class OhlcCandle:
    """OHLC bucket consumed by Feature.compute (§7.2 schema mirror, §9.3 input).

    Domain candle held by packages.features. T-110 feature-engine
    converts the §8.4 wire OhlcCandlePayload → this domain candle at
    the seam — only closed candles cross that seam, so this type does
    not carry an `is_closed` field (the wire-level classification has
    already discriminated by the time compute is called).
    """
    symbol: str
    interval: str        # "1m" | "5m" | "15m" | ...
    bucket_start: datetime
    open: Decimal
    high: Decimal
    low: Decimal
    close: Decimal
    volume: Decimal
    source: str          # "binance" | "bybit"

@dataclass(frozen=True, slots=True)
class FeatureValue:
    """Computed feature value (§8.4 FeatureUpdate wire schema mirror, lean shape).

    Exactly one of value_num/value_bool/value_json is non-None — enforced
    via __post_init__. value_num kept as Decimal preserves NUMERIC(30,12)
    candle precision through compute; T-110 converts Decimal -> float at
    the FeatureUpdate publish seam to match §8.4. Metadata (feature_name,
    symbol, computed_at, source_version) is the feature-engine and
    scoring concern — held alongside FeatureValue in those layers, not
    on this lean compute-return type.
    """
    value_num: Decimal | None = None
    value_bool: bool | None = None
    value_json: Mapping[str, object] | None = None

    def __post_init__(self) -> None:
        non_none = sum(
            1 for v in (self.value_num, self.value_bool, self.value_json) if v is not None
        )
        if non_none != 1:
            raise ValueError(
                f"FeatureValue must have exactly one of "
                f"(value_num, value_bool, value_json) set; got {non_none}"
            )
```

```python
# packages/features/protocols.py
class Feature(Protocol):
    """§9.3 Feature port — implemented by built-ins (T-107) and plugins.

    name_template uses {symbol}, {interval} (and indicator-specific
    placeholders like {period}) substituted at registration time by
    the feature registry (T-111). source_version pins the output
    against the algorithm version; a behaviour change requires a
    fresh suffix and triggers backfill (T-112). compute is sync
    because it is pure CPU over the warmup_candles-bounded window —
    no I/O.
    """

    source_version: str      # "builtin.ema.v1"
    interval: str            # "15m"
    warmup_candles: int      # required history candle count

    def compute(self, candles: Sequence[OhlcCandle]) -> FeatureValue: ...
```

## Decisions committed (operator-approved)

1. **`FeatureValue.value_num` type = `Decimal`** — preserves NUMERIC(30,12) precision through compute; T-110 converts to float at the FeatureUpdate publish seam.
2. **`OhlcCandle` source = fresh dataclass in `packages/features/types.py`** — keeps `packages.features` decoupled from `packages.bus` and `packages.market`. T-110 maps wire `OhlcCandlePayload` → domain `OhlcCandle` at the seam (~10 LOC mapping fn there, out of T-106 scope).
3. **`Feature` protocol = no `@runtime_checkable`** — would only check methods (not data attributes like `name_template`/`interval`), giving a false sense of security. T-111 plugin discovery does explicit attribute introspection if needed.
4. **`FeatureValue.value_json` type = `Mapping[str, object]`** — read-only abstraction; typed union would prematurely fix the schema (Bollinger upper/middle/lower vs MACD signal/histogram have different shapes).
5. **`pyproject.toml` = stays virtual stub** — no external runtime deps, no other workspace member depends on `scalper-v2-features` yet. Promotion deferred to T-107 (subpackage triggers ADR-0002 surprise 1) or T-110 (first service workspace dep).

## Test strategy

`tests/test_types.py` (~10 cases):

- `OhlcCandle` is frozen + slotted (assignment raises `FrozenInstanceError`).
- `OhlcCandle` round-trips Decimal at NUMERIC(30,12) precision floor (matches T-104a fixture pattern: `Decimal("50000.123456789012")`).
- `FeatureValue(value_num=Decimal("70.5"))` constructs.
- `FeatureValue(value_bool=True)` constructs.
- `FeatureValue(value_json={"upper": Decimal("1"), "lower": Decimal("-1")})` constructs.
- `FeatureValue()` (zero non-None) raises `ValueError`.
- `FeatureValue(value_num=Decimal("70"), value_bool=True)` (two non-None) raises `ValueError`.
- `FeatureValue(value_num=..., value_bool=..., value_json=...)` (three non-None) raises `ValueError`.
- `FeatureValue` is frozen (assignment raises).
- `FeatureValue` is hashable for `value_num` and `value_bool` variants (set-membership). NOTE per plan-reviewer: `value_json` variant is NOT hashable because `Mapping` (e.g. `dict`) is not hashable; the hashable test is scoped to the two scalar variants only. This is documented behaviour, not a bug.

`tests/test_protocols.py` (~3 cases):

- Toy `_AlwaysOneFeature` class with the 4 required attributes + `compute` method; `_assert: Feature = _AlwaysOneFeature()` typechecks (mypy-time guard via direct annotation under `if TYPE_CHECKING:` or runtime assignment).
- `_AlwaysOneFeature().compute([])` returns `FeatureValue(value_num=Decimal("1"))`.
- Attribute access on the instance for `name_template`, `source_version`, `interval`, `warmup_candles` returns the expected literals.

Coverage target: 100% on `packages/features/` (types + Protocol have minimal branches).
