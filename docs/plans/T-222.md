## Purpose

T-222 satisfies the three F2 phase exit criteria from BRIEF §19 Phase F2 lines 2526-2531:

- **E1**: A test webhook against Bybit testnet opens a position, places SL and TP, manages BE move and trail, closes correctly. (Manual operator-runnable; not CI per §11.6 line 1923 / §15 line 2276.)
- **E2**: PaperExchange produces identical event-stream shape to live for the same scenario. (CI test.)
- **E3**: Cumulative-delta reconciliation passes integration test under concurrent-bot simulated closes. (CI test.)

E1 is operator-driven; the deliverable is a markdown runbook. E2 + E3 are pytest tests. After T-222 ships, the operator runs E1 once on Bybit testnet and notes the run output in a follow-up `chore(F2-close)` commit; F2 phase is then declared complete.

This task does NOT add new business logic. All three components exercise existing T-202..T-221 surfaces; T-222 is the verification-and-runbook task that closes the phase.

### NEW `docs/runbooks/F2_E1_testnet_smoke.md` (markdown, ~150 lines)

Operator-runnable checklist with `[ ]` items for each step + the expected log key / SQL row to observe. Sections:

1. **Prerequisites** — Bybit testnet API key/secret in `.env`, `docker compose up -d`, sanity `/health` 200, `bots` table seeded with one testnet bot.
2. **Step 1 — webhook**: POST to `signal-gateway` with sample payload. Observe `signal_gateway.signal_accepted` log → `signal_id` in DB.
3. **Step 2 — order placement**: observe `placement.order_placed` + `OrderPlaced` event → `orders` row + `trades` row (`status='open'`) + `position_state` row.
4. **Step 3 — SL set**: observe `placement.set_trading_stop` + `SLMoved(sl_type='protective')`.
5. **Step 4 — TP set**: observe partial-TP `set_trading_stop(tpsl_mode='Partial')`.
6. **Step 5 — BE trigger**: wait until favorable move ≥ `be_trigger`, observe `lifecycle.be_triggered` + `SLMoved(sl_type='be')`.
7. **Step 6 — trail update**: observe `lifecycle.trail_sl_adjusted` + `SLMoved(sl_type='trail')`.
8. **Step 7 — close**: trigger close (manual TP-fill or SL-hit), observe `OrderClosed` + `trades.realized_pnl` updated + `position_state` row deleted.
9. **Step 8 — audit-loop verification**: wait `execution_audit_tick_interval_seconds` (5min), observe NO `audit.pnl_divergence` (cumulative-delta path matched closed-pnl reading).
10. **Sign-off** — record run timestamp + `correlation_id` + final realized_pnl in chore(F2-close) commit.

The runbook does NOT include destructive testnet actions; operator manually closes the testnet sub-account at end.

### NEW `services/execution/tests/test_paper_parity.py` (~140 LOC, ~3 tests)

Mock-based parity test — no DB, no real bus. Both adapter modes drive the same scripted event-injection flow; the test asserts that the resulting list of NATS-published `MessageEnvelope` payloads has the same shape (envelope `payload` model_dump dicts equal modulo timestamps and exchange IDs).

Approach:
1. Build `FakeNatsClient` that captures every `publish(subject, envelope)` call.
2. Drive a scripted lifecycle through:
   - bybit-mode: `BybitV5Adapter` mock with stubbed `get_positions`, `place_market_order`, `get_fill_price`, `set_trading_stop`, scripted `stream_executions` events.
   - paper-mode: `PaperExchange` real impl in-memory with same scripted price feed.
3. Compare published envelopes:
   - same count, same order, same `subject`s
   - per-envelope: same `event_type`, same field set (modulo `exec_id`, `order_id`, `exchange_order_id`, `timestamp` which are infrastructure-generated)
   - all 4 envelope types covered: `OrderPlaced`, `OrderFilled` (open + sl + close), `SLMoved` (protective + be + trail), `OrderClosed`

Tests:

1. `test_paper_and_bybit_emit_same_envelope_sequence_for_happy_path` — main parity assertion.
2. `test_paper_and_bybit_envelope_field_sets_match_per_event_type` — per `event_type` field-set comparison.
3. `test_paper_and_bybit_emit_sl_moved_for_each_sl_change` — SL move count parity (protective + BE + trail).

E2 OQ-2 verbatim: parity is on `model_dump(mode='json')` exact field-set match modulo timestamps/IDs (infrastructure-generated).

### NEW `services/execution/tests/test_concurrent_bot_cumulative_delta.py` (~165 LOC, ~3 tests)

PG-testcontainer-backed integration test gated on `POSTGRES_TEST_DSN` (per existing F2 pattern at `packages/exchange/paper/tests/test_paper_persistence.py:1125`). Tests T-219 D4 lock semantics under simulated concurrent close events from 2 bots sharing a single sub_account.

Fixture topology:
- 2 paper-mode bots `alpha` + `beta`, both opened on sub_account `shared-sub` per ADR-0004 (shared sub_account allowed).
- Both bots open `BTCUSDT` qty=0.1 trades concurrently.
- Adapter pool builds 2 paper adapters; T-219 reconcile.py shares one `asyncio.Lock` keyed on `shared-sub`.
- Mock `get_closed_pnl_cumulative` returns scripted values: BEFORE=100, AFTER=125 (delta=25 attributed to alpha) and BEFORE=125, AFTER=145 (delta=20 attributed to beta), in close-order serialized by D4 lock.
- Trigger 2 close events via `asyncio.gather` of dispatcher.

Assertions:
- Per-trade `realized_pnl` in `trades` table matches expected delta (25 for alpha, 20 for beta).
- D4 lock serialization: only one BEFORE→sleep→AFTER triplet runs at a time per sub_account.
- No cross-bot bleed: alpha's realized_pnl never includes beta's delta nor vice versa.

Tests:

1. `test_two_bots_share_sub_account_concurrent_closes_apportion_correctly` — main E3 hazard pin (verbatim brief §19 line 2529).
2. `test_lock_serializes_before_sleep_after_triplet_per_sub_account` — D4 mechanism pin.
3. `test_no_cross_bot_pnl_bleed_under_concurrent_close` — invariant pin.

E3 fixture validates ADR-0006 D4 lock scope: lock is keyed on sub_account string, NOT bot_id; multiple bots sharing a sub_account (per ADR-0004) share one Lock instance.

## Edge cases

- **E1 Bybit testnet outage**: runbook step 0 has "if step fails, abort and re-test once API status green" instruction.
- **E1 partial fill scenarios**: explicit "partial fills are normal on testnet under low liquidity; verify SL/TP cover the smallest fill quantum" caveat.
- **E2 timestamp/ID divergence**: explicitly skipped via comparison key set; only field shape + `event_type` + non-infrastructure values compared.
- **E2 paper SLMoved emission**: verifies paper adapter emits SLMoved on `set_trading_stop` (parity with bybit; if missing, brief-reviewer flags as paper bug, not test bug).
- **E3 testcontainer skip**: when `POSTGRES_TEST_DSN` unset, all 3 E3 tests pytest.skip with same reason string as existing pattern.
- **E3 lock fairness**: not asserted (asyncio.Lock is FIFO by implementation, but test only asserts serialization, not ordering).
- **E3 closed_pnl_cumulative scripting**: per-call return values mocked deterministically; no Bybit API contact.

## Hand verification

E1 has no math; E2 has no new math (just shape comparison). E3 has Decimal subtraction only:

- **Fixture A (E3 alpha apportion)**: BEFORE=Decimal("100"), AFTER=Decimal("125"). delta = 125 - 100 = 25. alpha trade realized_pnl = Decimal("25").
- **Fixture B (E3 beta apportion)**: BEFORE=Decimal("125"), AFTER=Decimal("145"). delta = 145 - 125 = 20. beta trade realized_pnl = Decimal("20").
- **Fixture C (E3 sum invariant)**: alpha.realized_pnl + beta.realized_pnl = 25 + 20 = 45 = 145 - 100 = total cumulative delta. Conservation pin.

These mirror T-219 hand-fixtures A-B (subtraction-only). No new arithmetic kinds.
