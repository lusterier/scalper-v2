## Purpose

The single canonical forward-write primitive for `trades.lifecycle_state`: a pure PK-`UPDATE` setting the column to a given `TradeLifecycleState`. T-533b2's exact need ‚Äî every post-trades-row state-transition site calls this (with a direct enum literal, OQ-3=A) alongside its unchanged legacy write. T-530/T-534b1/T-533a-shaped **no-caller** foundation leaf ‚Äî T-533b2 (wiring, blocked-by this) is the first consumer.

## Why a NEW dedicated helper, UNDECORATED (the two load-bearing calls)

- **NEW dedicated helper (OQ-2=A)**: every legacy write helper (`insert_trade`/`update_trade_close`/`update_position_state_*`) stays byte-unchanged ‚Äî T-533b2 calls `update_trade_lifecycle_state` ALONGSIDE the unchanged legacy write at each site. This is the strongest OQ-1=A additive guarantee (legacy capital-path writers untouched; single new write site; 0 ¬ß20 regression) vs folding `lifecycle_state` into existing helpers' SET clauses (capital-path blast radius, regression risk).
- **UNDECORATED (NOT `@non_idempotent`) ‚Äî T-534b1 plan-vs-reality precedent class**: the sibling trades/position_state PK-`UPDATE`-`SET`-column helpers `update_trade_close` (execution.py:292), `update_trade_fees_incremental` (:697), `update_position_state_sl` (:849), `update_position_state_monitor_tick` (:809) carry **NO** `@idempotent`/`@non_idempotent` marker (grep-verified ‚Äî only INSERT/place-order non-idempotent writers like `insert_trade`:174 are `@non_idempotent`). `UPDATE trades SET lifecycle_state=$1 WHERE id=$2` is idempotent-by-construction (re-running writes the identical value, no accumulation, retry-safe). ¬ßN3 ("every external write classified") is satisfied by the established convention: PK-`SET`-column UPDATE helpers are idempotent-by-construction and the codebase deliberately does NOT decorate them; markers are reserved for INSERT/place-order non-idempotent writes. Following the ACTUAL sibling precedent (undecorated) ‚Äî a marker-pin test asserts `is_idempotent==False AND is_non_idempotent==False` (mirror T-534b1 `test_tracked_close_marker_mirrors_undecorated_emergency_close` ‚Äî pre-empts the exact ¬ßN3-marker plan-reviewer concern that hit T-534b1).
