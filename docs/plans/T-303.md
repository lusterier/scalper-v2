# T-303: Series conditions (rising, falling, ema_stack) per §10.2

**Status**: DRAFT for plan-reviewer (2026-05-02)
**Phase**: F3 (fourth numbered F3 task)
**Trace**: F3 §10.2 lines 1700-1702 (3 of 14 condition types — series + multi-feature relationship)

## Purpose

Land 3 series condition variants per BRIEF §10.2 lines 1700-1702:

- `rising`: feature strictly increasing over last N samples
- `falling`: feature strictly decreasing over last N samples
- `ema_stack`: ordered relationship of 3 feature values, direction-aware (e.g. `ema_20 > ema_50 > ema_200` = stack-up)

`rising` + `falling` require **history** — N consecutive samples of ONE feature. `ema_stack` is **multi-feature single-snapshot** — 3 different features at the current evaluation time. T-303 extends `RuleContext` with `feature_history: Mapping[str, list[FeatureValue]]` (chronological, oldest → newest) populated by T-306 resolver. The new field defaults to empty mapping so T-302 simple conditions remain functional without churn.

T-302's 8 simple conditions are NOT modified. T-304 composite + T-305 plugin will land later.

## Validation rules

1. **`RisingCondition` / `FallingCondition`**: read `ctx.feature_history[ctx.feature_ref]`. If missing → `(False, {"error": "feature_history_missing", "feature_ref": ...})`. If `len(history) < n_samples` → `(False, {"error": "feature_history_too_short", "have": len(history), "need": n_samples})`.
2. **Strict monotonicity** per BRIEF §10.2:1701-1702 ("increasing" / "decreasing"): rising requires `history[-N+1] < history[-N+2] < ... < history[-1]` (strict `<`); falling requires strict `>`. Equal-adjacent values fail.
3. **Numeric-only history**: each `FeatureValue.value_num` must be non-None across the N-sample window. Any `value_num is None` → `(False, {"error": "type_mismatch", "expected": "value_num"})`.
4. **`EmaStackCondition`**: ignores `ctx.feature_ref`; uses its own `features` list (exactly 3 refs). Each must be present in `ctx.feature_snapshot` with `value_num is not None`. Missing → `(False, {"error": "feature_missing"|"type_mismatch"})`.
5. **`direction="up"`** asserts `features[0] > features[1] > features[2]` (strict, ema_fast > ema_medium > ema_slow). `direction="down"` asserts `features[0] < features[1] < features[2]` (strict).
6. All conditions are `frozen=True, strict=True`.

## Edge cases

- **`n_samples=2`**: minimum (Pydantic `Field(ge=2)`). `[Decimal("1"), Decimal("2")]` rising → True.
- **History exactly N samples**: takes the whole list. `len(history) == n_samples` is sufficient.
- **History longer than N**: takes last N (`history[-n_samples:]`).
- **Equal adjacent values in rising**: `[Decimal("1"), Decimal("2"), Decimal("2")]` rising n=3 → False (strict).
- **EmaStack with 3 missing features**: returns first-encountered missing-error (alpha → beta → gamma scan order; first miss reports its ref). Subsequent features not checked.
- **EmaStack with `direction="up"` but values are equal**: `ema_20 == ema_50` fails strict `>` → returns False. Operator-config error catch.
- **T-302 simple conditions called with RuleContext that doesn't pass feature_history**: backward-compatible — `feature_history` defaults to `{}` when omitted. T-302 conditions don't read it. Verified by re-running T-302 test suite unchanged.

## Hand verification

T-303 introduces ordinal **comparison math** (Decimal `<`, `>`) over windowed sequences + multi-feature snapshot. No arithmetic. Math-validator gate is in scope (`packages/scoring/`).

Hand-fixtures (per condition):

- **RisingCondition(n_samples=3)** with history=`[Decimal("1"), Decimal("2"), Decimal("3")]` → `(True, None)`.
- **RisingCondition(n_samples=3)** with history=`[Decimal("1"), Decimal("2"), Decimal("2")]` → `(False, None)` (strict <, equal pair fails).
- **RisingCondition(n_samples=3)** with history=`[Decimal("1"), Decimal("2")]` (only 2 samples) → `(False, {"error": "feature_history_too_short", "have": 2, "need": 3})`.
- **RisingCondition(n_samples=3)** with history=`[Decimal("1"), Decimal("3"), Decimal("2"), Decimal("4"), Decimal("5"), Decimal("6")]` → `(True, None)` — uses last 3 = `[4, 5, 6]`.
- **FallingCondition(n_samples=3)** with `[Decimal("3"), Decimal("2"), Decimal("1")]` → `(True, None)`.
- **FallingCondition(n_samples=3)** with `[Decimal("1"), Decimal("2"), Decimal("3")]` → `(False, None)`.
- **EmaStackCondition(features=["ema20", "ema50", "ema200"], direction="up")** with snapshot `{ema20: 60000, ema50: 55000, ema200: 50000}` → `(True, None)` (60000 > 55000 > 50000).
- **EmaStackCondition(direction="up")** with `{ema20: 50000, ema50: 55000, ema200: 60000}` → `(False, None)` (descending order; not stack-up).
- **EmaStackCondition(direction="up")** with `{ema20: 60000, ema50: 60000, ema200: 50000}` → `(False, None)` (equal pair fails strict).
- **EmaStackCondition(direction="down")** with `{ema20: 50000, ema50: 55000, ema200: 60000}` → `(True, None)` (50000 < 55000 < 60000).
- **EmaStackCondition** with `features=[]` or `features=["a", "b"]` or 4 features → ValidationError at construction (`Field(min_length=3, max_length=3)`).
- **RisingCondition / FallingCondition with `n_samples=1`** → ValidationError at construction (`Field(ge=2)`).
