## Architecture: in-process simulated restart (mirror T-512b)

**Flow per OQ-4 Path A**:

```
1. Pre-seed bot row + ohlc_1m candles spanning [created_at, now].
2. Build ShadowRejectedStartPayload + envelope.
3. Build ShadowRejectedWorker A; await start() — subscribes shadow.rejected.start.>.
4. await bus.publish("shadow.rejected.start.<bot_id>", envelope).
5. await asyncio.sleep(0.5) — NATS delivery + handler dispatch + INSERT shadow_rejected row.
6. Capture rejected_id via DB SELECT WHERE signal_id = $1 AND terminated_at IS NULL.
7. Verify NULL (mid-flight invariant).
8. Capture cancelled task BEFORE stop() (1:1 keyed by rejected_id).
9. await shadow_rejected_worker_A.stop() — cancels task mid-flight.
10. Drain cancelled task via contextlib.suppress(CancelledError, Exception) + await.
11. Verify NULL post-stop (cancellation-doesn't-finalize contract per shadow_rejected_worker.py:390-395 try/finally — terminal write INSIDE try, NOT in finally).
12. Build ShadowRejectedWorker B (no start; just constructor — used as resume target).
13. await resume_active_observations_on_startup(...) directly (T-513b1 shipped lifespan hook).
14. Assert per test design.
```

**Hand verification (math sanity check)**:

Per L-014 active control + math-validator gate. T-513b2 reuses T-513b1 + T-513a math (`_compute_thresholds`, `_compute_mfe_mae_pcts`); test fixtures pin classification only (per T-512b WG#7 pattern). Worked example for Test #1 SL trigger:

```
entry = Decimal("65000")
sl_pct = Decimal("0.005")
side = "buy" (action="LONG")
sl_threshold = entry × (1 - sl_pct) = 65000 × 0.995 = 64675 (Decimal-exact)

Candle 4: low = 64600 < sl_threshold (64675) → SL crosses.
T-513a `_make_observation_candle_handler` factory (line 119+) sets obs_state["outcome"] = WOULD_SL
+ terminal_future.set_result() on threshold cross. T-513b1 `_replay_observation_candle_loop` detects
terminal via terminal_future.done() post-each-candle. `_finalize_replay_terminal` writes
`update_shadow_rejected_terminal(... terminal_outcome=WOULD_SL ... mfe_pct=... mae_pct=...)`.

mfe_pct = (best_price - entry) / entry  (best_price ~ entry pre-trigger)
       ≈ 0
mae_pct = (entry - worst_price) / entry = (65000 - 64600) / 65000 ≈ 0.006154
```

These values are not literally asserted in the test (T-513b1 unit tests verify magnitude precision); the test asserts CLASSIFICATION (`'would_sl'`) + non-null mfe_pct/mae_pct.
