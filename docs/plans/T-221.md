## Purpose

`services/execution/app/restart.py` is the execution-service startup-flow per H-020. On boot:

1. Per adapter (per `bot_id`): fetch `Position`s from exchange + `position_state` rows from DB.
2. Compute three sets: matching, orphan_db, orphan_exchange.
3. **Orphan DB** (DB has row, exchange doesn't): close trade with `close_reason='reconcile_gone'` + delete position_state. Audit job (T-220b) catches realized_pnl divergence later via cumulative-delta.
4. **Orphan exchange** (exchange has open position, DB doesn't): per H-026, do NOT match on qty approximations; instead time-window+orderId test:
   - If a `trades` row exists for `(bot_id, symbol)` with `opened_at >= now - race_window_seconds` → race window from in-flight placement → log INFO + skip.
   - Otherwise → unmanaged orphan → `place_market_order(opposite_side, qty, reduce_only=True)` + log WARN.
5. **Matching**: spawn `run_position_monitor_for_trade` task (rehydrate FSM state from `position_state` row) into the lifespan-owned `position_lifecycle_tasks` registry.

All paths emit structured audit-grade log entries (§N2).

## New query helpers

`packages/db/queries/execution.py` (+~55 LOC):

```python
async def select_position_states_for_bots(
    conn: _DbExecutor,
    bot_ids: list[str],
) -> list[PositionStateRow]:
    """Return all open position_state rows for the given bot_ids.

    Empty bot_ids short-circuits to [] without DB roundtrip (defensive
    against empty adapter pool). Reuses PositionStateRow shape from
    select_position_state.
    """
```

```python
async def select_recent_open_trade_exists(
    conn: _DbExecutor,
    *,
    bot_id: str,
    symbol: str,
    since: datetime,
) -> bool:
    """Return True if any trades row exists for (bot_id, symbol) with
    opened_at >= since AND status='open'. H-026 race-window test.

    Note: only ``status='open'`` trades count — a closed trade in the
    last 60s does not indicate an in-flight placement.
    """
```

Both are `@idempotent` SELECT-only (composed under caller marker per T-213b precedent).

SQL constants (verbatim — drift-checker reference):

```sql
-- _SELECT_POSITION_STATES_FOR_BOTS_SQL
SELECT bot_id, symbol, trade_id, side, entry_price, qty, remaining_qty,
       sl_price, tp_price, sl_type,
       best_price, mfe_price, mae_price, running_pnl
FROM position_state
WHERE bot_id = ANY($1::text[])
ORDER BY bot_id, symbol

-- _SELECT_RECENT_OPEN_TRADE_EXISTS_SQL
SELECT 1 FROM trades
WHERE bot_id = $1 AND symbol = $2 AND opened_at >= $3 AND status = 'open'
LIMIT 1
```

`>= $3` (NOT `> $3`) is a verbatim test pin per Fixture D + test #16; boundary tick at `since` exact equals race-window match.


Per-bot loop in `reconcile_on_startup`:

```
positions = await adapter.get_positions()        # H-020 step 1
ps_rows = await select_position_states_for_bots(conn, [bot_id])
                                                  # H-020 step 2

ex_keys = {p.symbol for p in positions if p.size > 0}
db_keys = {row.symbol for row in ps_rows}

orphan_db = [row for row in ps_rows if row.symbol not in ex_keys]
orphan_ex = [p for p in positions if p.size > 0 and p.symbol not in db_keys]
matching  = [row for row in ps_rows if row.symbol in ex_keys]

for row in orphan_db:           # H-020 step 3
    # Path A per operator approval 2026-05-02: resolve close_order_id via
    # select_open_order_id_by_trade_id (synthetic close precedent from
    # T-219 reconcile.py:92-103). Semantic note: close_order_id ==
    # open_order_id for reconcile_gone closes (no actual close order
    # placed). Audit-grade readers MUST filter on
    # close_reason='reconcile_gone' to avoid mismatch interpretation.
    open_oid = await select_open_order_id_by_trade_id(conn, row.trade_id)
    if open_oid is None:
        bound_logger.error(
            "reconcile.orphan_db_open_order_missing",
            bot_id=bot_id, trade_id=row.trade_id, symbol=row.symbol,
        )
        continue
    # Atomicity: update + delete in single tx so a crash between them
    # cannot leave trade row closed with position_state still present.
    # Re-run idempotency: status='closed' trade re-update only shifts
    # closed_at by seconds; close_order_id (audit-grade key) stays stable.
    async with conn.transaction():
        await update_trade_close(
            conn, trade_id=row.trade_id,
            exit_price=Decimal("0"),    # OQ-1 placeholder; T-220b audit catches
            realized_pnl=Decimal("0"),  # OQ-1 placeholder
            fees_paid=None,
            closed_at=now_fn(),
            close_reason='reconcile_gone',
            close_order_id=open_oid,    # Path A — open_order_id reused
        )
        await delete_position_state(conn, bot_id=bot_id, symbol=row.symbol)
    bound_logger.warning("reconcile.orphan_db_closed", ...)

for pos in orphan_ex:           # H-020 step 4 + H-026
    in_race = await select_recent_open_trade_exists(
        conn,
        bot_id=bot_id,
        symbol=pos.symbol,
        since=now_fn() - timedelta(seconds=race_window_seconds),
    )
    if in_race:
        bound_logger.info("reconcile.orphan_exchange_in_race_window", ...)
        continue
    opposite_side = "sell" if pos.side == "buy" else "buy"
    try:
        await adapter.place_market_order(
            symbol=pos.symbol,
            side=opposite_side,
            qty=pos.size,
            reduce_only=True,
        )
        bound_logger.warning("reconcile.orphan_exchange_market_closed", ...)
    except Exception as exc:
        bound_logger.error("reconcile.orphan_exchange_close_failed", ..., error=str(exc))
        # Continue loop — operator alerted via WARN; don't fail-stop startup.

for row in matching:            # H-020 step 5
    task = asyncio.create_task(
        run_position_monitor_for_trade(
            bot_id=bot_id, symbol=row.symbol, trade_id=row.trade_id,
            side=row.side, entry_price=row.entry_price, qty=row.qty,
            pool=pool, bus=bus, adapter=adapter,
            bound_logger=bound_logger,
            poll_interval_s=position_poll_interval_s,
            stale_ticks_threshold=position_poll_stale_ticks,
            now_fn=now_fn,
        ),
        name=f"lifecycle_{bot_id}_{row.trade_id}",
    )
    position_lifecycle_tasks[row.trade_id] = task
    bound_logger.info("reconcile.matching_resumed", ...)
```

OQ-5 fail-fast policy applies to `adapter.get_positions()` failure: re-raise from lifespan. `place_market_order` failure is per-position best-effort (log ERROR + continue) — operator-actionable but not startup-fatal.

## Edge cases

- **Empty adapter pool**: function early-returns; no positions, no DB rows.
- **Adapter has zero positions and zero DB rows**: no orphans, no matching; loop completes silently.
- **`get_positions()` returns `Position` with `size == 0`**: filtered out (size > 0 guard) — flat positions per type docstring convention.
- **Multiple bots share a sub_account**: `get_positions()` is per-adapter (bot-scoped); DB rows filtered per bot_id. No cross-bot bleed.
- **Paper-mode parity (OQ-6 verified)**: paper.get_positions() at packages/exchange/paper/adapter.py:880 returns paper sim state. Same flow.
- **`update_trade_close` failure for orphan_db**: tx rolls back (no partial state — atomicity) + caught at lifespan + re-raised. Reconciliation correctness > availability per OQ-5.
- **`select_open_order_id_by_trade_id` returns None for orphan_db row**: trades-row already gone (DB inconsistency). Log ERROR `reconcile.orphan_db_open_order_missing` + continue loop. T-220b audit and operator alerting catch the broken trade lineage.
- **Race-window query returns trades that are themselves orphans**: edge case — if DB has open trade with no position_state row (T-216b2 partial-failure scenario), race-window check still reports True. Correct: don't market-close exchange position that has a pending DB trade. Operator alerted via T-216 placement_persist_tx_failed log.
