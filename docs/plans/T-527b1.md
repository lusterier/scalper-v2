# T-527b1 — `get_mark_price` adapter-protocol extension (foundation leaf, no consumer)

**Status:** Consolidated plan + ADR-0013 (Proposed), ready for plan-reviewer Gate-1 (reviews the ADR alongside this plan per CLAUDE.md NEEDS-DISCUSSION→ADR path). **Operator OQs (resolved, baked — T-527b plan-stage 2026-05-16, all recommended):** OQ-1=A compute home = execution-service (ADR-0013); OQ-2=A 2-leaf recursive split (this = the protocol-extension foundation); OQ-3=A score→multiplier mapping (binds T-527b2 only). No unresolved questions.

## Purpose

Add a pre-trade reference-price source to the exchange-adapter port so T-527b2 can convert §B.1 notional USD → order qty. `ExchangeClient` exposes no pre-trade price method today (`get_fill_price` is post-fill; `get_instrument_info` returns qty-step/min-notional, no price) — ADR-0013 §Context. NEW `get_mark_price(symbol) -> Decimal` (Bybit `/v5/market/tickers` `markPrice`; paper deterministic stub). **NO consumer** — T-527b2 is first consumer (T-530/T-534a-shaped no-caller foundation leaf). Zero behavior change (a new read method, nothing calls it yet).
