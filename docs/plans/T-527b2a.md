Four pure functions (exact contracts — the math is the deliverable):

1. **`select_tier(total_equity: Decimal, tiers: list[SizingTier]) -> SizingTier | None`**
   `tiers` is strictly-ascending by `balance_min` and non-empty (the T-527a `SizingSection._structural_guards` validator already guarantees this — documented as a precondition, NOT re-validated here). Return the **highest** tier whose `balance_min <= total_equity` (`>=` boundary inclusive). If `total_equity < tiers[0].balance_min` → return **`None`** (the OQ-4=A sub-lowest-tier sentinel; T-527b2b translates `None` → skip-the-signal-before-`place_market_order`). Pure scan, no arithmetic.

2. **`apply_score_multiplier(notional: Decimal, score: float | None, score_multipliers: dict[str, Decimal]) -> Decimal`** (OQ-3=A)
   - `score is None` → return `notional` unchanged (×1.0).
   - else: `k = floor(score)` (int). `present = sorted(int(key) for key in score_multipliers)` (`SizingSection` validator guarantees keys are digit-strings). `clamped = min(max(k, present[0]), present[-1])`. `key = str(clamped)`.
   - if `key in score_multipliers` → return `notional * score_multipliers[key]` (Decimal × Decimal — exact).
   - else (sparse-key gap after clamp, e.g. keys `{"4","9"}`, `clamped=6`) → return `notional` unchanged (×1.0, per OQ-3=A "no usable key → ×1.0").
   `score` (float) is used ONLY for `floor()`→int (a key-selection index, not money); the multiplier and notional are Decimal — no float in the value path (§N1).

3. **`cap_notional(notional: Decimal, max_notional_per_symbol: dict[str, Decimal], symbol: str) -> Decimal`**
   `cap = max_notional_per_symbol.get(symbol, max_notional_per_symbol["default"])` (the `SizingSection` validator guarantees the `"default"` key exists — documented precondition). Return `min(notional, cap)` (Decimal min — caps the post-multiplier notional per §B.1).

4. **`compute_qty_from_sizing(*, total_equity: Decimal, mark_price: Decimal, tiers: list[SizingTier], score: float | None, score_multipliers: dict[str, Decimal], max_notional_per_symbol: dict[str, Decimal], symbol: str) -> Decimal | None`**
   Orchestrator: `tier = select_tier(total_equity, tiers)`; **if `tier is None` → return `None`** (sub-lowest-tier skip sentinel propagates). `notional = apply_score_multiplier(tier.size, score, score_multipliers)`; `notional = cap_notional(notional, max_notional_per_symbol, symbol)`. **Guard: if `mark_price <= 0` → raise `ValueError`** (defensive div-by-zero/negative; mirror `quantize.py`'s pre-flight validation posture — a non-positive mark price is an upstream/exchange anomaly, fail loud, never silently size). Return `notional / mark_price` — a **Decimal division** (Python `Decimal` context, default 28-digit precision; full precision preserved). **No rounding/quantize here** — T-527b2b feeds this into the existing shipped T-529 `quantize_qty` which does the single qty_step round-down (single rounding point, mirror `quantize.py:7-9` round-down doc; introducing a second rounding here would double-round). No `float()` anywhere.

## Hand verification (Gate-4 — the core deliverable; uses the §B.1 alpha.yaml block verbatim)

§B.1 (BRIEF §22 alpha.yaml 3130-3145): `tiers=[{500,700},{1000,1400},{2000,2100},{4000,2800}]`, `score_multipliers={"4":0.75,"5":1.0,"6":1.25,"7":1.5,"8":1.5,"9":1.5}`, `max_notional_per_symbol={"default":3000,"BTCUSDT":5000}`. All values exact `Decimal`.

- **select_tier**: `Decimal("1500")` → 1500 ≥ 1000 and < 2000 → tier `{1000,1400}`. `Decimal("500")` → boundary-inclusive ≥500, <1000 → `{500,700}`. `Decimal("499.99")` → < 500 (lowest) → **`None`**. `Decimal("50000")` → ≥4000 → highest `{4000,2800}`.
- **apply_score_multiplier** (notional `Decimal("1400")`): `score=6.7` → `floor=6`, present `[4,5,6,7,8,9]`, clamp(6)→6, `"6"`→`Decimal("1.25")` → `1400 * 1.25 = Decimal("1750.00")`. `score=2.5` → floor 2, clamp→4, `"4"`→0.75 → `1400*0.75=Decimal("1050.00")`. `score=11.0` → floor 11, clamp→9, `"9"`→1.5 → `Decimal("2100.0")`. `score=None` → `Decimal("1400")` (×1.0). Sparse `{"4":0.75,"9":1.5}`, `score=6.0` → floor 6, present `[4,9]`, clamp(6)→6, `"6"` ∉ → `Decimal("1400")` (×1.0).
- **cap_notional**: `cap_notional(Decimal("1750.00"), max_notional, "ETHUSDT")` → no `ETHUSDT` key → `"default"`=3000 → `min(1750.00, 3000)=Decimal("1750.00")`. `cap_notional(Decimal("6000"), max_notional, "BTCUSDT")` → `BTCUSDT`=5000 → `min(6000,5000)=Decimal("5000")`.
- **compute_qty_from_sizing** (`total_equity=Decimal("1500")`, `mark_price=Decimal("42500.75")`, `score=6.7`, `symbol="ETHUSDT"`): tier `{1000,1400}` → `apply` "6"=1.25 → `1750.00` → cap default 3000 → `1750.00` → `qty = Decimal("1750.00") / Decimal("42500.75")` = `Decimal("0.04117575…")` (full Decimal context precision, no float; T-527b2b's downstream `quantize_qty` rounds to qty_step). `mark_price=Decimal("0")` → `ValueError`. `total_equity=Decimal("499")` → `select_tier`→`None` → returns **`None`** (skip sentinel).
