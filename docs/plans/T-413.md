1. **Open positions table** — symbol / side / entry / current / unrealized P&L / SL / TP / running MFE / MAE

**Live updates via T-408 SSE** per `/events/stream?types=positions,signals,trades`. Hybrid liveness per OQ-2=C — positions refetch every 5s baseline + SSE lifecycle event triggers immediate `queryClient.invalidateQueries(...)`. ### `ui/src/store/sse.ts` (NEW, ~50 LOC)

```tsx
type SSEStatus = "unknown" | "connecting" | "connected" | "disconnected";

interface SSEStoreState {
  status: SSEStatus;
  lastEventAt: number | null; // monotonic ms timestamp; null = never
  subscriberCount: number;     // ref-count for parallel route mounts
  setStatus: (s: SSEStatus) => void;
  recordEvent: () => void;
  incrementSubscribers: () => void;
  decrementSubscribers: () => void;
}

export const useSSEStore = create<SSEStoreState>((set) => ({...}));
```

Plain Zustand store; no persistence. Ref-counted subscribers ensures only one EventSource is open even if multiple components subscribe. **Per WG#X — no global EventSource singleton in module scope**: hook creates EventSource on first subscriber, closes on last.

### `ui/src/lib/hooks/useSSEStream.ts` (NEW, ~70 LOC)

```tsx
type SSEEventType = "positions" | "signals" | "trades";
type SSEEventHandler = (event: SSEEvent) => void;

export function useSSEStream(
  types: ReadonlyArray<SSEEventType>,
  onEvent: SSEEventHandler,
): { status: SSEStatus };
```

Mount → registers subscriber, opens EventSource on first one. Unmount → unregisters; if last, closes EventSource. Status mirrors store. Handler dispatches received events to consumer's callback. Inside the hook the EventSource is module-scoped via Zustand store (so multiple consumer hooks share one connection).

Per L-001 active control: SSE backoff knobs are NOT new business-logic timing — `EventSource` browser API has automatic reconnection (default 3s) + we don't override; backoff seam exists but is browser-controlled. No new Settings exposure needed.

### Behavioral contract — SSE lifecycle

Mount sequence (PerBotPage):

1. `useSSEStream(["positions","signals","trades"], handler)` registers subscriber.
2. If first subscriber: `useSSEStore.subscriberCount === 0` → `incrementSubscribers()` flips to 1 → opens `EventSource("/events/stream?types=positions,signals,trades")` → `setStatus("connecting")` → on `onopen` → `setStatus("connected")` → on every `onmessage` → `recordEvent()` + dispatch to handlers.
3. If not first: `incrementSubscribers()` → handler is appended to a closure list (Zustand keeps `handlers: Array<SSEEventHandler>`).

Unmount sequence:

1. `decrementSubscribers()` → if reaches 0 → `eventSource.close()` → `setStatus("disconnected")`.
2. Handler removed from closure list.

**Per WG#X — handler list lives outside Zustand state** to avoid re-renders on every event. Use module-scoped `Set<SSEEventHandler>` in `useSSEStream.ts` (Zustand only stores serializable state).

**Per WG#X — EventSource error handling**: browser `onerror` fires on disconnect; `EventSource` auto-reconnects (3s default); handler sets `status: "disconnected"` and waits for `onopen` to flip back to `connected`. No manual retry logic.
