3. **OQ-3 — Composition order**: **A** — extend lifespan to 6 steps:
   1. pool (existing)
   2. bus (existing)
   3. **`SharedRateLimiter`** instantiated with bus + Settings rate-limit kwargs
   4. **Load active bots** via `select_active_bots(conn)` filter `status='active'`
   5. **Per-bot construct**: live/testnet → `BybitV5Client + BybitV5PrivateWs + BybitV5Adapter` 3-tuple; paper → `PaperExchange` single instance. **Spawn** `asyncio.create_task(ws.run())` per Bybit bot, append to `app.state.ws_tasks`. 4. **OQ-4 — Shutdown order**: **A** —
   1. `bus.close()` (drains publishes + subscriptions) — must run BEFORE adapter.close so any in-flight publish that touches bus completes against open bus.
   2. **Cancel all ws_tasks + paper_consumer_tasks**, then `await asyncio.gather(*tasks, return_exceptions=True)`.
   3. **Per-bot `adapter.close()`** — for Bybit, calls `await ws.close(); await client.close()`; for Paper, no-op.
   4. `pool.close()`.

3. **Per-bot construction (Bybit live/testnet)**:
   ```python
   def _construct_bybit_adapter(
       *,
       bot_row: BotRow,
       env: Mapping[str, str],
       bus: NatsClient,
       rate_limiter: SharedRateLimiter,
       settings: Settings,
       logger: BoundLogger,
   ) -> tuple[BybitV5Adapter, asyncio.Task[None]]:
       api_key = _require_env(env, f"BOT_{bot_row.bot_id.upper()}_BYBIT_API_KEY")
       api_secret = _require_env(env, f"BOT_{bot_row.bot_id.upper()}_BYBIT_API_SECRET")
       sub_account = _require_env(env, f"BOT_{bot_row.bot_id.upper()}_BYBIT_SUB_ACCOUNT")
       base_url = _BYBIT_TESTNET_REST_URL if bot_row.exchange_mode == "testnet" else _BYBIT_PROD_REST_URL
       ws_url = _BYBIT_TESTNET_WS_URL if bot_row.exchange_mode == "testnet" else _BYBIT_PROD_WS_URL
       client = BybitV5Client(api_key=api_key, api_secret=api_secret, base_url=base_url)
       ws = BybitV5PrivateWs(api_key=api_key, api_secret=api_secret, ws_url=ws_url, logger=logger)
       adapter = BybitV5Adapter(
           client=client,
           ws=ws,
           limiter=rate_limiter,
           bus=bus,
           sub_account=sub_account,
           leverage_cache_ttl_s=settings.bybit_leverage_cache_ttl_s,
       )
       ws_task = asyncio.create_task(ws.run(), name=f"bybit_ws_{bot_row.bot_id}")
       ws_task.add_done_callback(_make_task_exception_logger(logger, bot_row.bot_id))
       return adapter, ws_task
   ```

7. **Lifespan extension** in `services/execution/app/main.py`:
   ```python
   # 3. Shared rate limiter.
   rate_limiter = SharedRateLimiter(
       bus=bus,
       orders_rate=settings.rate_limit_orders_rate,
       orders_capacity=settings.rate_limit_orders_capacity,
       positions_rate=settings.rate_limit_positions_rate,
       positions_capacity=settings.rate_limit_positions_capacity,
       ip_global_rate=settings.rate_limit_ip_global_rate,
       ip_global_capacity=settings.rate_limit_ip_global_capacity,
       pause_ms=settings.rate_limit_pause_ms,
   )

   # 4-5. Adapter pool composition.
   result = await build_adapter_pool(
       pool=pool, bus=bus, rate_limiter=rate_limiter,
       settings=settings, logger=logger,
   )

   # 6. State attach.
   app.state.pool = pool
   app.state.bus = bus
   app.state.rate_limiter = rate_limiter
   app.state.adapters = result.adapters
   app.state.ws_tasks = result.ws_tasks
   app.state.paper_consumer_tasks = result.paper_consumer_tasks

   logger.info("service_started", http_port=settings.http_port, bots_loaded=len(result.adapters))
   try:
       yield
   finally:
       # Reverse shutdown.
       await bus.close()
       all_tasks = [*result.ws_tasks, *result.paper_consumer_tasks]
       for t in all_tasks:
           t.cancel()
       if all_tasks:
           await asyncio.gather(*all_tasks, return_exceptions=True)
       for adapter in result.adapters.values():
           await adapter.close()
       await pool.close()
       logger.info("service_stopped")
   ```

8. **`_make_task_exception_logger`** — done-callback factory:
   ```python
   def _make_task_exception_logger(
       logger: BoundLogger, bot_id: str,
   ) -> Callable[[asyncio.Task[None]], None]:
       def _log(task: asyncio.Task[None]) -> None:
           if task.cancelled():
               return
           exc = task.exception()
           if exc is not None:
               logger.error(
                   "execution.background_task_died",
                   bot_id=bot_id,
                   task_name=task.get_name(),
                   error=str(exc),
               )
       return _log
   ```
   `task.cancelled()` early-out: cancellation is normal shutdown, not error.

## Lifecycle

execution-service lifecycle (post-T-215):

1. Startup:
   - asyncpg pool created.
   - NATS bus connected.
   - SharedRateLimiter constructed (DI bus).
   - `select_active_bots` reads bots table (filter status='active').
   - Per-bot construct adapter; spawn ws_task (Bybit) or paper_consumer_task (Paper).
   - State attach.
2. Steady-state:
   - Future T-216 `orders.requests.<bot_id>` consumers read `app.state.adapters[bot_id]`.
   - Future T-217/T-218 lifecycle / dispatcher tasks own per-trade lifecycle.
   - ws_tasks / paper_consumer_tasks run forever in background.
3. Shutdown:
   - bus.close() drains publishes + subscriptions.
   - All tasks cancelled + gathered.
   - Per-adapter close() called (ws.close + httpx close for Bybit; no-op for Paper).
   - pool.close().

## Edge cases

1. **No bots in DB**: empty `adapters` dict; service starts; /ready 200. Future T-216 receives `orders.requests.<bot_id>` with no matching adapter → DLQ per §9.5 step 2.

2. **Bot row with `status='paused'` or `'archived'`**: filtered out by SQL; no adapter constructed. Operator must update bot status to 'active' + restart service to bring online.

3. **Bot row with unknown `exchange_mode`** (e.g., 'demo'): `_validate_exchange_mode` raises `ValueError` at row mapping; lifespan crashes. Defends against operator typos.

4. **Concurrent pool startups** (multiple workers): F2 design is single-process per service per §3.1; pool composition runs once per process. Multi-worker deployment is out of scope.

5. **Bot with exchange_mode='live' but missing API_SECRET**: `_require_env` raises `MissingBotCredentialsError` at the moment of that bot's construction loop iteration; lifespan crashes BEFORE other bots complete construction. Partial-construction-then-fail is acceptable: subsequent bots are never reached because the exception propagates. **`ORDER BY bot_id` v `select_active_bots` query** (Decision #6) zaisťuje že partial-failure vždy ohlási ten istý prvý-zlyhaný bot pri opakovanom štarte — debugability + reproducible-failure invariant. Bez deterministic ordering by každý reštart mohol crashnúť na inom bote, čo by maskovalo root cause.

6. **Paper bot with malformed `SLIPPAGE_PARAMS_JSON`** (invalid JSON): `json.loads` raises `JSONDecodeError`; lifespan crashes. Same fail-fast invariant.

7. **Paper bot with valid JSON but missing required keys per slippage_model** (e.g., `fixed_pct` requires `fixed_slippage_pct`, env supplies different keys): `PaperExchange.__init__` raises `ValueError` per its own validation; lifespan crashes. T-215 doesn't duplicate that validation.

8. **ws_task crashes (programmer error in T-209 reconnect loop)**: done-callback logs ERROR; service continues with this bot WS-degraded. REST methods on the adapter still work; T-218 dispatcher will see no execution events for this bot. F5+ alerting hook fires on `execution.background_task_died` log line.

9. **Service shutdown mid-construction**: `build_adapter_pool` runs synchronously within lifespan startup (only `asyncio.create_task` is non-blocking). Mid-construction signals (SIGTERM) interrupt the lifespan startup; partial state is not attached to app.state because `result = await build_adapter_pool(...)` awaits to completion before state attach. CancelledError propagates cleanly.

10. **ENV vs `os.environ`**: `build_adapter_pool` accepts `env: Mapping[str, str] | None = None` defaulting to `os.environ`. Tests inject canned `dict[str, str]`. Production uses `os.environ`.
