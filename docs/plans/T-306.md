## Purpose

Land the feature-reference resolver per BRIEF §10.3 lines 1715-1727 — the runtime layer that bridges scoring rules' templated `feature: ind.${signal.symbol}.15m.ema_20` references to actual `FeatureValue` instances. T-307 evaluator will populate `RuleContext.feature_snapshot` (and `feature_history` per T-303) by calling this resolver per-rule.

§10.3 verbatim flow:
1. Substitute `${signal.symbol}` with `signal.symbol.lower()`.
2. Look up in NATS KV `feature_latest` by key `f"{feature_name}:{symbol}"`.
3. If missing, query DB `features` table for latest row matching `(feature_name, symbol)`.
4. If DB also missing, report `data_missing` status.

Staleness: `(now - computed_at).total_seconds() > max_staleness_sec` → status `data_stale`. `max_staleness_sec` default = `2 * interval_seconds` (parsed from feature_ref `ind.btcusdt.15m.ema_20` segment 2).

T-307 will consume resolver outputs to build `feature_snapshot` + map status to `RuleResult.result` field.

## Validation rules

1. **Template substitution** only supports `${signal.symbol}` per BRIEF §10.3:1722 verbatim. Any other template (`${signal.symbol_orig}`, `${signal.action}`, etc.) → `ValueError("unsupported template: <X>")`. Forward-compat: T-306 sets the contract; future tasks extend.
2. **Feature-ref format**: must split into ≥4 dot-segments — `<prefix>.<symbol>.<interval>.<rest...>`. T-306 only validates segments 1+2 (`symbol`, `interval`). `interval` must be in `INTERVAL_DELTA`. Anything else → `ValueError`.
3. **KV value decoding**: bytes → `FeatureUpdate.model_validate_json` → convert to `FeatureValue`. KV decode errors logged WARN + treated as KV miss (fall through to DB).
4. **DB query**: uses `features_latest` index `(feature_name, symbol, computed_at DESC)` per migration 0004:104. Single fetch with `ORDER BY computed_at DESC LIMIT 1`.
5. **Staleness**: if `(now_fn() - computed_at).total_seconds() > max_staleness_sec` → status `data_stale`, value `None`. Default `max_staleness_sec` is `2 * INTERVAL_DELTA[interval].total_seconds()` (parsed from feature_ref).
6. **`now_fn` UTC contract**: `now_fn()` returns tz-aware UTC datetime per §N1. Resolver's compare math uses `(now_fn() - computed_at).total_seconds()` — both must be tz-aware to avoid TypeError.
7. **`signal.symbol` lowercased** per §10.3:1722 verbatim — `signal.symbol.lower()`.

## Edge cases

- **KV bucket missing entirely**: NatsClient.kv_get raises (NotConnectedError or PublishError) — resolver catches at top level, falls through to DB. Logged WARN.
- **KV decode fails** (corrupt payload): log WARN, fall through to DB.
- **DB query returns no row + KV missing**: status `data_missing`, value `None`, error_info `{"path": "kv+db missing"}`.
- **DB query returns row but `computed_at` is stale**: status `data_stale`, value `None`, error_info `{"computed_at": "...", "age_sec": <int>, "max_staleness_sec": <int>}`.
- **DB query returns row + KV missing**: status `ok` if not stale; resolver does NOT back-fill KV (KV is producer-side cache; T-306 is consumer-side resolver).
- **`max_staleness_sec=0`**: any non-zero age triggers `data_stale`. Operator-config edge case.
- **Feature-ref interval not in INTERVAL_DELTA** (e.g. `2m`): `ValueError` at `parse_feature_ref`. Operator YAML config error.
- **FeatureUpdate→FeatureValue conversion**: FeatureUpdate has `value_num: float`, `value_bool: bool`, `value_json: dict`. FeatureValue uses `Decimal` for value_num. Resolver converts `float → Decimal(str(float))` to preserve representation. Hand-verification fixture pin.
- **Both KV and DB present**: KV wins (newer + faster). Resolver returns from KV without DB roundtrip.

## Hand verification

T-306 has **time-arithmetic** for staleness check: `(now - computed_at).total_seconds() > max_staleness_sec`. No financial Decimal compare at this layer. Math-validator gate is in scope (`packages/scoring/`) but content is timestamp comparison + dict marshaling.

Hand-fixtures:

- **Substitute happy path**: `"ind.${signal.symbol}.15m.ema_20"` + `signal.symbol="BTCUSDT"` → `"ind.btcusdt.15m.ema_20"` (lowercased per §10.3:1722).
- **Substitute unsupported template**: `"foo.${signal.unknown_attr}"` → `ValueError("unsupported template")`.
- **Parse feature-ref happy path**: `"ind.btcusdt.15m.ema_20"` → `(symbol="btcusdt", interval="15m")`.
- **Parse feature-ref unknown interval**: `"ind.btcusdt.2m.ema_20"` → `ValueError("unknown interval")`.
- **Parse feature-ref too few segments**: `"ema_20"` → `ValueError("malformed feature-ref")`.
- **Resolve KV hit**: bus returns valid FeatureUpdate JSON → status `"ok"`, value populated, no DB call.
- **Resolve KV miss + DB hit**: bus returns None → fall to DB → row exists → status `"ok"`, value populated.
- **Resolve KV miss + DB miss**: both empty → status `"data_missing"`, value `None`.
- **Resolve KV decode fail + DB hit**: KV returns garbage bytes → log WARN, fall to DB → status `"ok"`.
- **Resolve staleness fail (KV path)**: KV returns FeatureUpdate with `computed_at = now - 3600s`, `max_staleness_sec=1800` → status `"data_stale"`.
- **Resolve staleness fail (DB path)**: KV miss, DB row stale → status `"data_stale"`.
- **Default `max_staleness_sec` derivation**: `15m` interval → `2 * 900 = 1800s`. `1h` → `7200s`. `1m` → `120s`. `1d` → `172800s`.
- **`max_staleness_sec=0` boundary**: any non-zero age (`age_sec=1`) triggers `data_stale`.
- **FeatureUpdate→FeatureValue conversion**: `FeatureUpdate(value_num=1.5)` → `FeatureValue(value_num=Decimal("1.5"))` (string conversion preserves representation).
