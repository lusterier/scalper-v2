# packages/features/errors.py
from packages.core import ScalperError

class FeaturesError(ScalperError):
    """Base class for errors raised by :mod:`packages.features`."""

class FeatureUnderflowError(FeaturesError, ValueError):
    """Raised by :meth:`Feature.compute` when ``len(candles) < warmup_candles``.

    The engine MUST respect the warmup contract; a violation indicates
    an upstream bug (warmup buffer not primed, off-by-one in the
    engine, cagg returning fewer rows than expected).

    Inherits from both :class:`FeaturesError` and :class:`ValueError`:

    - **Primary contract:** ``except FeaturesError`` (project hierarchy).
      §5.4 deprecates raising bare ``ValueError`` for domain errors;
      the typed family is the primary catch.
    - **Convenience:** ``except ValueError`` also matches, so callers
      that already expect ``ValueError`` for argument-shape violations
      need not change. This is *additional* surface, not the contract.
    """
```

# packages/features/builtins/ema.py
class EmaFeature:
    """EMA — TradingView/Wilder convention.

    Seed = SMA of first ``period`` close prices. Subsequent values use
    smoothing factor ``alpha = 2 / (period + 1)``::

        EMA_t = alpha * close_t + (1 - alpha) * EMA_{t-1}

    Stateless: each :meth:`compute` call re-processes the entire
    window from scratch. The feature-engine (T-110) holds the rolling
    buffer; we do not cache prior state on the instance. Engine-side
    incremental compute is a F1+ optimisation if it ever matters at
    sub-10-bot scale.
    """

    def __init__(self, period: int, interval: str = "15m") -> None:
        if period < 1:
            raise ValueError(f"period must be >= 1, got {period}")
        self.period = period
        self.interval = interval
        self.name_template = f"ind.{{symbol}}.{{interval}}.ema_{period}"
        self.source_version = "builtin.ema.v1"
        self.warmup_candles = period

    def compute(self, candles: Sequence[OhlcCandle]) -> FeatureValue:
        if len(candles) < self.warmup_candles:
            raise FeatureUnderflowError(
                f"EMA(period={self.period}) requires >= {self.warmup_candles} "
                f"candles, got {len(candles)}"
            )
        closes = [c.close for c in candles]
        sma_seed = sum(closes[: self.period], Decimal(0)) / Decimal(self.period)
        alpha = Decimal(2) / Decimal(self.period + 1)
        ema = sma_seed
        for close in closes[self.period :]:
            ema = close * alpha + ema * (Decimal(1) - alpha)
        return FeatureValue(value_num=ema)
```

class RsiFeature:
    """Wilder's RSI.

    From a sequence of closes:

    - delta_i = close_i - close_{i-1} for i in 1..N (so we need
      ``period + 1`` closes to get ``period`` deltas).
    - gain_i = max(delta_i, 0); loss_i = max(-delta_i, 0).

## Decisions committed (operator-approved)

1. **EMA seed = SMA of first `period` closes** (TradingView convention).
2. **RSI = Wilder smoothing with SMA seed of first `period` gains/losses** (period + 1 closes needed).
3. **ATR = Wilder smoothing with SMA seed of first `period` true ranges** (period + 1 candles needed).
4. **Bollinger std_dev = population**, **VWAP = daily-only**, **VWAP typical price = (h+l+c)/3**, **MACD output = `{"macd", "signal", "histogram"}` via value_json**, **Bollinger output = `{"upper", "middle", "lower"}` via value_json** — operator decisions baked in from open-questions list; T-107a does not implement these but lists them so T-107b inherits the same paradigm.
5. **Underflow → raise `FeatureUnderflowError(FeaturesError, ValueError)`**. Engine MUST respect warmup contract; violation = bug, fail loud per §0.4. Multiple inheritance: primary contract is `FeaturesError`; `ValueError` is convenience surface only.
6. **Decimal output everywhere** (per T-106 contract); T-110 converts to float at the §8.4 wire seam.
7. **`name_template` instance-baked** with indicator-specific tokens; only `{symbol}` and `{interval}` are left for T-111 registry substitution.
8. **Built-ins are regular classes** (not dataclasses) — stateful service objects with `period`/`interval`/`name_template` attributes set in `__init__`. Per `OhlcBackfill` (T-105) and `OhlcPipeline` (T-104b) precedent. Convention: frozen+slotted dataclass = immutable value type, regular class = stateful service object.
9. **Stateless `compute`** — re-processes entire window each call. O(N) per indicator per closed candle, N = warmup_candles + small. Negligible CPU at sub-10-bot scale; simplicity worth the cost.
10. **Decimal precision = default 28-digit context**, no global mutator (§N6).
11. **TDD per §N4**: write test file with hand-computed expected values FIRST (red), then implement (green), then next indicator. Within a single commit — single-author flow doesn't require test-impl pair to be split across two commits.
