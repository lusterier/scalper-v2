## Purpose

Periodic verification that, for every actively-trailing position, the live Bybit-side stop-loss still matches what the FSM's trail logic would set given the current `best_price` + `trail_pct`. A divergence beyond a configurable relative tolerance ⇒ the FSM's `set_trading_stop` silently failed / partially-applied, or the trail did not keep pace with `best_price` ⇒ emit a `trail_sl_drift_detected` `trading_events` audit row + WARN. Completes the ADR-0011 SL/TP-verification cluster (T-534b2 watchdog owns SL *removal*/naked; T-535 owns out-of-band SL *modification*; T-536 owns trail *drift* — three disjoint diagnostic arms). Emit-only/non-destructive: the per-trade lifecycle monitor remains the single authoritative trail writer and re-asserts on its next tick.

## Hand verification (Gate-4 — REAL Decimal arithmetic)

`expected = _compute_trail_sl_price(side, best_price, trail_pct)` (reused, T-217b-math-validated): Long `best*(1-trail_pct)`, Short `best*(1+trail_pct)`. `drift = abs(exchange_sl - expected) / expected`. Emit iff `drift > tolerance_pct`. Python `Decimal` 28-digit ROUND_HALF_EVEN; div-by-zero guarded by `if expected <= Decimal("0"): continue` (degenerate only — for Long, `expected = best*(1-trail_pct) <= 0` iff `trail_pct >= 1`, an invalid config; for Short `best*(1+trail_pct) > 0` always when `best>0`; the guard is defensive, never hit under valid config).

Worked examples (tolerance default `Decimal("0.001")` = 0.1%):

| side | best_price | trail_pct | expected = compute | exchange_sl | drift = abs(ex−exp)/exp | > 0.001? | emit |
|---|---|---|---|---|---|---|---|
| buy | `100` | `0.005` | `100*(1-0.005)=99.500` | `99.500` | `0/99.5 = 0` | no | no |
| buy | `100` | `0.005` | `99.500` | `99.000` | `0.5/99.5 = 0.005025…` | **yes** | **emit** |
| buy | `100` | `0.005` | `99.500` | `99.450` | `0.05/99.5 = 0.000502…` | no (≤0.001) | no |
| sell | `100` | `0.005` | `100*(1+0.005)=100.500` | `100.600` | `0.1/100.5 = 0.000995…` | no (≤0.001, boundary-just-under) | no |
| sell | `100` | `0.005` | `100.500` | `100.610` | `0.11/100.5 = 0.0010945…` | **yes** | **emit** |
| buy | `45000.50` | `0.003` | `45000.50*(0.997)=44865.4985` | `44865.50` | `0.0015/44865.4985 ≈ 3.34e-8` | no | no |

`abs()` on `Decimal` is exact; division is `Decimal.__truediv__` (28-digit context, ROUND_HALF_EVEN — exact-enough for a tolerance comparison; no `float()`). `> tolerance_pct` is a `Decimal`-vs-`Decimal` comparison (strict `>`; boundary `drift == tolerance` ⇒ NOT emitted — "within tolerance" is `<=`). No EMA/Wilder/seed/closed-form; the only computation is the reused FSM trail formula + one division + one abs + one comparison. Fixtures are hand-authored Decimal string literals (not implementation-against-itself).
