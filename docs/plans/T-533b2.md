6 distinct forward states: **OPEN / BREAKEVEN_SET / TRAILING_ACTIVE / PARTIALLY_CLOSED / CLOSED / RECONCILED**. **OQ-3=A clarified (B1/B2 resolution)**: "no per-site conditional" prohibits introducing a NEW state-selection conditional (e.g. `state = X if … else Y`); it does NOT prohibit wiring a *fixed* literal into each arm of the HOST CODE's pre-existing structural branch. Two sites use a host-existing structural discriminator, each arm a fixed literal: (a) site #9 — `_close_orphan_db` is the sole `reconcile_gone` call-site (→ `RECONCILED`) vs every other `update_trade_close` call-site (→ `CLOSED`); the discriminator is *which call-site*, fixed literal per site. (b) site #7 — the dispatcher's pre-existing `remaining_qty == Decimal("0")` close-trigger `if`/(new)`elif`: the `if` arm's CLOSED is written by site #8 inside `reconcile_close`; the additive `elif` arm writes the fixed literal `PARTIALLY_CLOSED`. No exec_type conditional, no `state = … if … else …` — each arm a fixed literal on a branch the dispatcher already computes. Sites #2+#3 (emergency_close births then closes) write OPEN then CLOSED (uniform "every `insert_trade`→OPEN, every `update_trade_close`→CLOSED except `_close_orphan_db`→RECONCILED" rule; final observable CLOSED — correct). The dispatcher close path (`remaining_qty==0`) is NOT double-written: site #7's `elif` is structurally mutually-exclusive with the close-trigger `if`, and CLOSED is written once by site #8 inside `reconcile_close`.

**Placement**: the `update_trade_lifecycle_state` call goes **immediately after the legacy write, on the same `conn`, inside the caller's existing transaction** (atomic with the legacy state change; no new tx, no new conn, no new I/O round-trip beyond the one UPDATE). ## NOT forward-wired (OQ-4=A fold/skip — code-cited, no clean forward site)

- **`SIGNAL_RECEIVED` / `ORDER_REQUESTED` / `ORDER_PLACED`** — pre-`insert_trade`; no `trades` row exists yet (the row is born at `insert_trade`). OQ-4=A: enum-defined, deliberately not persisted (no carrier; §0.8 — no consumer; a future signals/orders-lifecycle task if one surfaces).
- **`CLOSING`** — the close path is `reconcile_close`-atomic (dispatcher remaining_qty==0 → `reconcile_close` → `update_trade_close`+`delete_position_state` in one flow); there is NO distinct "close order placed, fill unconfirmed" site that holds a `trades` row mid-close. Folds into `CLOSED` (OQ-4=A).
- **`TP_HIT`** — a partial-TP fill is one instance of "a non-open fill that did not zero `remaining_qty`", which is exactly the condition site #7's `elif` arm captures (the corrected site #7 fires for ANY non-open exec_type whose fill leaves the position open — partial_tp/trail/sl/unknown — not partial_tp-only; re-confirmed under the B1 fix). So a TP-hit lands in the same `elif` arm and is observably `PARTIALLY_CLOSED`. No separate momentary `TP_HIT` site distinct from that write. Folds into `PARTIALLY_CLOSED` (OQ-4=A — still holds after the site #7 re-pin: the fold target is unchanged; only the discriminator moved from a wrong exec_type claim to the host's real `remaining_qty` branch).
- **`ORPHANED`** — the only orphan-DB path (`restart._close_orphan_db`) immediately *resolves* the orphan by closing it `reconcile_gone` → `RECONCILED` (site #9). There is no persistent forward site that leaves a trade observably `ORPHANED` (it is a transient resolved within the same reconcile pass). Backfill-only (T-533a maps `status='open'+no-ps → ORPHANED` for historical divergence rows); no forward writer. Documented; not wired.
- **`FAILED`** — **exhaustive grep confirms no code writes `trades.status='error'`/`'failed'`** (only the T-533a `lifecycle.py`/`types.py` mapping docstrings reference it; corroborates the T-533a plan-reviewer exhaustive-grep finding). A failed placement does not persist a `trades` row (DLQ/error path returns before `insert_trade`). No forward `trades`-row carrier → enum-defined, not wired (backfill-defensive only, T-533a `status='error'→FAILED` for a hypothetical future T-221 `error` writer / manual repair). Documented; not wired.

This is OQ-4=A applied with code citations (L-018 — no "X not wired" claim without evidence): every not-wired state either has no `trades` row (pre-row 3), no distinct site (CLOSING/TP_HIT fold), or no forward writer (ORPHANED resolved-to-RECONCILED; FAILED no `status='error'` writer).
