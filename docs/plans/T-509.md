## Goal

Add backtest worker to analytics-api lifespan per BRIEF §12.2 (worker is the connect side of the F4 `/api/backtests/` POST endpoint that operators trigger from UI Step 4 of F4 E1 smoke runbook). Worker:

1. Polls `backtest_runs` for `status='queued'` rows
2. Atomically claims one via `UPDATE...RETURNING SKIP LOCKED` per OQ-2=A (race-safe for future multi-worker)
3. Invokes T-507b `main()` programmatically with the claimed row's `run_id` (small T-507b refactor adds Optional `run_id` arg per OQ-1=A — if set, skip `insert_backtest_run`; use existing row)
4. Loops with operator-tunable poll interval (default 5s; env-configurable per §N9 / L-001)

Per backlog OQ-1=A: in-process in analytics-api lifespan (NOT separate `services/backtest-worker/` service). Per OQ-2=A: atomic UPDATE...RETURNING + SKIP LOCKED for race-safe claim. Per OQ-1=A operator decision: import-call into refactored T-507b `main()` (NOT subprocess invocation).

End-to-end FSM:
1. analytics-api lifespan adds `backtest_worker_task = asyncio.create_task(_run_backtest_worker_loop(...))` if `Settings.backtest_worker_enabled` is True
2. Worker loop: poll → claim → invoke → loop (sleep `poll_interval_s` between iterations when no queued rows)
3. Worker shutdown on lifespan exit: cancel task + suppress CancelledError + drain in-flight invocation
4. T-507b `main(args)` accepts new optional `args.run_id`; if set, skips insert_backtest_run and uses existing row (writes started_at + transitions to 'running' per existing logic, then runs through normal completion path)

## Architecture — composition root sequence

### `services/analytics_api/app/backtest_worker.py` (NEW)

```python
"""T-509: backtest worker — polls backtest_runs queue + dispatches to T-507b CLI.

Per backlog OQ-1=A: in-process in analytics-api lifespan. Per OQ-2=A:
atomic UPDATE...RETURNING + SKIP LOCKED claim semantic (race-safe for
future multi-worker; SKIP LOCKED is no-op for single-worker today).

Dispatches to T-507b main() in-process via import-call (per worker OQ-1=A
operator decision). Refactored T-507b main() accepts optional run_id arg —
if set, skips insert_backtest_run + uses existing row.
"""

from __future__ import annotations

import argparse
import asyncio
import contextlib
from datetime import UTC, datetime
from typing import TYPE_CHECKING

from packages.core.types import BacktestStatus
from packages.db.queries.analytics import (
    claim_next_backtest_run,
    update_backtest_run_completion,
)

if TYPE_CHECKING:
    import asyncpg
    from structlog.stdlib import BoundLogger

    from services.analytics_api.app.config import Settings


__all__ = ["run_backtest_worker_loop"]


async def run_backtest_worker_loop(
    *,
    pool: asyncpg.Pool,
    settings: Settings,
    logger: BoundLogger,
) -> None:
    """T-509 worker loop: poll queue → claim → dispatch → loop.

    Loop body:
    1. Acquire pool conn → claim_next_backtest_run (atomic UPDATE...RETURNING)
    2. If row claimed: dispatch to T-507b main() with claimed run_id
    3. If no row: sleep settings.backtest_worker_poll_interval_s
    4. On asyncio.CancelledError (lifespan shutdown): break out of loop

    Per OQ-1=A in-process import-call dispatch; T-507b main() with
    args.run_id set skips insert_backtest_run + uses existing row.
    """
    logger.info("backtest.worker.started",
                poll_interval_s=settings.backtest_worker_poll_interval_s)
    try:
        while True:
            run_row = None
            async with pool.acquire() as conn:
                run_row = await claim_next_backtest_run(conn)
            if run_row is None:
                await asyncio.sleep(settings.backtest_worker_poll_interval_s)
                continue
            # Claimed a run; dispatch to T-507b.
            logger.info("backtest.worker.claimed",
                        run_id=str(run_row.id), bot_id=run_row.bot_id)
            try:
                await _dispatch_to_t507b(run_row, settings, logger)
            except Exception as exc:  # noqa: BLE001 — worker resilience
                # T-507b internally writes status='failed' on its own
                # exception path through ITS OWN codec-clean CLI pool;
                # this except catches dispatch-layer bugs (e.g., invalid
                # args mapping). BLOCKER #1 fix: worker pool registers
                # _register_jsonb_codec → must pass codec_registered=True
                # so update_backtest_run_completion uses dict-direct bind
                # (NOT json.dumps text-mode → would double-encode).
                logger.error("backtest.worker.dispatch_failed",
                             run_id=str(run_row.id), error=str(exc))
                async with pool.acquire() as conn:
                    await update_backtest_run_completion(
                        conn,
                        run_id=run_row.id,
                        status=BacktestStatus.FAILED,
                        summary={"error": f"worker dispatch failed: {exc!s}"[:500]},
                        finished_at=datetime.now(UTC),
                        codec_registered=True,  # L-013 BLOCKER #1 fix
                    )
    except asyncio.CancelledError:
        logger.info("backtest.worker.cancelled")
        raise


async def _dispatch_to_t507b(
    run_row: Any,  # BacktestRunRow
    settings: Settings,
    logger: BoundLogger,
) -> None:
    """T-509 OQ-1=A: import-call dispatch to T-507b main() with external run_id.

    Constructs argparse.Namespace from run_row fields. T-507b main() reads
    args.run_id (if set, skip insert_backtest_run); args.config_path NOT
    used (config_yaml read directly from row); args.bot/from/to read from
    row. Creates tmpfile for config_yaml since T-507b _load_bot_config_with_overrides
    expects a Path (refactor of _load_bot_config_with_overrides to accept
    YAML-text inline is OUT OF SCOPE; tmpfile bridge is operationally fine).
    """
    from pathlib import Path
    import tempfile

    from scripts.backtest import main as t507b_main

    # Write config_yaml to tmp so T-507b's path-based loader reads it.
    with tempfile.NamedTemporaryFile(
        mode="w", suffix=".yaml", delete=False,
    ) as tmp:
        tmp.write(run_row.config_yaml)
        tmp_path = Path(tmp.name)

    try:
        args = argparse.Namespace(
            bot=run_row.bot_id,
            from_at=run_row.date_range_start,
            to_at=run_row.date_range_end,
            config_path=tmp_path,
            overrides=[],  # worker doesn't apply additional overrides
            pace="max",
            source="binance",  # default; future: read from run_row.meta if added
            plugin_registry_path=None,
            db_url=settings.database_url,
            name=None,
            notes=None,
            compare=None,  # T-509 worker only handles run-mode dispatch
            run_id=run_row.id,  # T-507b refactor: skip insert_backtest_run
        )
        rc = await t507b_main(args)
        logger.info(
            "backtest.worker.completed",
            run_id=str(run_row.id),
            return_code=rc,
        )
    finally:
        tmp_path.unlink(missing_ok=True)
```

### `packages/db/queries/analytics.py` `update_backtest_run_completion` codec-aware refactor (BLOCKER #1 fix)

```python
@idempotent
async def update_backtest_run_completion(
    conn: _DbExecutor,
    *,
    run_id: UUID,
    status: BacktestStatus,
    summary: dict[str, Any],
    finished_at: datetime,
    codec_registered: bool = False,  # T-509 BLOCKER #1 fix per L-013
) -> None:
    """T-507b: transition backtest_runs to terminal status + persist summary.

    L-013 codec-state-immune convention via explicit flag:
    - codec_registered=False (default; CLI invocation path; pool created
      via `create_pool(...)` without `init` hook): pass
      `json.dumps(_to_jsonable(summary))` text-mode + `$N::jsonb` bind.
      asyncpg without codec rejects raw dict for `$N::jsonb` cast.
    - codec_registered=True (T-509 worker invocation path; analytics-api
      pool registers `_register_jsonb_codec` per `services/analytics_api/
      app/main.py:121`): pass `_to_jsonable(summary)` dict directly.
      Codec encoder serializes via internal `json.dumps`; text-mode +
      codec would double-encode (L-011 regression class).

    Backwards-compat: default False matches existing T-507b CLI behavior
    (commit fcdc453); worker passes True at single dispatch_failed call site.
    """
    if codec_registered:
        bind_value: Any = _to_jsonable(summary)
    else:
        bind_value = json.dumps(_to_jsonable(summary))
    await conn.execute(
        _UPDATE_BACKTEST_RUN_COMPLETION_SQL,
        status.value,
        bind_value,
        finished_at,
        run_id,
    )
```

### `packages/db/queries/analytics.py` claim helper (NEW)

```python
_CLAIM_NEXT_BACKTEST_RUN_SQL = """
    UPDATE backtest_runs
       SET status='running',
           started_at=$1
     WHERE id IN (
        SELECT id FROM backtest_runs
         WHERE status='queued'
         ORDER BY created_at
         LIMIT 1
         FOR UPDATE SKIP LOCKED
     )
    RETURNING id, name, bot_id, config_yaml, config_hash,
              date_range_start, date_range_end, status,
              started_at, finished_at, summary, notes, created_at
"""


@non_idempotent
async def claim_next_backtest_run(
    conn: _DbExecutor,
) -> BacktestRunRow | None:
    """T-509 OQ-2=A: atomic UPDATE...RETURNING + SKIP LOCKED claim.

    Race-safe for future multi-worker (FOR UPDATE SKIP LOCKED ensures
    multiple workers don't claim same row). Single-worker today: SKIP
    LOCKED is no-op. Returns None when no queued rows.

    @non_idempotent because state transition queued→running is one-shot;
    re-running on same row would no-op (status filter excludes it).

    started_at populated Python-side per §N1 (no SQL NOW()).
    """
    row = await conn.fetchrow(_CLAIM_NEXT_BACKTEST_RUN_SQL, datetime.now(UTC))
    if row is None:
        return None
    return _row_to_backtest_run(row)
```
