## OQ resolutions (operator-approved 2026-04-29 — "use defaults")

1. **OQ-1 — `entry_fee` reconstruction at hydrate**: **default A** — JOIN-query `paper_executions WHERE order_id = paper_trades.open_order_id`. The OPEN execution's fee is the entry fee. Math-correct after restart for both partial-TP'd and clean positions. +1 SELECT helper.

2. **OQ-2 — `tpsl_mode` + `tp_size` reconstruction at hydrate**: **default B** — hardcode `tpsl_mode='Full'`, `tp_size=None` on hydrated dict. Operator/T-216 must re-issue `set_trading_stop` after restart to re-establish Partial mode. Documented operational gap. The existing raise in `_check_sl_tp_crosses` (adapter.py:237-240) catches inconsistent state if `tp_price` is set but `tpsl_mode='Partial'` later resurfaces without `tp_size` — fail-loud surface preserved.

3. **OQ-3 — `get_positions(symbol=None)` flat semantics**: **default A** — empty list when no `paper_positions` rows. Consistent with Bybit V5 `/v5/position/list` REST behavior (returns only active positions; flat = empty array, NOT `Position(side=None, size=0)`).

4. **OQ-4 — `Position.size` mapping**: **default A** — `Position.size = paper_positions.remaining_qty`. "Current state right now" semantics per `types.py:55-58`. After partial TP, `remaining_qty < qty` and consumers (T-216 placement, T-221 reconciliation) see the correct open size.

5. **OQ-5 — `get_closed_pnl_cumulative(sub_account)` validation**: **default A** — exact str equality `sub_account == str(self._bot_id)`; mismatch raises `ValueError(f"sub_account mismatch: got {sub_account!r}, expected {self._bot_id!r}")`. T-213b OQ-4 default A binding (paper interprets sub_account as bot_id-equivalent).

6. **OQ-6 — Hydrate timing in lifecycle**: **default A** — hydrate at the START of `start_consuming()`, BEFORE the NATS subscribe. Ordering: `__init__` (sync, allocates queues + dict) → `start_consuming()` (async, hydrates dict from DB → subscribes to `market.ohlc.1m.>`). Composition root flow: `paper = PaperExchange(...); await paper.start_consuming()`. No constructor surface change; T-215 unchanged.

7. **OQ-7 — Hydrate failure handling**: **default A** — exception propagates; `start_consuming()` raises; composition root crashes. Symmetry-of-life with T-202/T-203 — DB-unavailable at startup is hard-fail per F2 invariant. No degraded-mode (which would mute SL/TP fills on existing positions silently — capital-loss risk per H-013 spirit).

8. **OQ-8 — Test fixture strategy**: **default A** — mirror T-213b verbatim. Integration tests (env-gated `POSTGRES_TEST_DSN`) in `test_paper_persistence.py` for SELECT helpers + restart hydrate end-to-end; unit tests (mock pool) in `test_paper_emission.py` for read-method shape + ValueError raising.

## Decisions committed

Per the T-204 → T-212 → T-213a → T-213b calibration trajectory.

1. **OQ-defaults applied throughout**: this plan-doc bakes all 8 OQ defaults; no unresolved decisions remain.

2. **Reads compose under the public method's `@idempotent` marker** (mirror T-213b W#4 stance A — public-method-marker authoritative): `get_positions`, `get_fill_price`, `get_closed_pnl_cumulative` carry `@idempotent` (already on T-201/T-211 stubs). New private SELECT helpers in `persistence.py` are NOT individually decorated (read-only — idempotency is degenerate; write-helper INSERT decoration is for grep-friendliness on row-creation sites only, NOT applicable to reads).

3. **Read-side connection acquisition via `async with self._pool.acquire() as conn`** — single connection, no transaction (no atomicity requirement on reads). Mirror T-213b write-side idiom, drop the `conn.transaction()` wrapper.

4. **Hydrate is best-effort per-position with one inner JOIN query**: at `start_consuming()` start, single SELECT joining `paper_positions` × `paper_trades` × `paper_executions(open exec)` returns all rows needed to populate dict. Fallback per-position queries NOT introduced — JOIN is canonical (one round-trip; row count bounded by symbols-per-bot which is ≤10 per F2 spec).

5. **`_active_positions` hydrated dict shape**: identical to `_persist_open` (adapter.py:418-430) modulo OQ-2 default B (`tpsl_mode='Full'`, `tp_size=None`):

   | Field | Source | Notes |
   |-------|--------|-------|
   | `trade_id` | `paper_positions.trade_id` | |
   | `side` | `paper_positions.side` | |
   | `qty` | `paper_positions.remaining_qty` | "current open qty" — unifies with mid-life invariant. `_drain_partial_tp` at adapter.py:628 already mutates `position["qty"] = new_qty` post-partial-TP, so dict's `qty` reflects current open qty regardless of partial-TP history. Hydrate writes `remaining_qty` to mirror that mid-life invariant after restart. |
   | `entry_price` | `paper_positions.entry_price` | |
   | `entry_fee` | `paper_executions.fee WHERE order_id = paper_trades.open_order_id` | OQ-1 default A |
   | `fees_paid` | `paper_trades.fees_paid` | post-partial-TP accumulator preserved |
   | `sl_price` | `paper_positions.sl_price` | |
   | `tp_price` | `paper_positions.tp_price` | |
   | `tp_size` | `None` | OQ-2 default B |
   | `tpsl_mode` | `'Full'` | OQ-2 default B |
   | `tp_hit` | `paper_positions.tp_hit` | preserved for restart-after-partial-TP |

   Note: when `tp_hit=TRUE` on hydrate, the position has already had a partial TP fill before restart. Dict's `qty` field is the SL/TP fill-trigger qty (consumed by `_check_sl_tp_crosses` line 226 → `PendingSLTPFill.qty` → `fill.qty` at `_drain_full_close` line 653). Reading from `paper_positions.remaining_qty` instead of `paper_positions.qty` ensures that after restart, an SL cross drains the actual open size (which is `remaining_qty` post-partial-TP), matching the live mid-life path that mutates dict's `qty` to `new_qty` at adapter.py:628 after partial fills.

6. **Edge case discipline**: OQ-2 default B documented gaps surfaced as test pins, not silently accepted. Specifically `test_hydrate_partial_tp_position_with_tp_hit_true_and_qty_field_pins_known_compromise` documents the fact that re-issuing `set_trading_stop` after restart is REQUIRED for partial-TP'd positions to re-enter Partial mode correctly.

7. **`get_fill_price` returns first match by `executed_at ASC`**: paper has one paper_executions row per order in the canonical case (open exec OR synthetic SL/TP exec — never both for the same order_id). LIMIT 1 ORDER BY executed_at ASC defends against schema drift if a future task synthesises additional executions per order; returns the earliest fill price (chronological "first fill" — Bybit V5 REST `get_fill_price` semantics).

async def start_consuming(self) -> None:
    """T-213c: hydrate _active_positions from paper_positions BEFORE NATS subscribe.

    OQ-2 default B — tpsl_mode='Full' + tp_size=None on hydrated entries;

async def _hydrate_active_positions(self) -> None:
    """Hydrate ``_active_positions`` dict from ``paper_positions`` JOIN
    on ``paper_trades`` + ``paper_executions`` (open).

    OQ-7 default A: query failure raises; composition root crashes.
    """
    async with self._pool.acquire() as conn:
        rows = await persistence.select_paper_positions_for_hydrate(
            conn, bot_id=str(self._bot_id),
        )
    partial_tp_count = 0
    for row in rows:
        if row["tp_hit"]:
            partial_tp_count += 1
        self._active_positions[row["symbol"]] = {
            "trade_id": int(row["trade_id"]),
            "side": row["side"],
            "qty": Decimal(row["remaining_qty"]),  # Decision #5 — current open qty.
            "entry_price": Decimal(row["entry_price"]),
            "entry_fee": Decimal(row["entry_fee"]),  # OQ-1 default A.
            "fees_paid": Decimal(row["fees_paid"]),
            "sl_price": Decimal(row["sl_price"]) if row["sl_price"] is not None else None,
            "tp_price": Decimal(row["tp_price"]) if row["tp_price"] is not None else None,
            "tp_size": None,                          # OQ-2 default B.
            "tpsl_mode": "Full",                      # OQ-2 default B.
            "tp_hit": bool(row["tp_hit"]),
        }
    logger.info(
        "paper_exchange.hydrate_complete",
        extra={
            "bot_id": str(self._bot_id),
            "symbols_hydrated": len(rows),
            "partial_tp_positions": partial_tp_count,
        },
    )
    if partial_tp_count > 0:
        # OQ-2 default B compromise — partial-TP positions hydrated as Full
        # mode. Operator MUST re-issue set_trading_stop with tpsl_mode='Partial'
        # to restore Partial behavior on those positions; otherwise the next
        # TP cross uses dict's qty (= remaining_qty, post-partial) for a Full
        # close instead of a partial-of-partial. WARN-level for fail-loud
        # observability per plan-reviewer CONCERN 2.
        logger.warning(
            "paper_exchange.hydrate_partial_tp_positions_require_set_trading_stop",
            extra={
                "bot_id": str(self._bot_id),
                "partial_tp_positions": partial_tp_count,
                "required_action": (
                    "re-issue set_trading_stop with tpsl_mode='Partial' on each "
                    "partial-TP'd symbol to restore Partial mode after restart"
                ),
            },
        )

@idempotent
async def get_positions(
    self, symbol: str | None = None,
) -> list[Position]:
    """OQ-3 default A: empty list when no rows. OQ-4 default A: size=remaining_qty."""
    async with self._pool.acquire() as conn:
        rows = await persistence.select_paper_positions(
            conn, bot_id=str(self._bot_id), symbol=symbol,
        )
    return [
        Position(
            symbol=row["symbol"],
            side=row["side"],
            size=Decimal(row["remaining_qty"]),
            entry_price=Decimal(row["entry_price"]),
            leverage=None,         # paper has no leverage (Decision #13 of T-213b).
            unrealized_pnl=None,   # REST snapshot; mark price not available at this surface.
        )
        for row in rows
    ]


@idempotent
async def get_fill_price(
    self, symbol: str, order_id: str,
) -> Decimal | None:
    """Decision #7: LIMIT 1 ORDER BY executed_at ASC. Returns None if no match."""
    async with self._pool.acquire() as conn:
        return await persistence.select_paper_execution_price_by_order_id(
            conn, exchange_order_id=order_id,
        )


@idempotent
async def get_closed_pnl_cumulative(self, sub_account: str) -> Decimal:

    if sub_account != str(self._bot_id):
        raise ValueError(
            f"sub_account mismatch: got {sub_account!r}, expected {str(self._bot_id)!r}"
        )
    async with self._pool.acquire() as conn:
        return await persistence.sum_paper_trades_realized_pnl(
            conn, bot_id=str(self._bot_id),
        )

## Edge cases

1. **Hydrate with stale `paper_positions` (no matching paper_trades / paper_executions)**: JOIN returns 0 rows for that broken position. Better than crash but indicates DB inconsistency. Test pin: `test_hydrate_skips_paper_position_without_open_execution_logs_warning` (deferred — would require synthetic schema-violating fixture; not needed at F2; logged but not blocking).

2. **`get_positions(symbol="BTCUSDT")` for symbol bot doesn't trade**: empty list per OQ-3 default A.

3. **`get_fill_price` for an order_id from a different bot**: query uses `paper_orders.exchange_order_id` only (no bot_id filter — exchange_order_id is globally unique per UUID4 generation per T-213b). Returns price for that order regardless of bot_id. Acceptable because exchange_order_id collision probability is ~0; concrete consumers (T-217) only query their own bot's orders. NOT a security boundary in F2.

4. **`get_closed_pnl_cumulative` returns 0 for never-traded bot**: paper_trades has no rows; SUM(NULL)→Decimal("0") per Decision #8.

5. **Hydrate writes `paper_positions.remaining_qty` to dict's `qty` field — consistency restoration after restart, NOT a semantic shift**: dict's `qty` ALWAYS reflects "current open qty" across the full lifecycle:
   - `_persist_open` (adapter.py:421): writes `qty=qty` at open time. No partial TP has fired yet → entry qty == remaining qty == "current open qty".
   - `_drain_partial_tp` (adapter.py:628): mutates `position["qty"] = new_qty` after partial TP fires mid-life. Dict's `qty` shrinks to the actual remaining size.
   - **T-213c hydrate**: writes `paper_positions.remaining_qty` to dict's `qty`. After restart, dict's `qty` matches what live mutation would have set it to before restart — same "current open qty" semantic.

   `_check_sl_tp_crosses` (adapter.py:213/226/235) consumes dict's `qty` as the SL/TP fill-trigger qty. `_drain_full_close` (adapter.py:653) then passes `fill.qty` to `_compute_realized_pnl`. The hydrate-from-`remaining_qty` choice keeps post-restart SL/TP fills sized correctly for partial-TP'd positions. Test pin: `test_hydrate_after_partial_tp_uses_remaining_qty_for_dict_qty_field`.

6. **Hydrate with `entry_fee` lookup failing (paper_executions row missing for open exec)**: JOIN inner-clauses; missing row → no result for that position. Test pin defers to edge case #1 (same surface).

7. **Concurrent `start_consuming` calls**: protocol/lifespan semantics — single call per instance per F2 spec. Not test-pinned.
