# T-530 — `ExchangeClient.get_account_balance()` protocol extension

**Status:** APPROVED (plan-reviewer 3rd-pass) → SHIPPED 2026-05-15 (commit `93faa9e`; all 4 gates passed: plan APPROVE / drift ON TRACK / brief SHIP WG 5/5 / math-validator VERIFIED).
**Phase:** F5 — ADR-0011 pre-live operational hardening cluster (account-balance / equity sub-cluster; top-of-DAG).
**Branch:** `feat/T-530-account-balance`
**Mirror precedent:** `get_instrument_info` (T-529 / H-036) — the most recent read-only `@idempotent` `ExchangeClient` protocol method returning a frozen dataclass, with a Bybit `GET /v5/*` live impl + a paper impl, golden-response decode tests. `get_closed_pnl_cumulative` (the closest *account-level* read — `sub_account` param + validate-before-limiter + paper derive-from-`paper_trades`).
**Unblocks:** T-531 (equity snapshot table + APScheduler tick + `virtual_balance{bot_id}` Prom gauge — calls `get_account_balance()` on its N-min tick).

## Purpose

Add a read-only `get_account_balance()` method to the `ExchangeClient` port so upper layers (T-531 equity snapshots; future balance-driven sizing T-527/T-528) can read a bot's account financial state through the same adapter abstraction as orders/positions. Live = Bybit `GET /v5/account/wallet-balance` (UNIFIED account-level totals); paper = derived from `paper_trades` (seed + cumulative realized P&L), mirroring the shipped `get_closed_pnl_cumulative` paper precedent. T-530 ships the protocol method + both adapter impls + the `AccountBalance` value type ONLY — no caller (T-531 is the first consumer, separate task).

### `packages/exchange/bybit_v5/adapter.py::BybitV5Adapter.get_account_balance`

Mirror `get_instrument_info` HTTP pattern + `get_closed_pnl_cumulative` sub_account-validate-before-limiter:

1. `if sub_account != self._sub_account: raise ValueError(...)` — BEFORE `limiter.acquire` (mirror `get_closed_pnl_cumulative` OQ-10 default A: caller mistake costs no rate-limit token).
2. `await self._limiter.acquire(self._sub_account, "positions")` — `"positions"` is the SAME limiter key `get_closed_pnl_cumulative` uses (`adapter.py:409`; `EndpointGroup = Literal["orders","positions","market"]` already includes it — no `EndpointGroup` extension). `/v5/account/wallet-balance` shares the `positions` bucket — consistent with the account-level-read precedent; a dedicated `/v5/account/*` bucket is an out-of-scope micro-decision (no §N9 knob). Pinned in test (`acquire(self._sub_account, "positions")` mock-captured).
3. `result = await self._client.request("GET", "/v5/account/wallet-balance", params={"accountType": "UNIFIED"}, retries=3)` (mirror `get_instrument_info` `retries=3` + the `except RateLimitError: await self._on_rate_limit_hit("positions"); raise` wrapper).
4. `items = result.get("list", [])`; `if not items: raise ExchangeError("wallet-balance: empty account list")` (a valid authed UNIFIED key always returns `list[0]`; empty = auth/account anomaly — §11.3 `ExchangeError`, NOT `OrderRejected` which is order-semantic).
5. `acct = items[0]`; OQ-1=A account-level totals → `Decimal(str(...))` (no float; mirror `get_instrument_info` `Decimal(lot["qtyStep"])`):
   - `wallet_balance = Decimal(str(acct["totalWalletBalance"]))`
   - `available_balance = Decimal(str(acct["totalAvailableBalance"]))`
   - `total_equity = Decimal(str(acct["totalEquity"]))`
   - `margin_balance = Decimal(str(acct["totalMarginBalance"]))`
   - `unrealized_pnl = Decimal(str(acct["totalPerpUPL"]))`
6. `return AccountBalance(...)`.

§5.13/§N1: Bybit returns these as JSON strings; `Decimal(str(x))` is exact (the `str()` wrapper defends if a field arrives as a number — never `Decimal(float)`; mirror the `Decimal(lot["qtyStep"])` get_instrument_info precedent which relies on string input — `str()` belt-and-suspenders here because account totals have higher float-risk than lotSize). No `float()` anywhere.

### `packages/exchange/paper/adapter.py::PaperExchange.get_account_balance`

Mirror `get_closed_pnl_cumulative` (sub_account validate → `persistence.sum_paper_trades_realized_pnl`):

1. `if sub_account != str(self._bot_id): raise ValueError(...)` (verbatim mirror `get_closed_pnl_cumulative`).
2. `async with self._pool.acquire() as conn: realized = await persistence.sum_paper_trades_realized_pnl(conn, bot_id=str(self._bot_id))` (reuse the shipped helper — same as `get_closed_pnl_cumulative`).
3. `wallet_balance = self._seed_balance + realized` (Decimal + Decimal, exact).
4. `unrealized_pnl = Decimal("0")` — **OQ-2=A documented limitation**: paper does NOT mark-to-market open positions in T-530 scope (would require a market-data mark-price feed coupling into the exchange adapter, out of a protocol-extension task's scope; TASKS.md says "from in-memory state" and paper has no live mark in-memory). Forward-compat for a future market-data-coupled task.
5. `total_equity = wallet_balance` (= wallet + unrealized; unrealized=0).
6. `margin_balance = wallet_balance` (Bybit semantic `marginBalance = walletBalance + unrealisedPnl`; = wallet + 0).
7. `available_balance = wallet_balance` (paper has no margin-lockup model — full wallet available; conservative + consistent).
8. `return AccountBalance(wallet_balance, available_balance, total_equity, margin_balance, unrealized_pnl)`.

`@idempotent` (mirror `get_closed_pnl_cumulative`).

## Hand verification (§6.2 step 5 — math-validator cross-check: Decimal money decode + paper arithmetic; no float)

**Bybit decode (no float)**: golden fixture `result.list[0] = {"totalWalletBalance":"10000.50","totalAvailableBalance":"9875.25","totalEquity":"9875.25","totalMarginBalance":"10000.50","totalPerpUPL":"-125.2500"}`. `Decimal(str("10000.50"))` = `Decimal("10000.50")` exact (string in → exact; the `str()` wrapper is a no-op on a str and a float-artefact guard if a number ever arrives). The 5 fields decode 1:1 to the 5 `AccountBalance` fields verbatim — no arithmetic, no rounding, no float. `unrealized_pnl == Decimal("-125.2500")` (negative UPL preserved with scale).

**Paper computation (Decimal arithmetic, no float)**: `seed_balance = Decimal("10000")`, `realized = sum_paper_trades_realized_pnl(...)` (shipped helper; NUMERIC(20,4) → `Decimal`). Worked example: realized = `Decimal("250.5000")` → `wallet_balance = Decimal("10000") + Decimal("250.5000") = Decimal("10250.5000")` (exact Decimal addition, scale preserved). `unrealized_pnl = Decimal("0")` (OQ-2=A limitation). `total_equity = wallet_balance = Decimal("10250.5000")`. `margin_balance = wallet_balance = Decimal("10250.5000")` (Bybit `marginBalance = walletBalance + unrealisedPnl` with UPL=0). `available_balance = wallet_balance = Decimal("10250.5000")` (no margin-lockup model). Loss case: realized = `Decimal("-500")` → wallet = `Decimal("10000") + Decimal("-500") = Decimal("9500")`. No `float()` anywhere; one Decimal addition; the other 3 fields are aliases of `wallet_balance` (no further arithmetic). The Decimal invariant: `total_equity == margin_balance == available_balance == wallet_balance` in paper (a parity test pins this — the simplification's defining identity).
