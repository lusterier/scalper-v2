The outer `json.dumps(...)` then serializes JSON-native dict → text → SQL `$N::jsonb` cast → JSONB object stored. **No `default=str` arg on `json.dumps`** because `_to_jsonable` already pre-stringifies (mirror audit.py module docstring §17-19 rationale). **Switch trigger documented in module docstring**: if execution-service registers `_register_jsonb_codec` in F5+, drop the outer `json.dumps(...)` wrapper — pass `_to_jsonable(meta)` dict directly to asyncpg; codec encoder serializes via `json.dumps` (single-encoded). The `_to_jsonable` wrapper is the constant element across both modes. Tests assert: (a) bind arg is `str` (text-mode under no-codec; switch test to `dict` after codec registration); (b) decoded JSON contains stringified UUID/datetime/Decimal leaves; (c) `_to_jsonable` recurses into nested dicts/lists. Pre-emptive value: `_to_jsonable` eliminates the F4 E1 `TypeError: Object of type UUID is not JSON serializable` regression class regardless of codec state.
