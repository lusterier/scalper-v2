## Edge cases

- **Empty `rule_results` list**: `[]` is valid JSONB array; T-310 may emit when `bot_config.scoring.rules == []` (per T-300 test pin ‚Äî empty rules accepted). Migration test covers.
- **Empty `feature_snapshot`**: `{}` valid; happens on `decision="reject"` with `reason="required_feature_missing"` if rule failed before any feature lookup. Migration test covers.
- **Concurrent inserts** (same `(bot_id, signal_id)` from duplicate signal-consumer):
  - No DB-level UNIQUE constraint (brief schema has none); duplicate prevention is T-310 application-layer concern via consumer dedup ring (mirror H-009 dispatcher dedup ring).
  - Migration test does NOT pin uniqueness (none exists at DB level).
- **Hypertable chunk boundary** at 30-day interval per ¬ß7.2:1053 ‚Äî migration test verifies chunk_time_interval setting.
- **Decimal in feature_snapshot inner values**: T-307 evaluator (future) will serialize FeatureValue.value_num as Decimal-string per ¬ßN1 ‚Äî T-301 migration treats inner shape as opaque JSONB blob.
- **Long correlation_id**: TEXT field accepts any length; no DB-side limit. T-300 BotConfig.bot_id has charset regex; correlation_id is opaque.
