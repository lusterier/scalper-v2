## Goal

Add `ReplayBus` in-process pub/sub class in `packages/bus/replay_bus.py` that mirrors the `NatsClient` interface subset T-507 CLI orchestrator uses (`publish(subject, envelope)` + `subscribe(subject_pattern, handler)`), but delivers messages in **timestamp order** (priority queue keyed on `envelope.published_at`) instead of arrival order. Per BRIEF §12.2:1953: *"in-process NATS-compatible publish/subscribe; messages delivered in timestamp order."*

Output is consumed by T-507 CLI orchestrator: T-503 HistoricalOHLCSource + T-504 HistoricalSignalSource publish historical OHLC + signals into ReplayBus (envelopes carry `published_at = bucket_start / received_at`); strategy-engine + execution-service subscribe via `subscribe(subject_pattern, handler)` — receive interleaved messages in true chronological order regardless of producer-side yield order.

## Algorithm specification

```python
import heapq
from dataclasses import dataclass, field
from typing import Awaitable, Callable

Handler = Callable[[MessageEnvelope], Awaitable[None]]


@dataclass(slots=True)
class ReplaySubscription:
    subject_pattern: str
    handler: Handler
    active: bool = True


class ReplayBus:
    """In-process NATS-compatible pub/sub with timestamp-ordered delivery.

    See module docstring for spec citation + heap-stable tie-break rationale.
    """

    def __init__(self) -> None:
        self._heap: list[tuple[datetime, int, str, MessageEnvelope]] = []
        self._subscriptions: list[ReplaySubscription] = []
        self._insertion_seq = 0
        self._closed = False

    async def publish(self, subject: str, envelope: MessageEnvelope) -> None:
        if self._closed:
            raise RuntimeError("ReplayBus: publish on closed bus")
        heapq.heappush(
            self._heap,
            (envelope.published_at, self._insertion_seq, subject, envelope),
        )
        self._insertion_seq += 1

    def subscribe(self, subject_pattern: str, handler: Handler) -> ReplaySubscription:
        if self._closed:
            raise RuntimeError("ReplayBus: subscribe on closed bus")
        sub = ReplaySubscription(subject_pattern=subject_pattern, handler=handler)
        self._subscriptions.append(sub)
        return sub

    async def run_until_empty(self) -> None:
        while self._heap:
            _ts, _seq, subject, envelope = heapq.heappop(self._heap)
            for sub in self._subscriptions:
                if not sub.active:
                    continue
                if not _subject_matches(sub.subject_pattern, subject):
                    continue
                try:
                    await sub.handler(envelope)
                except Exception:  # noqa: BLE001 — mirror NatsClient _dispatch swallow
                    pass  # handler exceptions don't kill drain

    async def close(self) -> None:
        if self._closed:
            return
        self._closed = True
        self._subscriptions.clear()
        self._heap.clear()
```

Subject matching:
```python
def _subject_matches(pattern: str, subject: str) -> bool:
    """NATS subset: exact + '*' single-token + '>' multi-token tail."""
    pat_tokens = pattern.split(".")
    sub_tokens = subject.split(".")
    for i, pt in enumerate(pat_tokens):
        if pt == ">":
            return True  # tail wildcard absorbs rest
        if i >= len(sub_tokens):
            return False
        if pt == "*":
            continue  # single-token wildcard
        if pt != sub_tokens[i]:
            return False
    return len(pat_tokens) == len(sub_tokens)
```

## Hand verification

### §A — Timestamp ordering across subjects

Publisher publishes 3 messages: `(t=100s, "market.ohlc.1m.BTC")` + `(t=50s, "signals.validated")` + `(t=150s, "market.ohlc.1m.BTC")`. Subscriber on `>` wildcard receives in order: t=50, t=100, t=150 (NOT publish order).

### §B — Exact subject match

Publisher: `(t=100, "market.ohlc.1m.BTC")` + `(t=200, "market.ohlc.1m.ETH")`. Subscriber on `"market.ohlc.1m.BTC"` exact: receives only first.

### §C — Wildcard `*`

Publisher: `(t=100, "market.ohlc.1m.BTC")` + `(t=200, "market.ohlc.5m.BTC")`. Subscriber on `"market.ohlc.*.BTC"`: receives both.

### §D — Wildcard `>`

Publisher: `(t=100, "market.ohlc.1m.BTC")` + `(t=200, "signals.validated")`. Subscriber on `">"`: receives both.

### §E — Multiple subscribers same subject

Publisher: `(t=100, "X")`. 2 subscribers on `"X"`. Both handlers invoked once each (no broadcast deduplication).

### §F — Same-timestamp tie-break

Publisher: `(t=100, "A")` + `(t=100, "B")`. Heap tie broken by `insertion_seq` ASC → A before B.

### §G — Handler exception swallowed

Publisher: `(t=100, "X")` + `(t=200, "X")`. Subscriber handler raises on first message. Second message still delivered.

### §H — `run_until_empty` semantic

After drain, heap empty + further `publish()` works (re-fills heap; next `run_until_empty()` drains the new content).
