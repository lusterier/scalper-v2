## Purpose

Read-only analytics aggregates for the dashboard's Overview + per-bot views (BRIEF §9.6:1628 + §14.3:2060). Four endpoints:

1. **`GET /api/analytics/expectancy`** — single-row aggregate: expectancy + win rate (WR) + total trades + avg_win + avg_loss + summary stats over `(bot_id, from_at, to_at)` filter window
2. **`GET /api/analytics/heatmap/hourly`** — 24×7 grid (hour-of-day × day-of-week, UTC) of `(trade_count, avg_realized_pnl)`; backs heatmap chart UI
3. **`GET /api/analytics/pnl-series`** — cumulative P&L time-series for chart UI; bucketed hour|day; capped at 5000 points
4. **`POST /api/analytics/monte-carlo`** — bootstrap simulation; CPU-heavy via `asyncio.to_thread` with **5-min in-memory cache** per BRIEF §9.6:1641

**Math scope.** T-406 ships financial aggregates (expectancy, WR, MC bootstrap). Per CLAUDE.md Gate 4 conditional list (line 121), `services/analytics_api/` is NOT in math-binding scope (only `packages/features/`, `packages/pnl/`, `services/feature-engine/+execution/+scoring/`). Math-validator stays OUT OF SCOPE per the letter of Gate 4. Plan ships `## Hand verification` section per §6.1 with hand-computed expectancy + WR + MC seed + percentile interpolation; brief-reviewer second-look gates math correctness.

```python
@dataclass(frozen=True, slots=True)
class ExpectancyMetrics:
    """Statistical expectancy result (§5.13 float domain — statistical not money)."""
    expectancy: float           # E = (WR × avg_win) − (LR × avg_loss)
    win_rate: float             # WR = wins / total
    loss_rate: float            # LR = losses / total
    avg_win: Decimal            # mean of positive realized_pnl (Decimal preserves NUMERIC precision)
    avg_loss: Decimal           # mean of |negative realized_pnl|; Decimal positive
    total_trades: int
    win_count: int
    loss_count: int


@dataclass(frozen=True, slots=True)
class HeatmapCell:
    """One cell of the 24×7 hour-of-day × day-of-week grid."""
    weekday: int                # 0=Monday, 6=Sunday (ISO 8601)
    hour: int                   # 0..23 UTC
    trade_count: int
    avg_pnl: Decimal | None     # None when trade_count=0


type PnlBucket = Literal["hour", "day"]


@dataclass(frozen=True, slots=True)
class PnlSeriesPoint:
    """One point on the cumulative P&L time-series chart."""
    bucket_at: datetime         # bucket start (truncated to hour or day boundary)
    bucket_pnl: Decimal         # sum of realized_pnl in this bucket
    cumulative_pnl: Decimal     # running sum since first bucket


@dataclass(frozen=True, slots=True)
class MonteCarloResult:
    """Bootstrap MC result; percentile distribution at end-of-period equity."""
    n_simulations: int
    seed: int
    p5: Decimal
    p25: Decimal
    p50: Decimal
    p75: Decimal
    p95: Decimal


def compute_expectancy(rows: list[TradeRealizedPnlRow]) -> ExpectancyMetrics:
    """Compute statistical expectancy + WR + avg_win + avg_loss.

    Formula (per OQ-6 default A — standard expectancy):
        E = (WR × avg_win) − (LR × avg_loss)
    where:
        WR = wins / total                    (win = realized_pnl > 0)
        LR = losses / total                  (loss = realized_pnl < 0)
        avg_win = mean of positive realized_pnl values
        avg_loss = mean of |negative realized_pnl| values (positive Decimal)

    Trades with realized_pnl == 0 (rare; flat trade) are EXCLUDED from
    win/loss buckets but counted in total. Empty input → all-zero result
    with avg_win = avg_loss = Decimal('0').

    expectancy + win_rate + loss_rate are float per §5.13 (statistical
    metrics, not money). avg_win + avg_loss preserve Decimal NUMERIC
    precision per §N1 / §5.3.
    """


def compute_hourly_heatmap(rows: list[TradeRealizedPnlRow]) -> list[HeatmapCell]:
    """Bucket trades into 24×7 grid; empty cells emitted with count=0, avg_pnl=None.

    Returns 168 cells in deterministic order: weekday 0..6 × hour 0..23
    (ISO 8601 weekday: Monday=0, Sunday=6). UTC for closed_at extraction.
    """


def compute_pnl_series(
    rows: list[TradeRealizedPnlRow],
    *,
    bucket: PnlBucket,
) -> list[PnlSeriesPoint]:
    """Bucket trades into hour or day buckets; emit cumulative P&L series.

    Empty buckets between non-empty ones are NOT emitted (sparse series;
    UI handles gaps client-side via timestamp deltas). Truncation to
    hour/day boundary uses ``datetime.replace(...)`` in UTC.
    """


def compute_monte_carlo(
    rows: list[TradeRealizedPnlRow],
    *,
    n_simulations: int,
    seed: int,
) -> MonteCarloResult:
    """Bootstrap-with-replacement MC; deterministic with seed.

    Algorithm:
        1. Extract sample = [r.realized_pnl for r in rows]
        2. For sim in range(n_simulations):
             draw = random.choices(sample, k=len(sample))  # bootstrap
             equity_at_end = sum(draw)
             results.append(equity_at_end)
        3. Sort results; compute percentiles via linear interpolation
           (numpy-free; matches NumPy `numpy.percentile(..., method='linear')`
           default per OQ-3 default A).

    Empty rows → all percentiles = Decimal('0') (no exception). Single-
    row input → all percentiles equal that one row's pnl × n_trades = 1.

    `seed: int` is caller-derived (router computes from bot_id +
    from_at + to_at + n_simulations to make MC reproducible for same
    request shape — see ``_derive_mc_seed`` per WG#6).
    """
```

### `services/analytics_api/app/analytics_cache.py` (NEW)

```python
class AnalyticsCache:
    """In-process cache for CPU-heavy analytics endpoints (Monte-Carlo only).

    Mirrors execution-service ``closed_pnl_locks: dict[str, asyncio.Lock]``
    per-key dict pattern (ADR-0006 D4). Lock-per-key prevents thundering
    herd: concurrent identical MC requests share a single compute pass
    rather than N parallel computes.

    F5+ revisit: cross-process distributed cache (Redis / NATS KV) if
    analytics-api is sharded. F4 single-process per §3.1.
    """

    def __init__(self) -> None:
        self._values: dict[str, tuple[float, Any]] = {}  # key → (expiry_unix_ts, value)
        self._locks: dict[str, asyncio.Lock] = {}
        self._global_lock = asyncio.Lock()  # protects _locks dict mutation

    async def get_or_compute(
        self,
        key: str,
        ttl_seconds: int,
        compute_fn: Callable[[], Awaitable[Any]],
        *,
        now_fn: Callable[[], datetime],
    ) -> Any:
        """Return cached value if fresh; else compute under per-key lock + cache.

        Per-key lock prevents thundering herd. Expired entries are
        recomputed lazily on next access. No background eviction (F5+
        if memory pressure surfaces).
        """
```

Cache key derivation (router-side):
```python
def _cache_key(endpoint: str, params: dict[str, Any]) -> str:
    """SHA256 over endpoint name + sorted params; deterministic same-request hash."""
    sorted_pairs = sorted((k, str(v)) for k, v in params.items())
    raw = endpoint + "|" + "|".join(f"{k}={v}" for k, v in sorted_pairs)
    return hashlib.sha256(raw.encode("utf-8")).hexdigest()
```

```python
    pool, cache: AnalyticsCache, now_fn,
    bot_id: str | None, from_at: datetime | None, to_at: datetime | None,
    n_simulations: int = _MC_DEFAULT_SIMULATIONS,  # Query(ge=1, le=10000)
) -> MonteCarloResponse:
    """Bootstrap MC; CPU-heavy via asyncio.to_thread; cached 5min per request shape."""
```

MC endpoint flow:
1. Build cache key from `(bot_id, from_at, to_at, n_simulations)`
2. `cache.get_or_compute(key, _CACHE_TTL_SECONDS, _compute, now_fn=now_fn)` where `_compute`:
   a. SELECT trades from PG via `select_trades_for_analytics`
   b. Derive `seed` from `(bot_id, from_at, to_at, n_simulations)` via `_derive_mc_seed` (SHA256-truncated int) per WG#6 — deterministic for same request shape, makes MC reproducible
   c. `result = await asyncio.to_thread(compute_monte_carlo, rows, n_simulations=n, seed=seed)` — offloads CPU bootstrap to thread pool (FastAPI/uvicorn default ThreadPoolExecutor)
   d. Build `MonteCarloResponse` from result + metadata
3. Return cached or freshly-computed response

## Edge cases

- **Empty trades window** → expectancy = `(0.0, 0.0, ...)`, heatmap = 168 zero cells, pnl-series = [], MC = all-zero percentiles
- **All-wins or all-losses** → LR=0 or WR=0; expectancy = +avg_win or -avg_loss respectively
- **`realized_pnl == 0` (flat trade)** → excluded from win/loss buckets; counted in `total_trades`
- **`?bucket=garbage`** → 422 from FastAPI Literal Query validation
- **`?n_simulations=10001`** → 422 from `Query(ge=1, le=10000)`
- **`?from=<future>&to=<past>`** → empty window → all-zero result (mathematically correct, not 4xx)

## Hand verification

T-406 ships **financial math** (expectancy, WR, MC bootstrap). Per CLAUDE.md §6.1 + Gate 4 letter (math-validator OUT OF SCOPE for `services/analytics_api/`), this section provides hand-computed worked examples for brief-reviewer math second-look.

### Expectancy formula

Standard expectancy per OQ-6 default A:
```
E = (WR × avg_win) − (LR × avg_loss)
```
where:
- `WR = win_count / total_trades` (win = `realized_pnl > 0`)
- `LR = loss_count / total_trades` (loss = `realized_pnl < 0`)
- `avg_win = sum(positive realized_pnl) / win_count` (Decimal)
- `avg_loss = sum(|negative realized_pnl|) / loss_count` (positive Decimal)

Trades with `realized_pnl == 0` (flat) are excluded from win/loss buckets but counted in `total_trades`. This means `WR + LR ≤ 1` (gap = `zero_count / total`).

### Worked example (test #2 fixture)

Input: `realized_pnl = [+10, +5, -3, +8, -2]` (Decimals)

- `total_trades = 5`
- Wins: `[+10, +5, +8]` → `win_count = 3`, sum_wins = `Decimal('23')`
- Losses: `[-3, -2]` → `loss_count = 2`, sum_losses_abs = `Decimal('5')`
- `WR = 3/5 = 0.6`
- `LR = 2/5 = 0.4`
- `avg_win = Decimal('23') / Decimal('3') = Decimal('7.6666...')`
- `avg_loss = Decimal('5') / Decimal('2') = Decimal('2.5')`
- `expectancy = 0.6 × float(avg_win) − 0.4 × float(avg_loss)`
              `= 0.6 × 7.6667 − 0.4 × 2.5`
              `= 4.6 − 1.0`
              `= 3.6` (within 1e-4 float epsilon)

Test #2 asserts `abs(metrics.expectancy - 3.6) < 1e-4` and `metrics.avg_win == Decimal('23') / Decimal('3')` (exact Decimal equality).

### Edge: empty input

Empty `rows` → `total_trades = 0`, `win_count = 0`, `loss_count = 0`. To avoid division-by-zero:
- `WR = 0.0` if total = 0 (sentinel; no division)
- `LR = 0.0` similarly
- `avg_win = Decimal('0')` (empty sum / sentinel divisor = 0 by convention)
- `avg_loss = Decimal('0')`
- `expectancy = 0.0`

Test #1 pins this.

### Edge: all-wins / all-losses

All wins: `loss_count = 0` → `avg_loss = Decimal('0')`, `LR = 0.0`, `expectancy = WR × avg_win` (no loss contribution).

All losses: `win_count = 0` → `avg_win = Decimal('0')`, `WR = 0.0`, `expectancy = -LR × avg_loss = -avg_loss × LR` (negative).

Tests #3, #4 pin both edges.

### Monte-Carlo bootstrap

Algorithm (pure-Python, no numpy):
```python
import random
import hashlib
from decimal import Decimal

def compute_monte_carlo(rows, *, n_simulations, seed):
    sample = [r.realized_pnl for r in rows]
    if not sample:
        return MonteCarloResult(n_simulations, seed, *([Decimal('0')] * 5))
    rng = random.Random(seed)
    end_equities = []
    for _ in range(n_simulations):
        draw = rng.choices(sample, k=len(sample))
        end_equities.append(sum(draw))
    end_equities.sort()
    p5, p25, p50, p75, p95 = (
        _percentile_linear(end_equities, p) for p in (5, 25, 50, 75, 95)
    )
    return MonteCarloResult(n_simulations, seed, p5, p25, p50, p75, p95)


def _percentile_linear(sorted_values: list[Decimal], pct: int) -> Decimal:
    """Linear interpolation between sorted values; matches NumPy method='linear'.

    rank = (pct/100) × (n-1)
    lower_idx = floor(rank); upper_idx = ceil(rank); frac = rank - lower_idx
    result = sorted[lower_idx] + frac × (sorted[upper_idx] - sorted[lower_idx])
    """
```

### MC seed derivation (router-side per WG#6)

```python
def _derive_mc_seed(bot_id: str | None, from_at: datetime | None,
                    to_at: datetime | None, n_simulations: int) -> int:
    """SHA256-truncated int seed; deterministic for same request shape."""
    parts = [str(bot_id or ""), str(from_at or ""), str(to_at or ""), str(n_simulations)]
    raw = "|".join(parts).encode("utf-8")
    digest = hashlib.sha256(raw).digest()[:8]  # 8 bytes = 64-bit seed range
    return int.from_bytes(digest, byteorder="big", signed=False)
```

This makes MC results **reproducible** across calls with same inputs (cache-coherent + debug-friendly). Test #12 pins via two `compute_monte_carlo` calls with same seed returning identical percentiles.

### Percentile interpolation

NumPy's default `numpy.percentile(method='linear')` interpolation:
- `rank = pct/100 × (n-1)` (0-indexed)
- `lower = floor(rank); upper = ceil(rank); frac = rank - lower`
- `result = sorted[lower] + frac × (sorted[upper] - sorted[lower])`

Hand-computed example for `[1, 2, 3, 4, 5]`:
- p50: rank = 0.5 × 4 = 2.0 → result = sorted[2] = 3
- p75: rank = 0.75 × 4 = 3.0 → result = sorted[3] = 4
- p25: rank = 0.25 × 4 = 1.0 → result = sorted[1] = 2

For non-integer ranks: e.g. p30 of `[1, 2, 3, 4, 5]`: rank = 0.3 × 4 = 1.2 → lower=1, upper=2, frac=0.2 → result = 2 + 0.2 × (3 − 2) = 2.2.

## Write-time guidance (plan-reviewer APPROVE 2026-05-03 — 8 pins)

1. **`AnalyticsCache.get_or_compute` lock granularity** — `_global_lock` must be held ONLY for `_locks.setdefault(key, asyncio.Lock())` mutation, NEVER during `await compute_fn()` callback. Holding global during compute eliminates per-key anti-thundering-herd benefit. Pattern verbatim from ADR-0006 D4. Add test asserting concurrent A/A serialize between themselves, B/B serialize between themselves, A/B run in parallel.

2. **MC endpoint `_compute` resource ordering** — DB pool connection MUST be released BEFORE `asyncio.to_thread(compute_monte_carlo, ...)` call. Pattern: `async with pool.acquire() as conn: rows = await select_trades_for_analytics(...)` exits context manager → THEN `await asyncio.to_thread(compute_monte_carlo, rows, ...)`. CPU bootstrap is 50-100ms; holding pool slot during compute multiplies pool occupancy 5-10×. Document inline in `_compute` body.

3. **`compute_expectancy` empty-input short-circuit** — explicit `if not rows: return ExpectancyMetrics(0.0, 0.0, 0.0, Decimal('0'), Decimal('0'), 0, 0, 0)` BEFORE any division/iteration. Otherwise `Decimal('0') / Decimal('0')` raises `decimal.DivisionUndefined`. Test #1 pins result; WG#3 pins implementation path.

4. **Decimal precision determinism** — compute functions MUST NOT call `decimal.getcontext().prec = ...` nor `Decimal.quantize(...)` on intermediate values (avg_win/avg_loss/percentiles). Default `getcontext().prec = 28` must be consistently used test-side and implementation-side. Otherwise `metrics.avg_win == Decimal('23') / Decimal('3')` is flaky on context change. Add module-level docstring note: "compute functions operate under default Decimal context (prec=28); no quantize on intermediate values."

5. **`compute_hourly_heatmap` weekday API** — use `dt.weekday()` (returns `0=Monday..6=Sunday`), NOT `dt.isoweekday()` (returns `1..7`). Plan says "0=Monday, 6=Sunday (ISO 8601)" matching `weekday()` semantics. Test #7 explicit assert for fixture with `Sunday` trade → cell `weekday=6`. Test name: `test_compute_hourly_heatmap_uses_weekday_not_isoweekday`.
