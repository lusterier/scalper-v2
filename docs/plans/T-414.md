### Behavioral contract — UTC timestamp formatting (per CONCERN #1 fix — extract 2 helpers, path A)

Per T-413 SignalFeed precedent, audit-relevant timestamps (signal `received_at`, trade `opened_at` / `closed_at`, evaluation `evaluated_at`) MUST render with explicit " UTC" suffix. Extract **2 helpers** to `ui/src/lib/format-time.ts` (path A from pass-2 BLOCKER #1 reconcile — different format intents):

```tsx
// Compact time-only "HH:MM:SS UTC" — for high-frequency feeds (e.g.,
// SignalFeed last-50 list where date is implicit from listing scope).
export function formatUtcTimeOnly(iso: string): string {
  const d = new Date(iso);
  const HH = String(d.getUTCHours()).padStart(2, "0");
  const MM = String(d.getUTCMinutes()).padStart(2, "0");
  const SS = String(d.getUTCSeconds()).padStart(2, "0");
  return `${HH}:${MM}:${SS} UTC`;
}

// Full "yyyy-MM-dd HH:MM:SS UTC" — for audit-trail-grade display (trade
// summary opened_at/closed_at, signal detail received_at, scoring
// evaluation evaluated_at). Date matters when surface mixes events from
// different days.
export function formatUtcDateTime(iso: string): string {
  const d = new Date(iso);
  const yyyy = d.getUTCFullYear();
  const mm = String(d.getUTCMonth() + 1).padStart(2, "0");
  const dd = String(d.getUTCDate()).padStart(2, "0");
  return `${yyyy}-${mm}-${dd} ${formatUtcTimeOnly(iso)}`;
}
```

SignalFeed.test.tsx test #1 continues to pass (asserts cross-bot banner + symbol + ordering, NOT exact timestamp string format). New test in plan §`Test strategy` locks the choice: "SignalFeed continues to render time-only (formatUtcTimeOnly) — date components MUST NOT appear in feed row".
