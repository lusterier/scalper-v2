## Mapping table (the central artifact — backfill SQL CASE ≡ `derive_lifecycle_state`)

Steady-state subset only (historical rows); transient states (`SIGNAL_RECEIVED`/`ORDER_REQUESTED`/`ORDER_PLACED`/`CLOSING`/`TP_HIT`) are **forward-only via T-533b dual-write** — backfill never produces them (no legacy column records them).

**`trades.status` vocabulary = `{'open','closed','error'}`** (B1 fix) — `packages/core/types.py:144-154` `class TradeStatus(StrEnum): OPEN='open'/CLOSED='closed'/ERROR='error'` (docstring: `error` for "orphan / partial-failure trades flagged by T-221"). Only writers: `insert_trade`→`'open'` (`packages/db/queries/execution.py:189`); `update_trade_close`→`'closed'` (execution.py:11 — the SOLE UPDATE site; reconcile/restart close via `update_trade_close` so even `reconcile_gone` rows are `status='closed'`). No code currently writes `'error'` (exhaustive grep — zero), but the `TradeStatus.ERROR` domain value is defined for the T-221 intent → mapped **defensively / enum-vocab-complete** (`error→FAILED`; a future T-221 `error` writer or manual repair is then correctly classified, no migration change). `status` is the **sole top-level discriminator** (B2 — no competing 'failed' rule).

| `trades.status` | `trades.close_reason` | `position_state` (LEFT JOIN `ps.trade_id = trades.id`) | → `lifecycle_state` |
|---|---|---|---|
| `'error'` | (any / NULL) | (n/a) | `FAILED` |
| `'closed'` | `'reconcile_gone'` | (n/a — no live ps for this trade_id) | `RECONCILED` |
| `'closed'` | else (`'emergency'`/`'manual'`/`'sl'`/`'trail'`/`'unknown'`/NULL) | (n/a) | `CLOSED` |
| `'open'` | (NULL) | `ps.sl_type='trail'` OR `ps.trailing_active=true` | `TRAILING_ACTIVE` |
| `'open'` | (NULL) | `ps.sl_type='be'` (and not trail) | `BREAKEVEN_SET` |
| `'open'` | (NULL) | `ps.tp_hit=true` (and not be/trail) | `PARTIALLY_CLOSED` |
| `'open'` | (NULL) | `ps` present, else (`sl_type='protective'`/NULL, `tp_hit=false`) | `OPEN` |
| `'open'` | (NULL) | **no `ps` row for this `trade_id`** | `ORPHANED` |

Top-level precedence (status): `error`→`FAILED` | `closed`→{`reconcile_gone`→`RECONCILED`, else→`CLOSED`} | `open`→{ps-substate}. Within `status='open'` (first match wins, top-down): trail → be → tp_hit-partial → plain-open; **no `ps` row for `trades.id` → `ORPHANED`**. `close_reason` vocabulary evidence (the `closed`-else enumeration): `_EXEC_TYPE_TO_CLOSE_REASON` (`services/execution/app/reconcile.py:53-57`) = `{close:'manual', sl:'sl', trail:'trail', unknown:'unknown'}` + `'emergency'` (`placement_persist.py:239/299/444/457`) + `'reconcile_gone'` (`restart.py:156`) + NULL → ONLY `'reconcile_gone'`→`RECONCILED`, all others→`CLOSED`. **ORPHANED evidence (C1)**: a `status='open'` trade normally HAS a `position_state` row — `insert_position_state` in the placement tx (`placement_persist.py:582`), `delete_position_state` only on close (`placement_persist.py:460`); so `status='open'` + no `ps` for `trade_id` is the genuine DB-divergence/orphan case (H-020 territory) → `ORPHANED` is correct (NOT `OPEN`). The Python `derive_lifecycle_state` precedence is identical (same top-down order); the L-003 cross-check pins SQL≡Python for every combination.
