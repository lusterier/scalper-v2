## Scope (T-217a)

T-217a ships the **state-tracking** half of the PositionLifecycle FSM monitor per §9.5:1585-1592:

1. Per-trade `asyncio.Task` spawned post-T-216b2 emit boundary; named `lifecycle_<bot_id>_<trade_id>`.
2. Tick at `Settings.position_poll_interval_s` (default 1.0; configurable per §N9).
3. Read latest price from NATS KV bucket `latest_price`, key `{symbol}` (Decimal-as-string bytes).
4. Compute MFE / MAE / best_price / running_pnl per tick.
5. UPDATE `position_state` via PK-keyed helper (H-018); no SL adjustments in T-217a.
6. Self-cancel when `select_position_state` returns `None` (T-219 close DELETE → graceful exit).
7. Lifespan integration: `app.state.position_lifecycle_tasks: dict[int, asyncio.Task]` keyed on `trade_id`; reverse-shutdown drain BEFORE dispatcher_tasks.

**Out of scope (T-217b owns)**:
- BE trigger detection + `set_trading_stop(tpsl_mode='Full', sl_price=new_be)` invocation.
- Trail SL update logic (best_price ± trail_pct → set_trading_stop).
- `position_state.sl_type` / `sl_price` / `tp_hit` / `trailing_active` writes.

**Out of scope (T-F2+ opportunistic)**:
- Producer side of `latest_price` KV bucket — T-F2+ task adds `kv_put` to `packages/market/ohlc.py:_publish` post-publish call (~5 LOC).
- REST fallback `adapter.get_latest_price(symbol)` — KV-only strategy in T-217a; if KV stale >5 ticks, log WARN + pause monitor; T-221 reconciliation is the recovery path. Per OQ-C scope-reduce.

## NEW `services/execution/app/lifecycle.py` (~150 LOC)

```python
"""§9.5:1585-1592 PositionLifecycle state monitor (T-217a).

Per-trade ``asyncio.Task`` spawned post-T-216b2 emit boundary. Each tick:

1. Sleeps ``settings.position_poll_interval_s`` seconds.
2. Reads latest price from NATS KV bucket ``latest_price`` keyed on symbol.
3. Reads current ``position_state`` row (PK ``(bot_id, symbol)``); if None → exit
   (T-219 close DELETE'd the row).
4. Computes MFE / MAE / best_price / running_pnl per side.
5. UPDATEs ``position_state`` via PK-keyed helper (H-018).
6. Loops.

T-217a does NOT do SL adjustments (BE trigger / trail update); T-217b owns those.

Stale-price handling: if KV read returns None or empty for
``settings.position_poll_stale_ticks`` consecutive ticks, log WARN
``execution.lifecycle_price_stale_pause`` and continue (does NOT exit).
T-221 reconciliation owns post-restart recovery from prolonged staleness.

Cancellation: ``asyncio.CancelledError`` propagates cleanly without log noise
(graceful shutdown). Lifespan reverse-shutdown cancels position_lifecycle_tasks
BEFORE dispatcher_tasks so monitor stops UPDATE'ing position_state before
dispatcher's fill flow could race.
"""

from __future__ import annotations

import asyncio
from decimal import Decimal
from typing import TYPE_CHECKING

from packages.db.queries.execution import (
    select_position_state,
    update_position_state_monitor_tick,
)

if TYPE_CHECKING:
    from collections.abc import Callable
    from datetime import datetime

    import asyncpg
    from structlog.stdlib import BoundLogger

    from packages.bus import NatsClient
    from packages.core import BotId


__all__ = ["run_position_monitor_for_trade"]


_LATEST_PRICE_BUCKET = "latest_price"


async def run_position_monitor_for_trade(  # noqa: PLR0913 — single ctor-style spawn site
    *,
    bot_id: BotId,
    symbol: str,
    trade_id: int,
    side: str,
    entry_price: Decimal,
    qty: Decimal,
    pool: asyncpg.Pool,
    bus: NatsClient,
    bound_logger: BoundLogger,
    poll_interval_s: float,
    stale_ticks_threshold: int,
    now_fn: Callable[[], datetime],
) -> None:
    """T-217a per-trade state-tracking monitor body.

    Lifecycle: spawn → loop ticks until position_state DELETE'd or task cancelled.
    """
    side_factor = Decimal("1") if side == "buy" else Decimal("-1")
    stale_count = 0
    while True:
        try:
            await asyncio.sleep(poll_interval_s)
        except asyncio.CancelledError:
            raise

        # 1. Read latest price.
        latest = await bus.kv_get(_LATEST_PRICE_BUCKET, symbol)
        if latest is None:
            stale_count += 1
            if stale_count == stale_ticks_threshold:
                bound_logger.warning(
                    "execution.lifecycle_price_stale_pause",
                    bot_id=bot_id,
                    trade_id=trade_id,
                    symbol=symbol,
                    stale_ticks=stale_count,
                )
            continue
        price_bytes, _revision = latest
        try:
            current_price = Decimal(price_bytes.decode("utf-8"))
        except (ValueError, UnicodeDecodeError) as exc:
            bound_logger.warning(
                "execution.lifecycle_price_decode_error",
                bot_id=bot_id,
                trade_id=trade_id,
                symbol=symbol,
                error=str(exc),
            )
            stale_count += 1
            continue
        stale_count = 0

        # 2. Read position_state; None → exit (T-219 close DELETE).
        async with pool.acquire() as conn:
            ps = await select_position_state(conn, bot_id=bot_id, symbol=symbol)
            if ps is None:
                bound_logger.info(
                    "execution.lifecycle_exit_position_closed",
                    bot_id=bot_id,
                    trade_id=trade_id,
                    symbol=symbol,
                )
                return

            # 3. Compute monitor metrics.
            running_pnl = (current_price - entry_price) * qty * side_factor
            new_best = _update_best_price(side, ps.sl_price, current_price, ps_best=getattr(ps, "best_price", None))
            new_mfe, new_mae = _update_mfe_mae(side, current_price, ps_mfe=getattr(ps, "mfe_price", None), ps_mae=getattr(ps, "mae_price", None))

            # 4. UPDATE position_state PK-keyed (H-018-symmetric for composite PK).
            await update_position_state_monitor_tick(
                conn,
                bot_id=bot_id,
                symbol=symbol,
                best_price=new_best,
                mfe_price=new_mfe,
                mae_price=new_mae,
                running_pnl=running_pnl,
                updated_at=now_fn(),
            )


def _update_best_price(
    side: str,
    sl_price: Decimal | None,  # unused in T-217a; kept for T-217b symmetry
    current_price: Decimal,
    *,
    ps_best: Decimal | None,
) -> Decimal:
    """best_price tracks the most-favorable price observed since entry.

    Long side → best is the **highest** price seen.
    Short side → best is the **lowest** price seen.
    Initial (None) → seed with current_price.
    """
    if ps_best is None:
        return current_price
    if side == "buy":
        return current_price if current_price > ps_best else ps_best
    return current_price if current_price < ps_best else ps_best


def _update_mfe_mae(
    side: str,
    current_price: Decimal,
    *,
    ps_mfe: Decimal | None,
    ps_mae: Decimal | None,
) -> tuple[Decimal, Decimal]:
    """MFE = maximum favorable excursion; MAE = maximum adverse excursion.

    Long: MFE = highest price; MAE = lowest price.
    Short: MFE = lowest price; MAE = highest price.
    Initial (None, None) → seed both with current_price.
    """
    new_mfe = ps_mfe if ps_mfe is not None else current_price
    new_mae = ps_mae if ps_mae is not None else current_price
    if side == "buy":
        if current_price > new_mfe:
            new_mfe = current_price
        if current_price < new_mae:
            new_mae = current_price
    else:  # sell
        if current_price < new_mfe:
            new_mfe = current_price
        if current_price > new_mae:
            new_mae = current_price
    return new_mfe, new_mae
```

```python
async def select_trade_fsm_params(
    conn: _DbExecutor,
    trade_id: int,
) -> dict[str, Decimal] | None:
    """Read FSM runtime params from ``trades.meta`` JSONB.

    Returns ``{'be_trigger': Decimal(...), 'be_sl_level': Decimal(...),
    'trail_pct': Decimal(...)}`` or None if trade row missing.
    Decimal-as-string serialization expected (Pydantic v2 default).
    """
    row = await conn.fetchrow(
        "SELECT meta FROM trades WHERE id = $1",
        trade_id,
    )
    if row is None:
        return None
    meta = row["meta"]  # asyncpg returns dict via JSONB codec
    return {
        "be_trigger": Decimal(meta["be_trigger"]),
        "be_sl_level": Decimal(meta["be_sl_level"]),
        "trail_pct": Decimal(meta["trail_pct"]),
    }


async def update_position_state_monitor_tick(
    conn: _DbExecutor,
    *,
    bot_id: str,
    symbol: str,
    best_price: Decimal,
    mfe_price: Decimal,
    mae_price: Decimal,
    running_pnl: Decimal,
    updated_at: datetime,
) -> None:
    """Composite-PK UPDATE for monitor-loop fields (best_price, MFE, MAE, running_pnl).

    H-018-symmetric: composite PK ``(bot_id, symbol)`` only; no LIKE-style
    multi-row updates possible. Distinct from T-218a ``update_position_state_after_fill``
    (which writes remaining_qty + sl_type) — this writer owns monitor-only fields,
    no overlap with fill-flow writes.
    """
    await conn.execute(
        """
        UPDATE position_state
        SET best_price = $1, mfe_price = $2, mae_price = $3,
            running_pnl = $4, updated_at = $5
        WHERE bot_id = $6 AND symbol = $7
        """,
        best_price, mfe_price, mae_price, running_pnl, updated_at,
        bot_id, symbol,
    )
```

## Hand verification (math-validator critical scope)

T-217a introduces **arithmetic on Decimal values** in `_update_best_price`, `_update_mfe_mae`, and the inline `running_pnl = (current_price - entry_price) * qty * side_factor`. Hand-computable fixtures:

**Fixture 1: Long position, price moves favorably**
- entry=100, qty=10, side='buy' (side_factor=+1)
- Tick 1: current=105 → running_pnl = (105-100) * 10 * 1 = **50.00**; best_price=105 (initial seed via None branch); MFE=105, MAE=105 (initial seed both)
- Tick 2: current=110 → running_pnl=100; best_price=110 (110>105); MFE=110, MAE=105
- Tick 3: current=98 → running_pnl=-20; best_price=110 (98<110, no update); MFE=110 (favorable still), MAE=98 (98<105 unfavorable peak)

**Fixture 2: Short position, price moves favorably**
- entry=100, qty=5, side='sell' (side_factor=-1)
- Tick 1: current=95 → running_pnl=(95-100)*5*-1 = **25.00**; best_price=95 (initial); MFE=95, MAE=95
- Tick 2: current=90 → running_pnl=50; best_price=90 (90<95 favorable for short); MFE=90, MAE=95
- Tick 3: current=102 → running_pnl=-10; best_price=90 (102>90, unfavorable); MFE=90, MAE=102

**Fixture 3: Decimal precision pin (NUMERIC(20,4) column-aware)**
- entry=`Decimal("45000.50")`, qty=`Decimal("0.01")`, current=`Decimal("45100.25")`, side='buy'
- running_pnl = `(Decimal("45100.25") - Decimal("45000.50")) * Decimal("0.01") * Decimal("1")` = `Decimal("99.75") * Decimal("0.01")` = `Decimal("0.9975")` exact at NUMERIC(20,4) — fits column scale without rounding (per WG#2 plan-reviewer fix).
- **Avoid `qty=Decimal("0.001")` fixture variant** because the 5-dp `Decimal("0.09975")` would be PG-rounded to `Decimal("0.0998")` at NUMERIC(20,4) column scale, breaking the "exact, no float coercion" claim — column truncation is not float coercion but it does break direct `Decimal("0.09975") == row["running_pnl"]` assertions.

**Decimal precision invariants**:
- `Decimal("1") if side == "buy" else Decimal("-1")` — string-constructed Decimal literals; no float intermediate.
- All arithmetic operates on Decimal × Decimal → Decimal (exact).
- KV bytes decoded `Decimal(price_bytes.decode("utf-8"))` is string→Decimal exact round-trip.
- No `getcontext().prec` mutations.

## Lifecycle

**Pre-T-217a master**: T-216b2 placement persists trades + position_state but no monitor task is spawned; position_state.{best_price, mfe_price, mae_price, running_pnl} stay NULL/0 (server defaults from migration 0006); FSM params from OrderRequest are dropped after placement tx.

**Post-T-217a master**: every successful placement spawns a per-trade lifecycle task that updates monitor fields each tick. T-219 close DELETE → monitor self-exits. Lifespan reverse-shutdown drains lifecycle tasks before dispatcher_tasks. FSM params persisted in trades.meta JSONB ready for T-217b.

**Unblocks T-217b** (BE trigger + trail SL adjustment using FSM params from trades.meta + monitor body extension).

## Edge cases

- **KV read returns None** (producer not writing yet, or transient bus failure): stale_count++ until threshold; WARN log + continue. Monitor never exits on KV failure (T-221 owns recovery).
- **KV value malformed** (not Decimal-parsable bytes): WARN + stale_count++; same as None case.
- **`select_position_state` returns None mid-loop** (T-219 closed the trade): graceful exit + INFO log `execution.lifecycle_exit_position_closed`. Task completes without error.
- **CancelledError mid-sleep**: re-raise without log emit (graceful shutdown).
- **CancelledError mid-tx** (asyncio.shield NOT used): `pool.acquire()` ctx mgr handles cleanup; tx rolls back if INSIDE transaction (we're not; only SELECT + UPDATE without explicit transaction(), since composite-PK UPDATE is atomic at row level).
- **Concurrent ticks on same trade**: not possible — single asyncio.Task per trade; sequential ticks via `await asyncio.sleep`.
- **Race with T-218b dispatcher fill UPDATE on position_state**: dispatcher writes `remaining_qty` + `sl_type`; monitor writes `best_price/mfe/mae/running_pnl`. Disjoint column sets → no row-level UPDATE conflict at PG (last writer wins per column; both compose under T-216b2's distinct tx boundaries). Reverse-shutdown order (cancel monitor BEFORE dispatcher) ensures shutdown is monitor-quiescent before any dispatcher tx commits.

## Write-time guidance (for plan-reviewer carry-forward)

1. **Decimal runtime import**: `from decimal import Decimal` at module-top in lifecycle.py (NOT TYPE_CHECKING) — runtime arithmetic uses Decimal literals + Decimal ctor on KV bytes.
2. **`side_factor` is `Decimal("1")` / `Decimal("-1")`** — never bare `1` / `-1` (Python int × Decimal = Decimal but explicit Decimal literal is consistent with §5.3 precision invariant).
3. **PositionStateRow extension**: T-217a extends the dataclass with `best_price: Decimal | None`, `mfe_price: Decimal | None`, `mae_price: Decimal | None`, `running_pnl: Decimal` (column already exists per migration 0006). `select_position_state` SELECT column list must be extended; existing T-218a callers continue to work since field access is name-based not positional.
4. **`select_position_state` is now reading 4 more columns** — ensure existing T-218a tests still pass (their fixtures must not over-constrain row shape).
5. **`select_trade_fsm_params` JSONB read**: asyncpg returns JSONB columns as native Python dicts (per default codec); `Decimal(meta["be_trigger"])` constructs from string. Tests pin Decimal-as-string round-trip.
6. **`insert_trade` `meta` default**: when omitted, `meta='{}'::jsonb` per current behavior. T-216b1's `emergency_close` path does NOT pass meta — emergency-closed trades carry `'{}'` (no FSM params; never monitored).
7. **Lifecycle task naming**: `name=f"lifecycle_{bot_id}_{trade_id}"` (mirror `dispatcher_{bot_id}` precedent from T-218a). Test pin verifies via `task.get_name()`.
8. **Reverse-shutdown ordering**: cancel position_lifecycle_tasks BEFORE dispatcher_tasks. Lifespan ordering is now: `bus.close → cancel position_lifecycle_tasks + gather → cancel dispatcher_tasks + gather → cancel ws_tasks + paper_consumer_tasks + gather → adapter.close per bot → pool.close`. Test pin verifies cancel order.
9. **Per-tick conn acquire**: `async with pool.acquire() as conn:` per tick — NOT held across ticks. Acquire-and-release pattern minimizes pool contention (1s tick × N trades × 1 conn-per-tick = bounded).
10. **No transaction context** — SELECT + UPDATE per tick are independently atomic at PG row level. Composite-PK UPDATE is atomic for the row's columns being written (best_price/mfe/mae/running_pnl/updated_at).
11. **Stale-tick threshold WARN at exactly N consecutive misses** (not every miss after N) — `if stale_count == stale_ticks_threshold:` (equality not ≥) avoids log spam. Test pin verifies.
12. **Stale count resets to 0 on first successful read** — no decay logic; binary recovery.
13. **`getattr(ps, "best_price", None)`** anti-pattern — DO NOT use; T-217a extends PositionStateRow with the field, so `ps.best_price` is direct attribute access. Plan draft used getattr for write-time clarity; in implementation use direct access (mypy strict catches typos).
14. **Mid-write LOC checkpoint per L-007**: lifecycle.py ≤ 200 LOC during write; halt at 200, hard ceiling 240. Per WG#3 carry from T-218b precedent.
15. **Forward-pointer comment** in `lifecycle.py` module docstring: T-217b will extend `run_position_monitor_for_trade` body with BE/trail SL adjustment logic + `select_trade_fsm_params(trade_id)` call. T-217a leaves a clear extension seam (post-`update_position_state_monitor_tick` is the natural insertion point).
