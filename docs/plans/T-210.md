## Distinct mechanism ŌĆö NOT JetStream `Nats-Msg-Id` / `duplicate_window`

Per the operator's standing memory note, T-210 dedup is **consumer-side, in-process, ring-bounded**, fundamentally distinct from the JetStream server-side `duplicate_window` (T-012 SIGNALS stream, `Nats-Msg-Id` header). Two layers:

| Layer | Where | What | Scope |
|-------|-------|------|-------|
| JetStream `duplicate_window` | NATS server | Publisher-side dedup via `Nats-Msg-Id` header | Cross-publisher idempotency window (e.g., 2m) |
| **DedupingConsumer** (T-210) | execution-service process | Consumer-side dedup of WS-replayed events keyed by `execId` | In-process ring of last N seen keys |

WS reconnect can replay events the publisher already considered "delivered" ŌĆö JetStream-side dedup is irrelevant for private-WS streams (no NATS in the loop). T-210 is the consumer-side correctness guard for the H-009 hazard.

**Also distinct from `services.signal_gateway.app.dedup.DedupRing`** (T-015b1) ŌĆö that ring is **TTL-based** (10s purge of `idempotency_key`s) for HTTP webhook idempotency. T-210 is **size-bounded** (capacity 10k FIFO eviction of `execId`s) for WS event dedup. Different contracts; the two coexist. T-210 lives in `packages/bus` (cross-service), T-015b1 lives in `services/signal_gateway/app` (service-local).

## Public interface

### `packages/bus/dedup.py` ŌĆö module + class

```python
"""┬¦20 H-009 + ┬¦9.5 line 1591 DedupingConsumer base class.

Generic, in-memory, size-bounded FIFO ring keyed by configurable
extractor. Subclass overrides ``_process(message)`` and binds
``key_fn`` at construction time.

NOT a JetStream-side dedup mechanism (publisher-side
``Nats-Msg-Id`` / ``duplicate_window`` is server-managed; this is
consumer-side). NOT the signal-gateway DedupRing (TTL-based; this
is size-bounded).
"""
from __future__ import annotations

import asyncio
from collections import OrderedDict
from typing import TYPE_CHECKING, Generic, TypeVar

if TYPE_CHECKING:
    from collections.abc import Callable

    from structlog.stdlib import BoundLogger

__all__ = ["DedupingConsumer"]

T = TypeVar("T")
_DEFAULT_CAPACITY: int = 10_000


class DedupingConsumer(Generic[T]):
    """┬¦20 H-009: consumer-side dedup of replay-prone event streams.

    Tracks the last ``capacity`` keys seen via ``key_fn``. Calls to
    :meth:`consume` invoke :meth:`_process` exactly once per distinct
    key within the ring window; subsequent duplicates are dropped
    silently with a DEBUG log entry.

    Subclass binding:

        class ExecutionDispatcher(DedupingConsumer[ExecutionEvent]):
            def __init__(self, ...):
                super().__init__(key_fn=lambda e: e.exchange_exec_id, ...)
            async def _process(self, event: ExecutionEvent) -> None:
                ...  # business logic

    Single event loop assumption: the internal :class:`asyncio.Lock`
    serialises concurrent ``consume`` invocations from the same loop.
    Cross-process / cross-loop coordination is out of scope (use a
    DB-side dedup or a shared NATS dedup window for that).
    """

    def __init__(
        self,
        *,
        key_fn: Callable[[T], str],
        capacity: int = _DEFAULT_CAPACITY,
        logger: BoundLogger | None = None,
    ) -> None: ...

    async def consume(self, message: T) -> None:
        """Run ``_process(message)`` iff ``key_fn(message)`` is fresh.

        Duplicate keys (still resident in the ring) are dropped with
        a DEBUG log entry; ``_process`` is not invoked.
        """
        ...

    async def _process(self, message: T) -> None:  # noqa: ARG002
        """Subclass override ŌĆö handle a single distinct-key message."""
        raise NotImplementedError(
            "Subclasses of DedupingConsumer must override _process"
        )
```

**Implementation sketch (load-bearing details):**

- `self._seen: OrderedDict[str, None]` ŌĆö ordered for FIFO eviction on overflow. Value is `None` (we only need keys; using `dict` over `set` because `OrderedDict.popitem(last=False)` is the cleanest FIFO eviction primitive).
- `self._lock: asyncio.Lock` ŌĆö serialise `consume` invocations.
- `self._capacity: int` ŌĆö bounded.
- `self._key_fn: Callable[[T], str]` ŌĆö extractor.
- `self._logger: BoundLogger` ŌĆö `structlog`. Default: `packages.observability.get_logger(__name__)`.

`consume` algorithm:
1. Acquire lock.
2. Compute `key = self._key_fn(message)`.
3. If `key in self._seen`: log DEBUG `dedup_dropped` event with `key`, return.
4. Else: `self._seen[key] = None`. If `len(self._seen) > self._capacity`: `self._seen.popitem(last=False)`.
5. Release lock.
6. Call `await self._process(message)` (outside lock ŌĆö long-running handler should not block ring updates).

**Crucial invariant**: the lock protects the **ring** (keys-seen tracking), not the **handler** (`_process`). Two consumes with different keys can run their handlers concurrently; two consumes with the same duplicate key see one process call. Mirror signal-gateway DedupRing single-event-loop assumption.

## Edge cases

- **Concurrent same-key consume**: `asyncio.gather(consumer.consume(msg), consumer.consume(msg))` with same key ŌåÆ `_process` invoked once; second call sees the key in the ring + returns early. Lock makes this race-free within a single event loop.
- **Restart**: in-memory state lost. Acceptable per Decision 5 ŌĆö T-221 reconciliation handles the "what is exchange state now" path via REST snapshot, not by replaying old WS events.
- **Capacity 0 or 1**: edge case for tests. Capacity 1 means each new key evicts the previous ŌåÆ no dedup beyond immediate adjacent duplicates. Used in unit tests for FIFO verification. Capacity 0 raises `ValueError` at construction (rejected as nonsensical).
- **`key_fn` raises**: if extractor blows up (malformed message), `consume` propagates the exception. T-218 plan-doc owns the wrapping policy (e.g., DLQ + log).
- **Long-running `_process`**: lock is released before invoking `_process`, so a slow handler doesn't block the ring. Conversely, two distinct-key messages can have `_process` running concurrently ŌĆö subclass must be re-entrancy-safe at the `_process` level if that matters (T-218 uses asyncpg pool / KV writes; both concurrency-safe).
- **Identity vs. equality on keys**: `key_fn` returns `str` per the type bound. Other types (int, bytes) would also work via `OrderedDict` semantics, but `str` is the brief-implied type (`execId` is a string).

## OQ resolutions (operator-approved 2026-04-27 ŌĆö "use defaults on all 5")

1. **OQ-1 ŌĆö API shape**: **default A ŌĆö template-method pattern**. `consume(message)` does the dedup check internally + invokes `_process(message)` which subclass overrides. 2. **OQ-2 ŌĆö Logging on dedup-drop**: **default A ŌĆö every drop emits DEBUG structured log**. Each duplicate emits `dedup_dropped` event with the dedup key. WS reconnect dedup is rare in normal operation; high-signal during incident triage. ┬¦N2 compliant. If future incident shows log-volume issues, downgrade to sampled is a single-line change.

3. **OQ-3 ŌĆö Prometheus metrics scope**: **default A ŌĆö defer to T-218**. 4. **OQ-4 ŌĆö Hypothesis property test**: **default A ŌĆö include**. Mirror T-015b1 `_ReferenceRing` oracle pattern: naive-but-correct reference, generate `(key, ...)` sequences via `hypothesis.strategies`, replay through both ring and reference, assert results match. ~40-50 LOC. Catches sizing-edge bugs (off-by-one in eviction) that hand-written unit tests miss.

5. **OQ-5 ŌĆö Lock granularity**: **default A ŌĆö `asyncio.Lock` per instance** around ring mutation (steps 2-4 of consume algorithm). Mirror T-015b1 DedupRing precedent. Non-controversial defensive correctness; future-maintenance-safe (does not rely on subtle "no awaits in mutation block" invariant).
