## Purpose

The enforcement + trip half of the daily-loss kill-switch. Reads the T-525a1 persistent latch (`bot_kill_switch_state`); lazily computes the bot's cumulative realized P&L for the current UTC trading day (derived from `trades`/`paper_trades`); trips + latches when the sum тЙд `-daily_loss_limit_usd`; blocks every signal while the latch is non-stale-tripped (sticky intra-day тАФ a recovering win does NOT un-trip). Pre-scoring gate in `consumer.py` AFTER the T-524 caps gate (chain cooldown тЖТ caps тЖТ loss-limit), silent-skip pattern. A long-running strategy-engine process that crosses UTC midnight clears a stale prior-day latch in the gate itself (mirrors `reconcile_kill_switch_on_startup`, so the day-rollover does not depend on a restart).

### NEW `packages/db/queries/trades.py::sum_realized_pnl_since` (extend T-526/T-524 module)

```python
async def sum_realized_pnl_since(
    conn: _DbExecutor, *, table_name: TradeTableName, bot_id: str, since: datetime,
) -> Decimal:
    """COALESCE(SUM(realized_pnl), 0) of closed trades for bot since `since` (UTC).
    SQL: WHERE bot_id = $1 AND status = 'closed' AND realized_pnl IS NOT NULL
         AND closed_at >= $2::timestamptz
    Zero rows тЖТ Decimal('0') (never None). NO float() cast (┬з5.13/┬зN1)."""
```

**L-021 тАФ RESOLVED IN-ADVANCE, NOT "verify at impl"**: `closed_at >= $2` is a comparison-operator parameter context тАФ exactly the L-021 failure class ("comparison operators across unioned types"); T-537a1's ci-full crash was a `timestamptz <=` mis-inference that mock tests missed. The SQL string ships with an **explicit `$2::timestamptz` cast** (3-char fix, mirror the L-021 active control verbatim). `$1` (bot_id) is column-direct TEXT equality тЖТ L-021-safe, no cast. Mock test pins `assert "$2::timestamptz" in sql`. Per WG (carried): no `float()` on the P&L path; `COALESCE(SUM(realized_pnl), 0)` returns NUMERIC тЖТ asyncpg `Decimal`.

### NEW `services/strategy_engine/app/loss_limit_gate.py`

```python
@dataclass(frozen=True, slots=True)
class LossLimitDecision:
    blocked: bool
    reason: str | None               # binding latch reason ('daily_loss_limit' on fresh
                                      # trip; state.trip_reason if pre-existing latch тАФ e.g.
                                      # 'max_drawdown' once T-525b ships) | None
    cumulative_loss_usd: Decimal | None
    limit_usd: Decimal | None

async def check_daily_loss_limit(
    *, pool, bot_id: BotId, exchange_mode: Literal["live","testnet","paper"],
    now: datetime, risk_config: RiskSection,
) -> LossLimitDecision
```

**Logic** (carried from `docs/plans/T-525a.md` Hand verification + operator OQ 2026-05-15):

1. `risk_config.daily_loss_limit_usd <= 0` тЖТ return `_NOT_BLOCKED` **before any DB** (short-circuit; mirror T-524/T-526; a disabled feature ignores even a lingering latch).
2. `async with pool.acquire() as conn:` (single acquire, no explicit txn тАФ read-then-conditional-`@idempotent`-write; a concurrent double-trip is convergent per T-525a1 ON CONFLICT).
3. `state = await select_kill_switch_state(conn, bot_id=str(bot_id))`.
4. If `state is not None and state.tripped`:
   - `is_stale_daily_latch(state, now)` True тЖТ `await clear_kill_switch(conn, bot_id=str(bot_id), updated_at=now)`; **fall through to recompute** (new UTC day; gate handles day-rollover for a non-restarting process тАФ mirrors reconcile Day-2 in T-525a Hand verification).
   - Else (non-stale tripped) тЖТ **blocked**, **reason-agnostic per operator OQ 2026-05-15**: `reason=state.trip_reason` (a kill-switch is a kill-switch тАФ block whether it tripped on `daily_loss_limit` or future `max_drawdown`; forward-compat with T-525b, no chain-order fragility), `cumulative_loss_usd=state.cumulative_loss_usd`, `limit_usd=daily_loss_limit_usd`. **Recompute SKIPPED** (sticky latch тАФ T-525a Hand verification cases 5/7; the +60 recovering win does NOT un-trip).
5. (not latched, or latch was stale-cleared) `since = now.astimezone(UTC).replace(hour=0, minute=0, second=0, microsecond=0)` (UTC-midnight of the current trading day; ┬зN1).
6. `table_name = "trades" if exchange_mode in ("live","testnet") else "paper_trades"` (mirror T-524/T-526 dispatch).
7. `total = await sum_realized_pnl_since(conn, table_name=table_name, bot_id=str(bot_id), since=since)`.
8. `total <= -daily_loss_limit_usd` (Decimal `<=`; boundary `-100 <= -100` тЖТ trip, per T-525a Hand verification) тЖТ `await upsert_kill_switch_trip(conn, bot_id=str(bot_id), trip_reason="daily_loss_limit", tripped_at=now, daily_anchor_date=now.date(), cumulative_loss_usd=total)` тЖТ **blocked** (`reason="daily_loss_limit"`, `cumulative_loss_usd=total`, `limit_usd=daily_loss_limit_usd`).
9. Else тЖТ `_NOT_BLOCKED`.
