### `ui/src/lib/hooks/useDebouncedValidation.ts` (NEW, ~50 LOC)

```tsx
interface UseDebouncedValidationResult {
  valid: boolean;
  errors: string[];
  parsedVersion: number | null;
  isPending: boolean;
}

export function useDebouncedValidation(
  yamlText: string,
  botId: string,
  debounceMs: number = 500,
): UseDebouncedValidationResult;
```

Internal pattern: `useEffect` with `setTimeout` wrapper around `apiFetch<ConfigValidateResponse>("/api/configs/validate", { method: "POST", body: { bot_id, yaml_text } })`. State machine: idle → debouncing → fetching → result → debouncing → ... Cleanup on `yamlText` change cancels prior fetch via AbortController.

**Per WG#X — empty yamlText skip**: do NOT POST when `yamlText.trim() === ""` (saves backend cycles + avoids 422 on Pydantic min_length=1 violation; backend test exercises this). Result returns `{ valid: false, errors: ["yaml_text empty"], parsedVersion: null, isPending: false }` synchronously.

### Behavioral contract — validation flow

`useDebouncedValidation(yamlText, botId, 500)`:

1. On `yamlText` change → cancel prior debounce timer + AbortController.
2. If `yamlText.trim() === ""` → return synchronous `{ valid: false, errors: ["yaml_text empty"], parsedVersion: null, isPending: false }`.
3. Otherwise → set `isPending=true` after 500ms debounce → fire `apiFetch<ConfigValidateResponse>("/api/configs/validate", { method: "POST", body: { bot_id, yaml_text }, signal })`.
4. On response → update state with backend `valid` / `errors` / `parsed_version`.
5. On AbortError → keep prior result (no flicker).
6. Cleanup on unmount → cancel timer + abort.

**Per WG#X — 200 always semantic**: backend `/api/configs/validate` returns 200 with `valid: bool + errors` (NOT 422 on validation failure). Hook treats `apiFetch` resolution as success regardless of `valid` value. UI renders status based on `valid` field.

### Behavioral contract — apply flow

`useMutation` against POST `/api/configs/{bot_id}/apply`:

```tsx
const applyMutation = useMutation({
  mutationFn: (body: ConfigApplyRequest) =>
    apiFetch<BotConfig>(`/api/configs/${botId}/apply`, { method: "POST", body }),
  onSuccess: (newVersion) => {
    queryClient.invalidateQueries({ queryKey: ["config-current", botId] });
    queryClient.invalidateQueries({ queryKey: ["config-versions", botId] });
    setApplyModalOpen(false);
    // Toast not implemented in T-416 (deferred); modal closes silently on success.
  },
});
```
