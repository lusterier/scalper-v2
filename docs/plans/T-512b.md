## Architecture: in-process simulated restart

Rationale (per operator OQ-1=A 2026-05-08): repo precedent for integration tests is in-process FastAPI TestClient + testcontainer (signal-gateway e2e at `services/signal_gateway/tests/integration/test_webhook_e2e.py` is canonical). True subprocess+SIGTERM introduces large new test infrastructure (process management, readiness probes, log capture, race-windows on startup) for marginal additional coverage versus the in-process flow. BRIEF §13.7:2037 verbatim says *"Integration: full variant lifecycle under testcontainers with simulated ticks"* — does not specifically mandate subprocess.

**Restart simulation flow:**

```
1. Pre-seed: parent paper_trade or trade row + shadow_variant row (terminated_at NULL) + ohlc_1m candles.
2. Run ShadowWorker A briefly (~50ms) — variant in-flight, FSM running.
3. Simulate kill: shadow_worker_A.stop() cancels variant task.
   Per shadow_worker.py:388 try/finally semantics, cancellation does NOT finalize the DB row
   (update_shadow_variant_terminal is INSIDE try:, NOT in finally:; finally only does H-016
   bus_unsubscribe + _active_tasks pop). Row stays terminated_at NULL.
4. bus.close() drains subscriptions (mirror real shutdown order from main.py:365).
5. Construct fresh bus_B + ShadowWorker B + call resume_active_variants_on_startup directly
   (the lifespan hook from T-512a). Equivalent to the lifespan invocation in main.py:268.
6. Assert via DB SELECT.
```

This faithfully exercises:
- Cancellation-does-not-finalize contract (shadow_worker.py:388 try/finally invariant).
- T-512a `select_all_active_shadow_variants` cross-bot enumeration filtering on `terminated_at IS NULL`.
- T-512a `replay_shadow_variant_to_now` + downstream `_check_parent_open` + `_replay_candle_loop` + (a) `_finalize_replay_terminal` finalize path OR (b) `_live_continuation` resume path.

**What it does NOT exercise:**
- uvicorn's SIGTERM signal handler (FastAPI lifespan shutdown is invoked the same way regardless of trigger).
- Cross-process state isolation (in-process pool/bus instances are deliberately re-created to mimic two distinct subprocesses).

The deltas above are operationally minor for the in-process flow. The H-023 hazard is about *replay correctness across restart*, which the in-process test fully verifies.

## Test strategy

**Test #1: replay-finalize path (the BRIEF §20:2787 verbatim test name).**

Setup:
- Parent paper_trade row: bot_id="alpha", symbol="BTCUSDT", side="buy", entry_price=Decimal("65000"), qty=Decimal("0.001"), open_at=now-10min.
- Shadow variant row: parent_trade_id=above, name="default", side="buy", entry_price=Decimal("65000"), qty=Decimal("0.001"), created_at=now-10min, terminated_at=NULL, meta={"symbol":"BTCUSDT","overrides":{"sl_pct":"0.005","tp_pct":"0.01","tp_qty_pct":"1","be_trigger":"0","be_sl_level":"0","trail_pct":"0","max_duration_hours":"4"}}.
- ohlc_1m candles spanning [now-10min, now-1min]:
  - Candle 0..3: prices around 65000 (no trigger).
  - Candle 4: low = 64600 (CROSSES SL at 64675 = 65000 × (1 - 0.005)) → SL_HIT terminal.
  - Candle 5..8: don't matter (replay returns True after candle 4 terminal).

Execution:
1. Run ShadowWorker A briefly: emit synthetic ShadowStartPayload via bus (or direct `_run_shadow_variant` invocation if simpler) → wait 50ms → stop().
2. Verify DB: `SELECT terminated_at FROM shadow_variants WHERE id=...` → NULL.
3. Construct fresh ShadowWorker B + bus_B → `resume_active_variants_on_startup(...)`.
4. Assert: row.terminated_at NOT NULL + row.terminal_outcome == 'sl_hit' + row.realized_pnl ≠ NULL + row.mfe_pct + row.mae_pct populated.

**Test #2: replay-resume path.**

Setup:
- Same paper_trade + variant as #1 BUT with overrides max_duration_hours = 1 (so resume timeout fits in test budget).
- Shadow variant created_at = now - 30min (still within 1h max_duration).
- ohlc_1m candles in [now-30min, now-1min] with prices oscillating around 65000 (NO triggers — all in [64850, 65150] range; SL=64675 not crossed; TP=65650 not crossed).

Execution:
1. Run ShadowWorker A briefly → stop().

3. Construct fresh ShadowWorker B → resume_active_variants_on_startup.
4. Assert via shadow_worker_B internal state: `parent_trade_id` registered in `_active_tasks` via `register_resume_task` → live continuation task is running.
5. Verify via DB: still NULL (no terminal yet).
6. Cleanup: cancel the registered task to drain test (mirror await asyncio.gather pattern from main.py shutdown).

**Hand verification (math sanity check):**

Per L-014 active control + math-validator gate. Worked example for Test #1 SL trigger:

```
entry_price = 65000 (Decimal)
sl_pct = 0.005 (Decimal)
side = buy
sl_price = entry × (1 - sl_pct) = 65000 × 0.995 = 64675 (Decimal-exact)

Candle 4: low = 64600 < sl_price (64675) → SL crosses.
Per PaperExchange._compute_intra_candle_path: price-walk crosses SL FIRST (long side; SL below entry, crossed when low ≤ SL).
Terminal outcome: ShadowVariantTerminal.SL_HIT.
realized_pnl = (sl_price - entry_price) × qty - fees
            ≈ (64675 - 65000) × 0.001 = -0.325 USD - fees
mfe_pct = (best_price - entry) / entry  (best_price == entry since price never went above entry)
       = 0
mae_pct = (entry - worst_price) / entry = (65000 - 64600) / 65000 ≈ 0.006154 (the candle low)
```

These values are not literally asserted in the test (PaperExchange internal slippage + fee math has its own settings); the test asserts the TERMINAL CLASSIFICATION (`'sl_hit'`) + non-null realized_pnl/mfe/mae rather than exact magnitudes. Magnitude precision is verified by T-512a + T-511b1 unit tests. Test #1 asserts: outcome value == "sl_hit" string; realized_pnl is not None; mfe_pct is not None; mae_pct is not None.

Math-validator should verify: (a) the SL price computation 65000 × 0.995 = 64675 matches `_compute_initial_sl/tp` math from T-511b1 / shadow_replay.py:263-265 verbatim; (b) candle 4 low = 64600 < SL 64675 should trigger PE intra-candle path; (c) no Decimal→float cast in the assertion path.
