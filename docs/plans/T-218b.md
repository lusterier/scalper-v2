```python
async def select_open_order_id_by_trade_id(
    conn: _DbExecutor, trade_id: int,
) -> int | None:
    """Return ``trades.open_order_id`` given trades.id PK; None if trade missing.

    T-218b dispatcher uses this in the synthetic SL/TP/trail fill path
    where ``executions.order_id`` is NOT NULL FK and we have no orders
    row issued for the synthetic fill — we attribute the FK to the
    entry order context (the order that opened the position the
    synthetic fill is closing). Defensible: every fill is anchored to
    a placed-order context even if the fill itself wasn't issued
    against that order.
    """
    row = await conn.fetchrow(
        "SELECT open_order_id FROM trades WHERE id = $1",
        trade_id,
    )
    return None if row is None else int(row["open_order_id"])
```

```python
async def _process(self, message: ExecutionEvent) -> None:
    """§9.5:1591 + H-024 (v2 per ADR-0005) — derive exec_type from DB state, persist fill, forward-pointer to T-219.

    Single ``conn.transaction()`` wraps lookups + INSERT + UPDATEs +
    (optional) close forward-pointer. NotImplementedError from T-219
    stub propagates through the tx → rollback → ring entry remains
    (in-memory; cleared on restart, so retry-on-restart is clean).
    """
    async with self._pool.acquire() as conn, conn.transaction():
        order_id_match = await select_order_id_by_exchange_id(
            conn, message.exchange_order_id,
        )

        exec_type, trade_id, _sl_type_observed = await _derive_exec_type(
            conn=conn,
            bot_id=self._bot_id,
            event=message,
            order_id_match=order_id_match,
            bound_logger=self._bound_logger,
        )

        # Defensive halt for over-fill (B2 fix): event.qty > remaining_qty would
        # drive position_state.remaining_qty negative on UPDATE, violating the
        # §9.5:1613 invariant `qty_closed + remaining_qty == entry_qty`. Treat
        # over-fill identically to unattributable: ERROR log + RuntimeError →
        # tx rollback → operator alert. T-221 reconciliation is the recovery path.
        # `_derive_exec_type` already emitted WARN with reason='qty_exceeds_remaining'
        # in this branch; here we promote to a halt rather than persisting a 'unknown'
        # row that would silently corrupt remaining_qty.
        if exec_type == "unknown" and trade_id is not None:
            ps_check = await select_position_state(
                conn, bot_id=self._bot_id, symbol=message.symbol,
            )
            if ps_check is not None and message.qty > ps_check.remaining_qty:
                self._bound_logger.error(
                    "execution.dispatcher_overfill_halt",
                    bot_id=self._bot_id,
                    exchange_exec_id=message.exchange_exec_id,
                    event_qty=str(message.qty),
                    remaining_qty=str(ps_check.remaining_qty),
                )
                msg = "over-fill: event.qty > position_state.remaining_qty"
                raise RuntimeError(msg)

        # FK resolution for executions.order_id (NOT NULL FK to orders.id):
        # 'open'/'close' → order_id_match (matched orders.id);
        # synthetic SL/TP/trail / 'unknown' with trade_id → trades.open_order_id
        # (entry order context — defensible: every fill anchored to placed-order context);
        # unattributable / orphan order match / orphan synthetic → raise (B4 + Pass-3 B-invariant
        # fix: every dead-end derivation path raises here BEFORE any INSERT/UPDATE, so close-trigger
        # branch below has trade_id != None invariant by construction; no sentinel, no assert).
        if order_id_match is not None and trade_id is None:
            # Orphan order match: orders row exists for event.exchange_order_id but
            # no trade has it as open_order_id or close_order_id. _derive_exec_type
            # already emitted WARN `dispatcher_orphan_order_match`. _process halts
            # symmetrically with the orphan_synthetic_fill / unattributable paths
            # so reconcile_close can rely on trade_id != None invariant.
            self._bound_logger.error(
                "execution.dispatcher_orphan_order_halt",
                bot_id=self._bot_id,
                exchange_order_id=message.exchange_order_id,
                exchange_exec_id=message.exchange_exec_id,
                order_id_match=order_id_match,
            )
            msg = (
                f"orphan order match: orders.id={order_id_match} matched but no trade "
                f"references it as open or close"
            )
            raise RuntimeError(msg)
        if order_id_match is not None:
            order_id_for_insert = order_id_match
        elif trade_id is not None:
            resolved = await select_open_order_id_by_trade_id(conn, trade_id)
            if resolved is None:
                self._bound_logger.error(
                    "execution.dispatcher_orphan_synthetic_fill",
                    bot_id=self._bot_id,
                    trade_id=trade_id,
                    exchange_exec_id=message.exchange_exec_id,
                )
                msg = f"orphan synthetic fill: trade_id={trade_id} not found"
                raise RuntimeError(msg)
            order_id_for_insert = resolved
        else:
            self._bound_logger.error(
                "execution.dispatcher_unattributable_fill",
                bot_id=self._bot_id,
                exchange_exec_id=message.exchange_exec_id,
                exchange_order_id=message.exchange_order_id,
            )
            msg = "unattributable fill: no order match and no position_state"
            raise RuntimeError(msg)

        await insert_execution(
            conn,
            exchange_exec_id=message.exchange_exec_id,
            order_id=order_id_for_insert,
            trade_id=trade_id,
            bot_id=self._bot_id,
            symbol=message.symbol,
            side=message.side,
            price=message.price,
            qty=message.qty,
            fee=message.fee,
            exec_type=exec_type,
            executed_at=message.executed_at,
        )

        # UPDATE position_state — subtract qty; on 'partial_tp' promote sl_type to 'trail'
        # per ADR-0005 (T-218a OQ-5 v2 semantic).
        new_sl_type: Literal["protective", "be", "trail"] | None = (
            "trail" if exec_type == "partial_tp" else None
        )
        await update_position_state_after_fill(
            conn,
            bot_id=self._bot_id,
            symbol=message.symbol,
            qty_delta=message.qty,
            new_sl_type=new_sl_type,
            updated_at=self._now_fn(),
        )

        # UPDATE trades.fees_paid incremental (PK-only WHERE id; H-018).
        if trade_id is not None:
            await update_trade_fees_incremental(
                conn,
                trade_id=trade_id,
                fee_delta=message.fee,
            )

        # Close forward-pointer (T-219). B1 fix: trigger is purely state-based —
        # `ps_after.remaining_qty == Decimal("0")` — independent of exec_type or
        # sl_type. Re-read inside tx (cheap; defensive against intra-tx race).
        ps_after = await select_position_state(
            conn, bot_id=self._bot_id, symbol=message.symbol,
        )
        if ps_after is not None and ps_after.remaining_qty == Decimal("0"):
            # Invariant: every code path reaching here has trade_id != None.
            # Sources of trade_id:
            #   - orders-lookup match → trade_open.id or trade_close.id (non-None);
            #   - synthetic fill → ps.trade_id (non-None per PositionStateRow.trade_id: int).
            # Dead-end paths (unattributable / orphan_order / orphan_synthetic) raise
            # BEFORE this point, so the type checker's `int | None` reduces to `int` here.
            # No assert (would be stripped by `python -O`); no sentinel; FK resolution
            # raise-pattern is the structural guarantee.
            await reconcile_close(
                conn=conn,
                bound_logger=self._bound_logger,
                bot_id=self._bot_id,
                symbol=message.symbol,
                trade_id=trade_id,
                close_order_id=order_id_match,
                final_fill_price=message.price,
                final_fill_qty=message.qty,
                final_fill_fee=message.fee,
                closed_at=message.executed_at,
            )
```

## NEW private helper `_derive_exec_type` in dispatcher.py (~50 LOC)

(B6 typing decision: `bot_id: BotId` propagates through `_derive_exec_type` to query helpers; `BotId = NewType("BotId", str)` is implicit-assignable to `bot_id: str` parameters in `packages/db/queries/execution.py` per mypy NewType semantics. No `str(bot_id)` cast at call sites.)

```python
async def _derive_exec_type(
    *,
    conn: _DbExecutor,
    bot_id: BotId,
    event: ExecutionEvent,
    order_id_match: int | None,
    bound_logger: BoundLogger,
) -> tuple[str, int | None, str | None]:
    """Return ``(exec_type, trade_id, sl_type_after_fill)`` per H-024.

    Branches:

    * ``order_id_match is not None`` → look up trade by open_order_id then
      close_order_id; return ``'open'`` or ``'close'`` per match.
    * ``order_id_match is None`` (synthetic SL/TP/trail) → read position_state;
      compare event.side vs ps.side (opposite required) and event.qty vs
      ps.remaining_qty:

        * ``event.qty < ps.remaining_qty`` → ``'partial_tp'`` (sl_type→'trail')
        * ``event.qty == ps.remaining_qty`` AND ``ps.sl_type == 'trail'`` → ``'trail'``
        * ``event.qty == ps.remaining_qty`` AND ``ps.sl_type IN ('protective','be')`` → ``'sl'``
        * else → ``'unknown'`` + WARN log ``execution.dispatcher_exec_type_unknown``
    """
    if order_id_match is not None:
        trade_open = await select_trade_by_open_order_id(conn, order_id_match)
        if trade_open is not None:
            return ("open", trade_open.id, None)
        trade_close = await select_trade_by_close_order_id(conn, order_id_match)
        if trade_close is not None:
            return ("close", trade_close.id, None)
        bound_logger.warning(
            "execution.dispatcher_orphan_order_match",
            bot_id=bot_id,
            exchange_order_id=event.exchange_order_id,
            exchange_exec_id=event.exchange_exec_id,
        )
        return ("unknown", None, None)

    ps = await select_position_state(conn, bot_id=bot_id, symbol=event.symbol)
    if ps is None:
        bound_logger.warning(
            "execution.dispatcher_exec_type_unknown",
            bot_id=bot_id,
            reason="no_position_state",
            exchange_exec_id=event.exchange_exec_id,
        )
        return ("unknown", None, None)

    is_opposite = (event.side != ps.side)
    if not is_opposite:
        bound_logger.warning(
            "execution.dispatcher_exec_type_unknown",
            bot_id=bot_id,
            reason="same_side_synthetic_fill",
            exchange_exec_id=event.exchange_exec_id,
        )
        return ("unknown", ps.trade_id, ps.sl_type)

    if event.qty < ps.remaining_qty:
        return ("partial_tp", ps.trade_id, "trail")
    if event.qty == ps.remaining_qty:
        if ps.sl_type == "trail":
            return ("trail", ps.trade_id, ps.sl_type)
        if ps.sl_type in ("protective", "be"):
            return ("sl", ps.trade_id, ps.sl_type)
        bound_logger.warning(
            "execution.dispatcher_exec_type_unknown",
            bot_id=bot_id,
            reason=f"sl_type_unrecognized:{ps.sl_type!r}",
            exchange_exec_id=event.exchange_exec_id,
        )
        return ("unknown", ps.trade_id, ps.sl_type)

    # event.qty > ps.remaining_qty — exchange over-fill (shouldn't happen)
    bound_logger.warning(
        "execution.dispatcher_exec_type_unknown",
        bot_id=bot_id,
        reason="qty_exceeds_remaining",
        exchange_exec_id=event.exchange_exec_id,
        event_qty=str(event.qty),
        remaining_qty=str(ps.remaining_qty),
    )
    return ("unknown", ps.trade_id, ps.sl_type)
```

## Hand verification (Gate 4 math-validator critical)

## Edge cases

- **Same-side synthetic fill** (e.g., a manual Bybit user trade on the same sub-account in the same direction; shouldn't happen in our flow but possible): `_derive_exec_type` emits WARN with `reason="same_side_synthetic_fill"` and returns `('unknown', ps.trade_id, ps.sl_type)`. `_process` body proceeds normally (trade_id is set, qty <= remaining checked separately) and INSERTs the row with `exec_type='unknown'`. Operator alerted via WARN; T-221 reconciliation can investigate. **Sub-edge**: if `event.qty == ps.remaining_qty` in the same-side path, the unconditional `update_position_state_after_fill` qty subtraction will zero remaining_qty and trigger `reconcile_close` for what was NOT actually a close. Pre-T-219 this is masked benignly (NotImplementedError → tx rollback → WARN already emitted). Post-T-219 the close flow would mis-fire — T-221 reconciliation owns the recovery path. Foresight pin for T-219 plan-reviewer to ensure `reconcile_close` body is robust against `exec_type='unknown'` close triggers (e.g., reject + WARN rather than apportion P&L for an unknown-typed close).
- **No position_state for synthetic fill** (e.g., reconciliation orphan): `_derive_exec_type` emits WARN with `reason="no_position_state"` and returns `('unknown', None, None)`. `_process` body falls through to the unattributable `else: raise RuntimeError` branch — tx rollback, NO row persisted. (See "Unattributable fill" bullet for full description; this bullet documents the upstream WARN trail.)
- **`event.qty > ps.remaining_qty`** (exchange over-fill, shouldn't happen): `_derive_exec_type` emits WARN with `reason="qty_exceeds_remaining"` (diagnostic context with `event_qty` and `remaining_qty` fields). `_process` body promotes to ERROR `execution.dispatcher_overfill_halt` + raises RuntimeError → tx rollback. NO INSERT (B2 fix preserves §9.5:1613 invariant). Test pin `test_process_overfill_halts_with_runtime_error_preserves_remaining_qty`.
- **NotImplementedError from `reconcile_close`** (T-219 stub): tx rolls back cleanly; ring entry remains until restart; on restart ring is fresh and event re-processed cleanly. Operator workflow: ship T-219 immediately after T-218b OR ensure no live execution flows hit close events between T-218b and T-219.
- **Synthetic fill `order_id_for_insert`**: `executions.order_id` is NOT NULL FK to orders.id. Synthetic SL/TP/trail fills have no orders row — resolved via `select_open_order_id_by_trade_id(trade_id)` (the entry order context). Defensible: the FK semantically anchors every fill to a placed-order context, even if the fill itself wasn't issued against that order.
- **Unattributable fill** (no order match AND no position_state): cannot satisfy FK NOT NULL constraint. ERROR log `execution.dispatcher_unattributable_fill` + raise `RuntimeError` → tx rollback → operator alert via lifespan exception gather. Defensive boundary: should never happen in production; if it does, T-221 audit catches via dropped exec_id in dedup ring + zero `executions` rows for that exec_id.
- **Orphan order match** (orders row matches but no trade references it as open or close — possible via data drift, manual SQL, or T-220 audit-fix gone wrong): ERROR log `execution.dispatcher_orphan_order_halt` + raise `RuntimeError` → tx rollback. Symmetric with orphan_synthetic_fill / unattributable defensive halts. Establishes the structural invariant that every code path reaching the close-trigger branch has `trade_id != None`.
- **Orphan synthetic fill** (synthetic fill with derived trade_id but trade row missing — race with T-219 close DELETE): ERROR log `execution.dispatcher_orphan_synthetic_fill` + raise `RuntimeError` → tx rollback. Defensive parallel to orphan_order_match.

## Write-time guidance (for plan-reviewer carry-forward)

1. **Single `conn.transaction()` boundary**: all of {derivation lookups, FK resolution, INSERT execution, UPDATE position_state, UPDATE fees, optional reconcile_close} run under ONE `async with self._pool.acquire() as conn, conn.transaction():`. Any raise inside → full rollback.
2. **`reconcile_close` invocation is purely state-based (B1)**: trigger is `ps_after.remaining_qty == Decimal("0")` — re-read inside tx after UPDATE. NOT gated on `sl_type_after` or `exec_type` enum. Test pin `test_process_close_trigger_pure_state_based_independent_of_exec_type` verifies.
3. **Over-fill defensive halt (B2)**: `event.qty > ps.remaining_qty` (only reachable in 'unknown' path) → ERROR log `execution.dispatcher_overfill_halt` + RuntimeError BEFORE INSERT. Preserves §9.5:1613 invariant. Test pin `test_process_overfill_halts_with_runtime_error_preserves_remaining_qty`.
4. **Unattributable fill (B4)**: no `order_id_match` AND no `trade_id` → ERROR log `execution.dispatcher_unattributable_fill` + RuntimeError. NO `INSERT 'unknown' with NULL order_id`. Edge cases section consistent with this.
5. **Adapter NOT injected (B3-(a))**: dispatcher ctor signature unchanged from T-218a. `reconcile_close` signature has no `adapter` parameter. T-219 will refactor when implementing `closed_pnl_total` snapshot. Lifespan main.py wiring unchanged.
6. **H-024 v2 semantic per ADR-0005 (B5)**: hazard binding row in plan's hazards table cites ADR-0005. Test name v2-aligned (`test_post_tp_close_fill_labeled_per_db_sl_type_not_exchange_orderlink`); module-level docstring cross-references brief's `test_sl_fill_after_partial_tp_labeled_sl_not_tp` for navigation.
7. **`bot_id: BotId` typing (B6)**: `_derive_exec_type` accepts `BotId`; passes through to `select_position_state(conn, bot_id=bot_id, ...)` etc. — `BotId = NewType("BotId", str)` is implicit-assignable to `bot_id: str` via mypy NewType semantics. NO `str(bot_id)` cast.
8. **`_derive_exec_type` is a module-private function (NOT method)**: testable in isolation; receives conn + bot_id + event + order_id_match + bound_logger. No class state needed.
9. **`'unknown'` exec_type admissibility (re-affirmed)**: migration 0005:202 has no CHECK constraint. Each `'unknown'` write logs WARN `execution.dispatcher_exec_type_unknown` with explicit `reason=` field (no_position_state, same_side_synthetic_fill, sl_type_unrecognized, qty_exceeds_remaining). T-220 audit loop reads these for operator visibility — `'unknown'` rows are operator-actionable, not silent dead writes.
10. **`Decimal("0")` runtime import shift (C5)**: T-218a-shipped dispatcher.py has `from decimal import Decimal` inside `if TYPE_CHECKING:` block (line 35). T-218b promotes to runtime import (top-level) for `Decimal("0")` literal comparison in `_process`. Test pin verifies the comparison branches correctly when remaining_qty is `Decimal("0.0")` vs `Decimal("0.00000")` (asyncpg may return either depending on column scale).
11. **`new_sl_type` Literal narrowing**: per T-218a write-side asymmetry, `update_position_state_after_fill(new_sl_type=...)` receives `Literal["protective", "be", "trail"] | None`. T-218b call-site uses `"trail" if exec_type == "partial_tp" else None` — mypy must narrow correctly (verify with `mypy --strict` pass).
12. **§N3 marker on `_process`**: `_process` itself is non-idempotent at the body level (INSERT executions is non-idempotent). The `@non_idempotent` decorator lives on `insert_execution` (T-218a-shipped). DO NOT add a duplicate decorator on `_process` — the dedup ring is the idempotency boundary upstream of any DB write.
13. **Logger emission within tx**: WARN logs in `_derive_exec_type` happen BEFORE `insert_execution` — so even if INSERT fails or tx rolls back, the WARN is emitted (logs are I/O-flushed independently of DB tx). Intentional: operator visibility into 'unknown' fills should not depend on DB tx outcome. **Over-fill dual-log**: the over-fill path emits two records — WARN `execution.dispatcher_exec_type_unknown` (reason=qty_exceeds_remaining) from `_derive_exec_type`, then ERROR `execution.dispatcher_overfill_halt` from `_process` before the RuntimeError raise. Intentional: WARN gives the diagnostic context (qty values), ERROR gives the operator-actionable alert level.
14. **Mid-write LOC checkpoint (C1)**: if `services/execution/app/dispatcher.py` exceeds 320 src LOC mid-write, halt + request operator path-A waiver. Hard ceiling: 360 LOC.
15. **Close-trigger `trade_id` invariant via raise-pattern, NOT assert**: `reconcile_close` accepts `trade_id: int` (non-Optional). The structural guarantee is the FK resolution block: every dead-end path (`unattributable`, `orphan_order_match`, `orphan_synthetic_fill`) raises RuntimeError BEFORE the INSERT/UPDATE/close-trigger sequence. After the FK resolution block, `trade_id is None` is unreachable. mypy narrowing reduces `int | None` to `int` because the raise paths exit via `raise RuntimeError(...)`. NO `assert` (would be stripped under `python -O`); NO sentinel `0`. Three T-218b-introduced test pins cover the three halt paths: `test_process_unattributable_fill_raises_runtime_error_and_rolls_back`, `test_process_orphan_order_match_raises_runtime_error_and_rolls_back`, `test_process_orphan_synthetic_fill_raises_runtime_error_and_rolls_back`.
