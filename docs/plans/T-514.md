## Goal

Add `ShadowVariant` + `ShadowConfig` pydantic models to `packages/scoring/types.py` per BRIEF §13.2 verbatim YAML schema. Add optional `shadow: ShadowConfig | None = None` field to existing `BotConfig`. Extend `packages/scoring/yaml_loader.py:load_bot_config_from_string` to parse the `shadow:` block (per existing `_parse_exchange / _parse_signals / _parse_execution` keyword-extraction precedent). Backward-compat: existing `alpha.yaml` / `beta.yaml` fixtures have NO `shadow:` block → ShadowConfig stays `None` + downstream consumers (T-511 worker) check `if bot.shadow is not None and bot.shadow.enabled`.

Validation rules per OQ-3=A: variant names unique, override keys subset of valid ExecutionSection paths, `max_duration_hours` bounded (0 < x ≤ 24), `variants` non-empty if `enabled=True`.

## Pydantic schema

```python
# packages/scoring/types.py — append after ExecutionSection

_EXECUTION_FIELD_NAMES: frozenset[str] = frozenset({
    "qty", "leverage", "sl_pct", "tp_pct", "tp_qty_pct",
    "be_trigger", "be_sl_level", "trail_pct", "fee_rate",
})


class ShadowVariant(BaseModel):
    """Per-variant override per BRIEF §13.2 YAML schema."""

    model_config = ConfigDict(frozen=True, extra="forbid")

    name: str = Field(..., min_length=1)
    overrides: dict[str, Decimal] = Field(default_factory=dict)

    @field_validator("overrides")
    @classmethod
    def _overrides_target_execution_fields(cls, value: dict[str, Decimal]) -> dict[str, Decimal]:
        unknown = set(value) - _EXECUTION_FIELD_NAMES
        if unknown:
            msg = f"shadow variant overrides target unknown ExecutionSection fields: {sorted(unknown)}"
            raise ValueError(msg)
        return value


class ShadowConfig(BaseModel):
    """Per-bot shadow simulation config per BRIEF §13.2."""

    model_config = ConfigDict(frozen=True, extra="forbid")

    enabled: bool = False
    variants: list[ShadowVariant] = Field(default_factory=list)
    max_duration_hours: float = Field(default=4.0, gt=0.0, le=24.0)

    @field_validator("variants")
    @classmethod
    def _variant_names_unique(cls, value: list[ShadowVariant]) -> list[ShadowVariant]:
        names = [v.name for v in value]
        if len(names) != len(set(names)):
            msg = f"shadow variant names must be unique: {names}"
            raise ValueError(msg)
        return value

    @model_validator(mode="after")
    def _enabled_requires_variants(self) -> Self:
        if self.enabled and not self.variants:
            msg = "shadow.enabled=True requires at least 1 variant"
            raise ValueError(msg)
        return self


# BotConfig — append field
class BotConfig(BaseModel):
    # ... existing fields ...
    shadow: ShadowConfig | None = None
```

## Yaml_loader extension

```python
# packages/scoring/yaml_loader.py — append after _parse_execution

def _parse_shadow(spec: Mapping[str, Any] | None) -> ShadowConfig | None:
    """Parse `shadow:` YAML block per BRIEF §13.2; return None if absent.

    Decimal coercion on `overrides` values mirrors ``_parse_execution`` pattern.
    Pydantic validators on ShadowConfig + ShadowVariant enforce structural rules
    (unique names + valid override keys + max_duration_hours bounds + enabled
    requires variants).
    """
    if not spec:
        return None
    raw_variants = spec.get("variants", []) or []
    coerced_variants: list[ShadowVariant] = []
    for v in raw_variants:
        if not isinstance(v, dict):
            continue
        name = v.get("name", "")
        overrides_raw = v.get("overrides", {}) or {}
        overrides: dict[str, Decimal] = {
            k: _to_decimal(val) for k, val in overrides_raw.items()
        }
        coerced_variants.append(ShadowVariant(name=name, overrides=overrides))
    return ShadowConfig(
        enabled=bool(spec.get("enabled", False)),
        variants=coerced_variants,
        max_duration_hours=float(spec.get("max_duration_hours", 4.0)),
    )


# load_bot_config_from_string — add to kwargs
return BotConfig(
    # ... existing kwargs ...
    shadow=_parse_shadow(data.get("shadow")),
)
```

## Hand verification

### §A — BRIEF §13.2 verbatim 5-variant example loads cleanly

Input YAML (verbatim from spec):
```yaml
shadow:
  enabled: true
  variants:
    - name: baseline
    - name: no_be
      overrides: { be_trigger: 0 }
    - name: full_tp
      overrides: { tp_qty_pct: 1.0, trail_pct: 0 }
    - name: sl_tight
      overrides: { sl_pct: 0.005 }
    - name: sl_wide
      overrides: { sl_pct: 0.015 }
  max_duration_hours: 4
```

Result: `BotConfig.shadow` is `ShadowConfig(enabled=True, variants=[5 ShadowVariants], max_duration_hours=4.0)`. Each variant decoded with Decimal-coerced overrides (e.g. `Decimal("0")`, `Decimal("0.005")`).

### §B — Backward-compat: alpha.yaml without shadow → None

Input YAML omitting `shadow:` block (current alpha.yaml). Result: `BotConfig.shadow is None`.

### §C — Validation: duplicate variant names

Input variants `[{"name": "x"}, {"name": "x"}]`. Result: `ValueError` containing "must be unique".

### §D — Validation: invalid override key

Input variant `{"name": "v", "overrides": {"unknown_field": 0}}`. Result: `ValueError` containing "unknown ExecutionSection fields".

### §E — Validation: enabled without variants

Input `{"enabled": True, "variants": []}`. Result: `ValueError` containing "requires at least 1 variant".

### §F — Validation: max_duration_hours bounds

Input `max_duration_hours=0` or `25`. Result: pydantic validator `gt=0` / `le=24` raises ValueError.

### §G — Override Decimal coercion

Input `overrides: {sl_pct: 0.005}` (YAML float). Result: `ShadowVariant.overrides["sl_pct"] == Decimal("0.005")`.
