## Verified correctness defect (why T-549a exists)

All 5 strategy-engine risk gates select the source table/position via an explicit allowlist `... if exchange_mode in ("live","testnet") else <paper_table>`: `cooldown_gate.py:223`, `concurrent_caps_gate.py:109`, `drawdown_gate.py:120`, `loss_limit_gate.py:116`, `opposite_side_gate.py:106`. With `exchange_mode="demo"`, `"demo" not in ("live","testnet")` → `else` → `paper_trades`/`paper_position_state`. A demo bot places REAL Bybit demo-trading orders → its trades/positions are real rows in `trades`/`position_state`. So cooldown-after-loss, concurrent-caps, drawdown kill-switch, daily-loss-limit kill-switch, and the H-005 opposite-side guard would all read empty paper tables for a demo bot → **silently never trigger** = safety defect. T-549a adds `"demo"` to each allowlist tuple (→ real-table realm) + §N4 pins it.

**Demo-real-table rationale:** demo is a *real Bybit demo-trading account* — fills are real rows in `trades`/`position_state`, NOT paper-simulated; demo MUST route to the real-table realm exactly like live/testnet. The real-table *semantic* is decided in this foundation; the *endpoint/URL* by which demo reaches Bybit is T-549b.
