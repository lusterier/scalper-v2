## Purpose

Ship the SSE (Server-Sent Events) multiplexed endpoint per BRIEF §9.6:1633-1638. Single connection per dashboard client → multiplexed stream of NATS-published events. Subscribed event types selected via query param (`?types=positions,signals,trades`). Each event encoded as `data: {"type": "...", "payload": {...}}`. Consumed by T-413 Per-bot live view (live signals feed last 50, position updates).

**Explicitly out of scope for T-408**: Last-Event-ID replay (F5+ — needs JetStream replay infrastructure); per-IP auth/rate-limit (F5+ when cloudflared lands); SSE compression; backpressure handling beyond per-client async queue with drop-oldest policy; partial subscription mid-stream (one connection = static type set).

### Event types (5 whitelisted per OQ-1=A)

| Type | NATS subjects subscribed | Payload semantic |
| --- | --- | --- |
| `positions` | `orders.events.>` (filter to `event_type IN ('order_closed', 'sl_moved')`) | Live position updates: close events + SL move indicators (T-413 position card needs both) |
| `signals` | `signals.validated` | Validated inbound signals (signal-gateway → strategy-engine) |
| `trades` | `orders.events.>` (filter to `event_type IN ('order_placed', 'order_filled')`) | Trade lifecycle fills |
| `scoring` | `signals.rejected.>` | Rejected signals across all bots (shadow tracking) |
| `alerts` | `system.alerts` | Alert-worthy events (alerting-svc consumer; T-409) |

**`sl_moved` routing per CONCERN #3**: SL move events emitted by execution-service (per `OrderEventBase` discriminated union with `event_type='sl_moved'` + `new_sl_price` field) flow through `orders.events.>` subject. They are routed under `positions` type because UI live position card displays current SL price + a visual indicator when SL moves (T-413 spec). NOT routed under `trades` (`trades` is fill lifecycle: placed/filled). TRADES/POSITIONS share subject `orders.events.>` with disjoint event_type filters — no double-emission to single client subscribed to both types.

**Wildcard `>` subjects**: NATS supports `>` as multi-token wildcard. `orders.events.>` matches `orders.events.alpha`, `orders.events.beta`, etc. — analytics-api receives all bots' events.

**Type → multiple subjects mapping**: `positions` and `trades` BOTH subscribe to `orders.events.>` but server-side filter on `event_type` discriminator. Per OQ-2=A per-connection subscriptions: if one client requests both types, ONE subscription to `orders.events.>` is created with handler dispatching to both type queues. Connection close → subscription drains.

**Unknown type in `?types=`**: 422 with detail `f"unknown event type {t!r}; allowed: positions, signals, trades, scoring, alerts"`.
**Empty `?types=`**: 422 (must subscribe to at least one type).

### `SSEMultiplexer` class — `services/analytics_api/app/sse.py` (NEW)

Owns per-connection lifecycle:

```python
# Sentinel posted to client queues by unregister_client / shutdown to break the
# StreamingResponse generator loop cleanly. Distinct from any real event payload.
_QUEUE_SENTINEL: Final = None


class SSEMultiplexer:
    """Per-connection NATS subscription + per-client async queue.

    Lifespan-attached singleton (app.state.sse_multiplexer). Each call to
    `register_client(types) -> ClientHandle` creates:
      * Per-client asyncio.Queue(maxsize=settings.sse_client_queue_maxsize) —
        buffers events before SSE flush (OQ-3=A drop-oldest policy).
      * Per-client NATS subscriptions (one per unique subject; some types
        share subjects so dedup logic). Subscription handlers push into
        client queue.
      * Active client counter; reject new clients beyond
        settings.sse_max_connections (OQ-7=A).

    `unregister_client(handle)` drains subscriptions + posts queue sentinel.
    Idempotent — called from router's StreamingResponse `finally` block AND
    from `shutdown()`; double-call is safe (no-op on second invocation).

    `shutdown()` (called from lifespan reverse-shutdown BEFORE bus.close())
    iterates all active handles + calls `unregister_client` on each. Itself
    idempotent. Sentinel posting wakes up any generator stuck in
    `await queue.get()`; generator's finally block then runs cleanup which
    re-calls unregister_client (idempotent, no-op).
    """

    def __init__(
        self,
        *,
        bus: NatsClient,
        logger: Any,
        heartbeat_interval_s: int,
        client_queue_maxsize: int,
        max_connections: int,
        overflow_log_interval_s: int,
    ) -> None:
        """Constructor takes all 4 tuning knobs as DI (per BLOCKER #1 fix).

        Lifespan in main.py injects from settings. NO module-level constants
        for these values per §N9 + L-001 active control.
        """

    async def register_client(self, types: frozenset[EventType]) -> "ClientHandle":
        """Allocate queue + create per-type subscriptions.

        Raises SSEConnectionLimitError if active_client_count >= max_connections
        (check-then-increment in single asyncio loop = no race per CONCERN #8).
        Raises BusError if any per-subject subscribe fails (rolls back partial
        subscriptions before re-raising).
        """

    async def unregister_client(self, handle: "ClientHandle") -> None:
        """Unsubscribe + post sentinel to wake generator + decrement counter.

        IDEMPOTENT — second call is no-op (handle.is_active flag toggled).
        Required idempotency: router finally + shutdown both call this.
        """

    async def shutdown(self) -> None:
        """Drain all active client subscriptions before bus.close().

        Iterates snapshot of active handles + calls unregister_client on each.
        IDEMPOTENT — second call is no-op (active_handles frozen empty).
        Posts sentinel to each client queue → generator loop reads sentinel,
        breaks out, finally block runs unregister_client (no-op idempotent).
        """

    @property
    def active_client_count(self) -> int: ...


@dataclass
class ClientHandle:
    """Returned to router; iterates queue + manages subscriptions."""

    client_id: str  # UUID4 for log tagging
    types: frozenset[EventType]
    queue: asyncio.Queue[dict[str, Any] | None]  # None = sentinel, dict = SSE event
    subscriptions: list[Subscription]  # NATS subscriptions to drain on unregister
    overflow_count: int = 0  # for log rate-limit
    last_overflow_log_monotonic: float = 0.0  # last time overflow log emitted
    is_active: bool = True  # toggled to False by unregister_client (idempotency guard)


class SSEConnectionLimitError(Exception):
    """Raised by register_client when max_connections reached."""
```

### Router — `services/analytics_api/app/routers/events.py` (NEW)

```python
@router.get("/events/stream")
async def stream_events(
    request: Request,
    multiplexer: Annotated[SSEMultiplexer, Depends(get_sse_multiplexer)],
    types: Annotated[
        str,
        Query(description="Comma-separated event types: positions,signals,trades,scoring,alerts"),
    ],
) -> StreamingResponse:
    parsed_types = _parse_types(types)  # 422 on empty / unknown
    try:
        handle = await multiplexer.register_client(parsed_types)
    except SSEConnectionLimitError as exc:
        raise HTTPException(
            status_code=status.HTTP_503_SERVICE_UNAVAILABLE,
            detail=f"max SSE connections reached ({multiplexer.max_connections}); retry later",
        ) from exc

    heartbeat_interval = multiplexer.heartbeat_interval_s

    async def event_stream() -> AsyncIterator[bytes]:
        try:
            yield f": connected types={','.join(sorted(t.value for t in parsed_types))}\n\n".encode()
            heartbeat_at = time.monotonic() + heartbeat_interval
            while True:
                # is_disconnected() is best-effort per CONCERN #6 — finally block
                # is the load-bearing cleanup mechanism (broken pipe on yield to
                # closed connection raises, finally runs unregister_client).
                if await request.is_disconnected():
                    break
                timeout = max(0.0, heartbeat_at - time.monotonic())
                try:
                    event = await asyncio.wait_for(handle.queue.get(), timeout=timeout)
                except asyncio.TimeoutError:
                    yield b": heartbeat\n\n"  # SSE comment line
                    heartbeat_at = time.monotonic() + heartbeat_interval
                    continue
                if event is None:  # sentinel from unregister_client / shutdown
                    break
                payload_json = json.dumps(event)
                yield f"data: {payload_json}\n\n".encode()
                heartbeat_at = time.monotonic() + heartbeat_interval
        finally:
            await multiplexer.unregister_client(handle)  # idempotent

    return StreamingResponse(
        event_stream(),
        media_type="text/event-stream",
        headers={"Cache-Control": "no-cache", "X-Accel-Buffering": "no"},
    )
```

### NATS subject derivation (sse.py)

```python
def _subjects_for_types(types: frozenset[EventType]) -> dict[str, list[EventType]]:
    """Compute unique NATS subjects + which types each subject feeds.

    Returns {subject: [types]}. Some subjects feed multiple types
    (orders.events.> serves both positions + trades) — handler filters
    by event_type discriminator before queuing per type.
    """
    out: dict[str, list[EventType]] = {}
    if EventType.POSITIONS in types or EventType.TRADES in types:
        out["orders.events.>"] = [t for t in (EventType.POSITIONS, EventType.TRADES) if t in types]
    if EventType.SIGNALS in types:
        out["signals.validated"] = [EventType.SIGNALS]
    if EventType.SCORING in types:
        out["signals.rejected.>"] = [EventType.SCORING]
    if EventType.ALERTS in types:
        out["system.alerts"] = [EventType.ALERTS]
    return out
```

### NATS handler → SSE event mapping (sse.py)

```python
def _envelope_to_sse_event(envelope: MessageEnvelope, event_type: EventType) -> dict[str, Any]:
    """Per OQ-5=A: strip envelope to {type, payload, correlation_id, published_at}.

    Drops envelope.message_id, schema_version, publisher (UI doesn't need).
    Preserves correlation_id (UI debug) + published_at (ordering).
    """
    return {
        "type": event_type.value,
        "payload": envelope.payload,
        "correlation_id": envelope.correlation_id,
        "published_at": envelope.published_at.isoformat(),
    }


# event_type whitelist sets per type; explicit constants for L-008 spirit
# (no string scattering across functions; single source of truth).
_POSITIONS_EVENT_TYPES: frozenset[str] = frozenset({"order_closed", "sl_moved"})
_TRADES_EVENT_TYPES: frozenset[str] = frozenset({"order_placed", "order_filled"})


def _filter_for_type(envelope: MessageEnvelope, event_type: EventType) -> bool:
    """Server-side discriminator filter for shared subjects.

    `positions` emits on event_type IN ('order_closed', 'sl_moved') — close
    events drive position card removal/refresh; sl_moved drives current-SL
    indicator update on the open position card (CONCERN #3 fix per T-413
    UI spec).
    `trades` emits on event_type IN ('order_placed', 'order_filled') —
    fill lifecycle for trade explorer feed.
    Disjoint sets: a single client subscribed to both POSITIONS and TRADES
    receives each event exactly once (no double-emission per shared subject).
    Other types are 1:1 subject → type, no filter.
    """
    if event_type is EventType.POSITIONS:
        return envelope.payload.get("event_type") in _POSITIONS_EVENT_TYPES
    if event_type is EventType.TRADES:
        return envelope.payload.get("event_type") in _TRADES_EVENT_TYPES
    return True
```

## Edge cases

1. **Empty `?types=`**: 422 with detail `"types query param is required and non-empty"`.
2. **Unknown type in `?types=`**: 422 with detail enumerating allowed types.
3. **Duplicate type in `?types=`** (e.g., `?types=signals,signals`): dedup silently into frozenset.
4. **51st concurrent client**: 503 with detail `f"max SSE connections reached ({max_connections}); retry later"`. Rejected at `register_client` BEFORE StreamingResponse opens.
5. **Client disconnect mid-stream — primary cleanup path**: `request.is_disconnected()` is polled every queue-iteration BUT is **best-effort** per CONCERN #6 — known FastAPI/Starlette behavior is that `is_disconnected()` may return False for some time after the client closed the TCP connection. The **load-bearing cleanup mechanism is the generator's `finally` block**: when the next `yield` writes to a closed connection, `BrokenPipeError` / `ConnectionResetError` propagates, the `finally` block runs `multiplexer.unregister_client(handle)` (which drains NATS subscriptions + posts queue sentinel + decrements active counter). NATS unsubscribe is fast and bounded.
6. **Per-client queue overflow** (OQ-3=A drop-oldest): handler attempts `queue.put_nowait(event)`. On `QueueFull` exception → drop oldest via `queue.get_nowait()` then retry put. Increment `handle.overflow_count`. Rate-limited log warning `sse_client_buffer_overflow` per `settings.sse_overflow_log_interval_s` (default 60s — 1/min/client).
7. **NATS subscription failure during `register_client`**: any per-subject subscribe raises → `register_client` rolls back partial subscriptions (best-effort `unsubscribe()` on each so-far-successful sub; swallows secondary errors in rollback) + re-raises (router catches → 500). Client sees 500, reconnects.
8. **Heartbeat during quiet period**: `asyncio.wait_for(queue.get(), timeout)` with timeout = remaining time until `heartbeat_at`; `TimeoutError` → emit `: heartbeat\n\n` SSE comment line. Reset `heartbeat_at`.
9. **Slow client + many types**: per-client queue is the only buffer. `settings.sse_client_queue_maxsize` (default 1000) gives ~1k events headroom; slow client beyond that → drop-oldest. NATS subscription stays healthy (drops are app-side, not bus-side).
10. **NATS message envelope parse failure**: handler swallows per `NatsClient.subscribe` contract (logs `bus_handler_failed`) — never reaches client queue.
11. **NATS handler exception during T-408 dispatch** (per CONCERN #5): `NatsClient._dispatch` swallows handler exceptions as `bus_handler_failed` log line. If T-408 handler raises (e.g., non-JSON-serializable type in payload), the message is silently dropped from that client's stream and the failure is visible only in service logs. Test pin `test_stream_skips_malformed_event_logged` exercises this.
12. **Bus disconnect during active stream**: NATS subscriptions go silent (no events arrive); heartbeat keeps connection alive; client sees no data flow but stays connected. F5+ may add NATS state push event.
13. **Lifespan shutdown with N active clients**: `multiplexer.shutdown()` iterates snapshot of active handles + calls `unregister_client(handle)` on each (sets `is_active=False`, posts `_QUEUE_SENTINEL=None` to queue, drains NATS subs, decrements counter). Generator awaits `queue.get()` → receives `None` sentinel → breaks loop → finally calls `unregister_client(handle)` again (no-op idempotent because `is_active` already False).
14. **Connection cap counter atomicity** (per CONCERN #8): single asyncio event loop guarantees check-then-increment is not interleaved across concurrent `register_client` calls. Implementation uses a simple `if self._active_count >= self._max_connections: raise SSEConnectionLimitError; self._active_count += 1` sequence — no `asyncio.Lock` needed because no `await` between check and increment. Comment in code makes invariant explicit.

## Write-time guidance (plan-reviewer APPROVE pass-2 — must be addressed during implementation; brief-reviewer verifies verbatim at SHIP gate)

1. **`pyproject.toml` PEP 735 block creation**: `services/analytics_api/pyproject.toml` currently has no `[dependency-groups]` table. Create `[dependency-groups]` table with `dev = ["asgi-lifespan==2.1.0"]` (exact pin). Verify `uv sync --group dev` resolves; do NOT add to runtime `[project.dependencies]`. Brief-reviewer will grep staged diff for both the new table and the `==2.1.0` exact pin.

2. **Sentinel typing pin**: `ClientHandle.queue: asyncio.Queue[dict[str, Any] | None]` literal in dataclass annotation (not `asyncio.Queue[Any]`). `_QUEUE_SENTINEL: Final[None] = None` (with `Final[None]` not bare `Final` so mypy --strict pins disjointness). Generator uses `if event is None: break` (identity check, not equality).

3. **Counter atomicity invariant comment**: in `register_client`, add a code comment immediately above the `if self._active_count >= self._max_connections: ... self._active_count += 1` block stating: `# Single asyncio loop invariant: no `await` between check and increment, so this sequence is atomic without asyncio.Lock. Decrement in unregister_client gated by handle.is_active flag (idempotency).`. Decrement path must mirror: `is_active` guard read + counter `-= 1` with no `await` interleaved between them.

4. **Idempotency guard ordering in `unregister_client`**: first statement is `if not handle.is_active: return`, second statement is `handle.is_active = False`, then NATS unsubscribes (may await), then queue sentinel post (`queue.put_nowait(_QUEUE_SENTINEL)` — never await; if QueueFull, drop oldest first then retry as in normal overflow path), then counter decrement. Test #2 must exercise second-call no-op path with a side-effect-tracking mock to assert NATS `unsubscribe` called exactly once (not twice).

5. **Shutdown snapshot-iteration**: `shutdown()` must iterate `list(self._active_handles)` (snapshot copy) NOT the live set, because `unregister_client` mutates `_active_handles` mid-iteration. Test #9 should register 3 handles, call `shutdown()`, assert all 3 sentinels delivered + `active_client_count == 0` + second `shutdown()` no-op (no exception, no spurious unregister calls).

6. **Lifespan reverse-shutdown ordering pin**: `await app.state.sse_multiplexer.shutdown()` MUST be called BEFORE `await bus.close()` in the lifespan teardown (after `yield`). `bus.close()` drains tracked subscriptions; calling shutdown after would race with bus drain. Test #11 (`test_main_shutdown_drains_multiplexer_before_bus_close`) pins call order via patched mocks recording invocation order.

7. **Test #6 docstring**: `test_stream_emits_event_on_nats_message` injects via `handle.queue.put_nowait(envelope_to_sse_event(env, EventType.SIGNALS))` — test docstring must explicitly state "queue-level injection; envelope→SSE mapping coverage lives in test #8 (`strips_envelope_internal_fields`)" so it's clear test #6 doesn't accidentally bypass mapping coverage.

8. **422 detail message exact strings**: empty `?types=` returns FastAPI 422 with body containing `"types query param is required and non-empty"`; unknown type returns 422 with `f"unknown event type {t!r}; allowed: positions, signals, trades, scoring, alerts"`. Tests #1/#2/#3 must assert these substrings (not just status code) so a future refactor can't silently change UX-facing error text.

9. **`asgi-lifespan` import within fixture body**: `from asgi_lifespan import LifespanManager` inside the fixture function (not at module top level of conftest.py) so non-streaming tests that don't use `async_client` don't pay the import cost and so a missing dev-dep failure surfaces only when the fixture is actually used (clearer error path).

10. **`text/event-stream` + `X-Accel-Buffering: no` header pin**: tests #5/#6/#7 should assert `response.headers["content-type"].startswith("text/event-stream")` AND `response.headers["x-accel-buffering"] == "no"` AND `response.headers["cache-control"] == "no-cache"` so any future header strip / proxy buffering regression is caught.
