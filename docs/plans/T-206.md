- T-211 — "PaperExchange does what its skeleton says".
- T-206 — "every adapter conforms to the ExchangeClient contract".

When T-207 BybitV5Adapter ships, T-206 picks it up automatically (one-line append to `_ADAPTERS_UNDER_TEST`); T-211 stays focused on PaperExchange. No duplication of intent.

## Public interface

### `packages/exchange/tests/test_protocol_conformance.py`

```python
"""§11.6 adapter protocol conformance contract test.

Parametrised over every ExchangeClient implementation registered in
``_ADAPTERS_UNDER_TEST``. Each adapter is asserted against two
invariants per §11.6 verbatim:

* every Protocol method is present on the adapter class.
* every Protocol method NOT in
  :data:`packages.exchange.protocols._UNLABELED_METHODS` carries
  ``@idempotent`` or ``@non_idempotent`` marker per §N3.

H-013 binding (per OQ-2 default A): :meth:`set_trading_stop` on every
adapter has ``tpsl_mode`` parameter without default value (compile-
error if omitted; mirrors T-201 protocol surface invariant).

T-207 (BybitV5Adapter) appends its class to ``_ADAPTERS_UNDER_TEST``
when it ships — one-line edit. Future testnet/sandbox/mock adapters
similarly. The framework loops existing assertions over the new
adapter; no per-adapter test duplication.
"""

from __future__ import annotations

import inspect

import pytest

from packages.core import is_idempotent, is_non_idempotent
from packages.exchange import ExchangeClient, PaperExchange
from packages.exchange.protocols import _UNLABELED_METHODS


# Adapter registry. T-207 BybitV5Adapter appends here.
_ADAPTERS_UNDER_TEST: list[type] = [PaperExchange]


def _protocol_method_names() -> set[str]:
    """Public callable names declared on the ExchangeClient Protocol."""
    return {
        name
        for name, member in inspect.getmembers(ExchangeClient)
        if callable(member) and not name.startswith("_")
    }


@pytest.mark.parametrize("adapter_cls", _ADAPTERS_UNDER_TEST)
def test_adapter_implements_every_protocol_method(adapter_cls: type) -> None:
    """§11.6: every adapter implements every ExchangeClient method."""
    expected = _protocol_method_names()
    actual = {
        name
        for name, member in inspect.getmembers(adapter_cls)
        if callable(member) and not name.startswith("_")
    }
    missing = expected - actual
    assert not missing, (
        f"{adapter_cls.__name__} missing ExchangeClient methods: {missing}"
    )


@pytest.mark.parametrize("adapter_cls", _ADAPTERS_UNDER_TEST)
def test_adapter_methods_carry_correct_idempotency_markers(
    adapter_cls: type,
) -> None:
    """§N3 + T-201 W#1: every Protocol method NOT in _UNLABELED_METHODS
    carries @idempotent or @non_idempotent on every adapter.

    Streams + close are exempt per the operator-locked rationale block in
    ``packages/exchange/protocols.py`` (T-201 W#1).
    """
    unlabeled: list[str] = []
    for name in _protocol_method_names():
        if name in _UNLABELED_METHODS:
            continue
        method = getattr(adapter_cls, name)
        if not (is_idempotent(method) or is_non_idempotent(method)):
            unlabeled.append(name)
    assert not unlabeled, (
        f"{adapter_cls.__name__} has unlabeled non-exempt methods: {unlabeled}"
    )


@pytest.mark.parametrize("adapter_cls", _ADAPTERS_UNDER_TEST)
def test_adapter_unlabeled_methods_carry_no_markers(adapter_cls: type) -> None:
    """T-201 W#1 negative: streams + close MUST NOT carry markers.

    A future implementer accidentally decorating ``stream_executions``
    with ``@idempotent`` would silently change the §5.8 marker
    invariant; this test catches that regression.
    """
    for name in _UNLABELED_METHODS:
        method = getattr(adapter_cls, name)
        assert not is_idempotent(method), (
            f"{adapter_cls.__name__}.{name} is in _UNLABELED_METHODS but is decorated @idempotent"
        )
        assert not is_non_idempotent(method), (
            f"{adapter_cls.__name__}.{name} is in _UNLABELED_METHODS but is decorated @non_idempotent"
        )


@pytest.mark.parametrize("adapter_cls", _ADAPTERS_UNDER_TEST)
def test_adapter_set_trading_stop_tpsl_mode_has_no_default(
    adapter_cls: type,
) -> None:
    """H-013 binding per OQ-2 default A: tpsl_mode no-default invariant
    is enforced on every adapter, not just on the Protocol type level.

    The ExchangeClient Protocol declares ``tpsl_mode`` with no default;
    structural typing at mypy time would catch a default-bearing adapter
    method, but mypy strict is a lint not a runtime guard. This test
    pins the H-013 invariant at the contract-test layer so the no-default
    contract holds even if a future adapter is added without mypy in CI.
    """
    sig = inspect.signature(adapter_cls.set_trading_stop)
    assert sig.parameters["tpsl_mode"].default is inspect.Parameter.empty, (
        f"{adapter_cls.__name__}.set_trading_stop tpsl_mode must have no default "
        f"(H-013); got default={sig.parameters['tpsl_mode'].default!r}"
    )


def test_unlabeled_methods_set_is_complete() -> None:
    """Sanity: _UNLABELED_METHODS is a non-empty subset of Protocol methods.

    Catches typos in the frozenset (e.g., ``"stream_execution"`` singular
    instead of plural) — the misspelled entry would be a no-op exemption
    that fails to mask a future stream-method marker omission.
    """
    protocol_methods = _protocol_method_names()
    bad = _UNLABELED_METHODS - protocol_methods
    assert not bad, (
        f"_UNLABELED_METHODS contains names not in ExchangeClient Protocol: {bad}"
    )
    assert _UNLABELED_METHODS, "_UNLABELED_METHODS must be non-empty"
```

## Edge cases

- **Empty registry** (impossible at T-211 ship; defensive): if `_ADAPTERS_UNDER_TEST = []`, parametrize collapses to zero invocations. The sanity test `test_unlabeled_methods_set_is_complete` still runs and would fail if `_UNLABELED_METHODS` is empty — that's intentional fail-loud.
- **Future adapter missing __init__.py registration**: a developer creating a new adapter class but forgetting to append to `_ADAPTERS_UNDER_TEST` produces no T-206 test failure (silent omission). Mitigation: the adapter's own per-class test file (mirror T-211 pattern) catches surface bugs; T-206 is the cross-adapter consistency layer. **L-NNN candidate** if the omission ever bites: propose a `subclasses(ExchangeClient)` walker, but that requires `runtime_checkable` Protocol which T-201 explicitly rejected. Document as known limitation in module docstring.
- **`_UNLABELED_METHODS` typo** (e.g., `"stream_execution"` singular): the sanity test `test_unlabeled_methods_set_is_complete` catches this — the misspelled entry would be in `_UNLABELED_METHODS` but NOT in `_protocol_method_names()`, triggering the assert.
- **Adapter inheritance**: if a future adapter inherits from another (e.g., `BybitTestnetAdapter(BybitV5Adapter)`), `inspect.getmembers` walks MRO so inherited methods + markers resolve correctly. No special handling needed.
- **Protocol method count**: ExchangeClient has 10 methods (T-201 verbatim). Tests are parametrized over adapters, NOT over methods, so test count is `len(_ADAPTERS_UNDER_TEST) * 4 + 1` (4 parametrized tests + 1 sanity). For T-206 ship today: 4 * 1 + 1 = 5 test invocations.

## OQ resolutions (operator-approved 2026-04-27 — "use defaults on all 4")

1. **OQ-1 — Adapter registry mechanism**: **default A — hardcoded module-level list** `_ADAPTERS_UNDER_TEST: list[type] = [PaperExchange]` at top of test file. T-207 appends `BybitV5Adapter` via 1-line edit at ship time. Underscore-prefixed = test-internal, not re-exported. Pytest fixture / conftest entry-point / `__init_subclass__` auto-register all rejected: indirection adds nothing for a static list; auto-register couples adapter authoring to test framework (violates §N6 no-globals-from-class-definition).

2. **OQ-2 — H-013 `tpsl_mode` no-default check**: **default A — include**. `test_adapter_set_trading_stop_tpsl_mode_has_no_default[adapter_cls]` parametrised over `_ADAPTERS_UNDER_TEST`. mypy strict + Protocol structural typing catches default drift at lint time, but T-206 contract test must hold even if mypy is bypassed (e.g., a future adapter `# type: ignore` slipping past review). Hazard-listed (§20 H-013) + adapter-pool-wide → pin at contract layer means every future adapter inherits the check automatically. ~10 LOC marginal duplication with T-201/T-211 acceptable for capital-loss-preventing invariant.

3. **OQ-3 — H-024 `ExecutionEvent` shape check**: **default B — skip**. T-201 `test_execution_event_does_not_carry_exec_type` already pins this at the type level via `dataclasses.fields` introspection. T-206 is the adapter-conformance layer, not type-shape; the `ExecutionEvent` dataclass is shared by all adapters — there is no per-adapter variation to check. Re-asserting in T-206 would be scope drift + calibration discipline says: don't re-test invariants that don't have per-adapter variation.

4. **OQ-4 — Full signature parity check beyond markers**: **default B — skip**. mypy strict + Protocol structural typing IS the signature parity check — at every mypy invocation in CI, every adapter is structurally typechecked against `ExchangeClient`. Re-implementing at the contract-test layer would be duplicative AND fragile (forward-ref string equivalence, `from __future__ import annotations` deferred-evaluation quirks). The H-013 default-value check (OQ-2 default A) is the single exception worth pinning at the contract layer because the hazard cost is capital loss; broader signature parity is mypy's job.

## Write-time guidance (verbatim from plan-reviewer APPROVE 2026-04-27)

1. **`is_idempotent` / `is_non_idempotent` registry-key correctness.** The markers module keys decorations by `f"{func.__module__}.{func.__qualname__}"` (`packages/core/markers.py:39`). When the test calls `getattr(adapter_cls, name)` and passes that to `is_idempotent`, the resolved key is the **adapter's** qualname (`packages.exchange.paper.adapter.PaperExchange.set_trading_stop`), not the Protocol's. The plan code is correct as-written; do not "simplify" by walking `ExchangeClient.set_trading_stop` directly — that would test the Protocol's own decoration, not the adapter's. Keep the `getattr(adapter_cls, name)` pattern verbatim.

2. **Filter `Protocol`-inherited members in `_protocol_method_names()`.** `inspect.getmembers(ExchangeClient)` walks the MRO including `typing.Protocol` parent which contributes callables like `_proto_hook`, `_no_init_or_replace_init`, etc. The plan's `not name.startswith("_")` filter handles this (those names start with `_`), but watch out: `Protocol` may also expose public-named callables in some Python versions. Verify the resolved set has exactly 10 entries (the §11.1 surface) by adding an assertion `assert len(_protocol_method_names()) == 10` inside `test_unlabeled_methods_set_is_complete` — this catches future Python upstream additions silently expanding the surface, which would otherwise pass the contract test vacuously. Existing T-211 `test_paper_exchange_implements_full_exchange_client_surface` doesn't pin this count, so T-206 is the right place to lock it.

3. **`test_adapter_set_trading_stop_tpsl_mode_has_no_default` parameter-positional check.** The plan asserts `sig.parameters["tpsl_mode"].default is inspect.Parameter.empty`. Add a second assertion that `tpsl_mode` is the **third** parameter (after `self`, `symbol`) — H-013 invariant per T-201 W#2 plan-doc decision is not just no-default but also positional ordering (so optional fields keep their `None` defaults without breaking the no-default contract). Use `list(sig.parameters)[2] == "tpsl_mode"` or equivalent. This adds ~3 LOC and pins the full T-201 W#2 contract at the adapter-pool layer.

4. **Sanity-test entry-point pattern: hardcoded count for `_UNLABELED_METHODS`.** Plan asserts `_UNLABELED_METHODS` is non-empty and a subset of Protocol methods. Add an explicit `assert len(_UNLABELED_METHODS) == 3` (the 3 known exemptions: `stream_executions`, `stream_positions`, `close`). Reasoning: a future maintainer could add a fourth entry without thinking. Pinning the count at 3 forces a deliberate update path: if T-21x or future task adds a fourth exemption, both `_UNLABELED_METHODS` and this assertion change in the same diff, surfacing the decision for review. ~1 LOC.

5. **Module docstring "known limitation" callout.** Plan §Edge cases mentions "future adapter forgotten in registry" as a known limitation. Reflect this in the test file's module docstring verbatim (a sentence like "Adapter registration is manual; an adapter class not appended to `_ADAPTERS_UNDER_TEST` is silently skipped — per-adapter test files (mirror T-211) are the per-class safety net"). This makes the limitation discoverable when a future maintainer reads the file in isolation, not only via plan-doc traceability.
