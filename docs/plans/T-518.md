## Purpose

On feature-engine lifespan startup, detect newly-registered feature_names by diffing the current `configs/features/indicators.yaml` × symbols cross-product against the `feature_registry_seen` NATS KV bucket. For each NEW feature_name, schedule an async fire-and-forget backfill job over the configurable historical OHLC window (default 30d). On successful completion, write the feature_name + first-seen UTC timestamp into the KV bucket so subsequent restarts skip it (idempotency contract).

## Public interface

### NEW module `services/feature_engine/app/auto_backfill.py` (~150 LOC)

```python
"""Feature auto-backfill scheduler (T-518, BRIEF §9.3:1525-1528, ADR-0012).

On feature-engine lifespan startup, diff current indicators.yaml × symbols
registry against the `feature_registry_seen` NATS KV bucket; for each NEW
feature_name, fire-and-forget an async backfill task over the configurable
historical OHLC window (default 30d back). On successful completion, mark
the feature_name as seen in the KV bucket so subsequent restarts skip it.

Per ADR-0012: NATS KV bucket `feature_registry_seen` is pre-provisioned by
infra/nats/bootstrap.sh (4th bucket per BRIEF §8.2 amendment). NatsClient
kv_get / kv_put API per packages/bus/client.py:278+376.

Idempotency: Backfill body uses INSERT ON CONFLICT DO UPDATE per
packages/db/queries/feature_engine.py:insert_feature (T-110b). Re-running
a backfill yields identical rows. Marker write is delayed until full
success — partial completion (crash mid-window) leaves marker unset, so
next startup retries.
"""
```

Public API:

```python
async def schedule_auto_backfills(
    *,
    pool: asyncpg.Pool,
    bus: NatsClient,
    features_by_key: Mapping[tuple[str, str], list[tuple[str, Feature]]],
    window_days: int,
    max_batch_size: int,
    source: str,
    logger: BoundLogger,
    background_tasks: set[asyncio.Task[None]],
    now_fn: Callable[[], datetime] = lambda: datetime.now(UTC),
) -> int:
    """Detect NEW features (current registry minus seen-set) + schedule backfills.

    Returns the count of NEW feature_names scheduled (0 if all already seen).
    Each scheduled task is added to ``background_tasks`` for lifespan-shutdown
    cancellation; tasks self-remove on completion via ``add_done_callback``.

    Detection algorithm:
      1. Flatten ``features_by_key`` to set[feature_name] (post-cross-product
         with symbols — each (symbol, interval) entry yields N feature_names).
      2. For each feature_name: kv_get(`feature_registry_seen`, feature_name).
         If None → NEW; else → already seen, skip.
      3. For each NEW: asyncio.create_task(_backfill_and_mark(...)).

    Errors per feature are isolated (one feature backfill failure does not
    block sibling backfills nor crash the lifespan; logged at ERROR).
    """


async def _backfill_and_mark(
    *,
    pool: asyncpg.Pool,
    bus: NatsClient,
    feature: Feature,
    feature_name: str,
    symbol: str,
    interval: str,
    source: str,
    window_days: int,
    max_batch_size: int,
    logger: BoundLogger,
    now_fn: Callable[[], datetime],
) -> None:
    """Execute backfill for one feature; on success, mark seen in KV."""
```

Backfill body mirrors `scripts/backfill_features.py:_backfill_one_feature` (lines 134-200):

1. Compute `(from_dt, to_dt) = (now - window_days, now)`.
2. `fetch_ohlc_range(conn, symbol, interval, source, from_dt, to_dt)` → list[row].
3. `deque(maxlen=feature.warmup_candles)`.
4. For each candle: append + if buffer full → `feature.compute(tuple(buf))` → `insert_feature(...)`.
5. On full completion: `kv_put(feature_registry_seen, feature_name, now.isoformat().encode())`.
6. On exception: log `feature_auto_backfill_failed` at ERROR; do NOT mark seen; do NOT re-raise (fire-and-forget; lifespan continues).

`max_batch_size` enforcement: for now, backfill_one_feature naturally fetches the entire window in one `fetch_ohlc_range` call (existing helper does NOT chunk). Per WG plan-stage acceptance: `max_batch_size` setting is exposed for future-proofing but not enforced in T-518 scope (insert helper buffers per-row; OHLC fetch is bounded by 30d × intervals × N symbols — within asyncpg fetch limits at expected F5 scale). Note this honestly in plan + commit.

## Lifecycle

Auto-backfill tasks are fire-and-forget asyncio Tasks created in lifespan startup AFTER pipeline.start_consuming() (step 9.5). Tasks live in `app.state.auto_backfill_tasks` (`set[asyncio.Task[None]]`). On task completion, `add_done_callback(auto_backfill_tasks.discard)` removes self from the set. On lifespan shutdown (before `pipeline.stop()`), all remaining tasks get cancelled and joined via `asyncio.gather(..., return_exceptions=True)`.

This intentionally runs AFTER `pipeline.start_consuming()` so live-candle processing isn't blocked by backfill work — per BRIEF §9.3:1525-1528 spec semantics ("automatically triggered" with no latency requirement) the auto-backfill is opportunistic; live processing is the hot path.

## Edge cases

1. **All features already seen** — `kv_get` returns non-None for all → no tasks scheduled → returns 0; logger logs `auto_backfill_scheduled new_features=0`.
2. **Empty registry** (`features_by_key = {}`) — no iteration → 0 NEW; returns 0.
3. **NATS unavailable at startup** — bus.connect() already failed earlier in lifespan (step 5) → lifespan dies before reaching step 9.5; T-518 never invoked. No defensive guard needed inside auto_backfill.
4. **Bucket missing** — `kv_get` raises `PublishError` per `packages.bus.client.kv_get:289-291`. Per ADR-0012 + bootstrap.sh: bucket pre-provisioning is infra responsibility; missing bucket = config bug, fail loud per §0.4. T-518 propagates the error → lifespan dies. Operator reads bootstrap.sh log for guidance.
5. **OHLC fetch returns empty** (no historical data for symbol+interval+window) — backfill task completes with `inserted=0`; STILL marks as seen (per spec semantics: "feature has been processed; no data is acceptable; don't re-attempt"). Logged at INFO with row count.
6. **Backfill task crashes mid-window** (e.g., feature.compute raises on bad data) — task body exception is caught; `kv_put` SKIPPED; logged at ERROR. Next startup retries.
7. **Concurrent restarts with same NEW feature** — both lifespans see the feature as NEW; both schedule backfill. INSERT ON CONFLICT DO UPDATE handles concurrent writes idempotently. Last `kv_put` wins; harmless (both paths complete successfully). Operator-visible duplicate work in logs is acceptable for F5 scale.
8. **Lifespan shutdown during backfill** — task gets cancelled mid-flight via `task.cancel()` → CancelledError raised inside backfill body → exception path skips kv_put → next startup retries. Semantics preserved.
9. **`feature.warmup_candles` larger than OHLC window** — buffer never fills; loop exits with `inserted=0`. Same as edge case #5 (mark seen, skip future runs).
10. **Window boundary** — `from_dt = now - timedelta(days=window_days)`; existing `fetch_ohlc_range` uses inclusive `>=` on `from_dt`. No off-by-one risk per existing test coverage.

## Hazards bound

- **§N1 UTC** — `now_fn() -> datetime` defaults to `datetime.now(UTC)`; `from_dt` / `to_dt` always tz-aware UTC. KV value is `now.isoformat()` (UTC ISO-8601 string). No naive datetimes.
- **§N3** — `kv_put` is `@idempotent` per `packages.bus.client.kv_put:375` (last-write-wins per (bucket, key)). `insert_feature` write is `@non_idempotent` per existing decoration but ON CONFLICT DO UPDATE makes it operator-safe to re-run. Auto-backfill task itself is `@non_idempotent` (logically — re-running would re-process the window) but the underlying writes are idempotent.
- **§N6** — DI via function parameters (pool, bus, logger, etc.); no globals.
- **§N7** — thin scheduler module (~150 LOC); compute logic delegated to existing `feature.compute` + `insert_feature`.
- **§N9** — `backfill_window_days` + `backfill_max_batch_size` exposed via Settings; no hardcoded business config (per L-001 active control).
- **§20** — none F5-binding bind directly. Operator-detectable via auto-backfill log lines.
- **L-001** — backfill window + batch-size configurable via Settings env vars. Defaults documented in Settings docstring.
- **L-013/L-021** — N/A (no JSONB writes; no SQL — uses existing `fetch_ohlc_range` + `insert_feature` helpers).
