1. **L-006 active control lesson confirmed**: persistence-heavy F2 tasks systematically underestimated 50-100% by plan-estimate-time. Original T-216b plan estimate ~328; reality ~756. Even after WG#3 mid-write split, T-216b1 remains 42% over due to genuine persistence-density (6 INSERT/UPDATE helpers each carrying full SQL string + multi-param signatures + emergency-close 2 INSERTs + 1 UPDATE + 2 trading_events + payload-patch post-INSERT + post-commit emit).

3. **T-215 precedent extends**: T-215 ad-hoc waiver path A 13% over with cohesion rationale; T-216b1 42% over is bigger but the underlying pattern (cohesive composition root with load-bearing helpers spanning multiple files) matches.
4. **Alternative (B) triple-split** would force 3 review cycles (drift + brief + math × 3) for a cohesive flow with no architectural benefit; pure operational cost.
5. **Substantive LOC ~440** (excluding docstrings/blanks) is closer to cap than raw 568; trim of docstrings could land ~10% closer but doesn't fundamentally change the overage.

Commit message will document overage explicitly per T-215 precedent.
