## Purpose

Ship 6 reusable UI components per BRIEF §14.4:2070-2079 — the foundation that T-412..T-420 dashboard sections consume. **NO business logic** beyond what each component contractually requires (e.g., `BotSelector` fetches `/api/bots/`); no per-section glue.

Components:

1. **`DataTable`** — pagination + sorting + filtering + column visibility (TanStack Table headless adapter on top of shadcn `<Table>`)
2. **`TimeRangePicker`** — preset buttons (1h, 24h, 7d, 30d, custom) + native `<input type="datetime-local">` for custom range
3. **`BotSelector`** — single OR multi-select; fetches `/api/bots/` via TanStack Query
4. **`StatusBadge`** — colored badge with `kind: 'bot' | 'signal' | 'trade'` + `status` props; cva variants per kind
5. **`PriceDelta`** — sign-colored formatted price; pass-through string `value` (no Decimal→float per §5.3)
6. **`CorrelationIdChip`** — clickable chip; navigates to `/audit?correlation_id=<id>` by default; `onClick` override

**Explicitly out of scope for T-411**: any of the 9 dashboard sections (T-412..T-420 each ship one); production Dockerfile (F5+); auth (F5+); Playwright E2E (T-422); error boundary infrastructure (per-route in T-412+); state stores (zustand dep installed in T-410; first store in T-413); charts (recharts dep installed; first chart in T-413); ECharts (deferred to candlestick consumer T-413/T-417 per T-410 WG#15).

## Public interface

### Component contracts

#### `DataTable<TData>` (`ui/src/components/DataTable.tsx`)

```tsx
import { type ColumnDef } from "@tanstack/react-table";

interface DataTableProps<TData> {
  columns: ColumnDef<TData, unknown>[];
  data: TData[];
  pageSize?: number; // default 50
  enableSorting?: boolean; // default true
  enableFiltering?: boolean; // default true (global filter input)
  enableColumnVisibility?: boolean; // default true (visibility menu)
  onRowClick?: (row: TData) => void; // optional row click handler
  emptyMessage?: string; // default "No data"
}

export function DataTable<TData>(props: DataTableProps<TData>): JSX.Element;
```

Wraps `useReactTable` from `@tanstack/react-table` (headless). Renders shadcn `<Table>` body + pagination footer + global filter input (debounced 200ms) + column visibility menu via **shadcn `<Dialog>` + checkbox list** (per BLOCKER #2 fix). Dialog is heavyweight for the toggle-3-columns use case but already present in T-410 baseline (no new shadcn primitive needed). F5+ may swap to `<DropdownMenu>` when the 7th shadcn baseline lands. Pagination state internal (UI controls only); server-side pagination is consumer concern (consumer slices `data` per current page externally if backend pagination needed — T-414 Trade explorer pattern).

#### `TimeRangePicker` (`ui/src/components/TimeRangePicker.tsx`)

```tsx
type TimeRange = { from: Date; to: Date; preset: "1h" | "24h" | "7d" | "30d" | "custom" };

interface TimeRangePickerProps {
  value: TimeRange;
  onChange: (range: TimeRange) => void;
}

export function TimeRangePicker(props: TimeRangePickerProps): JSX.Element;
```

5 preset buttons (active state colored). Selecting "Custom" reveals 2 `<input type="datetime-local">` (from + to); on blur emits `onChange({from, to, preset: 'custom'})`. Native `Date` instance produced from input value via `new Date(input.value)`.

#### `BotSelector` (`ui/src/components/BotSelector.tsx`)

```tsx
interface BotSelectorProps {
  value: string | string[]; // single bot_id or list
  onChange: (selection: string | string[]) => void;
  multi?: boolean; // default false
  placeholder?: string; // default "Select bot(s)"
}

export function BotSelector(props: BotSelectorProps): JSX.Element;
```

Wraps shadcn `<Select>` (single) or custom multi-select via `<Dialog>` + checkboxes (multi). Fetches via TanStack Query:

```tsx
const { data: bots, isLoading, error } = useQuery({
  queryKey: ["bots"],
  queryFn: () => apiFetch<BotListResponse>("/api/bots/"),
});
```

Loading → disabled select with "Loading..." placeholder. Error → red border + tooltip with error.message. Empty `bots.bots[]` → disabled with "no bots configured" placeholder. Component does NOT cache locally — TanStack Query cache (30s staleTime per main.tsx default) is shared across all selector instances.

`BotListResponse` type imported from new `ui/src/lib/api-types.ts` (T-411 introduces this thin TS mirror of analytics-api response shapes; T-412+ extend per consumer endpoint).

#### `StatusBadge` (`ui/src/components/StatusBadge.tsx`)

```tsx
type BadgeKind = "bot" | "signal" | "trade";

interface StatusBadgeProps {
  kind: BadgeKind;
  status: string; // narrowed via cva variants per kind
}

export function StatusBadge(props: StatusBadgeProps): JSX.Element;
```

`cva` variants:
- `kind=bot`: `active` → green / `paused` → yellow / `archived` → gray
- `kind=signal`: `validated` → green / `duplicate` → yellow / `invalid` → red
- `kind=trade`: `open` → blue / `closed` → gray / `error` → red

Unknown `(kind, status)` combo → render `status` text in muted-foreground (no color). Inline `cva` per OQ-4=A — no centralized `STATUS_COLORS` map (premature abstraction; only consumer is StatusBadge).

#### `PriceDelta` (`ui/src/components/PriceDelta.tsx`)

```tsx
interface PriceDeltaProps {
  value: string | number;
  currency?: string; // default "USD"
  showSign?: boolean; // default true (prepend + on positive)
}

export function PriceDelta(props: PriceDeltaProps): JSX.Element;
```

Pass-through string per OQ-5=A. Component reads numeric sign via `parseFloat` ONLY for color decision; renders `value` verbatim. positive → green, negative → red, zero → muted. Currency suffix appended (e.g., `+12.34 USD`).

**§5.3 Decimal preservation**: callers MUST pass already-formatted Decimal-as-string from backend (`/api/trades` `realized_pnl` is JSON string per T-402 §5.3). PriceDelta does NOT convert via `Number(...)` — would silently lose precision on Decimal values like `"0.000000123456789"`.

#### `CorrelationIdChip` (`ui/src/components/CorrelationIdChip.tsx`)

```tsx
interface CorrelationIdChipProps {
  correlationId: string;
  onClick?: () => void; // optional override
}

export function CorrelationIdChip(props: CorrelationIdChipProps): JSX.Element;
```

Renders chip with truncated `correlationId` (first 8 chars + ellipsis). Tooltip shows full ID on hover. **Per CONCERN #2 fix — lazy useNavigate resolution**: if `onClick` prop provided, `useNavigate` hook is NOT called at all (preserves chip's reusability outside RouterProvider context — e.g., in standalone tests or future static-context renders). Default branch (no `onClick`) calls `useNavigate` and routes to `/audit?correlation_id=<encoded>` (T-419 Audit log viewer route).

### `ui/src/lib/api-types.ts` (NEW)

```tsx
export interface Bot {
  bot_id: string;
  display_name: string;
  // Pydantic serializes datetime to ISO-8601 string in JSON
  created_at: string;
  status: "active" | "paused" | "archived";
  exchange_mode: "live" | "testnet" | "paper";

  // T-411 BotSelector consumes only bot_id + display_name + status; remaining fields

  meta: Record<string, unknown>;

export interface BotListResponse {

  // (BRIEF expects <10 rows; no pagination). Verified 2026-05-04 against actual
  // Pydantic model line 40-43.
  bots: Bot[];
}
```

## Edge cases

1. **DataTable empty data** → renders `emptyMessage` row spanning all columns.
2. **DataTable with 1 row, multi-page enabled** → pagination footer hidden (no second page).
3. **DataTable column visibility menu** → state internal; no persistence across reloads (F5+ may add localStorage).
4. **TimeRangePicker custom range with from > to** → no validation in T-411; emits as-is (consumer route validates if needed; mirror T-407 backend pattern where inverted range returns empty list).
5. **TimeRangePicker preset click while custom inputs visible** → custom inputs hide, preset becomes active.
6. **BotSelector loading** → disabled with "Loading..." placeholder; user cannot interact.
7. **BotSelector fetch failure** → red border + tooltip with `error.message`; fallback to disabled select with "Failed to load bots" text.
8. **BotSelector empty `bots[]`** → disabled with "no bots configured" placeholder.
9. **BotSelector multi-select with `value=[]`** → "Select bot(s)" placeholder visible.
10. **StatusBadge unknown `(kind, status)` combo** → renders `status` in muted-foreground (no color); no throw.
11. **PriceDelta `value="not-a-number"`** → renders verbatim with muted color (parseFloat returns NaN, treat as zero per UX simplicity).
12. **PriceDelta `value=0`** → muted color, sign omitted regardless of `showSign`.
13. **CorrelationIdChip with empty `correlationId=""`** → renders chip with placeholder text "no-corr-id"; click is no-op (prevents navigating to `/audit?correlation_id=`).
14. **TanStack Router `useNavigate` outside RouterProvider context** → React error boundary catches (T-411 doesn't ship error boundary; T-411+ tasks add per-route).

## §N invariants

Test #5 augmented to assert `Date.toISOString()` returns Z-suffixed string.
- **§N2 structured logs**: N/A (browser).
- **§N3 idempotency**: N/A (no writes from components themselves; BotSelector fetch is read-only via TanStack Query).
