## Decisions committed

Per the T-204 → T-205 calibration trajectory.

1. **OQ-defaults applied throughout**; no unresolved decisions remain.

2. **Constructor surface (T-207 baseline)**: 5 keyword-only params:
   ```python
   BybitV5Client(
       *,
       api_key: str,
       api_secret: str,
       base_url: str = "https://api.bybit.com",
       connect_timeout: float = 5.0,
       read_timeout: float = 10.0,
   )
   ```
   T-208 will EXTEND this constructor with adapter-level wrapping (one ADR-0003 SharedRateLimiter ctor kwarg + bus + sub_account); T-207 baseline is REST-envelope-only.

3. **`async def close()` lifecycle method** + `__aenter__/__aexit__` per `BinanceRestClient` precedent. No `start_consuming` (no NATS subscribes — REST-only).

4. **HMAC-SHA256 signing per Bybit V5 spec**: signature payload format `f"{timestamp}{api_key}{recv_window}{query_or_body}"`; `recv_window=5000` (Bybit default per docs). Headers: `X-BAPI-API-KEY`, `X-BAPI-TIMESTAMP`, `X-BAPI-RECV-WINDOW`, `X-BAPI-SIGN`. Signature is hex-encoded HMAC of the payload using `api_secret` as key. **No new third-party dep** — `hmac` + `hashlib` are stdlib.

5. **`_request_with_retry(method, path, *, params=None, body=None, retries: int)` interface**: wraps `httpx.AsyncClient.request()`; returns parsed JSON dict (Bybit V5 envelope `{retCode, retMsg, result, time}`). Retry-decision tree:
   - On `httpx.ConnectError` / `httpx.ReadTimeout` / `httpx.PoolTimeout` (transport): if `retries > 0`, sleep `[0.5, 1.0, 2.0]s * (2**attempt) + jitter` and retry up to `retries` times; else raise `NetworkTimeout`.
   - On HTTP 5xx: same as transport — retryable.
   - On HTTP 401/403: raise `AuthError` (no retry).
   - On HTTP 429 OR retCode 10006/10016: raise `RateLimitError` (no retry — caller's job to back off via shared pause flag).
   - On retCode 10003/10004 (sign mismatch / API key invalid): raise `AuthError`.
   - On retCode 10001/10005 (param error / order rejected with reason): raise `OrderRejected(reason=retMsg)`.
   - On other non-zero retCode: raise generic `ExchangeError(f"retCode={retCode} retMsg={retMsg}")`.
   - On retCode == 0 (success): return `result` field.

6. **Jitter formula per §11.2**: `delay = base_delay[attempt] * (1.0 + random.uniform(-0.1, 0.1))` — ±10% jitter mirror `[0.5, 1.0, 2.0]s` schedule. NOT a §N1/§5.13 violation: jitter is operational backoff, not financial math (Decision parallel to T-205 Decision #12).

7. **Logger**: module-level `logger = logging.getLogger(__name__)` per T-205 precedent. Log events: `bybit_rest_request_started` (DEBUG), `bybit_rest_request_succeeded` (DEBUG), `bybit_rest_request_failed` (WARNING with attempt + retries_remaining), `bybit_rest_retcode_error` (WARNING with retCode + retMsg).

8. **No state** — T-207 is stateless (signing is per-call). All state is in T-208 (method semantics) + T-215 (composition root config).

9. **`@idempotent` / `@non_idempotent` markers**: T-207 ships LOW-LEVEL HTTP envelope (`_request_with_retry`, `_sign_request`); these are not user-facing methods on the `ExchangeClient` Protocol — markers are at T-208 method wrappers (`place_market_order @non_idempotent`, etc., already on Protocol per T-201). T-207 internal helpers carry NO markers (no public §N3 contract surface).

10. **Test vectors per Bybit V5 docs**: Hand verification §F.1 will provide deterministic timestamp+nonce+secret→signature triples computed from Bybit V5 spec (HMAC-SHA256 hex-encoded over signed payload string). 3 vectors: GET request (params in query), POST request (body in JSON), GET with empty params.

11. **`packages/exchange/bybit_v5/__init__.py`** re-exports `BybitV5Client`. T-208 will add `BybitV5Adapter` (ExchangeClient Protocol implementation that composes `BybitV5Client`).

12. **`asyncio.sleep` is the retry sleeper** — NO custom sleep mechanism. Test mocks via `monkeypatch.setattr("asyncio.sleep", fake_sleep)` per T-205 precedent.

### `packages/exchange/bybit_v5/client.py` (NEW)

Single module. ~220 LOC src.

```python
"""Bybit V5 REST envelope + HMAC-SHA256 signing + retry matrix (T-207).

T-207 ships the low-level HTTP layer. T-208 (BybitV5Adapter REST methods)
wraps this client with ExchangeClient Protocol semantics (place_market_order,
cancel_order, get_positions, ...) — including pre-call rate-limit acquire()
and post-call signal_upstream_rate_limit() per OQ-3/4 default A decoupling.

Constructor takes 5 kwargs (api_key, api_secret, base_url, connect_timeout,
read_timeout); T-208 will extend with adapter-level wrapping.
"""

from __future__ import annotations

import asyncio
import hashlib
import hmac
import json
import logging
import random
import time
from typing import TYPE_CHECKING, Any, Literal, Self

import httpx

from packages.exchange.errors import (
    AuthError,
    ExchangeError,
    NetworkTimeout,
    OrderRejected,
    RateLimitError,
)

if TYPE_CHECKING:
    from types import TracebackType


__all__ = ["BybitV5Client"]


_RECV_WINDOW_MS = 5000  # Bybit V5 default per docs.
_BASE_BACKOFF_S: tuple[float, float, float] = (0.5, 1.0, 2.0)
_JITTER_PCT = 0.1
_RETCODE_RATE_LIMIT: frozenset[int] = frozenset({10006, 10016})
_RETCODE_AUTH: frozenset[int] = frozenset({10003, 10004})
_RETCODE_REJECT: frozenset[int] = frozenset({10001, 10005})

logger = logging.getLogger(__name__)


HttpMethod = Literal["GET", "POST", "DELETE"]


class BybitV5Client:
    """Low-level Bybit V5 REST envelope.

    State-free: signing is per-call; httpx.AsyncClient owns connection
    pool. Caller (T-208) owns method-level decisions (retry count,
    rate-limit acquire/signal, body shape).
    """

    def __init__(
        self,
        *,
        api_key: str,
        api_secret: str,
        base_url: str = "https://api.bybit.com",
        connect_timeout: float = 5.0,
        read_timeout: float = 10.0,
    ) -> None:
        self._api_key = api_key
        self._api_secret = api_secret
        self._client = httpx.AsyncClient(
            base_url=base_url,
            timeout=httpx.Timeout(
                connect=connect_timeout,
                read=read_timeout,
                write=read_timeout,
                pool=read_timeout,
            ),
        )

    async def __aenter__(self) -> Self: ...
    async def __aexit__(
        self, exc_type: type[BaseException] | None,
        exc: BaseException | None, tb: TracebackType | None,
    ) -> None: ...
    async def close(self) -> None: ...

    async def request(
        self,
        method: HttpMethod,
        path: str,
        *,
        params: dict[str, Any] | None = None,
        body: dict[str, Any] | None = None,
        retries: int,
    ) -> dict[str, Any]:
        """Execute one Bybit V5 REST call with retry + retCode mapping.

        Returns the ``result`` field of the Bybit envelope on retCode==0.
        Raises typed exceptions per §11.3.
        """
        return await self._request_with_retry(
            method, path, params=params, body=body, retries=retries,
        )

    async def _request_with_retry(self, ...) -> dict[str, Any]:
        """Loop: sign → POST/GET → on transport-error retry; on retCode/HTTP map to typed exception."""
        ...

    def _sign_request(
        self, *, method: HttpMethod, params: dict[str, Any] | None,
        body: dict[str, Any] | None, timestamp_ms: int,
    ) -> dict[str, str]:
        """Compute X-BAPI-* headers per Bybit V5 signing spec.

        Payload: f"{timestamp_ms}{api_key}{recv_window}{query_string_or_body_json}"
        Signature: HMAC-SHA256(api_secret, payload).hexdigest()
        """
        ...

    @staticmethod
    def _translate_retcode_or_status(
        status_code: int, retcode: int, retmsg: str,
    ) -> ExchangeError | None:
        """Map HTTP status + Bybit retCode to typed exception or None on success."""
        ...

    @staticmethod
    def _backoff_delay(attempt: int) -> float:
        """Compute jittered backoff per §11.2 [0.5, 1.0, 2.0]s + ±10% jitter."""
        base = _BASE_BACKOFF_S[min(attempt, len(_BASE_BACKOFF_S) - 1)]
        return base * (1.0 + random.uniform(-_JITTER_PCT, _JITTER_PCT))
```

## Hand verification

### §F.1 — Signing vector triples (Bybit V5 spec)

Bybit V5 signing payload: `f"{timestamp_ms}{api_key}{recv_window}{query_or_body}"`; signature = `hmac.new(api_secret, payload, hashlib.sha256).hexdigest()`.

**Vector 1: GET with query params.**
- `api_key = "abc123"`, `api_secret = "secret"`, `timestamp_ms = 1700000000000`, `recv_window = 5000`
- query_string = `"category=linear&symbol=BTCUSDT"` (sorted alphabetically per Bybit V5 docs)
- payload = `"1700000000000abc1235000category=linear&symbol=BTCUSDT"`
- HMAC-SHA256 hex-digest computed at test time as `expected_signature_1`
- Test fixture: `assert _sign_request(method="GET", params={"symbol": "BTCUSDT", "category": "linear"}, body=None, timestamp_ms=1700000000000) == {"X-BAPI-API-KEY": "abc123", "X-BAPI-TIMESTAMP": "1700000000000", "X-BAPI-RECV-WINDOW": "5000", "X-BAPI-SIGN": expected_signature_1}`

**Vector 2: POST with JSON body.**
- Same key/secret/timestamp.
- body = `{"category": "linear", "symbol": "BTCUSDT", "side": "Buy", "qty": "0.5"}`
- body_json = `json.dumps(body, separators=(',', ':'))` (compact, no whitespace)
- payload = `f"1700000000000abc1235000{body_json}"`
- HMAC-SHA256 → `expected_signature_2`

**Vector 3: GET with empty params.**
- query_string = `""`
- payload = `"1700000000000abc1235000"`
- HMAC-SHA256 → `expected_signature_3`

**Test computes expected signatures via `hmac.new()` directly and compares against `_sign_request()` output. Hand-computable: each vector is a deterministic function of (timestamp, key, secret, query/body) — no randomness. Test fixture uses real `hmac` library so reference + implementation are the same primitive (NOT implementation-against-itself violation: testing OUR API surface invariants — header names, format, sorted-query-keys — against KNOWN library output).**

### §F.2 — Retry decision tree

Given `_request_with_retry(method="GET", path="/v5/account/wallet-balance", params={...}, retries=3)`:

1. **Attempt 0**: send request. If `httpx.ReadTimeout` → wait `0.5 * (1 ± 0.1)` = 0.45-0.55s → attempt 1.
2. **Attempt 1**: same. If timeout → wait `1.0 * (1 ± 0.1)` = 0.9-1.1s → attempt 2.
3. **Attempt 2**: same. If timeout → wait `2.0 * (1 ± 0.1)` = 1.8-2.2s → attempt 3.
4. **Attempt 3** (final): if timeout → raise `NetworkTimeout`; if HTTP 200 → parse retCode → return result OR raise mapped exception.

`retries=0` (place_market_order, H-003): attempt 0 only; on timeout → `NetworkTimeout` immediate.

Test fixtures:
- `test_request_with_retry_3_retries_succeeds_on_attempt_3` — 2 timeouts then OK.
- `test_request_with_retry_zero_retries_raises_immediately_on_timeout`.
- `test_request_with_retry_uses_jittered_backoff_per_attempt` — assert sleeps fall in `[0.45..0.55, 0.9..1.1, 1.8..2.2]s` ranges.

### §F.3 — Jitter formula

`delay = base * (1.0 + random.uniform(-0.1, 0.1))` where `base ∈ {0.5, 1.0, 2.0}`.

Bounds: `0.45 ≤ delay ≤ 0.55` for attempt 0; `0.9 ≤ delay ≤ 1.1` for attempt 1; `1.8 ≤ delay ≤ 2.2` for attempt 2.

Test pin `test_request_with_retry_uses_jittered_backoff_per_attempt`: monkeypatch `asyncio.sleep` → capture all sleep durations → assert each falls in expected range per attempt index.

Math-validator (Gate 4) ACTIVE per CLAUDE.md gate-4 conditional rule (diff touches `packages/exchange/`). Expected verdict: **VERIFIED** — closed-form arithmetic + hand-computable signing fixtures + bounded float jitter + no Decimal/precision boundary issues.

## Edge cases

1. **Bybit returns retCode==0 with empty `result`**: return empty dict `{}` (caller handles per-method).

2. **HTTP 200 with malformed JSON**: `json.JSONDecodeError` → wrap as `ExchangeError(f"malformed response body")` with original as `__cause__`.

3. **HTTP 500/502/503**: retryable (transport-class). Same path as `httpx.ReadTimeout`.

4. **HTTP 4xx other than 401/403/429**: raise generic `ExchangeError(f"HTTP {status} {body[:200]}")`.

5. **Connection pool exhaustion (`httpx.PoolTimeout`)**: same retry path as `ReadTimeout`. Pool size at httpx default (100); adapter pool composition root tunes if needed.

6. **Empty `result` field on success**: rare for Bybit V5; return empty dict — caller can validate per-method.

7. **System clock skew vs Bybit server time**: `recv_window=5000ms` allows ±5s drift. If consistently breaking, operator action (NTP) — out of T-207 scope.

8. **Signing timestamp generated via `time.time_ns() // 1_000_000`**: stdlib monotonic-aware sufficient. T-208 may DI a `now_fn` for test determinism if needed; T-207 baseline uses stdlib directly.

## Risks (post-resolution)

Mitigation: test pin `test_retcode_other_maps_to_generic_ExchangeError`; T-208 method tests will surface specific retCodes that need refined mapping.
