**L-037** (T-558a, plan-reviewer Gate-1 pass-1 BLOCKER, 2026-05-18): Python `Decimal // Decimal` truncates toward zero (the `divide-integer` op), it does NOT floor toward −∞ — a ceil built as `-(((-x)//t)*t)` or any floor-assuming negative-path construction is WRONG for negative quotients. `quantize_qty`'s docstring asserts "`//` is Python floor-div toward −∞ per §5.3" — false in general; it never bit because `quantize_qty` floors only POSITIVE qty (truncate==floor for positive operands). Active control: any tick/grid/precision rounding helper with a negative-operand or ceil path MUST use explicit `Decimal.to_integral_value(rounding=ROUND_FLOOR|ROUND_CEILING)` (or `.quantize(..., rounding=)`), NEVER `//`-derived floor/ceil; plan-reviewer + math-validator must reject any `//`-floor claim on a path that can see a negative operand and recompute the negative/ceil hand-case.
