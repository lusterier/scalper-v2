2. **`packages/bus/schemas/orders.py` shape** — **CONCERN #5 fix: subclass `event_type: Literal[...]` narrowing**. Brief §8.4 lines 1349-1362 shows base class `event_type: str` plus `class OrderPlaced(OrderEventBase): ...` (empty body) — the brief is silent on whether subclasses override `event_type`. We narrow each subclass with `event_type: Literal[<value>] = <value>` for type-narrowing + Pydantic discriminated union support. Brief is silent on this; intentional improvement, not deviation.
   ```python
   """`orders.requests` / `orders.events` / `trading.events` payloads (§8.4 lines 1316-1363)."""

   from __future__ import annotations
   from datetime import UTC, datetime, timedelta
   from decimal import Decimal
   from typing import Any, Literal
   from pydantic import BaseModel, ConfigDict, field_serializer, field_validator

   __all__ = [
       "OrderClosed", "OrderEventBase", "OrderFilled", "OrderPlaced",
       "OrderRequest", "SLMoved", "TradingEvent",
       "subject_for_orders_request", "subject_for_orders_event", "subject_for_orders_dlq",
   ]


   def subject_for_orders_request(bot_id: str) -> str:
       return f"orders.requests.{bot_id}"

   def subject_for_orders_event(bot_id: str) -> str:
       return f"orders.events.{bot_id}"

   def subject_for_orders_dlq(bot_id: str) -> str:
       return f"orders.dlq.{bot_id}"


   class OrderRequest(BaseModel):
       model_config = ConfigDict(frozen=True)
       schema_version: Literal["1.0"] = "1.0"
       bot_id: str
       signal_id: int
       symbol: str
       side: Literal["buy", "sell"]
       order_type: Literal["market"] = "market"
       qty: Decimal
       leverage: int
       sl_pct: Decimal
       tp_pct: Decimal
       tp_qty_pct: Decimal
       be_trigger: Decimal
       be_sl_level: Decimal
       trail_pct: Decimal
       exchange_mode: Literal["live", "testnet", "paper"]


   class OrderEventBase(BaseModel):
       model_config = ConfigDict(frozen=True)
       schema_version: Literal["1.0"] = "1.0"
       bot_id: str
       event_type: str  # discriminator
       order_id: int
       exchange_order_id: str
       symbol: str
       timestamp: datetime

       @field_validator("timestamp")
       @classmethod
       def _must_be_utc(cls, value: datetime) -> datetime:
           if value.tzinfo is None:
               raise ValueError("timestamp must be timezone-aware (tzinfo=datetime.UTC)")
           if value.utcoffset() != timedelta(0):
               raise ValueError("timestamp must be in UTC")
           return value if value.tzinfo is UTC else value.replace(tzinfo=UTC)

       @field_serializer("timestamp")
       def _serialize_utc(self, value: datetime) -> str:
           return value.isoformat()


   class OrderPlaced(OrderEventBase):
       event_type: Literal["order_placed"] = "order_placed"


   class OrderFilled(OrderEventBase):
       event_type: Literal["order_filled"] = "order_filled"
       exec_id: str
       price: Decimal
       qty: Decimal
       fee: Decimal
       exec_type: Literal["open", "partial_tp", "sl", "trail", "close"]


   class OrderClosed(OrderEventBase):
       event_type: Literal["order_closed"] = "order_closed"
       realized_pnl: Decimal
       close_reason: str


   class SLMoved(OrderEventBase):
       event_type: Literal["sl_moved"] = "sl_moved"
       new_sl_price: Decimal
       sl_type: Literal["protective", "be", "trail"]


   class TradingEvent(BaseModel):
       """`trading.events` payload — same shape as OrderEventBase + payload dict.

       Persisted by analytics-api (later F4 task) into ``trading_events``
       hypertable per §7.2 line 1091. `event_type` is the routing
       discriminator; ``payload`` carries the full OrderEvent body.
       """
       model_config = ConfigDict(frozen=True)
       schema_version: Literal["1.0"] = "1.0"
       occurred_at: datetime
       bot_id: str | None = None
       correlation_id: str | None = None
       event_type: str
       payload: dict[str, Any]
       # UTC validators identical to OrderEventBase.timestamp ...
   ```

3. **`services/execution/app/placement.py` shape** — **REVISED per BLOCKER #1 (b) + #4 (single handler shape) + CONCERN #6 (mismatch test) + CONCERN #7 (ctor Settings kwargs) + BLOCKER #3 (step-size TODO)**:
   ```python
   """§9.5 steps 1-5 order placement pipeline (T-216a).

   Per-bot subscriptions registered at lifespan step 6 (after adapter
   pool composition): one `bus.subscribe(orders.requests.<bot_id>, handler)`
   per bot in `app.state.adapters`. Subject is bot-bound at registration;
   `_make_per_bot_handler(bot_id, adapter, bus, logger, ...)` is a closure
   factory returning the per-bot async handler.

   Per-message handler (single `_make_per_bot_handler` body, NO dual
   `__call__/handle`):

   1. Validate envelope payload as `OrderRequest`; on failure → log ERROR
      + counter, drop (programmer-error fail-loud per OQ-7).
   2. **CONCERN #6 fix**: assert `request.bot_id == bot_id_from_subject` (subject is
      authoritative routing key; payload bot_id mismatch logs WARN
      `execution.subject_payload_botid_mismatch_using_subject` and
      processing continues using subject bot_id). Defends against payload
      spoofing.
   3. Adapter is closure-captured (NOT looked up by bot_id) — per-bot
      handler is bot-bound at registration. (Adapter lookup remains for
      defensive nil-check only; if adapter has been removed mid-run,
      DLQ to `orders.dlq.<bot_id>` + log ERROR. In F2 sub-10-bot scope
      this never happens; still defended.)
   4. **BLOCKER #3 step-size TODO**: log WARN `execution.qty_step_rounding_pending_t_f2_plus`
      with `request.qty` BEFORE place call. Operator visibility hook for
      Bybit retCode 110017 reject path until T-F2+ instruments-info cache lands.
   5. Call `adapter.set_leverage(symbol, leverage)` (LRU cache adapter-internal).
   6. Call `adapter.place_market_order(symbol, side, qty, reduce_only=False)`;
      on `UnknownState` raised → log ERROR + counter (NO DLQ, NO retry per H-003).
      On Bybit `OrderRejected` with retMsg matching qty/precision → log ERROR
      `execution.place_market_order_qty_rejected` (operator-actionable signal
      to wire T-F2+ step-size cache).
   7. Call `adapter.get_fill_price(symbol, exchange_order_id)`; if None →
      retry up to `settings.execution_fill_price_retry_attempts` × with
      `settings.execution_fill_price_retry_backoff_s` backoff; if still
      None → raise `FillPriceUnresolvedError` → outer try catches → DLQ.
   8. Raise `NotImplementedError("T-216b: SL+TP+persist+events")` —
      forward-pointer for T-216b owner; mirror T-208a stub-pin
      precedent. master working state preserved (execution-service not
      wired to live production until T-222 E1 testnet smoke).
   """

   from __future__ import annotations
   import asyncio
   import logging
   from decimal import Decimal
   from typing import TYPE_CHECKING

   from packages.bus.schemas.orders import (
       OrderRequest,
       subject_for_orders_dlq,
   )
   from packages.core import BotId
   from packages.exchange.errors import (
       AuthError, NetworkTimeout, OrderRejected, RateLimitError, UnknownState,
   )

   if TYPE_CHECKING:
       from collections.abc import Awaitable, Callable

       from structlog.stdlib import BoundLogger

       from packages.bus import MessageEnvelope, NatsClient
       from packages.exchange.protocols import ExchangeClient


   __all__ = [
       "FillPriceUnresolvedError",
       "make_per_bot_handler",
   ]


   logger = logging.getLogger(__name__)


   class FillPriceUnresolvedError(RuntimeError):
       """Adapter returned None for get_fill_price after configured max attempts.

       **Placement justification (CONCERN #8 fix)**: lives in
       `services/execution/app/placement.py`, NOT `packages/exchange/errors.py`.
       This represents an execution-service orchestration outcome (3× inline
       retry exhausted across the consumer side), not an adapter call failure.
       §11.3 error taxonomy in `packages.exchange.errors` covers
       adapter→caller error contracts only (AuthError, OrderRejected,
       NetworkTimeout, UnknownState, RateLimitError); upper-layer
       orchestration errors stay local to their consuming service.
       """


   def make_per_bot_handler(
       *,
       bot_id: BotId,
       adapter: ExchangeClient,
       bus: NatsClient,
       logger: BoundLogger,
       fill_price_retry_attempts: int,
       fill_price_retry_backoff_s: float,
   ) -> Callable[[MessageEnvelope], Awaitable[None]]:
       """Closure factory returning the per-bot `orders.requests.<bot_id>` handler.

       Bot identity + adapter + Settings-derived retry knobs are bound at
       registration time. Returned coroutine has the
       `Callable[[MessageEnvelope], Awaitable[None]]` shape required by
       `NatsClient.subscribe` (per `packages/bus/client.py:58`); subject
       is implicit (we registered to one subject per bot, so subject is
       always `orders.requests.<bot_id>`).
       """

       async def _handle(envelope: MessageEnvelope) -> None:
           try:
               request = OrderRequest.model_validate(envelope.payload)
           except Exception as exc:
               logger.error(
                   "execution.order_request_validation_failed",
                   bot_id=bot_id,
                   error=str(exc),
               )
               return
           if request.bot_id != bot_id:
               logger.warning(
                   "execution.subject_payload_botid_mismatch_using_subject",
                   subject_bot_id=bot_id,
                   payload_bot_id=request.bot_id,
               )
           # Defensive nil-check (sub-10-bot scope: never happens; still defended).
           if adapter is None:  # pragma: no cover — closure binding cannot land None
               logger.error("execution.adapter_missing", bot_id=bot_id)
               await bus.publish(subject_for_orders_dlq(bot_id), envelope)
               return
           # BLOCKER #3 — step-size TODO.
           logger.warning(
               "execution.qty_step_rounding_pending_t_f2_plus",
               bot_id=bot_id,
               symbol=request.symbol,
               qty=str(request.qty),
           )
           # 5. set_leverage (LRU cached adapter-internal).
           try:
               await adapter.set_leverage(request.symbol, request.leverage)
           except (AuthError, OrderRejected, NetworkTimeout, RateLimitError) as exc:
               logger.error(
                   "execution.set_leverage_failed",
                   bot_id=bot_id,
                   symbol=request.symbol,
                   error=str(exc),
               )
               return
           # 6. place_market_order (H-003 zero-retry; UnknownState catch path).
           try:
               place_result = await adapter.place_market_order(
                   request.symbol, request.side, request.qty,
               )
           except UnknownState as exc:
               logger.error(
                   "execution.place_market_order_unknown_state",
                   bot_id=bot_id,
                   symbol=request.symbol,
                   error=str(exc),
               )
               return
           except OrderRejected as exc:
               # Defensive: surface qty/precision rejects with explicit log key
               # for operator-actionable signal (T-F2+ step-size cache trigger).
               logger.error(
                   "execution.place_market_order_qty_rejected"
                   if "precision" in exc.reason.lower() or "qty" in exc.reason.lower()
                   else "execution.place_market_order_rejected",
                   bot_id=bot_id,
                   symbol=request.symbol,
                   reason=exc.reason,
               )
               return
           except (AuthError, NetworkTimeout, RateLimitError) as exc:
               logger.error(
                   "execution.place_market_order_failed",
                   bot_id=bot_id,
                   symbol=request.symbol,
                   error=str(exc),
               )
               return
           # 7. get_fill_price with inline retry per OQ-8 + CONCERN #7 ctor kwargs.
           fill_price: Decimal | None = None
           for attempt in range(fill_price_retry_attempts):
               fill_price = await adapter.get_fill_price(
                   request.symbol, place_result.exchange_order_id,
               )
               if fill_price is not None:
                   break
               if attempt + 1 < fill_price_retry_attempts:
                   await asyncio.sleep(fill_price_retry_backoff_s)
           if fill_price is None:
               logger.error(
                   "execution.fill_price_unresolved",
                   bot_id=bot_id,
                   exchange_order_id=place_result.exchange_order_id,
               )
               await bus.publish(subject_for_orders_dlq(bot_id), envelope)
               raise FillPriceUnresolvedError(
                   f"fill_price None after {fill_price_retry_attempts} attempts"
               )
           # 8. T-216a stops here. T-216b owns SL+TP+persist+events.
           raise NotImplementedError(
               "T-216b: SL+TP+persist+events — placement pipeline post-fill_price"
           )

       return _handle
   ```

## Lifecycle

T-216a placement handler lifecycle:

1. **Construction** (per bot at lifespan step 5): `make_per_bot_handler(bot_id=, adapter=, bus=, logger=, fill_price_retry_attempts=, fill_price_retry_backoff_s=)` returns a per-bot async handler. Bot identity + adapter + Settings retry knobs bound in closure.
2. **Subscription registration** (async at lifespan step 5, per bot): for each bot in `adapter_pool.adapters`, call `bus.subscribe(subject_for_orders_request(bot_id), handler)`. Subject bound at registration; no wildcard `.>`. WG#7: subscribe failure crashes the lifespan (fail-fast).
3. **Per-message handle** (async, driven by NATS):
   a. Validate envelope payload as `OrderRequest`; on failure → log + drop.
   b. Subject-payload bot_id mismatch → WARN log; continue using closure-bound bot_id.
   c. Pre-place step-size WARN (operator visibility until T-F2+).
   d. set_leverage → place_market_order (H-003 catch + OrderRejected qty/precision differentiation) → get_fill_price (Settings-tunable retry).
   e. Raise NotImplementedError (T-216b owner).
4. **Shutdown**: bus.close() (existing T-215 step) drains subscriptions; handler closures garbage-collected.

## Edge cases

1. **Per-bot subscription registration**: each bot has exactly one subject `orders.requests.<bot_id>`. NATS routes only exact subject matches — no multi-segment / empty-suffix / wrong-prefix subjects can be received. (Eliminated by BLOCKER #1 (b) resolution; subject parser unnecessary.)

2. **Bot row exists in `bots` table but adapter NOT bound at registration**: impossible by construction — handler is registered exactly when adapter is in `adapter_pool.adapters`. Defensive nil-check in handler exits via `if adapter is None` guard but is `# pragma: no cover` (closure binding cannot land None).

3. **`OrderRequest.bot_id` differs from closure-bound bot_id** (CONCERN #6): payload could spoof. Handler logs WARN `execution.subject_payload_botid_mismatch_using_subject` and continues using closure-bound bot_id as authoritative. Test pin `test_handler_logs_warn_on_subject_payload_botid_mismatch_uses_subject_as_authoritative`.

4. **`get_fill_price` returns None forever**: configured-attempts retry exhausted → DLQ + raise `FillPriceUnresolvedError`. Operator manual cleanup OR T-221 reconcile via REST.

5. **Adapter raises `RateLimitError`**: handled at adapter layer (T-205 SharedRateLimiter pause flag). T-216a catches generic ExchangeError family → log + return. Handler will not retry; next strategy-engine signal will retry naturally.

6. **`place_market_order` succeeds but envelope state lost mid-handler** (e.g., service crash before fill_price retrieved): orphan position on Bybit. T-221 reconciliation cleans up by matching open positions vs DB orders (no DB row means orphan).

7. **NatsClient.subscribe handler signature**: `Callable[[MessageEnvelope], Awaitable[None]]` per existing T-104/T-110c precedent (verified `packages/bus/client.py:58`). Path (b) per-bot registration aligns naturally — no signature change needed.

8. **DLQ publish failure during fill-price unresolved path** (WG#6): if `bus.publish(orders.dlq.<bot_id>, envelope)` itself fails (NATS disconnect), the publish exception is caught + logged `execution.dlq_publish_failed`; `FillPriceUnresolvedError` still propagates to caller. Test pin `test_handler_dlq_publish_failure_still_raises_FillPriceUnresolvedError`.
