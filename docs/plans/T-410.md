### `src/lib/api-client.ts` skeleton

Native `fetch` wrapper per OQ-6=A. Type-safe response handling; throws on non-2xx. ZERO actual calls in T-410 — just the helper that T-412..T-420 import.

```ts
type FetchOptions = Omit<RequestInit, "body"> & { body?: unknown };

export async function apiFetch<T>(path: string, options: FetchOptions = {}): Promise<T> {
  const { body, headers, ...rest } = options;
  const res = await fetch(path, {
    ...rest,
    headers: { "content-type": "application/json", ...headers },
    body: body !== undefined ? JSON.stringify(body) : undefined,
  });
  if (!res.ok) throw new Error(`API ${path} failed: ${res.status}`);
  return res.json() as Promise<T>;
}
```

### `src/lib/sse-client.ts` skeleton

Native `EventSource` wrapper per OQ-7=A. T-413 will consume `/events/stream?types=positions,signals,trades`.

```ts
export type SSEEvent = { type: string; payload: unknown; correlation_id: string | null; published_at: string };
export type SSEHandler = (event: SSEEvent) => void;

export function subscribeSSE(
  url: string,
  handler: SSEHandler,
  onError?: (e: Event) => void,
): () => void {
  const source = new EventSource(url);
  source.onmessage = (msg) => {
    try { handler(JSON.parse(msg.data) as SSEEvent); }
    catch (e) { console.error("SSE parse failed", e); }
  };
  if (onError) source.onerror = onError;
  return () => source.close();
}
```

## Edge cases

1. **Node 18 deprecation pressure on Vite 5.4**: Vite 5.x will receive security updates through ~2026 per Vite policy. F4+ may need to bump Vite + Node as part of operator host upgrade.
2. **`pnpm install` on first run**: ~30-60s on cold cache; ~1-2 GB `node_modules/`. `.gitignore` excludes; pnpm-lock.yaml committed.
3. **TanStack Router file-based routing requires generated `routeTree.gen.ts`**: vite plugin generates on save. We commit it to avoid `pnpm install` step generating it post-clone (test reproducibility).
4. **shadcn/ui upstream changes**: components are static copies; no auto-update. F5+ may add a script to re-sync on demand.
5. **Tailwind JIT cache**: `.vite/` + `.vitest-cache/` should be in `.gitignore`.
6. **Vite dev server on `127.0.0.1` only**: deliberate per §16.6 — operator can `ssh -L 5173:localhost:5173` to access remotely. Production exposure is F5+.
7. **CORS in dev**: Vite proxy `/api` + `/events` routes to `http://127.0.0.1:8000` (analytics-api dev port). Operator runs analytics-api locally before `pnpm dev`.
8. **TypeScript strict + `noUncheckedIndexedAccess`**: imposes more `?? defaultValue` patterns. Worth the safety; well-established per Python's `mypy --strict` analog.
9. **ESLint flat config (eslint v9)**: replaces legacy `.eslintrc.json`. Some plugins still on legacy; we use only ones that ship flat-config support.
10. **Vitest jsdom environment**: tests run in jsdom (browser-like); no real browser. Playwright (T-422) handles real-browser scenarios.
