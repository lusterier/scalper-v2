## Public interface

### `packages/exchange/protocols.py` — `ExchangeClient` Protocol

```python
"""§11.1 ExchangeClient port — protocol that BybitV5Adapter (T-207..T-208)
and PaperExchange (T-211..T-213) implement.
... per-method docstrings + class docstring covering retry semantics,
streaming pattern, and idempotency labeling per §5.8 / §N3 / H-003.
"""

from __future__ import annotations
from typing import TYPE_CHECKING, Literal, Protocol
from packages.core import idempotent, non_idempotent

if TYPE_CHECKING:
    from collections.abc import AsyncIterator
    from decimal import Decimal
    from .types import ExecutionEvent, OrderPlaceResult, Position, PositionEvent

__all__ = ["ExchangeClient"]


class ExchangeClient(Protocol):
    @idempotent
    async def set_leverage(self, symbol: str, leverage: int) -> None: ...

    @non_idempotent
    async def place_market_order(
        self,
        symbol: str,
        side: Literal["buy", "sell"],
        qty: Decimal,
        reduce_only: bool = False,
    ) -> OrderPlaceResult: ...

    @idempotent
    async def set_trading_stop(
        self,
        symbol: str,
        tpsl_mode: Literal["Full", "Partial"],   # NO DEFAULT — H-013 / Q11
        sl_price: Decimal | None = None,
        tp_price: Decimal | None = None,
        tp_size: Decimal | None = None,
    ) -> None: ...

    @idempotent
    async def cancel_order(self, symbol: str, order_id: str) -> None: ...

    @idempotent
    async def get_positions(self, symbol: str | None = None) -> list[Position]: ...

    @idempotent
    async def get_fill_price(self, symbol: str, order_id: str) -> Decimal | None: ...

    @idempotent
    async def get_closed_pnl_cumulative(self, sub_account: str) -> Decimal: ...

    def stream_executions(self) -> AsyncIterator[ExecutionEvent]: ...
    def stream_positions(self) -> AsyncIterator[PositionEvent]: ...

    async def close(self) -> None: ...
```

**Decisions baked in (already, before plan-reviewer):**

- **Q11 — `tpsl_mode` no-default.** Brief §11.1 line 1857 shows `tpsl_mode: Literal["Full", "Partial"] = "Full"`. Hazard H-013 line 2699 mandates "no default; compile error if omitted". H-013 is the authoritative bug-preventing invariant; §11.1 default `"Full"` is a drafting artifact and gets overruled. `tpsl_mode` is moved to FIRST positional parameter after `symbol` (so kwargs-only ordering of optional fields stays clean — `sl_price`, `tp_price`, `tp_size` keep their `= None` defaults). T-200 plan-doc already codified this (§Decisions committed Q11). Plan-doc author MUST include this note verbatim.
- **`Optional[X]`** in brief → **`X | None`** in code (modern syntax, mirrors rest of codebase: `packages/features/types.py`, `packages/core/types.py`).
- **`AsyncIterator[X]` return** without `async def`: see OQ-1 below.
- **`@idempotent` / `@non_idempotent` on Protocol methods**: marker decorators from `packages.core.markers` apply to function objects, including Protocol method stubs. `assert_all_methods_labeled(SomeAdapter)` then verifies subclasses inherit the labeling. Marker keys by module+qualname (`packages.exchange.protocols.ExchangeClient.set_leverage`), so the protocol-level labels are independently registered from any future adapter labels — they don't collide.
- **`stream_*` not `@idempotent`**: streaming methods are neither writes nor retry-eligible; they open an iterator. Per §5.8 the markers tag external **writes**; reads-and-streams sit outside that classification. T-201 ships a module-level frozenset codifying the exemption (see below).
- **`close()` not labeled**: lifecycle method, not a write; idempotent in spirit but not in the §5.8 retry sense. Same exemption as streams.

**`_UNLABELED_METHODS` exemption set.** `protocols.py` declares at module level:

```python
_UNLABELED_METHODS: frozenset[str] = frozenset({"stream_executions", "stream_positions", "close"})
```

**Rationale (locked for future readers):** Frozenset codifies the protocol-level exemption from §N3 marker invariant. T-206 conformance test (later F2 task) will read this set to assert: every Protocol method NOT in this set carries either `@idempotent` or `@non_idempotent`. `stream_*` + `close()` are lifecycle/IO patterns, not request-response writes — marker semantics don't apply.

The set is module-private (`_` prefix) but accessed by T-206 via direct module attribute read (not re-exported through `__init__.py`). Underscore signals "consumed only by sibling test machinery in this package, not external callers".

### `packages/exchange/errors.py` — taxonomy

```python
"""§11.3 exception hierarchy for `packages.exchange`.

All exchange-side errors inherit from `ExchangeError`, which itself
inherits from `packages.core.ScalperError`. Upper layers narrow to
`except ExchangeError` for retry/abort/reconcile decisions per §11.3.
"""

from __future__ import annotations
from dataclasses import dataclass
from packages.core import ScalperError

__all__ = [
    "AuthError",
    "ExchangeError",
    "NetworkTimeout",
    "OrderRejected",
    "RateLimitError",
    "UnknownState",
]


class ExchangeError(ScalperError):
    """Base class for errors raised by `packages.exchange` adapters."""


class RateLimitError(ExchangeError):
    """Bybit retCode 10006 / 10016 (per-endpoint or IP rate limit hit).
    Triggers shared coordinated pause per §11.4 / H-025."""


class AuthError(ExchangeError):
    """Auth/signing error (invalid HMAC, expired key, IP not whitelisted)."""


class OrderRejected(ExchangeError):
    """Order rejected at submit time. `reason` examples per §11.3:
    `'insufficient_margin'`, `'price_deviation'`, etc."""

    def __init__(self, reason: str, *args: object) -> None:
        super().__init__(reason, *args)
        self.reason = reason


class NetworkTimeout(ExchangeError):
    """Request timed out at network layer for an idempotent call.
    Caller may retry per the adapter's retry matrix (§11.2)."""


class UnknownState(ExchangeError):
    """Used for `place_market_order` timeout per H-003: order may have
    succeeded; reconciliation on next startup determines state.
    `last_known_action` carries adapter-side context for the audit log."""

    def __init__(self, last_known_action: str, *args: object) -> None:
        super().__init__(last_known_action, *args)
        self.last_known_action = last_known_action
```

**Decisions baked in:**

- `OrderRejected` and `UnknownState` get explicit `__init__` so the named field (`reason` / `last_known_action`) is settable as a positional arg AND attached as an attribute (not just an args[0]). This matches the brief's wire-level shape (`OrderRejected.reason: str`, `UnknownState.last_known_action: str`) — readable from the exception object, not via `args` indexing.
- No additional fields beyond what §11.3 specifies. Resist temptation to add `retCode: int`, `endpoint: str`, etc. Adapters may attach those via `meta` JSON in T-207, but T-201 does NOT pre-design the carrier (§0.8 anti-hypothetical).

### `packages/exchange/types.py` — domain types

```python
"""§11.1 / §11.3 domain types — return values of `ExchangeClient` methods.

These are Python-internal frozen dataclasses, NOT NATS wire schemas.
The §8.4 wire schemas (`OrderFilled`, etc.) are Pydantic and live in
`packages/bus/schemas.py`. T-216b / T-218 map domain → wire at the
publish boundary (mirror T-110 OhlcCandle pattern).
"""

from __future__ import annotations
from dataclasses import dataclass
from datetime import datetime  # noqa: TC003 — runtime annotation
from decimal import Decimal    # noqa: TC003 — runtime annotation
from typing import Literal

__all__ = ["ExecutionEvent", "OrderPlaceResult", "Position", "PositionEvent"]


@dataclass(frozen=True, slots=True)
class OrderPlaceResult:
    """Returned by `ExchangeClient.place_market_order`. Minimum shape
    upper layers need to (a) persist `orders.exchange_order_id`, (b)
    drive subsequent `set_trading_stop` calls. Fill price comes from a
    separate `get_fill_price()` call (§11.1 line 1869), NOT from this
    response — exchanges return the order ack before the fill record
    materializes."""

    exchange_order_id: str
    placed_at: datetime  # UTC; adapter constructs via packages.core.now_utc()


@dataclass(frozen=True, slots=True)
class Position:
    """REST snapshot of a single symbol's position state. Returned by
    `get_positions()`. Fields chosen to support T-216 (placement) and
    T-221 (post-restart reconciliation)."""

    symbol: str
    side: Literal["buy", "sell"] | None  # None = flat (no open position)
    size: Decimal                        # 0 if flat; absolute value
    entry_price: Decimal | None          # None if flat
    leverage: int | None                 # None if flat or not reported
    unrealized_pnl: Decimal | None       # None if flat


@dataclass(frozen=True, slots=True)
class ExecutionEvent:
    """Single fill emitted by the exchange's private WS stream
    (`stream_executions()`). Mirrors §7.2 `executions` table columns
    minus `exec_type` (per H-024 the dispatcher T-218 derives
    `exec_type` from DB `execId → order_id` matching, NOT from any
    field on this event)."""

    exchange_exec_id: str    # Bybit `execId`
    exchange_order_id: str
    symbol: str
    side: Literal["buy", "sell"]
    price: Decimal
    qty: Decimal
    fee: Decimal
    executed_at: datetime    # UTC


@dataclass(frozen=True, slots=True)
class PositionEvent:
    """Single position update from the exchange's private WS stream
    (`stream_positions()`). Same shape as `Position` plus `occurred_at`
    so consumers (T-218 dispatcher) can correlate with execution events
    by timestamp ordering."""

    symbol: str
    side: Literal["buy", "sell"] | None
    size: Decimal
    entry_price: Decimal | None
    leverage: int | None
    unrealized_pnl: Decimal | None
    occurred_at: datetime    # UTC
```

**Decisions baked in:**

- All four types: `@dataclass(frozen=True, slots=True)` mirroring `packages/features/types.py:OhlcCandle`. Frozen → safe to share across tasks; slots → smaller footprint.
- `Decimal | None` not `Optional[Decimal]` (project style).
- All `datetime` are UTC by §N1; constructor enforcement happens in adapter (`packages.core.now_utc()` returns aware UTC). T-201's dataclasses don't enforce tz at the type level (Python's `datetime` type is the same for naive/aware) — the contract is documented in field comments and tested in T-206 conformance + T-207/T-211 adapter tests.
- `ExecutionEvent` does NOT carry `exec_type` per H-024. The dispatcher T-218 derives it via DB lookup. Adapter cannot know the trade-level interpretation.
- `Position` and `PositionEvent` are distinct classes (not `Position` with optional timestamp). Two reasons: (a) brief uses two distinct names — preserving them keeps the protocol legible; (b) `Position` from REST is "current state right now" while `PositionEvent` is a discrete moment in a stream — the timestamp semantics differ.
- `OrderPlaceResult` does NOT carry fill price. §11.1 line 1869 has separate `get_fill_price` for that — Bybit's place-order ack is async w.r.t. fill materialization. Bundling fill into `OrderPlaceResult` would either force the adapter to wait (sync-on-async) or force callers to handle "filled?" None. Cleaner: separate call.

## Edge cases

- **Forward refs in `protocols.py`**: types referenced in method signatures (`Position`, `OrderPlaceResult`, `ExecutionEvent`, `PositionEvent`) are imported under `if TYPE_CHECKING:` from `.types`. At runtime, the protocol class body has only marker decorators applied; the type hints are strings via `from __future__ import annotations`. This avoids any runtime cycle even though `types.py` does NOT import from `protocols.py` (the dep is one-way).
- **`AsyncIterator` annotation on a non-`async def` method**: `def stream_executions(self) -> AsyncIterator[ExecutionEvent]: ...` is the legal form — caller does `async for x in client.stream_executions():`. The brief writes `async def` (line 1878) which would mean caller `await`s first to get the iterator, then iterates — usable but unusual. **Default chosen here = drop `async def`, use plain `def` returning `AsyncIterator`** (Python idiom for "method returns an async iterator"; mirrors how every `__aiter__` is typed). See OQ-1 below.
- **`UnknownState` after `place_market_order`**: H-003 says timeout raises `UnknownState`. T-201 ships only the type; T-207 wires it. Test in T-201 verifies the class accepts `last_known_action: str` positionally and exposes it as attribute.
- **Python ≥ 3.12 syntax**: `X | None`, type-parameterized generics. All matches project baseline `requires-python = ">=3.12"`.

## Decisions committed (operator-approved 2026-04-27 — "use defaults on all 10")

All ten OQs resolved with the proposed default. No overrides. Each entry below is now load-bearing for plan-reviewer Gate 1 and downstream readers.

1. **OQ-1 — `stream_*` signature**: **default A — `def` (sync method) returning `AsyncIterator[X]`.** Caller idiom: `async for event in client.stream_executions(): ...`. The brief's `async def` (§11.1 lines 1878-1879) is a drafting artifact, same class as Q11 — overruled in favor of Python convention. T-209/T-211 implement as `async def stream(self): yield event` async generators; the protocol stub `def stream(self) -> AsyncIterator[X]: ...` matches structurally because `async def + yield` is typed as `AsyncIterator` (via `AsyncGenerator` subtype). If a third §11.1 brief inconsistency emerges during T-205..T-222 implementation, escalate as NEEDS DISCUSSION rather than silently resolve.
