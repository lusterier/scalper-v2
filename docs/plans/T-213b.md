## Decisions committed (verbatim per §12.1 + §3.1 + T-200 + T-211 + T-213a + T-212)

Per the T-204 → T-212 → T-213a calibration trajectory.

1. **OQ-1 split applied**: this plan-doc scopes T-213b only (writes + emission). T-213c (reads + restart recovery) gets its own plan-doc later.

2. **Persist-then-emit ordering** per T-104b precedent + §9.5 step 9 ("Persist orders+trades+position_state in one transaction. Emit `orders.events.<bot_id>` …" — emit AFTER persist commit, NOT during). DB is canonical; NATS queue (in this case the in-memory `asyncio.Queue` for `stream_executions`) is the delivery channel. Mirror Q2/Q5 publish-after-persist.

3. **Single-transaction persist for `place_market_order`** per §9.5 step 8 verbatim: paper_orders + paper_trades + paper_executions + paper_positions persisted in single asyncpg tx via `async with pool.acquire() as conn: async with conn.transaction(): ...`. If any INSERT fails, the whole order rolls back; no partial state visible to readers. `_pending_sl_tp_fills` queue drain uses the same single-tx idiom.

   **`paper_trades.notional_usd` computation at INSERT time** (BLOCKER 3 resolution): `notional_usd = (qty * fill_price).quantize(Decimal("0.0001"))` per NUMERIC(20,4) schema. NOT NULL on schema (migration 0008). Production formula uses post-slippage `fill_price` (T-213a §C); Hand verification §E.1 fixture uses simplified `entry_price=65000` (no-slippage on entry test fixture for readable close-side math) — see §E.1 inline note. Immutable post-INSERT — close flow does NOT update notional_usd.

   **`paper_orders.idempotent` column mapping** (CONCERN 2 resolution):

   | Source path | order_type | idempotent column |
   |-------------|------------|-------------------|
   | `place_market_order(reduce_only=False)` open | `'market'` | `False` (mirror @non_idempotent decorator + H-003) |
   | `place_market_order(reduce_only=True)` close | `'market'` | `False` (same — manual market close is non-idempotent) |
   | `_drain_sl_tp_fill` SL synthetic | `'sl'` | `True` (mirror Bybit set_trading_stop @idempotent semantics) |
   | `_drain_sl_tp_fill` TP synthetic | `'tp'` | `True` (same) |

   `cancel_order` does NOT INSERT — UPDATE-only on existing row.

   **`paper_orders.correlation_id` sourcing** (BLOCKER 2 resolution): adapter generates per-call via `f"paper-corr-{uuid.uuid4()}"`. Decoupled from T-216 placement pipeline (which has its own correlation propagation via NATS message correlation_id). Trade-off: NATS upstream → paper_orders audit-link is lost; in F2 the correlation chain bot_id + signal_id + paper_orders.id covers audit traceability. If audit-link to NATS becomes load-bearing later, T-201 protocol amendment via ADR (extend `place_market_order` signature with `correlation_id: CorrelationId` kwarg) is the path. F2 simplification per OQ-1 split-scope discipline favors UUID4 generation.

4. **`exchange='paper'` discriminator written at INSERT time** per T-212 OQ-3 + §12.1 verbatim: every paper_orders row gets `exchange = 'paper'`. Other paper_* tables don't have an exchange column (mirror live shape per T-212).

5. **Synthetic paper_orders rows for SL/TP fills** per §3.1 line 268 + H-024 binding: when `_pending_sl_tp_fills` queue drains, each PendingSLTPFill creates a synthetic paper_orders row (`status='filled'`, `order_type='sl' | 'tp'`, `reduce_only=True`, `price=trigger_price`, `qty=fill.qty`, `idempotent=True` per Bybit set_trading_stop semantics, `exchange='paper'`, generated `exchange_order_id` for downstream matchability) before paper_executions INSERT. **Why**: T-218 execution dispatcher derives `exec_type` via DB `execId → order_id` matching per H-024 — paper_executions.order_id MUST point at a real paper_orders row, otherwise H-024's invariant breaks. Mirror live: Bybit's set_trading_stop with `tpsl_mode` creates a tracked stop order that gets its own exchange_order_id. Paper synthesises the same shape locally.

6. **`exchange_order_id` generation strategy**: paper has no upstream exchange to provide an external ID. Use UUID4 string per-row (`f"paper-{uuid.uuid4()}"`). Stored at INSERT time on paper_orders; re-used on paper_executions.exchange_exec_id (separate UUID4) + paper_executions.exchange_order_id link. **Mirror live invariant**: every paper_executions row resolves to exactly one paper_orders row via order_id FK (NOT via exchange_order_id text-match — the FK is the canonical link; exchange_order_id is metadata for downstream consumers).

7. **`reduce_only` parameter semantics for paper**: at `place_market_order(reduce_only=True)`, the call CLOSES the active paper_positions row for `(bot_id, symbol)`: UPDATE paper_trades SET exit_price, realized_pnl, fees_paid, closed_at, status='closed', close_reason='manual'; DELETE paper_positions; emit ExecutionEvent + PositionEvent. `reduce_only=False` opens a new trade — must NOT have an existing active paper_positions row for `(bot_id, symbol)` (single-shot trades per §9.5; raise `OrderRejected("position_already_open")` if existing position present and reduce_only=False; raise `OrderRejected("no_position_to_close")` if reduce_only=True without active position — caller bugs that exchange logically rejects per §11.3 OrderRejected semantics). Paper does NOT scale into positions in v2. **`OrderRejected` reused** (BLOCKER 4 resolution): no new exception class; mirrors live BybitV5Adapter caller-mistake mapping per §11.3.

8. **Realized P&L computation at close** per §9.5 / H-001-H-002 cumulative-delta convention: paper computes `realized_pnl` directly (no Bybit closed_pnl snapshot needed; paper IS the source of truth). Formula:
   - For long close: `realized_pnl = (exit_price - entry_price) * qty - fees_open - fees_close`
   - For short close: `realized_pnl = (entry_price - exit_price) * qty - fees_open - fees_close`
   - `fees_paid` = sum of all paper_executions.fee for this trade.

   Stored on paper_trades.realized_pnl as NUMERIC(20,4) (USD-denominated per T-212 schema). Decimal arithmetic in default 28-digit context; coerced to NUMERIC(20,4) at INSERT/UPDATE boundary by asyncpg.

9. **Partial TP semantics** per H-013 + §9.5 step 7 + OQ-3 default A (entry fee reserved on partial TP): `tpsl_mode='Partial'` with `tp_size < qty`. On TP cross detection (T-213a populated PendingSLTPFill with `qty=tp_size`), drain creates:
   - Synthetic paper_orders SL/TP row with `qty=tp_size` (NOT full position qty).
   - paper_executions row with `qty=tp_size`.
   - paper_trades stays OPEN with reduced qty: UPDATE paper_trades SET qty = qty - tp_size, fees_paid = fees_paid + tp_fee, realized_pnl = realized_pnl + partial_pnl (via `update_paper_trade_partial` helper per BLOCKER 5).
   - paper_positions stays OPEN with reduced remaining_qty: UPDATE paper_positions SET remaining_qty = remaining_qty - tp_size, tp_hit = TRUE.
   - Realized P&L on partial TP = `(tp_price - entry_price) * tp_size - tp_fee` (TP fee only; entry fee RESERVED for full close per OQ-3 default A — simpler accounting; matches v1 paper behavior; deterministic for backtest reproducibility).
   - Emit ExecutionEvent + PositionEvent (partial — size reduced, not zeroed).

   `tpsl_mode='Full'` with TP cross: full close. Same path as `reduce_only=True` close. Entry fee absorbed at full-close realized_pnl computation.

10. **Constructor extension** at T-213b: `pool: asyncpg.Pool` + `event_queue_maxsize: int = 1000` kwargs added (T-213a baseline = 7 params; T-213b → **9 params**, all keyword-only — W#3 count fix incl. BLOCKER 6's queue-size DI). T-213a's `bot_id`, `bus`, `slippage_params`, `now_fn` preserved. **Restart hydrate** of `_active_positions` from paper_positions deferred to T-213c per OQ-1 split.

11. **`stream_executions` / `stream_positions` AsyncIterator implementation** per OQ-5 default A: per-instance `asyncio.Queue[ExecutionEvent]` (and Queue[PositionEvent]); each write method `await queue.put(event)` post-persist; AsyncIterator yields via `async def _iter(): while True: yield await queue.get()`. **Constructor kwarg `event_queue_maxsize: int = 1000`** (BLOCKER 6 / L-001 active control resolution): bounded-queue size DI'd via constructor with sensible default; not hardcoded literal. Pure-blocking backpressure — if consumer falls behind, writer blocks at `queue.put()`. Mirror T-209 BybitV5Adapter shape (when shipped) so T-218 dispatcher cannot distinguish per §3.1 line 268. Queue lifecycle: created at construction, drained at `close()` (T-213a no-op stays — queue cleanup is process-exit only).

12. **`stream_executions` / `stream_positions` returned shape** per T-201 OQ-1 verbatim: `def stream_executions(self) -> AsyncIterator[ExecutionEvent]` (NOT `async def`); returns the AsyncIterator directly. Caller idiom `async for event in client.stream_executions(): ...`. Idempotent-on-call: each call returns a new generator yielding from the same shared queue (so multiple consumers fan out — though §3.1 expects single-consumer T-218 dispatcher per bot).

   **Caveat — single-consumer assumption**: shared `asyncio.Queue` round-robins across `await queue.get()` callers; if two consumers simultaneously call `stream_executions()`, each event lands in exactly one consumer (NOT broadcast). Brief §9.5 wires single dispatcher per bot, so this matches expected use. Document in docstring so a future multi-consumer attempt fails loudly via integration test (covered in OQ-5 below).

## Edge cases

- **Concurrent SL/TP cross + manual close**: candle delivery and manual `place_market_order(reduce_only=True)` are both async-task-driven. asyncio single-thread serialisation: only one of them executes at a time. If SL drains FIRST: paper_positions deleted → manual close raises `OrderRejected("no_position_to_close")`. If manual FIRST: paper_positions deleted → drain finds no position → drain skips fill → log warning + drop event. Test pin: `test_concurrent_sl_drain_and_manual_close_yields_one_close_path`.
- **`reduce_only=True` on partial-position close**: paper supports only full or partial-via-tp_size; manual `reduce_only=True` always closes whole remaining position (mirror live "market close all" semantics).
- **asyncpg.InterfaceError mid-persist**: single-tx rollback via `async with conn.transaction()` context manager; exception propagates; queue NOT mutated; ExecutionEvent NOT emitted. Caller (T-216) handles via H-003 unknown_state path.
- **Queue full (`maxsize=1000`)**: writer blocks at `queue.put()`. In F2 paper this is unreachable (single bot, ≤10 events/min). Surfaces as test pin `test_queue_full_blocks_writer_does_not_drop`.
- **Multiple `stream_executions()` consumers**: shared queue round-robins (Decision #12 caveat). Test pin: `test_two_consumers_split_events_does_not_broadcast` documents the limitation.
- **Synthetic exchange_order_id collision**: UUID4 collision probability is ~0; not handled.
- **Decimal precision overflow at INSERT**: NUMERIC(30,12) qty + NUMERIC(30,12) price + NUMERIC(20,8) fee + NUMERIC(20,4) usd handle all real-world scenarios. Default 28-digit context preserves T-213a Decimal exactness up to the asyncpg coerce boundary. Test pin: `test_max_precision_decimal_round_trips_through_paper_orders`.
- **Restart with stale `_active_positions` (deferred to T-213c)**: at T-213b, restart yields empty `_active_positions` dict + populated paper_positions DB rows. SL/TP cross detection won't fire on the open positions until T-213c hydrate ships. **OQ-2 below** asks if this gap is acceptable for T-213b independent ship.
