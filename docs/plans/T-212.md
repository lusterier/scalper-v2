## Edge cases

- **§12.1 vs T-200 inconsistency**: 3 tables verbatim (§12.1) vs 4 tables (T-200 plan + TASKS.md). OQ-1 surfaces this; default B (4 tables) preserves T-200 commit and live-ledger symmetry per §3.1 line 268.
- **`paper_trades` foreign-key target if 3-table option (OQ-1 default A)**: paper_executions.trade_id and paper_positions.trade_id would have no target — would degrade to plain BIGINT no-FK or be dropped entirely.
